# TUMLU: A Unified and Native Language Understanding Benchmark for Turkic Languages

**会议**: ACL 2025  
**arXiv**: [2502.11020](https://arxiv.org/abs/2502.11020)  
**代码**: 有 (https://github.com/ceferisbarov/TUMLU)  
**领域**: NLP / 多语言NLU / 基准评测  
**关键词**: MMLU, 突厥语系, 多语言基准, 原生语言评估, 低资源语言

## 一句话总结

提出 TUMLU 和 TUMLU-mini，首个面向突厥语系（9 种语言）的原生语言理解基准，包含 38139 道中高中学科的多选题，覆盖拉丁、西里尔和阿拉伯三种文字系统，为低资源语言的 LLM 评估提供了高质量的原生基准。

## 研究背景与动机

评估 LLM 的多语言理解能力需要高质量的原生语言基准，但现有方案存在三个主要问题：

**翻译导致的偏差**：大多数多语言 MMLU 基准（如 Global MMLU）通过机器翻译从英语生成，引入翻译腔（translationese）和文化不匹配

**低资源语言覆盖不足**：突厥语系中的维吾尔语、卡拉卡尔帕克语、鞑靼语、克里米亚鞑靼语此前从未被纳入 MMLU 类基准

**学科和难度不平衡**：现有跨语言基准在不同语言间的学科分布和知识水平差异巨大，难以直接比较

突厥语系具有独特的形态句法特征和丰富的文化背景，且使用拉丁、西里尔和阿拉伯三种文字系统，为 LLM 评估提供了独特的多维度测试场景。

## 方法详解

### 整体框架

TUMLU 由 9 种突厥语的原生中高中学科题目组成：阿塞拜疆语、克里米亚鞑靼语、卡拉卡尔帕克语、哈萨克语、吉尔吉斯语、鞑靼语、土耳其语、维吾尔语和乌兹别克语。

### 关键设计

1. **数据收集与质量控制**：

    - 从公开书籍和网站收集原始题目（非翻译），大部分是大学入学考试的样题或正式题目
    - 统一为 4 选项格式（缺少选项保持原样，多于 4 个则删除一个错误选项）
    - 母语者手动验证随机样本质量，发现约 10% 的题目无效或答案错误
    - 设计动机：避免翻译偏差，确保题目反映真实的文化和语言特性

2. **TUMLU-mini（精简验证版）**：

    - 每个学科-语言组合随机选取 100 题（不足 100 则全部使用）
    - 所有手动验证、选项打乱（防止记忆效应）
    - 移除仅在 1-2 种语言中有的学科
    - 用于主要实验以控制成本

3. **双文字平行数据集**：

    - 对使用双文字系统的语言（如哈萨克语的拉丁/西里尔），提供转写版本
    - 允许在相同内容上比较 LLM 在不同文字系统下的性能
    - 设计动机：研究文字系统对 LLM 理解能力的影响

4. **11 个学科覆盖**：

    - 数学、物理、化学、生物、地理、本国语言文学、历史、逻辑、人与社会、哲学、宗教伦理
    - 其中逻辑等 4 个学科仅在 1-2 种语言中有，主要实验中排除

### 损失函数 / 训练策略

- 纯评估性工作，不涉及训练
- 实验设置：5-shot 和 5-shot CoT 两种 prompting 方式
- 所有提示使用目标语言（而非英语），基于先前研究表明英语提示无性能增益
- 超参数：temperature=0, max_tokens=1024, top_p=1.0

### 关键发现

1. **闭源模型全面领先**：Claude 3.5 Sonnet 在所有语言上均为最优，前 5 名均为闭源模型
2. **规模效应显著**：Llama 3.1 8B→70B 提升约 17 个百分点，Qwen2.5 7B→72B 提升约 19 个百分点
3. **低资源语言不一定表现差**：克里米亚鞑靼语、卡拉卡尔帕克语等低资源语言的性能与中高资源语言可比
4. **本国语言文学最难**：几乎所有模型在"本国语言文学"学科上表现最差，因为需要深度文化和文学知识
5. **Llama 3.1→3.3 70B 有显著提升**：从 57.7 提升到 58.6，主要在土耳其语和鞑靼语上
6. **维吾尔语和乌兹别克语**整体得分最低，可能反映了这些语言在预训练数据中的匮乏

## 亮点与洞察

- **首次覆盖**维吾尔语、卡拉卡尔帕克语、鞑靼语和克里米亚鞑靼语的 MMLU 类基准
- **原生数据**避免了翻译偏差，更真实地反映各语言的文化特性
- **双文字平行数据集**的设计具有独特价值——可以控制内容变量，仅比较文字系统的影响
- 低资源语言在 LLM 中的表现出乎意料地不差，可能因为语言间的迁移学习效应
- 38139 道题目的规模在区域性多语言 MMLU 中算较大

## 局限性 / 可改进方向

- 不同语言的题目难度不统一：土耳其和乌兹别克的题目模拟大学入学考试（较难），其他语言的社区出题可能较简单
- 跨语言直接比较分数需谨慎，因为学科分布和难度不等
- 部分学科（如逻辑、哲学）仅在 1-2 种语言中有，限制了完整性
- CoT prompts 仅为 4 种语言创建，其他语言缺少 CoT 评估
- 数据集可能部分出现在 LLM 的预训练语料中（虽然做了选项打乱）

## 相关工作与启发

- 与 TurkishMMLU (Yüksel et al., 2024) 的土耳其语数据集整合，形成更完整的基准
- 与 INCLUDE (Romanou et al., 2025) 的原生数据理念一致，但 TUMLU 在单一语系内更加平衡
- 与 Global MMLU (Singh et al., 2024) 的翻译方法形成对照——原生数据质量更高但构建成本更大
- 启发：区域性语言家族基准（如突厥语系、德拉维达语系等）可能比大而全的全球基准更有评估价值
