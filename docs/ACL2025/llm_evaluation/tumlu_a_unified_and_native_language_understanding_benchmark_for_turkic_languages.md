# TUMLU: A Unified and Native Language Understanding Benchmark for Turkic Languages

**会议**: ACL 2025  
**arXiv**: [2502.11020](https://arxiv.org/abs/2502.11020)  
**代码**: [ceferisbarov/TUMLU](https://github.com/ceferisbarov/TUMLU)  
**领域**: NLP / 多语言NLU / 基准评测  
**关键词**: MMLU, 突厥语系, 多语言基准, 原生语言评估, 低资源语言

## 一句话总结

提出 TUMLU 和 TUMLU-mini，首个面向突厥语系 9 种语言的原生多任务语言理解基准，包含 38,139 道中高中学科多选题，覆盖拉丁/西里尔/阿拉伯三种文字系统，系统评估了 13 个开源与闭源 LLM，揭示了文字系统、语言资源量和 CoT 对模型性能的差异化影响。

## 研究背景与动机

**核心问题**：如何为形态句法独特的突厥语系构建高质量、无翻译偏差的 MMLU 类评估基准？

**现有多语言基准的翻译偏差**：Global MMLU 等主流多语言基准依赖机器翻译从英语生成，引入 translationese 和文化不匹配，无法反映目标语言的真实推理场景。例如，翻译后的数学题可能丢失原语言中的惯用表述。

**突厥语系的严重缺位**：在 TUMLU 之前，维吾尔语、卡拉卡尔帕克语、鞑靼语和克里米亚鞑靼语从未被纳入任何 MMLU 类基准；阿塞拜疆语、哈萨克语、吉尔吉斯语和乌兹别克语的已有基准覆盖也极为有限，现有数据集主要集中在土耳其语（TurkishMMLU）。

**跨语言可比性困难**：INCLUDE 等原生基准虽然不依赖翻译，但各语言的学科分布和知识水平差异巨大，难以在同一语系内进行公平的模型能力对比；突厥语系还涉及三种文字系统（拉丁、西里尔、阿拉伯），文字系统对 LLM 性能的影响未被系统研究。

## 方法详解

### 整体框架

TUMLU 是一个面向突厥语系的多语言多任务语言理解基准，涵盖阿塞拜疆语、克里米亚鞑靼语、卡拉卡尔帕克语、哈萨克语、吉尔吉斯语、鞑靼语、土耳其语、维吾尔语和乌兹别克语共 9 种语言。数据集包含 38,139 道四选一多选题，来源为各国公开的大学入学考试样题和教材，横跨 11 个中高中学科。评估采用 5-shot 和 5-shot CoT 两种 prompting 方式，所有提示均使用目标语言。

### 关键设计

1. **原生数据收集与多层质量控制**：所有题目直接从各语言的公开教育资源采集（非翻译），大部分为大学入学考试的正式或模拟题。统一为 4 选项格式（多于 4 个删除一个错误项，少于 4 个保留原样）。母语标注者对各学科随机样本进行人工审核，发现约 10% 的题目存在无效或答案错误问题并予以修正。选项打乱以降低记忆效应。

2. **TUMLU-mini 精简验证子集**：为控制实验成本和保证平衡性，从每个「学科×语言」组合中随机抽取 100 题（不足 100 则全部使用），全部经人工验证。移除仅在 1-2 种语言中存在的学科（逻辑、人与社会、哲学、宗教伦理），最终保留 7 个核心学科用于主实验。先前工作已证明 100 题/学科足以可靠估计大规模数据集的性能。

3. **双文字平行数据集**：针对同时使用两种字母系统的语言（克里米亚鞑靼语的拉丁/西里尔、哈萨克语的西里尔/拉丁、维吾尔语的阿拉伯/拉丁、乌兹别克语的拉丁/西里尔），提供自动转写的平行版本。在内容完全一致的条件下，控制变量地比较文字系统对 LLM 理解能力的影响。利用 FineWeb 2 语料统计验证了性能差异与预训练语料中各文字分布的一致性。

### 评估设置

| 配置项 | 设置 |
|-------|------|
| Prompting | 5-shot / 5-shot CoT（CoT 仅阿塞拜疆/哈萨克/土耳其/乌兹别克 4 种语言） |
| 提示语言 | 各目标语言原文（非英语） |
| Temperature | 0.0 |
| Max tokens | 1,024 |
| Top-p | 1.0 |
| API | OpenAI / Anthropic / Google Cloud / Together AI / Deep Infra |

### 关键发现

1. **闭源模型全面领先**：Claude 3.5 Sonnet 在全部 9 种语言上均为最优，前 5 名均为闭源模型；最佳开源模型 Qwen2.5 72B 比 Claude 低 19 个百分点
2. **规模效应显著**：Llama 3.1 8B→70B 提升约 17.4 pp，Qwen2.5 7B→72B 提升约 18.9 pp
3. **文字系统影响与语料分布一致**：模型在预训练语料中占比更高的文字系统上表现更好（如哈萨克西里尔 > 拉丁，克里米亚鞑靼拉丁 > 西里尔）
4. **本国语言文学最难**：Claude 在该学科平均仅 ~68%，远低于其他 STEM 学科（~85%+），因为需要深层文化和文学知识
5. **低资源语言的跨语言迁移**：克里米亚鞑靼等极低资源语言性能出人意料地与中资源语言可比，但生成文本语言检测显示其输出多为土耳其语与目标语言的混合
6. **CoT 对闭源模型稳定正向**：Claude CoT 平均提升 +2.7 pp，但对小型开源模型（Llama 3.1 8B）反而有负面影响（-5.1 pp）

## 亮点与洞察

- 首个覆盖维吾尔语、卡拉卡尔帕克语、鞑靼语和克里米亚鞑靼语的 MMLU 类基准，填补了突厥语系 LLM 评估的关键空白
- 双文字平行数据集的实验设计精巧：控制内容变量、仅改变文字系统，揭示了预训练语料分布对文字偏好的决定性影响
- 生成语言检测分析揭示了一个深层问题：LLM 在低资源语言上"答对了题"不代表"用对了语言"，模型实际上在做跨语言的码混（code-mixing）推理
- 38,139 道原生题目的规模在区域性语言家族基准中属于最大级别

## 局限与展望

- 不同语言的题目难度不统一（土耳其/乌兹别克为入学考试级别，其他为社区出题），跨语言比较需谨慎
- CoT 提示仅覆盖 4 种语言，其余 5 种缺乏 CoT 评估
- 部分学科（逻辑、哲学等）仅在 1-2 种语言中有数据，限制了完整性
- 数据集可能部分出现在 LLM 预训练语料中（虽然做了选项打乱处理）
- 未评估最新 LLM（如 GPT-4o-mini、Llama 3.2 等更新模型）

## 相关工作与启发

- TurkishMMLU (Yüksel et al., 2024) 的土耳其语子集被直接整合入 TUMLU，形成更完整的跨语系对比
- 与 INCLUDE (Romanou et al., 2025) 共享原生数据理念，但 TUMLU 通过聚焦单一语系实现了更均衡的学科和难度分布
- 与 Global MMLU (Singh et al., 2024) 的翻译路线形成对照，证明原生数据虽构建成本更高但质量显著更优
- 启发：区域性语言家族基准（突厥语系、德拉维达语系等）可能比全球性基准更有效地评估低资源语言能力

## 评分

- **新颖性**: ⭐⭐⭐⭐ — 首个突厥语系原生 MMLU，三种文字系统的平行对比实验具有独特价值
- **实验充分度**: ⭐⭐⭐⭐ — 13 个模型（开源+闭源）、9 种语言、7 个学科、两种 prompting 方式、文字系统对比分析
- **写作质量**: ⭐⭐⭐ — 内容翔实但部分章节冗长，跨语言比较的局限性讨论可以更前置
- **价值**: ⭐⭐⭐⭐ — 填补突厥语系 LLM 评估空白，双文字平行数据集有独立研究价值
