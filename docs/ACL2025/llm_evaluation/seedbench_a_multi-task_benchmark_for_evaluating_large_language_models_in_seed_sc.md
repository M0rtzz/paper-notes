# SeedBench: A Multi-task Benchmark for Evaluating Large Language Models in Seed Science

**会议**: ACL2025  
**arXiv**: [2505.13220](https://arxiv.org/abs/2505.13220)  
**代码**: [open-sciencelab/SeedBench](https://github.com/open-sciencelab/SeedBench)  
**领域**: llm_nlp  
**关键词**: benchmark, seed science, agriculture, LLM evaluation, domain-specific

## 一句话总结

提出 SeedBench——首个面向种子科学（种子育种）的多任务 LLM 评测基准，包含 2,264 道专家验证题目，覆盖基因信息检索、基因功能调控和品种选育三大育种流程，对 26 个 LLM 进行系统评估，揭示了当前 LLM 与真实育种需求之间的显著差距。

## 研究背景与动机

1. **领域重要性**：种子科学是现代农业的基石，直接影响作物产量和全球粮食安全，但学科交叉性强、经济回报低，导致育种专家持续短缺。
2. **LLM 在科学领域的潜力**：LLM 在金融、医疗、教育等领域已展现强大能力，但在种子育种这一 AI for Science 新兴方向上的应用仍十分有限。
3. **数字资源匮乏**：种子科学的在线文献和结构化数据远少于主流研究领域，制约了 LLM 的训练和评估。
4. **缺乏标准化基准**：现有农业 benchmark（AgEval 做植物胁迫表型、AgXQA 做农业推广、CROP 做作物知识）均未涉及种子育种的多步骤决策过程。
5. **育种流程复杂**：真实育种涉及基因检索→功能分析→品种选育的多阶段决策，需要综合遗传、环境、农艺数据，一般 benchmark 无法覆盖。
6. **研究空白明确**：在 LLM 评测领域，种子育种作为关键农业任务从未有过专门的基准测试，亟需填补这一空白。

## 方法详解

### 整体框架

SeedBench 以育种专家的实际工作流为蓝本，将评估任务划分为三大类别（对应育种三步骤）、十个子类别和十一种题型，共 2,264 道题目。构建流程为：数据收集（308,727 篇论文 → 1.1B token 语料） → 专家精选 279 段高质量文本 → GPT-4 自动生成题目 → 机器审查 + 人工专家双重验证。

### 关键设计一：三层级育种任务分类体系

- **做什么**：将题目按真实育种流程分为三大类十个子类别——（1）基因信息检索（基本信息/表达模式/细胞定位）、（2）基因功能与调控（实验观察/下游调控/功能预测）、（3）品种选育与农艺性状（选育过程/农艺性状/栽培技术/适宜种植区域推荐）。
- **核心思路**：模拟育种专家从「查基因 → 分析功能 → 选育品种」的完整决策链，覆盖从基础知识到应用的全部环节。
- **设计动机**：仅做单一维度评估（如问答）无法真实反映 LLM 在育种场景的可用性；需要多层级、多阶段的评测体系才能全面衡量模型在知识检索、推理和决策方面的综合能力。

### 关键设计二：11 种多样化题型

- **做什么**：设计了 Q&A（单选/多选/填空/生成）、摘要（简单摘要/关键信息提取）和阅读理解（单选/多选/填空/生成/子类别分类）共 11 种题型，采用 Accuracy、Macro-F1 和 ROUGE-L 等指标分别评估。
- **核心思路**：不同题型考察不同能力维度——选择题测知识记忆、生成题测表述能力、摘要题测信息压缩、阅读理解题测长文本中的检索与推理。
- **设计动机**：育种任务的多样性要求 LLM 同时具备精确召回、推理分析和文本生成等能力，单一题型无法全面评测，多题型设计确保评估的全面性和鲁棒性。

### 关键设计三：双重质量验证机制

- **做什么**：对 GPT-4 自动生成的题目实施两阶段验证——第一阶段由 GPT-4 自动检查连贯性、逻辑一致性和任务符合度（过滤 ~0.01%）；第二阶段由具有种子科学博士学位的领域专家人工审核相关性和准确性（过滤 ~20%），最终保留 2,264 道高质量题目。
- **核心思路**：机器审查快速排除明显错误，人工审查确保领域专业性和科学严谨性。
- **设计动机**：纯自动生成的题目质量参差不齐，尤其在高度专业化的育种领域，GPT-4 可能生成看似合理但实际不准确的问题；双重验证兼顾效率和质量。

## 实验关键数据

### 主实验：26 个 LLM 在 SeedBench 上的表现

| 模型 | 类型 | 参数量 | 平均分 |
|------|------|--------|--------|
| DeepSeek-V3 | 开源 | 671B | **63.30** |
| GPT-4 | 闭源 | - | 62.06 |
| GLM-4-Plus | 闭源 | - | 59.61 |
| GPT-4o mini | 闭源 | - | 58.40 |
| Qwen2-72B | 开源 | 72B | 57.62 |
| Qwen2.5-14B | 开源 | 14B | 54.21 |
| Claude-3.5-Sonnet | 闭源 | - | 55.45 |
| OpenAI o1-mini | 闭源 | - | 53.25 |
| QwQ-32B | 开源 | 32B | 33.55 |
| Aksara-v1 | 领域微调 | 7B | 35.04 |
| PLLaMa-13B | 领域微调 | 13B | 17.57 |

### 消融/分析实验

| 分析维度 | 关键发现 |
|----------|----------|
| 推理模型 vs 通用模型 | o1-mini (53.25)、QwQ (33.55) 均低于 GPT-4 (62.06)，显式推理链在知识检索型任务中反而引入噪声 |
| 领域微调 vs 通用模型 | PLLaMa-7B (16.46)、PLLaMa-13B (17.57) 远低于同参数量通用模型 Qwen2.5-7B (48.45)，微调导致指令遵循能力退化 |
| 模型规模影响 | 7B–14B 为最佳性价比区间；Qwen 系列从 14B 到 72B 几乎无提升（54.21→52.63），暗示训练数据分布比规模更重要 |
| 子类别难度 | C5（基因下游调控分析）最难，最高分仅 56.34；C3（细胞定位查询）最易，多个模型超过 70 |

### 关键发现

1. **DeepSeek-V3 以 63.30 分领先所有模型**，超过 GPT-4 的 62.06，但所有模型平均分均未突破 65，种子科学仍是巨大挑战。
2. **推理型模型在育种任务上表现不佳**——冗长的推理链在以知识检索为主的任务中引入无用信息，拉低 ROUGE 等指标。
3. **领域微调模型全面落后**——核心原因是微调后指令遵循和对话能力退化，PLLaMa 在 one-shot 评估中甚至将示例误当成查询。
4. **模型规模存在收益递减**——7B-14B 为最优区间，更大模型未必更好，训练数据质量比模型大小更关键。

## 亮点与洞察

1. **首创性**：填补了种子育种领域 LLM 基准测试的空白，是 AI for Science 在农业方向的重要基础设施。
2. **专家驱动设计**：由具有跨学科背景的博士级专家全程参与，确保任务设计贴合真实育种工作流。
3. **反直觉发现**：推理增强模型（o1、QwQ）在种子科学上反而不如通用模型，挑战了"推理能力越强越好"的假设。
4. **对领域微调的深刻反思**：揭示了微调过程中通用能力退化的问题，为未来领域适配策略提供了重要参考。

## 局限性

1. **数据来源单一**：主要依赖同行评审论文，未纳入在线数据库和专家知识库，数据多样性有限。
2. **语言偏差**：虽收集中英文论文但评测以英文为主，未充分考虑跨语言差异对模型表现的影响。
3. **缺乏多模态评估**：实际育种高度依赖图像（表型）和传感器数据，当前基准仅覆盖文本模态。
4. **静态评估**：真实育种是多轮迭代决策过程，当前基准为单次问答形式，无法评估模型的序贯决策能力。

## 相关工作与启发

| 对比维度 | SeedBench | AgEval / AgXQA / CROP |
|----------|-----------|----------------------|
| 领域覆盖 | 种子育种全流程（基因→功能→选育） | 单一方面（植物胁迫/农业推广/作物知识） |
| 任务复杂度 | 11种题型、10个子类别、多步骤决策模拟 | 题型单一，多为简单问答或分类 |
| 质量保证 | 双重验证（机器+博士级专家） | 多依赖网络数据和 GPT 重标注 |
| 评估规模 | 26个模型（闭源+开源+领域微调） | 通常仅评估少量模型 |

**vs 通用科学基准（GMAI-MMBench、LawBench）**：SeedBench 的核心区别在于其任务设计严格对齐领域专家的实际工作流程，而非简单汇集领域相关的问答对；这种"流程驱动"的设计范式可迁移到其他 AI for Science 领域。
