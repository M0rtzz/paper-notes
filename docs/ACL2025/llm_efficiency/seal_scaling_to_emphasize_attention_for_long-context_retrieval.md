# SEAL: Scaling to Emphasize Attention for Long-Context Retrieval

**会议**: ACL 2025  
**arXiv**: [2501.15225](https://arxiv.org/abs/2501.15225)  
**代码**: 无  
**领域**: LLM效率 / 长上下文  
**关键词**: 长上下文检索, 注意力头缩放, 零推理开销, Needle-in-Haystack, 注意力头剪枝

## 一句话总结
通过剪枝分析发现特定注意力头与长上下文检索高度相关，提出 SEAL（可学习标量/向量缩放注意力头/通道），仅需 ~1024 个额外参数 + 50 个合成样本训练，即可将 LongChat-7B 在 31K 上的检索准确率从 0.32 提升到 0.88，且缩放参数可离线合并到权重中实现零推理开销。

## 研究背景与动机

1. **领域现状**：长上下文 LLM（32K-128K）在理论支持的长度范围内仍会出现检索性能退化。
2. **现有痛点**：LongChat-7B 声称支持 32K，但在 19-31K 范围内 Needle-in-Haystack 准确率从 0.96 骤降到 0.32。微调整个模型代价高且可能影响其他能力。
3. **核心矛盾**：退化是少数注意力头的问题还是全局问题？如果是少数头，能否精准修复？
4. **本文要解决什么**：用最小参数变化修复长上下文检索退化。
5. **切入角度**：head-wise/channel-wise 剪枝发现个别头对检索准确率有 ±20% 的影响→缩放这些头而非修改整个模型。
6. **核心 idea**：学习标量（head级）或向量（channel级）缩放因子→合并进权重→零开销。

## 方法详解

### 关键设计

1. **注意力头剪枝分析**：逐个屏蔽注意力头，发现少数头对检索有巨大影响（±20%）

2. **SEAL-H（Head-wise Scaling）**：每个注意力头学一个标量 $\alpha_h$，仅 ~1024 参数（7B 模型）

3. **SEAL-C（Channel-wise Scaling）**：每个注意力头学一个向量缩放，更细粒度

4. **离线权重合并**：缩放因子可直接乘进 QKV 权重矩阵→推理时无任何额外计算

## 实验关键数据

### 主实验（LongChat-7B-32K, Line Retrieval）

| 上下文长度 | Baseline | SEAL-H | SEAL-C |
|-----------|----------|--------|--------|
| 9K | 0.98 | 1.00 | 0.98 |
| 14K | 0.96 | 1.00 | 0.96 |
| 31K | **0.32** | **0.80** | **0.88** |

### 关键发现
- **31K 从 0.32→0.88**：性能近乎恢复到短上下文水平
- **MMLU 保持稳定**（42.5→42.2）：未损害通用能力
- **适用多种架构**：LongChat、Mistral、Vicuna、Llama-3.1 均有效
- **50 个合成训练样本即足够**：极低数据量需求

## 亮点与洞察
- **"注意力头有专用功能"的实证**：少数头专门负责长距离检索，与其他功能头分离
- **零推理开销的参数合并**：缩放→合并→免费提升

## 局限性 / 可改进方向
- 需要 Needle-in-Haystack 迭代找到退化点
- Channel 级剪枝探索计算量大
- 仅针对检索任务验证，复杂推理任务效果未知
