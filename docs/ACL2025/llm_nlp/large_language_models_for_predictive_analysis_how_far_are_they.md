# Large Language Models for Predictive Analysis: How Far Are They?

**会议**: ACL 2025  
**arXiv**: [2505.17149](https://arxiv.org/abs/2505.17149)  
**代码**: 无  
**领域**: LLM/NLP  
**关键词**: predictive analysis, benchmark, code generation, text analysis, PredictiQ

## 一句话总结
提出 PredictiQ 基准（1130 查询×44 真实数据集×8 领域），从文本分析、代码生成和文本-代码对齐三个维度评估 12 个 LLM 的预测分析能力，发现现有 LLM 在此领域仍面临巨大挑战。

## 研究背景与动机

1. **领域现状**：预测分析（基于历史数据预测未来趋势）是决策支持的基石，LLM 作为交互式助手有潜力帮助非专业用户进行预测分析。
2. **现有痛点**：现有评估要么只看输出结果，要么只看生成代码，忽略了LLM预测分析需要"文本解释+代码实现+两者对齐"的完整能力。
3. **核心矛盾**：预测分析需要数据预处理、算法选择、结果解释等复合能力，现有基准不覆盖这种端到端流程。
4. **本文要解决什么？** 构建首个全面评估LLM预测分析能力的基准。
5. **切入角度**：三维度评估协议——文本分析（解释算法选择等）+ 代码生成（可执行代码）+ 文本-代码对齐（两者一致性）。
6. **核心idea一句话**：LLM 的预测分析能力远未成熟——文本解释和代码实现之间常常不一致。

## 方法详解

### 整体框架
收集 44 个真实数据集 × 8 个领域 -> 专家构造 1130 个预测查询 -> LLM 生成文本+代码回答 -> 三维度评估（文本分析质量 + 代码正确性 + 一致性）。

### 关键设计

1. **三维度评估协议**
   - 文本分析：数据理解、算法选择、结果解释、假设合理性
   - 代码生成：代码可执行性、结果正确性
   - 文本-代码对齐：文本中的算法描述与代码实现是否一致
   - 设计动机：预测分析不仅要做对，还要解释对

2. **专家+LLM 混合评估**
   - 人类专家评估 + GPT-4Turbo 自动评估
   - GPT-4Turbo 与人类专家一致率 90.5%
   - 设计动机：平衡评估质量和成本

3. **查询设计原则**
   - 每个查询是单一、自包含的预测问题
   - 仅依赖提供的数据集，不需要外部知识
   - 设计动机：确保可重复评估

## 实验关键数据

### 主实验 — 12 个 LLM 的预测分析能力
| 模型 | 文本分析 | 代码生成 | 文本-代码对齐 | 总分 |
|------|---------|---------|-------------|------|
| GPT-4o | ~70% | ~55% | ~50% | ~58% |
| Claude-3.5 | ~68% | ~52% | ~48% | ~56% |
| DeepSeek-Coder | ~55% | ~60% | ~45% | ~53% |
| Llama-3-70B | ~58% | ~42% | ~38% | ~46% |

### 领域差异分析
| 领域 | 最佳模型表现 | 最差模型表现 | 差距 |
|------|------------|------------|------|
| 经济/金融 | ~65% | ~30% | 35% |
| 医疗 | ~60% | ~25% | 35% |
| 交通 | ~55% | ~28% | 27% |

### 关键发现
- **代码微调提升文本分析**：代码专用模型在文本分析上也有提升，说明代码能力和分析能力互相促进
- **文本-代码对齐是最大短板**：模型常在文本中描述一种算法，但代码实现另一种
- **领域差异巨大**：同一模型在不同领域表现差距可达 35%
- **LLM 远未掌握预测分析**：最佳模型总分仅 ~58%

## 亮点与洞察
- **三维度评估**抓住了 LLM 辅助分析的真正挑战——不是能不能做，而是"说的和做的是否一致"
- **文本-代码不一致**是一个独特且重要的发现——直接影响用户信任

## 局限性 / 可改进方向
- 查询和评估成本高（300人时构造 + 900人时评估）
- 改进方向：自动化评估、更多领域、交互式预测分析

## 相关工作与启发
- **vs CodeBench**：CodeBench 只评代码，PredictiQ 评文本+代码+对齐
- **vs MMLU**：MMLU 评知识，PredictiQ 评应用能力
