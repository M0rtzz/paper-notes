# DICE-Bench: Evaluating the Tool-Use Capabilities of Large Language Models in Multi-turn Settings

**会议**: ACL 2025  
**arXiv**: [2506.22853](https://arxiv.org/abs/2506.22853)  
**代码**: 无  
**领域**: LLM/NLP  
**关键词**: tool use, multi-turn, benchmark, LLM evaluation, function calling

## 一句话总结
提出 DICE-Bench 基准评估 LLM 在多轮对话中的工具使用能力，涵盖工具发现、调用、错误恢复和组合使用四个维度，发现即使最强模型在复杂多轮场景下也仅达 ~60% 成功率。

## 研究背景与动机

1. **领域现状**：LLM 的工具使用（function calling）能力评估主要聚焦单轮调用。
2. **现有痛点**：真实应用中工具使用通常需要多轮交互（发现工具->调用->处理错误->组合），现有基准无法评估这一完整流程。
3. **核心矛盾**：单轮评估高估了 LLM 的实际工具使用能力。
4. **本文要解决什么？** 构建覆盖工具使用全流程的多轮评估基准。
5. **切入角度**：定义四个评估维度 DICE（Discovery, Invocation, Composition, Error-recovery）。
6. **核心idea一句话**：多轮工具使用远比单轮困难——错误累积和上下文管理是关键挑战。

## 方法详解

### 整体框架
DICE-Bench 包含四类任务：发现（从描述中识别合适工具）、调用（正确生成 API 参数）、组合（串联多工具完成复杂任务）、错误恢复（处理调用失败并重试）。每类任务设计为多轮对话。

### 关键设计

1. **四维度评估 (DICE)**
   - Discovery：给定任务，从工具库中选择正确工具
   - Invocation：生成正确参数调用工具
   - Composition：组合多个工具完成复杂任务
   - Error-recovery：工具调用失败后修正重试
   
2. **多轮交互模拟**：真实模拟用户-LLM-工具的交互链

## 实验关键数据

### 主实验
| 模型 | Discovery | Invocation | Composition | Error-recovery | 总体 |
|------|-----------|------------|-------------|---------------|------|
| GPT-4o | ~80% | ~75% | ~50% | ~55% | ~65% |
| Claude-3 | ~78% | ~72% | ~48% | ~50% | ~62% |
| Llama-3-70B | ~65% | ~60% | ~35% | ~40% | ~50% |

### 消融
| 维度 | 单轮 | 多轮 | 差距 |
|------|------|------|------|
| Invocation | ~90% | ~75% | -15% |
| Composition | ~70% | ~50% | -20% |

### 关键发现
- **多轮显著更难**：所有维度多轮都比单轮下降 15-20%
- **错误恢复是最大短板**：大多数模型在错误后无法有效修正
- **工具组合能力差距大**：小模型在组合任务上几乎无法完成

## 亮点与洞察
- **DICE 四维度**分解了工具使用的完整能力谱
- **错误恢复维度**是此前基准完全忽略的

## 局限性 / 可改进方向
- 工具库规模有限
- 未包含真实 API 调用

## 相关工作与启发
- **vs ToolBench**：ToolBench 主要评估单轮，DICE 扩展到多轮全流程
