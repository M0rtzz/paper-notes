# SCoP: Evaluating the Comprehension Process of Large Language Models from a Cognitive View

**会议**: ACL 2025  
**arXiv**: [2506.05000](https://arxiv.org/abs/2506.05000)  
**代码**: [SCUNLP/SCOP](https://github.com/SCUNLP/SCOP)  
**领域**: LLM / NLP  
**关键词**: 阅读理解评估, 认知过程, 理解技能, LLM可靠性, 支持句定位

## 一句话总结

SCoP 从认知科学视角将 LLM 的文档理解过程分解为五种递进技能（定位、推断、连接、组织、选择），构建了包含 4682 个样本的测试集来评估理解"过程"而非仅仅"答案"，发现 LLM 普遍在局部理解（~94%）上远强于全局理解（~31%），且即使答案正确其理解过程也可能是错误的。

## 研究背景与动机

**领域现状**：阅读理解是 LLM 最基础的能力之一，现有评估（SQuAD、HotpotQA、RACE 等）通过对比模型输出与标准答案的匹配度来衡量理解能力。在这些基准上，顶级 LLM 已经接近甚至超越人类表现。然而，在法律、医疗、教育等高风险场景中，人们仍然不敢完全信任 LLM 的理解结果。

**现有痛点**：现有评估只关注理解的"结果"（答案是否正确），完全忽略了理解的"过程"。模型可能通过记忆训练数据、利用表面捷径或猜测选项来给出正确答案，而非真正理解了文档内容。少数关注过程的工作（如 Sugawara et al.）又聚焦于理解"之前"的语言技能（共指消解、命名实体识别等），而不是理解过程本身。

**核心矛盾**：答案匹配分数高≠真正理解了文档。一个通过"捷径"得到正确答案的模型，在面对训练数据覆盖不到的新场景时可能完全失效。但我们缺乏系统性的框架来评估 LLM 的理解"过程"是否与专家一致。

**本文目标** 建立一套基于认知科学理论的框架，将文档理解过程分解为可测试的技能层次，构建测试数据评估 LLM 在每个技能上的表现，并揭示过程正确性与答案正确性之间的关系。

**切入角度**：作者借鉴 Bloom 分类法和阅读认知理论（Krathwohl 2002, Afflerbach et al. 2015），将理解过程从局部到全局分为三个层次，五种技能：定位（单句事实检索）、推断（多句信息整合）、连接（句间逻辑关系）、组织（段落结构理解）、选择（全文要点提取）。每种技能对应一个独立的测试任务。

**核心 idea**：从认知科学的三层次五技能框架出发评估 LLM 的文档理解"过程"而非"结果"，揭示即使最强 LLM 也远未实现专家级理解过程。

## 方法详解

### 整体框架

SCoP 包含三个组件：（1）基于认知理论的五种理解技能定义——定位（Locating，句级事实检索）、推断（Inferring，多句信息整合）、连接（Connecting，句间逻辑填充）、组织（Organizing，按小标题分段）、选择（Selecting，关键句提取）；（2）从 12 个现有数据集中严格筛选和构建的 4682 个测试样本；（3）对 4 个顶级 LLM（Qwen2-72B、Llama3.1-70B、Claude-3.5、GPT-4o）的系统评估。核心创新在于要求模型不仅给出答案，还必须输出支持句（理解过程的证据），从而可以检测"答案对但过程错"的不一致现象。

### 关键设计

1. **从认知理论到可测试任务的映射**:

    - 功能：将抽象的认知理解过程分解为五个具体的、可自动评估的 NLP 任务
    - 核心思路：定位→给出问题和文档，输出支持句+答案；推断→给出复杂问题，输出多个支持句+答案；连接→在文档中挖空若干句子，从候选中选择正确句子填入（类似完形填空但单位是句子）；组织→给出打乱的文档和小标题，预测每个小标题的正确位置；选择→给出文档和所需关键句数量，提取关键句
    - 设计动机：这五种技能形成从局部到全局的递进层次。认知科学研究表明人类阅读也遵循这一层次，因此用同样的框架评估 LLM 可以直接对比人机差异

2. **严格的数据构建与噪声过滤框架**:

    - 功能：确保测试样本确实需要模型理解文档才能回答，排除记忆和捷径
    - 核心思路：对于定位和推断任务，先用语义相似度（z-score 阈值）自动标注支持句，然后通过句法树分析判断问题是否需要多步推理。关键一步是"去记忆化过滤"——将问题送入 LLM（不提供文档），如果模型仅靠记忆就能答对则移除。这一步过滤了超过 90% 的样本（37023→4682）
    - 设计动机：高达 90% 的过滤率本身就是惊人的发现——说明大量现有阅读理解基准的问题实际上可以被 LLM 通过记忆回答，完全无需理解文档。这验证了评估理解过程的必要性

3. **不一致性分析（Inconsistency Analysis）**:

    - 功能：检测"答案正确但理解过程错误"的不一致现象
    - 核心思路：计算不一致性分数——在定位和推断任务中，统计支持句预测错误但最终答案正确的样本比例。GPT-4o 在推断层次的不一致率高达 18.29%，是定位层次（2.32%）的 8 倍
    - 设计动机：如果模型通过错误的理解过程得到正确答案，说明它依赖了记忆、猜测或表面捷径，而非真正理解了文档。这种"虚假正确"在高风险场景中是不可接受的

### 损失函数 / 训练策略

SCoP 是评估框架而非模型训练方法。所有模型以 zero-shot 方式评估，temperature 设为 0。

## 实验关键数据

### 主实验

四个 LLM 在五种技能上的准确率（%）：

| 技能 | 层次 | GPT-4o | Claude-3.5 | Llama3.1-70B | Qwen2-72B |
|------|------|--------|-----------|-------------|-----------|
| 定位 (Locating) | 局部 | 95.10 | 92.78 | 93.09 | 93.23 |
| 推断 (Inferring) | 中间 | 33.89 | 37.49 | 46.54 | 31.51 |
| 连接 (Connecting) | 全局 | 41.79 | 65.04 | 25.36 | 52.81 |
| 组织 (Organizing) | 全局 | 31.70 | 49.50 | 30.43 | 17.78 |
| 选择 (Selecting) | 全局 | 17.46 | 14.18 | 13.31 | 12.96 |

### 消融实验

| 分析维度 | 关键发现 |
|---------|---------|
| 提供正确支持句后推断性能 | GPT-4o: +15.97%, Claude: +8.03%, Qwen: +18.22% |
| 不一致率(定位) | GPT-4o: 2.32%, Claude: 4.77%, Llama: 4.45%, Qwen: 4.28% |
| 不一致率(推断) | GPT-4o: **18.29%**, Claude: 12.80%, Llama: 19.68%, Qwen: 22.50% |
| 去记忆化过滤率 | 37023→4682 (87.4% 被过滤) |

### 关键发现

- **局部理解 >> 全局理解**：所有模型在定位技能上超过 92%，但在选择技能上不超过 17.5%，差距高达约 76%。这与人类认知研究的发现一致——局部信息处理比全局综合容易
- **开源模型可以超越闭源模型**：Llama3.1-70B 在推断技能上超过 GPT-4o（46.54% vs 33.89%），Qwen2 在连接技能上超过 GPT-4o（52.81% vs 41.79%），说明参数量和训练数据量不是理解过程质量的唯一决定因素
- **理解过程正确性直接影响答案质量**：提供正确支持句后，推断准确率大幅提升（Qwen +18.22%），证明理解过程的正确性是答案质量的前提
- **高达 22.5% 的"虚假正确"**：Qwen2 在推断任务中，有 22.5% 的样本支持句错误但答案正确，说明模型大量使用捷径
- **去记忆化暴露了评估幻觉**：现有基准中 87.4% 的样本可以被 LLM 不看文档就答对，说明传统评估严重高估了模型的理解能力

## 亮点与洞察

- **"过程评估"视角的范式意义**：从评"答案"转向评"过程"，这个视角对所有 NLP 评估都有启发。SCoP 证明了答案正确不等于理解正确，这对 LLM 在高风险场景的部署有重要警示
- **认知科学与 NLP 评估的有机结合**：五种技能不是随意设计的，而是有认知科学理论支撑（Bloom 分类法、Kintsch 模型），使得框架具有理论合理性。技能间的相关性分析进一步验证了框架的内在一致性
- **90% 过滤率是最震撼的发现**：意味着当前几乎所有阅读理解基准都在很大程度上测试记忆而非理解，这对整个评估方法论是一个严重挑战

## 局限与展望

- **支持句标注的自动化质量**：定位和推断任务的支持句通过语义相似度自动标注，虽然 F1 达到 0.91-0.96，但仍有噪声
- **仅评估英语文档**：跨语言理解过程可能存在不同的认知模式
- **模型规模有限**：仅评估了 4 个 70B 级模型，缺少对更大规模模型（如 GPT-4-turbo、Claude-3-Opus）的评估
- **未提出改进方法**：框架揭示了问题但未提出如何训练模型改善理解过程的具体方案

## 相关工作与启发

- **vs SQuAD/HotpotQA 等答案评估**：这些基准只评答案对错，SCoP 同时评估支持句（理解过程）和答案，发现大量"答案对过程错"的情况
- **vs Sugawara et al. (2017) 的技能分析**：他们分析的是理解"之前"所需的语言技能（NER、共指等），SCoP 评估的是理解"过程中"的认知技能
- **vs MMLU/SuperGLUE 等综合评估**：这些基准混杂记忆和理解，SCoP 通过去记忆化过滤确保纯粹评估理解

## 评分
- 新颖性: ⭐⭐⭐⭐ 从认知过程角度评估理解是独特且有价值的视角，90% 过滤率的发现令人震撼
- 实验充分度: ⭐⭐⭐⭐ 跨 12 个数据集、4 个模型、5 种技能、多维度分析，但缺少更大规模模型
- 写作质量: ⭐⭐⭐⭐ 动机清晰、认知理论与 NLP 任务的对应关系阐述自然
- 价值: ⭐⭐⭐⭐ 对 LLM 评估方法论提出了根本性质疑，对高风险场景的 LLM 部署有重要警示
