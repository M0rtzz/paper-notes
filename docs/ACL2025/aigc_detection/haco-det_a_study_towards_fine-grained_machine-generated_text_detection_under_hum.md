# HACo-Det: A Study Towards Fine-Grained Machine-Generated Text Detection under Human-AI Coauthoring

**会议**: ACL 2025  
**arXiv**: [2506.02959](https://arxiv.org/abs/2506.02959)  
**代码**: 无  
**领域**: AIGC检测  
**关键词**: 细粒度检测, 人机共创文本, 词级别标注, 序列标注, 机器生成文本定位

## 一句话总结

提出人机共创场景下的细粒度MGT检测任务和数据集HACo-Det，通过多轮LLM改写构建了11,200篇带词级别标注的共创文本，将7种主流检测器适配到词级/句级任务后发现：微调方法（DeBERTa）远优于基于度量的方法，但整体检测水平仍远未解决该问题。

## 研究背景与动机

当前MGT检测研究主要聚焦于文档级别的二分类（全文判定为人写或机器生成）。然而，随着GPT-4o-canvas、Notion等人机协作写作工具的兴起，真实世界中越来越多的文本是人和AI**共同创作**的。这种混合文本无法简单地用二值标签描述——一篇文档中可能既有人写的段落也有AI改写的句子。

作者指出现有工作的三个盲区：
**标注不当**：一些数据集以人写开头+LLM续写的方式构建，但将全部输出标记为MGT，忽略了开头部分的人类贡献
**改写归属模糊**：一些工作将改写后的文本直接标记为机器生成，但改写中保留的原词实际上仍属于原作者
**交互简化**：现有工作仅考虑单轮协作，但真实的人机交互可能包含多轮修改

因此需要一种能在**词级别**准确识别文本归属的细粒度检测方法。

## 方法详解

### 整体框架

论文设计了一个完整的研究流水线：
1. 从四个领域收集人写原文
2. 用多轮LLM改写模拟人机共创过程
3. 基于词级别编辑对齐进行细粒度标注
4. 将检测器适配为序列标注任务（词级别→句级别投票聚合）

### 关键设计

1. **多轮改写的共创文本生成**: 不同于传统的"前缀+续写"模式，作者采用**多轮局部改写**策略。每一轮从文本中选择连续句子片段，指示LLM进行改写，然后用改写结果替换原文。重复多轮，每轮仅改写MGT片段以避免归属歧义。使用四种不同的指令调优模型（Llama-3、Mixtral、GPT-4o-mini、GPT-4o）分别处理四个领域，确保多样性。

2. **词级别标注机制**: 默认改写片段中的所有词标记为MGT。但如果改写后的某个词在改写前的对应片段中以相同形式出现（词形匹配），则保留其原始归属标签。这种基于词跨匹配的标注方法比简单的"整段为MGT"更精确合理。

3. **句级别聚合**: 对每个句子内的词级标签做多数投票，得到句级标签。这确保了词级和句级预测的一致性。

4. **任务形式化**: 将检测建模为序列标注任务——输入整篇文本，输出每个词的HWT/MGT标签。对于基于度量的方法（DetectGPT等），在词/token级别计算度量值并二值化；对于微调方法（DeBERTa），直接做序列分类。

### 数据集构建

- **原始文本来源**: XSum（新闻）、WritingPrompts（故事）、Dagpap24（科学论文）、Wikipedia_en（百科），共11,200篇
- **目标平均长度**: ~4,000词——足够长以支持多轮改写，且在该长度下不同检测器的性能差异可区分
- **预处理**: 过滤过短和低质量条目
- **MGT片段数**: 平均4.25个/篇，MGT占比平均36%

## 实验关键数据

### 主实验（域内检测）

| 检测器 | F1-W(词) | AUC-W | F1-S(句) | AUC-S |
|--------|----------|-------|----------|-------|
| Random | 0.433 | - | 0.497 | - |
| **DeBERTa** | **0.831** | - | **0.966** | - |
| SeqXGPT | 0.513 | - | 0.674 | - |
| Fast-DetectGPT | 0.501 | 0.507 | 0.533 | 0.510 |
| DetectGPT | 0.375 | 0.482 | 0.459 | 0.501 |
| NPR | 0.414 | 0.485 | 0.473 | 0.509 |
| LRR | 0.475 | 0.483 | 0.516 | 0.510 |

### 域外检测（OOD）

| 配置 | 关键指标 | 说明 |
|------|---------|------|
| DeBERTa OOD (News → others) | F1-W: 0.826, F1-S: 0.964 | 域外仍保持高水平，下降有限 |
| DeBERTa OOD (Story → others) | F1-W: 0.830, F1-S: 0.904 | 故事领域测试结果也不错 |
| DeBERTa OOD (Paper → others) | F1-W: 0.776, F1-S: 0.920 | 科学论文领域略有下降 |
| SeqXGPT OOD | F1-W下降至0.45~0.49 | 泛化能力严重不足 |
| 基于度量方法 OOD | F1-W约0.40~0.50 | 整体接近随机水平 |

### 不同生成器的影响

- GPT-4o的改写幅度最大（与原文相似度最低），理论上更容易检测
- Llama-3的改写最保守，检测更困难
- DeBERTa在不同生成器间具有较好的泛化性，但基于度量的方法变化较大

### 关键发现

- **微调方法远优于度量方法**：DeBERTa词级F1达0.831，句级达0.966；而度量方法平均F1仅0.462，接近随机
- **语义表征是关键**：DeBERTa基于预训练嵌入，能捕获词级语义特征；度量方法依赖统计量，在细粒度场景下失效
- **简单聚合不够**：度量方法在词级弱、句级更弱，说明简单投票无法弥补词级缺陷
- **域外泛化是挑战**：DeBERTa域外性能下降但仍可用，SeqXGPT和度量方法域外几乎失效
- **上下文窗口影响显著**：更大的上下文窗口有助于提升检测性能

## 亮点与洞察

- **任务定义有价值**：词级别MGT检测是一个重要且被低估的研究方向，对学术诚信、版权保护等有直接应用
- **数据构建流水线设计精细**：多轮改写+词匹配标注比简单拼接更接近真实人机协作场景
- **全面的基线评估**：将7种主流检测器适配到新任务，并在域内/域外两个维度评估，工作量巨大
- **问题发现有深度**：揭示了基于度量的方法在细粒度场景下的根本性不足，指明了研究方向
- **多角度分析**：考虑了不同生成器、不同领域、不同上下文窗口等多个影响因素

## 局限性 / 可改进方向

- 改写作为共创方式相对单一，真实人机协作可能涉及插入、删除、补充等更复杂操作
- 词级标注依赖词形匹配启发式规则，可能存在假阳性（相同词可能是AI独立生成的而非保留的）
- DeBERTa的512 token限制在处理长文本时需要截断，可能影响长文本性能
- 未考虑更先进的生成方式如RAG辅助写作、多模型协作等
- 数据集仅覆盖英语，多语言场景有待探索
- 缺乏对抗性场景的评估——如果用户故意混淆归属该如何应对

## 相关工作与启发

- 与Mixtext三分类（Gao等）、边界检测（Kushnareva等）、MGT定位（Zhang等）等相关工作互补
- 启发：文档级二分类检测已趋近饱和，社区应更关注细粒度检测
- 与RAID等鲁棒性基准结合，可以形成更全面的检测能力评估
- 序列标注范式为MGT检测提供了新视角：不再是"是否为AI"而是"哪些部分是AI"

- 实验充分度: ⭐⭐⭐⭐⭐ 7种检测器、4个领域、4种生成器、域内外评估，覆盖极为全面
