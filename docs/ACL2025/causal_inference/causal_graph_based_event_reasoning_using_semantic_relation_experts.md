# Causal Graph based Event Reasoning using Semantic Relation Experts

**会议**: ACL 2025  
**arXiv**: [2506.06910](https://arxiv.org/abs/2506.06910)  
**代码**: [github](https://github.com/StonyBrookNLP/causal-graphs)  
**领域**: 因果推理  
**关键词**: 因果图生成, 事件推理, 多智能体协作, 语义关系专家, 可解释预测

## 一句话总结

本文提出了一种基于多个语义关系专家（时间、篇章、前置条件、常识）协作讨论的因果事件图生成方法，用于显式建模事件间的因果连接，并在事件预测、事件预报等多个下游任务上取得了有竞争力的结果。

## 研究背景与动机

事件理解和推理在 NLP 中有着悠久的历史，从早期手工定义的 schema 到统计事件共现模型，再到事件语言模型和基于 LLM 的可控生成。虽然近年来事件推理能力有所提升，但大多数工作仍然主要依赖事件间的分布关系（即共现关系），而缺乏对事件间深层逻辑连接的建模。

因果关系识别本身就是一个困难的任务。即使是最先进的 LLM，在标准的 in-context learning 设置下，识别事件间的因果关系准确率仍然不高。这是因为因果判断需要考虑事件在更大全局语境中的嵌入方式。例如，地震本身很常见且不一定是灾难性的，但如果一个城市"面临资源短缺"且"遭受了损害"，这两个事件结合起来才"使得"当地官员请求援助。

作者认为，如果没有这些逻辑连接被显式地拼写出来，LLM 可能会错过这种微妙的推理。因此，他们提出使用因果事件图作为一种并行机制，帮助 LLM 在推理过程中显式表示因果关系。

## 方法详解

### 整体框架

本文的核心思想是使用 LLM 模拟多个专门关注不同语义关系的"专家"，通过多轮讨论协作生成因果事件图。整个流程包含三个阶段：

1. **专家初始化**：四个关系专家各自独立生成初始的因果关系判断
2. **多轮讨论**：专家之间进行多轮通信，每轮中每个专家获取其他所有专家的响应，分析后生成修订的因果链接列表及推理过程
3. **裁判整合**：一个因果裁判（Judge）LLM 汇总所有专家的讨论结果，最终确定因果链接列表

### 关键设计

**四类语义关系专家：**

1. **时间专家 (Temporal Expert)**：负责识别具有时间先后关系的事件对。时间关系是因果关系的必要条件（原因必须先于结果发生），因此时间专家可以缩小搜索空间。

2. **篇章专家 (Discourse Expert)**：关注共享实体的事件对。如果两个事件涉及同一实体，则它们可能存在因果关系——一个事件中对实体的操作可能导致该实体参与的后续事件。

3. **条件专家 (Conditional Expert)**：负责反事实推理。通过考虑替代场景，判断移除某事件是否会影响其他事件的发生，从而识别前置条件关系。

4. **常识专家 (Commonsense Expert)**：关注隐含的、未明确提及的知识。有些因果关系需要通过隐含信息来桥接，常识专家负责识别这些连接显式事件的隐含知识。

**协作机制的设计理由：**

直接提示 LLM 同时考虑所有方面在一次推理中完成是困难的。每个方面的推理本身可能存在错误，而这些错误最好通过检查论据和重新评估推断来解决。因此需要一种"关注点分离"的方法，让不同专家辩论、反思并修正推理。

**上下文的重要性：**

作者特别强调了上下文对处理传递性问题的重要性。在简单的 (verb, object) 事件表示中，缺乏连接上下文会导致不一致的传递性预测。而在本文的设置中，事件更加扎根于文本，因此有更丰富的连接上下文，减少了不一致关系的可能性。

### 损失函数 / 训练策略

本方法是零样本的，不需要在下游任务上微调。使用 GPT-4o 和 Llama-70B-instruct 作为基础 LLM，通过精心设计的 prompt 实现各专家的角色。讨论最多进行 3 轮或直到所有专家达成共识。

## 实验关键数据

### 主实验

**内在评估——因果图预测（CRAB 数据集）：**

使用 GPT-4o 作为基础 LLM 时：
- Direct 直接提示：BAcc 70.86, Macro F1 67.99（图级别）
- Pairwise 逐对提示：BAcc 73.93, Macro F1 72.68
- Experts wo collab（无协作）：BAcc 74.92, Macro F1 74.22
- **Collab with experts（完整方法）：BAcc 79.27, Macro F1 79.21**

使用 Llama 时完整方法同样最优：BAcc 73.69, Macro F1 72.49。

**外在评估——可解释事件似然预测 (EEL)：**

与 one-shot baseline 相比，CGEL 在因果性、信息量、连贯性三个维度上均以较大优势获胜：
- vs One-shot：CGEL 在信息量上赢 48.4%（baseline 仅 31.9%）
- vs One-shot w Reflexion：CGEL 在因果性上赢 39.3%

**事件预报 (ForecastQA)：**

CGEL 准确率 62.7%，显著超过 GPT-4 基线（51.3%）和 one-shot 基线（50.0%），接近微调模型的 63.1-67.4%。

**下一事件预测（Narrative Cloze）：**

CGEL with context 准确率 61.0%，优于所有事件语言模型基线（最好的 EGELM 仅 50.0%）。

### 消融实验

**专家角色消融（GPT-4o）：**

| 设置 | BAcc | Macro F1 |
|------|------|----------|
| Collab wo experts（无专家角色） | 75.39 | 75.51 |
| Collab with experts（完整） | 79.27 | 79.21 |
| 去掉时间专家 | 77.51 | 77.72 |
| 去掉篇章专家 | 78.32 | 78.29 |
| 去掉前置条件专家 | 77.48 | 77.26 |
| 去掉常识专家 | 78.88 | 78.85 |

所有专家均有贡献，去掉任一专家都会导致性能下降。

### 关键发现

1. **协作讨论的辩论轨迹分析**：时间专家初始最弱（13% 与 gold pair 重叠），但经过讨论后提升最大（33%），成为最准确的专家。前置条件专家虽然初始表现不错（17%），但由于有较多错误翻转（67% 的翻转是错误的），最终贡献下降。

2. **专家间的冲突模式**：前置条件和常识专家在已识别的因果对上冲突较少，但它们与时间专家的分歧最大。

3. 因果图方法不仅提升预测准确性，更重要的是**提供了可解释的因果事件链**作为预测依据，这是微调模型所不具备的。

## 亮点与洞察

1. **多智能体协作的巧妙设计**：通过让不同专家从时间、篇章、条件、常识四个互补维度讨论因果关系，实现了"关注点分离"的推理策略。这比直接让 LLM 一次性考虑所有因素更有效。

2. **零样本泛化能力**：该方法不需要在任何下游任务上微调，却能取得与微调模型相竞争的结果，证明了显式因果结构的通用价值。

3. **提出了新的可解释任务 EEL**：不仅要求预测事件似然，还要求提供因果事件链作为解释，这比传统的事件预测更具实用性和可解释性。

4. **辩论过程分析的透明性**：详细分析了专家在讨论中的行为模式，包括翻转、添加、冲突等，使得方法更加可解释和可调试。

## 局限性 / 可改进方向

1. 方法依赖 LLM（GPT-4 和 Llama）对因果关系的内在理解，可能与人类的因果感知存在差异。
2. 使用 GPT-4 作为评估器可能存在偏向 GPT-4 生成内容的偏差。
3. 没有涉及因果强度的等级化建模——人类可以提供因果关系的分级判断，建模这种分级判断可以提供更丰富的推理表示。
4. 计算成本较高，需要多个 LLM 调用进行多轮讨论。
5. 可以探索更多类型的语义关系专家，或将方法扩展到更多领域和语言。

## 相关工作与启发

本文与多智能体辩论框架（如 ChatEval、ReConcile）相似但有本质区别：每个智能体不是简单地复制同一任务，而是拥有不同的语义关系专长。这种"异构专家协作"的范式对其他需要多维度推理的 NLP 任务（如立场检测、论证挖掘）也有启发。

因果图作为事件推理的骨架结构，比简单的事件对预测更加全面。方法的可解释性使其特别适合需要透明推理链的应用场景，如新闻事件预报和情报分析。
