# One Missing Piece for Open-Source Reasoning Models: A Dataset to Mitigate Cold-Starting Short CoT LLMs in RL

**会议**: ACL 2025  
**arXiv**: [2506.02338](https://arxiv.org/abs/2506.02338)  
**代码**: 有  
**领域**: LLM推理  
**关键词**: 长思维链, 冷启动, RL训练, 推理数据集, 开源推理模型

## 一句话总结
构建 100K 长 CoT 数据集解决开源短 CoT LLM 在 RL 训练中的冷启动问题——用 1K 种子数据捕捉 o1 的推理流模式，然后用短 CoT LLM (GPT-4o) 扩展为长推理链，训练后的模型在 RLVR 初始化后获得 2-3 倍更大的收益。

## 研究背景与动机

1. **领域现状**：o1/R1 等模型展示了长 CoT 推理的强大能力，但开源 LLM 通常只有短 CoT 能力，直接做 RL 训练时存在冷启动问题。
2. **现有痛点**：(a) 直接在短 CoT 模型上做 RLVR 效果有限；(b) 蒸馏 o1/R1 的长推理输出受限于闭源 API。
3. **核心 idea 一句话**：用少量种子捕捉 o1 的推理流模式，然后用短 CoT LLM “扩写”为长推理链，满足 RL 训练的初始化需求。

## 方法详解

### 关键设计
1. **种子数据捕捉**：1K 个样本捕捉 o1 的推理流模式（回溯、自我纠错、多角度思考等）
2. **推理流生成**：为新问题先生成推理流框架，再用 GPT-4o 等短 CoT LLM 扩展为完整长推理链
3. **可控思维预算**：可调节生成的长度

## 实验关键数据
- 训练后模型用 RLVR 初始化时获得 **2-3× 更大的收益**
- 数据集质量与 R1 蒸馏数据相当
- 在 MATH500, GPQA, MMLU-Pro 上有显著提升

## 亮点与洞察
- **解决了开源推理模型的关键短板**：冷启动问题是 RL 训练的实际瘦颈，本文提供了低成本解决方案
- **种子数据+扩写的范式**可迁移到其他需要长推理的场景

## 局限性 / 可改进方向
- 依赖 GPT-4o 做扩写，成本不低
- 推理流模式的捕捉质量依赖种子数据选择
