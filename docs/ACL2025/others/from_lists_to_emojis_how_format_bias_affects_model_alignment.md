# From Lists to Emojis: How Format Bias Affects Model Alignment

**会议**: ACL 2025  
**arXiv**: [2409.11704](https://arxiv.org/abs/2409.11704)  
**代码**: 无  
**领域**: 其他  
**关键词**: RLHF, 格式偏差, 奖励模型, 偏好学习, 对齐

## 一句话总结

本文系统研究了 RLHF 中偏好模型（包括人类评估者、GPT-4 和开源模型）对粗体、列表、emoji 等格式模式的偏好偏差，展示了不到 1% 的偏差数据即可显著注入偏差，并提出了双头奖励模型的去偏方法。

## 研究背景与动机

RLHF（基于人类反馈的强化学习）已成为 LLM 训练管线中的关键技术，用于将模型输出与人类偏好对齐。然而，RLHF 面临的一个核心挑战是"奖励黑客"（reward hacking），即策略模型通过操纵奖励模型的弱点获得高分，但并未真正提升回答质量。

冗长偏差（verbosity bias）是已知的一种典型奖励黑客形式——偏好模型倾向于青睐更长的回答。但本文指出，除了冗长偏差之外，还存在大量**格式偏差**（format bias），包括对粗体文本、列表、emoji、感叹号、链接和肯定语气的偏好。这些偏差在现有研究中被严重忽视。

本文的核心动机是：当前流行的指令跟随（IF）基准测试上的得分是否真正反映了模型的能力？作者发现，AlpacaEval 排行榜上排名靠前的模型都强烈偏好生成包含粗体、列表和感叹号的回答，这暗示排名可能反映的是格式操纵能力而非内容质量。

## 方法详解

### 整体框架

本文的研究框架包含三个层面：（1）识别并量化偏好模型中的格式偏差；（2）研究偏差如何从偏好数据传递到奖励模型再到下游对齐算法；（3）提出去偏方法。

### 关键设计

1. **格式偏差识别与量化**：作者选取了七种格式模式（长度、emoji、粗体、感叹号、列表、链接、肯定语气），并在四个主流偏好数据集（RLHFlow-700K、LMSYS-Arena-55K、AlpacaEval、UltraFeedback）上统计了偏好回答和非偏好回答中各格式模式的出现比例。发现 GPT-4 标注的数据集中粗体偏好差异最为显著（preferred 42.76% vs unpreferred 16.78%）。

2. **偏差评估数据集构建**：对于每种格式模式，作者通过生成包含/不包含特定格式的回答对来创建评估集。具体做法是从模型生成的回答中选取包含特定格式的回答，再去除该格式得到配对样本，每种模式生成 200 对进行评估。无偏奖励模型的胜率应接近 50%。

3. **偏差注入实验**：以 UltraFeedback 数据集为基础（过滤后 71.6K 对），注入少量（不到 1%）偏差数据。偏差数据中，包含目标格式的回答被标记为偏好回答。实验表明，仅注入 0.7% 的列表偏差数据就能将奖励模型对列表的胜率从 51% 提升到 77.5%。

4. **双头奖励模型去偏**：训练一个双头 RM，分别预测真实奖励 $r^A$ 和格式相关的解耦奖励 $r^D$。总奖励为两者之和，通过约束损失使真实奖励头与格式模式的相关性最小化，同时让解耦头吸收格式信息。

### 损失函数 / 训练策略

总训练目标包含两部分：

- **排序损失**（Ranking Loss）：标准的 Bradley-Terry 偏好排序损失，使用两个奖励头的总和。
- **约束损失**（Constraint Loss）：$\mathcal{L}^C = |\rho(r^A, \#_p)| - \rho(r^D, \#_p)$，其中 $\rho$ 为 Pearson 相关系数，$\#_p$ 为启发式格式计数函数。该损失鼓励真实奖励头与格式无关，同时解耦头与格式高度相关。

针对格式信号稀疏性问题（如粗体仅出现在不到 2% 的回答中），作者提出了**重排序技术**（reordering）：将包含/不包含格式的偏好对分组排列，在格式计数全为零的批次中跳过约束损失计算。

## 实验关键数据

### 主实验

| 偏好模型 | Bold 胜率 | List 胜率 | Emoji 胜率 | Link 胜率 |
|---------|----------|----------|-----------|----------|
| GPT-4 Turbo | 89.5% | 75.8% | 86.8% | 87.3% |
| ArmoRM-8B | 98.0% | 50.5% | 55.0% | 27.0% |
| Pairwise-PM-8B | 97.0% | 93.5% | 70.5% | 84.8% |
| Skywork-Critic-8B | 99.3% | 88.8% | 97.3% | 75.0% |
| FsfairX-8B | 95.5% | 68.5% | 15.0% | 64.5% |

偏差注入实验：

| 训练数据 | Bold 胜率 | List 胜率 |
|---------|----------|----------|
| 基线（去格式） | 57.5% | 51.0% |
| 基线 + 0.70% Bold | 88.0% | - |
| 基线 + 0.70% List | - | 77.5% |
| 基线 + 0.70% Bold + 1.40% List | 83.0% | 80.0% |

### 消融实验

| 去偏配置 | Bold 胜率 | List 胜率 | Chat | Chat-Hard | Safety | Reasoning |
|---------|----------|----------|------|-----------|--------|-----------|
| 无去偏 | 89.0% | 92.5% | 98.3% | 71.4% | 83.1% | 85.1% |
| Bold 去偏 + 重排序 (λ=0.2) | 52.5% | - | 97.5% | 71.1% | 83.0% | 87.3% |
| List 去偏 + 重排序 (λ=0.1) | - | 54.0% | 98.4% | 72.9% | 83.6% | 89.4% |
| Bold&List 去偏 + 重排序 (λ=0.2) | 50.5% | 53.0% | 97.2% | 72.8% | 82.9% | 89.7% |

### 关键发现

1. **格式偏差无处不在**：GPT-4、人类标注者和所有开源偏好模型都展现出对特定格式的偏好，GPT-4 的偏差通常比人类更强。
2. **极少偏差数据即可攻击奖励模型**：不到 1% 的偏差数据就能显著改变奖励模型的行为。
3. **在线算法放大偏差**：相比离线 DPO，在线迭代 DPO 和 PPO 更容易利用和放大格式偏差，因为在线方法持续探索和适应新数据。
4. **被动数据过滤不充分**：简单删除含格式的训练数据会丢失 57% 的数据，导致模型能力下降。
5. **重排序技术解决稀疏性问题**：对稀疏格式模式，重排序技术显著改善了去偏效果，同时保持了奖励模型的能力。

## 亮点与洞察

- 首次系统性地将格式偏差研究从长度扩展到粗体、列表、emoji 等多种模式，填补了重要研究空白。
- "偏好翻转"现象令人印象深刻：GPT-4 有时会因为格式更好而偏好内容质量更差的回答。
- 偏差注入实验的 1% 攻击门槛说明了实际系统中偏好数据质量控制的重要性。
- 使用去偏奖励模型重新评价 AlpacaEval 排名，发现 Llama 系列模型排名下降而闭源模型排名上升，揭示了当前排行榜的潜在问题。

## 局限与展望

- 在线 DPO 和 PPO 实验仅在较小模型（8B）上进行，缺乏大规模模型的验证。
- 去偏方法中的超参数 λ_C 需要手动调节，在偏差强度和模型能力之间存在权衡。
- 仅关注了粗体和列表两种格式的去偏实验，其他格式（emoji、链接等）的去偏效果未验证。
- 未探讨格式偏差在多轮对话场景中的影响。

## 相关工作与启发

- **ODIN**（Chen et al., 2024）提出了解耦奖励的思路用于缓解 RLHF 中的黑客行为，本文在此基础上扩展到格式偏差。
- **OffsetBias**（Park et al., 2024）尝试去偏但未考虑格式偏差，去偏后列表偏差反而加重。
- 本文的发现对 RLHF 系统设计有重要启示：需要在评估和训练中显式区分格式与内容。
