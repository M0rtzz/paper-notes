# Building Better: Avoiding Pitfalls in Developing Language Resources when Data is Scarce

**会议**: ACL 2025  
**arXiv**: [2410.12691](https://arxiv.org/abs/2410.12691)  
**代码**: 无  
**领域**: 其他（NLP 伦理与数据实践）  
**关键词**: 低资源语言, 数据标注伦理, 参与式研究, 语言资源建设, 社区参与

## 一句话总结

通过对 81 位从事中低资源语言 NLP 研究的从业者进行问卷调查，系统性地揭示了低资源语言数据收集与标注中的质量问题和伦理隐患，特别是参与式研究框架中对社区贡献者的剥削问题，并提出了以语言社区为中心的改进建议。

## 研究背景与动机

NLP 研究近年来越来越关注以人为中心和社会意识导向的发展方向。语言不仅是符号系统，更承载着身份、文化和社会结构。因此，语言数据的收集和标注应当超越简单的"token 集合"视角。

然而，在中低资源语言的 NLP 研究中，存在一系列独特挑战：

**数据稀缺**：许多语言缺乏足够的数字化文本和标注数据

**合格标注员难觅**：在主流众包平台（AMT、Prolific）上几乎找不到特定低资源语言的母语标注员

**标准缺失**：英语等高资源语言已建立的数据质量和伦理标准未能延伸到低资源语言

**权力不对称**：资源充足的机构和知名研究者往往主导方向，忽视本地社区的需求

尽管已有一些关于 NLP 伦理实践的工作（如 Datasheets for Datasets、Model Cards），但很少有研究系统地调查**低资源语言从业者自身的经历和观点**。本文正是从社会科学方法出发，填补这一空白。

## 方法详解

### 整体框架

本文是一篇**立场论文（position paper）**，采用定量与定性相结合的调查研究方法：

1. **目标人群**：从事中低资源语言 NLP 项目的研究者和从业者
2. **触达渠道**：2024年6-10月间通过 X（Twitter）、LinkedIn、Google Groups、Slack 社区及直接邮件联系
3. **调查内容**：经历、实践、问题和激励机制

### 关键设计

**调查结构分为四个模块**：

1. **一般问题**：工作语言、参与项目、是否属于在线社区/参与式框架
2. **激励与局限性**：

    - 从事低资源语言研究的动机
    - 所在领域的工具和资源缺陷
    - 语言特定的技术挑战
3. **贡献认可**：

    - 是否获得经济补偿
    - 是否获得论文作者身份
    - 未获适当认可时的替代激励
    - 未被认可工作所花费的时间
4. **自由文本反馈**：允许受访者详细描述遇到的具体问题

**定义灵活性**：研究者允许受访者自行界定"低资源语言"的含义，承认该定义没有普遍标准。

### 数据收集与分析

- 共收到 81 份回复，覆盖 70+ 种语言
- 超过 90% 的受访者选择非匿名回复，80% 要求后续更新
- 采用定量统计和定性主题分析相结合的方法

## 实验关键数据

### 主实验（调查统计结果）

**受访者画像**：
- 57% 来自学术界，12% 来自工业界，31% 两者兼有
- 最常见任务：数据创建（47%）、数据标注（33%）、数据收集（33%）
- 主要动机：科学兴趣（81%）、构建语言技术（72%）、观察到的资源局限（60%）、LLM 研究（59%）

**报告的主要局限性**（可多选）：
| 局限性 | 占比 |
|---|---|
| 数据稀缺 | 78% |
| 数据缺乏代表性 | 58% |
| 现有工具性能不佳 | 54% |
| 工具与用户需求不匹配 | 54% |
| 标注质量低 | 25% |
| 数据缺乏实用性 | 18% |

**贡献认可问题**：
- 超过 67% 的受访者报告至少一次未被适当认可
- 在未获报酬的项目中，40% 的受访者花费了**一天到一个月以上**的时间
- 激励方式包括：社区成员身份、网站/论文致谢、被暗示有"职业益处"

### 消融实验（定性分析的三类问题）

**1. 现有资源的局限**：
- 许多资源不公开或规模太小
- 翻译数据导致"翻译腔"，不能捕捉目标语言的独特性
- 口语化/方言资源尤其匮乏（许多低资源语言本质上是口头语言）

**2. 建设新资源的实践问题**：
- 过度依赖机器翻译和 LLM 来构建资源
- 缺乏对文化和语言特殊性的理解
- 在标注平台上找不到母语者，转而依赖参与式框架/个人关系
- 参与式框架的滥用：社区贡献者被要求长期免费劳动

**3. 根本性问题**：
- 缺乏资金支持，常与"低声望语言"偏见挂钩
- 学术界和工业界同样存在此问题
- 某些语言被认为"不够重要"，研究经费申请困难

### 关键发现

1. **参与式研究框架的双刃剑**：虽然促进了包容性，但缺乏标准，导致初级研究者被剥削——被告知参与社区构建语言资源"很有声望"，但实际上只获得公司周边产品作为"报酬"
2. **工具为母语构建的动机强劲**：60% 的受访者更关心为自己的语言构建技术，而非扩展到尽可能多的语言——语言作为"符号资本"的力量
3. **数据质量 vs 数据量**：数据稀缺固然是首要问题，但数据**代表性**和**自然性**同样关键，频率几乎并列第二

## 亮点与洞察

- **从业者视角**：与大多数从技术角度讨论低资源 NLP 的论文不同，本文直接询问了"做这些工作的人"的经历和感受，提供了珍贵的第一手数据
- **揭露了参与式研究的阴暗面**：看似民主和包容的参与式框架，在缺乏标准的情况下可能变成新的剥削工具，这对 Masakhane 等社区具有警示意义
- **语言即身份**：受访者对自己语言的投入常常超越纯技术动机，反映了语言在社会权力结构中的地位
- **建议具有可操作性**：论文提出的建议围绕"以人为中心"展开，包括尊重标注者的劳动、选择合适数据源、设定公平预期、避免文化误传

## 局限性 / 可改进方向

- 样本量 81 份虽覆盖面广，但仍可能存在选择偏差（主要通过学术社区渠道招募）
- 由于隐私保护，未公开具体项目名称，可能限制了问题追踪和改进
- 未深入探讨不同地区/文化背景下问题的差异性
- 缺乏纵向追踪：一次性调查无法捕捉实践演变趋势
- 建议虽好，但缺乏具体的制度化或技术化落地方案

## 相关工作与启发

- **鸟类研究者的系列工作**（Bird 2020, 2022, 2024; Bird & Yibarbuk 2024）：倡导语言社区对自己语言的主权，强调许多低资源语言是口头+非制度化的
- **Masakhane 案例研究**（Birhane et al., 2022）：参与式框架的正面典范，但本文揭示了其潜在风险
- **Doğruöz & Sitaram (2022)**：强调不能将低资源语言 NLP 视为高资源语言技术的缩小版
- **Blaschke et al. (2024)**：关注方言使用者的需求，与本文关注的语言变体问题相呼应
- 对 NLP 社区的启发：在追求"多语言覆盖"的同时，需要认真对待每种语言背后的人和文化
