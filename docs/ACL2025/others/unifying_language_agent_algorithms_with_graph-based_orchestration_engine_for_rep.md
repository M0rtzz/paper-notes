# Unifying Language Agent Algorithms with Graph-based Orchestration Engine for Reproducible Agent Research

**会议**: ACL 2025  
**arXiv**: [2505.24354](https://arxiv.org/abs/2505.24354)  
**代码**: 有 (https://github.com/om-ai-lab/OmAgent)  
**领域**: NLP / Agent  
**关键词**: 语言代理, 图编排引擎, 推理算法, 基准评测, 模块化框架

## 一句话总结

提出 AGORA 框架，通过图编排引擎统一实现 10 种主流 Agent 推理算法，并建立标准化评测体系，系统比较不同算法在多种 LLM 上的表现，发现简单的 CoT 方法在成本效益上往往优于复杂算法。

## 研究背景与动机

语言代理（Language Agent）在理解、推理和执行复杂任务方面展现出巨大潜力，但实际开发面临三大挑战：（1）大量定制化工程开销；（2）缺乏标准化组件；（3）评测框架不足导致难以公平比较不同方法。现有框架如 LangChain、AutoGPT 等提供了通用基础设施，但缺少对不同推理范式的系统性比较和统一实现。

此外，已有的 Agent Leaderboard 主要评估 LLM 的工具调用和 API 交互能力，未能同时考察底层 LLM 能力和推理算法的有效性。AGORA 的目标是填补这一空白，让研究者在统一环境中实验各种推理策略，同时让开发者能低成本构建鲁棒的代理应用。

## 方法详解

### 整体框架

AGORA 构建于 OmAgent 框架之上，包含三个核心组件：（1）基于图的工作流编排引擎；（2）模块化代理算法支持；（3）多种客户端接口用于评测和交互。

### 关键设计

1. **图编排引擎 (Graph-based Workflow Orchestration)**：采用有向无环图（DAG）建模任务，每个节点是简单任务或逻辑任务（分支、循环等控制流）。基于 Conductor 库实现，支持可视化追踪调试，支持异步分布式执行，适合管理长期运行的复杂工作流。

2. **模块化算法 Operator 设计**：每种推理算法被实现为独立的 Operator 模块，包含清晰的输入输出接口，可复用 LLM 推理、工具调用、记忆访问等公共函数。框架共实现了 10 种算法：CoT、SC-CoT、ToT、ReAct、PoT、DnC、GoT、RAP、V*、ZoomEye。

3. **ReAct-Pro 增强**：受 Reflexion 启发，将原来合并的 Think 和 Action 步骤拆分为两次独立模型调用，并添加"You can take as many steps as needed"提示语，仅此一句就使 AQuA 上的准确率从 34.25% 跃升至 64.57%，提升近 90%。

4. **客户端接口**：提供 WebPageClient（实时交互）、ProgrammaticClient（批量自动评测）、DefaultClient（命令行调试）三种界面，可通过配置文件灵活切换。

### 训练策略

框架不涉及模型训练，核心在于实验设置的标准化：默认温度为 0，SC-CoT 温度为 1 且路径数为 5，ToT 使用 BFS 搜索（b=1, max_depth=6）。

## 实验关键数据

### 主实验 — 数学推理

| 模型 | 代理算法 | GSM8K | MATH-500 | AQuA | 平均 |
|------|---------|-------|----------|------|------|
| GPT-4o | CoT | 高 | 高 | 高 | 最优 |
| Qwen2.5-72B | CoT | 接近GPT-4o | - | - | 竞争力强 |
| deepseek-r1-1.5B | CoT | - | - | - | 超过InternLM-7B |
| GPT-3.5 | ReAct | - | - | 34.25% | 基线 |
| GPT-3.5 | ReAct-Pro | - | - | 64.57% | +90% |

### 主实验 — 多模态推理 (MME-RealWorld)

| Agent | VLM | Score | Pass Rate |
|-------|-----|-------|-----------|
| ZoomEye | Qwen2.5-VL-72B | 51.56 | 99.81% |
| ZoomEye | Qwen2.5-VL-7B | 48.06 | 96.50% |
| IO | Qwen2.5-VL-72B | 44.47 | 100% |
| IO | Qwen2.5-VL-7B | 42.86 | 100% |
| V* | seal_vqa & vsm | 15.14 | 72.37% |

### 关键发现

1. **简单算法表现稳健**：CoT 在大多数场景下准确率最高且 token 消耗最低，复杂算法如 ToT、PoT 反而因引入多步推理或代码生成依赖而表现更差。
2. **提示设计至关重要**：仅添加一句"可以使用任意多步骤"就能带来 90% 的性能提升。
3. **开源模型可匹敌商业模型**：70B 级别开源模型（Llama-3.3-70B、Qwen2.5-72B）可超越 GPT-4o；deepseek-r1-1.5B 仅 1.5B 参数即可超越 InternLM-7B。
4. **小模型更适合简单算法**：PoT 等依赖代码生成的方法在小模型上效果有限。
5. **多模态代理有效提升小模型**：ZoomEye 使 7B VLM 甚至超过 72B 直接推理。

## 亮点与洞察

- 统一框架下的系统性比较是本文最大价值，打破了各种 Agent 算法之间"各自为政"的评测困境。
- "简单即有效"的结论具有很强现实指导意义：在设计 Agent 系统时应优先使用 CoT，仅在必要时引入复杂性。
- ReAct-Pro 的案例生动展示了 prompt engineering 的巨大杠杆效应。

## 局限性 / 可改进方向

- 评测仅覆盖数学推理和多模态问答，缺少工具使用、网页交互等更复杂的真实世界任务场景。
- 缺乏对推理时间和延迟的系统比较。
- GoT、RAP、DnC 因 token 消耗过高而被排除在成本分析之外，可能低估了某些复杂方法的潜在优势。
- 未探索自适应选择推理策略的方法。

## 相关工作与启发

- 与 AgentBench、WebArena 等评测基准互补，AGORA 更侧重于推理算法本身的比较。
- Chinchilla scaling law 的思路可借鉴用于代理系统的计算资源优化。
- 未来可结合自适应策略选择（根据任务特征动态选取最优推理算法）进一步提升效率。

## 补充细节

### 成本效益分析

论文的 Score vs Cost 分析显示最优成本效益点集中在 CoT 配置下，复杂方法如 ToT 的 token 消耗增加 5-10 倍但准确率提升不超过 1-2%。这对工业部署意义重大——不是越复杂的推理算法越好。

### V* 模型的局限

V* 在 MME-RealWorld 上 pass rate 仅 72.37%，因其依赖特定训练的视觉搜索模型（seal_vqa/vsm），在高分辨率图像上搜索时间极长须限制搜索步数（10步），导致目标定位失败率高。说明专用模型的泛化性不足。
