# Unlocking Speech Instruction Data Potential with Query Rewriting

**会议**: ACL 2025  
**arXiv**: [2507.08603](https://arxiv.org/abs/2507.08603)  
**代码**: 无  
**领域**: NLP / 语音  
**关键词**: 语音指令数据, 查询重写, TTS, 多Agent标注, 知识融合

## 一句话总结

提出基于多 LLM 知识融合的查询重写框架，通过零样本重写将文本指令转换为更适合 TTS 模型合成的分布，结合多 Agent 标注与验证机制，将语音指令数据可用率从 72% 提升至 93%。

## 研究背景与动机

端到端大型语音语言模型（LSLM）在响应延迟和语音理解方面展现出巨大潜力，但由于缺乏高质量语音指令数据集且训练任务严重偏倚，其遵循语音指令的能力未被充分释放。传统对齐方法让模型重复或识别语音中的语言信息，导致模型缺乏意图感知能力。

已有工作通过 LLM 续写语音的语言内容来构造数据集，但 LLM 生成结果与真实人类回答存在差距，续写方法反而放大了这一缺陷。人工采集标注成本极高，使用语音合成构建数据集成为折中方案。然而 TTS 模型词汇有限，无法正确转换缩写、数学公式等分布外文本，导致语言信息丢失。

## 方法详解

### 整体框架

框架分三部分：多 Agent 标注验证、多 LLM 查询重写、知识融合。

### 关键设计

1. **多 Agent 标注与验证**：使用 3 个 ASR 模型（whisper-large-v3、canary-1b、parakeet-tdt-1.1b）+ 3 个 Embedding 模型模拟多标注者。对每条合成语音取多个识别结果，在嵌入空间计算与原文的平均余弦相似度，取最优值作为质量分数，避免单一 ASR 的一致性错误。

2. **查询重写框架**：用 Llama-3-8B、Phi-3-small、Qwen2-7B 三个 LLM 分别重写原始文本指令，生成候选文本集 {原文, llama重写, phi重写, qwen重写}。每个候选分别通过 TTS 合成语音，再经多 Agent 评估选择最优。同时用 GPT-4 生成 192 种说话人风格描述增加多样性。

3. **知识融合**：将成功重写的样本对 <原文, 重写文本> 作为训练数据，用 LoRA 微调 Llama-3-8B，融合多个 LLM 的重写知识。微调后的模型用于处理之前所有 LLM 都重写失败的困难样本。

### 训练策略

知识融合训练使用 LoRA（r=8, α=16），学习率 3e-4，余弦调度器。LSLM 微调用 LoRA（r=8, α=32），学习率 3e-5。4 张 A40 GPU 训练，梯度累积 64 样本。

## 实验关键数据

### 主实验 — TTS 模型合成质量（ParlerTTS-Large，Multi-Speaker Setting）

| 方法 | DROP | NarrativeQA | SQUAD1.1 | TAT-QA | 平均 SIM | 平均 PASS |
|------|------|------------|----------|--------|---------|---------|
| Original | 93.71 | 95.87 | 93.40 | 82.24 | 93.14 | 72.19 |
| TN (文本规范化) | 95.95 | 97.75 | 94.88 | 89.07 | 95.34 | 82.05 |
| Ours w/o KF | 98.02 | 98.14 | 97.31 | 97.21 | 97.85 | 93.15 |
| **Ours** | **98.11** | **98.25** | **97.52** | **97.32** | **98.06** | **93.62** |

### 消融实验 — 各 LLM 单独重写

| 方法 | 平均 SIM | 平均 PASS |
|------|---------|---------|
| Llama3 alone | 96.94 | 88.36 |
| Qwen2 alone | 95.84 | 83.72 |
| Phi3 alone | 97.05 | 88.48 |
| Multi-LLM (Ours) | **98.06** | **93.62** |

### 关键发现

1. 多 LLM 重写 + 最优选择的策略效果显著优于任何单一 LLM 重写。
2. 知识融合额外带来约 0.5% 的可用率提升，主要贡献在于解决困难重写样本。
3. 该方法跨 3 种 TTS 模型和 2 种设置（单说话人/多说话人）一致有效。
4. 用真实人类标注答案（Golden）训练的 LSLM 效果显著优于 LLM 生成答案（Continue），强调了学习真实回答的重要性。

## 亮点与洞察

- 巧妙利用多 LLM 的知识正交性：不同 LLM 在不同类型的重写任务上各有优势，取最优后组合效果远超单模型。
- 多 Agent 标注验证机制模拟了人类多标注者的流程，有效避免了单一 ASR 模型的系统性偏差。
- 嵌入空间相似度比 WER 更鲁棒，与人类评估更一致。

## 局限性 / 可改进方向

- 目前仅在英文 QA 数据集上验证，未覆盖多语言场景。
- TTS 风格描述由 GPT-4 生成，可能存在风格覆盖不完整的问题。
- 知识融合只在 failed 样本上额外提升，增益有限。
- 未探索更先进的 TTS 模型带来的影响。

## 相关工作与启发

- 与 Qwen2-Audio 等端到端语音模型的训练数据构建密切相关。
- 查询重写思路可推广到其他模态转换场景（如文本→手语视频）。

## 补充细节

### 对齐目标对 LSLM 微调的影响

论文比较了两种对齐目标下的 LSLM 训练效果。用高质量人工标注答案（Golden）训练的模型在所有 QA 任务上显著优于用 Llama-3 续写答案（Continue）训练的模型，差距约 5-15% 准确率。这表明学习真实人类回答对遵循语音指令能力至关重要。

### TTS 模型差异

三种 TTS 模型中 MeloTTS 基线可用率最高（85%），但 ParlerTTS-Large 受益于风格可控性最终提升空间最大。MMS-TTS 因词汇覆盖最差，原始可用率仅 63%，经重写后提升至 88%，改善最为显著。

### ASR 的句子模式错误

论文展示了 ASR 识别中的句子模式错误：原文为疑问句但 ASR 识别结果缺少标点导致语气变为陈述或蔑视。这类错误在评估语音指令质量时容易导致不恰当的数据拒绝。
