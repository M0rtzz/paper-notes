# Voting or Consensus? Decision-Making in Multi-Agent Debate

**会议**: ACL 2025  
**arXiv**: [2502.19130](https://arxiv.org/abs/2502.19130)  
**代码**: 有 (https://github.com/lkaesberg/decision-protocols)  
**领域**: NLP / 多智能体系统  
**关键词**: 多Agent辩论, 决策协议, 投票, 共识, 答案多样性

## 一句话总结

系统评估了 7 种决策协议（4 种投票 + 3 种共识）对多 Agent LLM 辩论的影响，发现投票在推理任务上优于共识 13.2%，共识在知识任务上优于投票 2.8%，并提出 AAD 和 CI 两种提升答案多样性的新方法。

## 研究背景与动机

多 Agent 辩论系统的成功很大程度上取决于参数选择，其中决策协议（如何从多个Agent的答案中产生最终决定）是最关键的选择之一。然而，已有研究通常将决策策略作为固定变量，同时改变多个讨论参数，使得系统比较几乎不可能。

已有工作的不足：Exchange-of-Thought 仅用共识但未对比其他方案；Yang et al. 引入多种投票方式但未跨任务比较；ReConcile 混合了投票和共识但改变了多个变量。缺少单一变量控制下的系统性比较。

## 方法详解

### 整体框架

多 Agent 辩论框架包含三个组件：讨论范式（定义沟通规则）、决策协议（定义终止和选择机制）、响应生成器（定义 Agent 回复风格）。每次讨论 3 个自动生成的专家 Persona。

### 关键设计

1. **7 种决策协议**：
   - 共识类（3 种）：多数共识（>50% 同意）、超多数共识（>66%）、一致共识（100%）
   - 投票类（4 种）：简单投票（每人一票）、排名投票（排序后汇总）、认可投票（无限票）、累积投票（25 分分配）

2. **All-Agents Drafting (AAD)**：强制每个 Agent 在第一轮独立生成解决方案（而非一个 Agent 提议后其他人改进），促进独立思维和答案多样性。

3. **Collective Improvement (CI)**：AAD 的扩展版——不仅第一轮独立生成，之后各轮也禁止直接沟通，仅展示上一轮各 Agent 的解决方案供参考。适用于投票协议（无需迭代达成共识）。

4. **响应生成器变体**：简单（中立回复）、批判（鼓励挑战前人答案）、推理（仅分享推理过程不给最终答案）。

### 实验设置

Llama 3 8B 和 70B，3 个 Agent（自动生成 Persona），至多 5 轮讨论。记忆窗口限制为最近 2 轮。6 个数据集：知识（MMLU, MMLU-Pro, GPQA）+ 推理（SQuAD 2.0, StrategyQA, MuSR）。

## 实验关键数据

### 主实验 — 投票 vs 共识（Llama 3 8B）

| 任务类型 | 共识优势 | 投票优势 |
|---------|---------|---------|
| MMLU（知识）| +2.3% | - |
| MMLU-Pro（知识）| +4.9% | - |
| GPQA（知识）| +1.3% | - |
| SQuAD 2.0（推理）| - | +13.1% |
| StrategyQA（推理）| - | +0.2% |
| MuSR（推理）| - | +26.4% |

### AAD 和 CI 效果（StrategyQA）

| 策略 | 答案余弦相似度 | 平均准确率 | vs 基线 |
|------|-------------|----------|--------|
| 基线 | 0.888 | 58.3% | 0.0% |
| AAD | 0.870 | 62.8% | **+3.3%** (更差) |
| **CI** | **0.845** | **65.7%** | **+7.4%** |
| 批判响应 | 0.843 | 59.4% | +1.1% |
| 推理响应 | 0.916 | 51.9% | -6.4% |

### Agent 数量 vs 讨论轮次

| 扩展维度 | 趋势 |
|---------|------|
| 增加 Agent 数量（1→10）| 准确率线性上升 |
| 增加讨论轮次（1→10）| 准确率线性下降 |

### 关键发现

1. **投票适合推理、共识适合知识**：投票允许探索多条推理路径（类似 self-consistency），共识通过多次复核减少事实错误。
2. **增加 Agent 数量优于增加讨论轮次**：更多 Agent 类似于增加采样数量，而更多轮次导致问题漂移和过度共识。
3. **答案多样性与性能正相关**：CI 产生最低的答案相似度和最高准确率；推理响应生成器反而降低多样性和性能。
4. **认可投票常失败**：Agent 倾向于投票赞成所有选项，导致 59% 的讨论无法达成决定。
5. **挑战最终答案无效果**：给 Agent 提供对最终答案的质疑机会并不能改善结果，特别在推理任务上甚至有负面影响。
6. **共识更快（1.42 轮 vs 投票 3.38 轮）**：共识的计算效率更高。

## 亮点与洞察

- 核心贡献在于**单一变量控制**的实验设计——所有实验仅改变决策协议，其他参数不变，因此结论高度可信。
- AAD 和 CI 的设计理念可概括为"独立思考→减少群体思维"，与人类决策研究中的经典发现一致。
- 投票在推理任务上的 26.4%（MuSR）巨大优势说明复杂推理需要路径探索而非路径精炼。
- 讨论轮次增加反而降低性能的发现挑战了"更多讨论=更好结果"的直觉。

## 局限性 / 可改进方向

- 计算成本高昂：共识约 5 倍于 CoT 基线，投票约 10 倍。性能提升需与计算成本权衡。
- 使用数据集子集+3 次重复，可能存在方差。
- 仅用 Llama 3（8B/70B），未测试闭源模型或更多架构。
- Persona 生成的影响未做系统消融。
- Agent 的随和性（agreeableness）是普遍问题，CI 虽缓解但未根治。

## 相关工作与启发

- 与 self-consistency 的联系：投票的优势本质上是利用了多样化推理路径的选择。
- 与人类决策理论的联系：共识在需要准确性的任务中更有效（类似于同行评审），投票在需要创造性的任务中更有效。
- 可探索分层决策结构（如"经理-员工"层级）对结果的影响。
