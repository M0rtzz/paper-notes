# Generating Pedagogically Meaningful Visuals for Math Word Problems: A New Benchmark and Analysis of Text-to-Image Models

**会议**: ACL 2025 (Findings)  
**arXiv**: [2506.03735](https://arxiv.org/abs/2506.03735)  
**代码**: 无  
**领域**: 图像生成 / 教育AI  
**关键词**: 数学应用题, 教育可视化, 文本到图像生成, 视觉语言, 教学设计

## 一句话总结

Math2Visual 提出了一个从数学应用题（MWP）文本描述自动生成教学可视化图像的框架，定义了基于教师访谈的视觉语言和设计空间，构建了 1,903 张标注数据集，并评估和微调了多个 TTI 模型，揭示了当前模型在数学关系表示上的关键不足。

## 研究背景与动机

**领域现状**：可视化教具在小学数学教育中被广泛使用，帮助学生将文字描述转化为数学表达式。例如"小明有3个苹果，又买了2个"这类应用题，配上直观的图片可以大幅提升低年级学生的理解。然而，目前这些教学图片几乎完全依赖教师手动制作，非常耗时费力。

**现有痛点**：虽然文本到图像（TTI）模型（如 DALL-E、Stable Diffusion）在通用图像生成上取得了巨大进步，但它们无法直接用于教学场景——生成的图像通常只是"好看的插图"，缺乏对数学关系的准确表达。例如，一道加法题需要图片明确展示"3个"和"2个"的分组关系，但通用 TTI 模型可能会生成5个随机排列的苹果，丢失了分组和运算的教学意义。

**核心矛盾**：教学可视化不仅需要视觉美观，更核心的是要准确传达数学关系（数量、运算、比较等）。现有 TTI 模型的训练数据和评估标准都不涉及这种"教学语义正确性"，导致生成结果在教育场景下不可用。

**本文目标**：(1) 定义一套教学可视化的设计规范（视觉语言 + 设计空间）；(2) 构建标注数据集用于评估和训练；(3) 评估现有 TTI 模型在教学可视化上的能力并分析不足。

**切入角度**：作者通过与多位数学教师的深度访谈，提炼出教学可视化的核心设计原则和常用视觉元素，将领域知识形式化为可计算的设计规范。

**核心 idea**：为数学教学可视化定义结构化的视觉语言（包含对象表示、数量编码、运算关系等元素），基于此构建标注数据集和自动化生成框架 Math2Visual。

## 方法详解

### 整体框架

Math2Visual 包含三个层次：(1) 基于教师访谈定义的视觉语言（Visual Language），描述教学图片中应包含哪些视觉元素及其组合规则；(2) 设计空间（Design Space），规定不同数学运算类型对应的可视化模板；(3) 自动化生成管线，将 MWP 文本解析为结构化表示，映射到设计空间中的模板，再转化为 TTI 模型的 prompt 或直接渲染。

### 关键设计

1. **视觉语言定义（Visual Language）**:

    - 功能：提供教学可视化元素的形式化描述
    - 核心思路：通过对数学教师的半结构化访谈，提炼出教学可视化中的核心组件——包括对象类型（具体物品如苹果、抽象形状如圆圈）、数量编码方式（逐个展示、分组展示、数字标注）、空间排列（行排列、列排列、分区排列）、运算关系表示（分组线、箭头、加号等辅助符号）。这些元素被组织成一个层次化的视觉词汇表。
    - 设计动机：没有形式化的视觉语言，就无法系统评估生成图片的"教学正确性"。教师的经验知识被转化为可操作的设计规则，使得评估可以从主观判断变为结构化的多维度检查。

2. **设计空间与模板系统（Design Space）**:

    - 功能：将不同数学运算类型映射到对应的可视化模板
    - 核心思路：根据 MWP 中涉及的数学运算类型（加法、减法、乘法、除法、比较），定义不同的可视化布局模板。例如，加法题使用"两组+合并箭头"的布局，比较题使用"对齐排列+差异高亮"的布局。每个模板包含必需元素（如正确数量、分组关系）和可选元素（如颜色编码、标注文字）。解析 MWP 文本得到运算类型和数值后，自动选择对应模板并填充参数。
    - 设计动机：不同的数学概念需要不同的视觉表达策略。一个通用的"画张数学图"的 prompt 远不够，需要针对运算类型的定制化设计才能传达正确的教学意图。

3. **数据集构建与标注（Dataset Construction）**:

    - 功能：为评估和微调提供高质量标注数据
    - 核心思路：从现有 MWP 数据集中选取覆盖不同运算类型的题目，使用 Math2Visual 框架为每道题生成标准可视化图像，共构建 1,903 张经人工验证的教学图片。每张图片附带结构化标注，记录包含的视觉元素（数量是否正确、运算关系是否体现、布局是否符合模板）。这些标注既用于自动评估，也用于人工评估的参考标准。
    - 设计动机：缺乏标注数据是教育AI研究的共性问题。本数据集填补了数学教学可视化评估的空白。

### 损失函数 / 训练策略

对 Stable Diffusion 等 TTI 模型进行微调时，使用标准的去噪扩散训练目标，但在数据集上使用 Math2Visual 生成的教学图片作为目标图像，prompt 为对应 MWP 的结构化描述。微调旨在让模型学会教学可视化的特定视觉模式（如准确计数、分组排列等）。

## 实验关键数据

### 主实验

评估多个 TTI 模型在教学可视化生成上的表现，使用结构化指标（数量准确率、运算关系正确率、布局合规率）和人工评学评分：

| 模型 | 数量准确率 | 运算关系正确率 | 教学有效性(人评) | 说明 |
|------|-----------|--------------|-----------------|------|
| DALL-E 3 | ~35% | ~25% | 低 | 最强通用模型但教学语义差 |
| Stable Diffusion XL | ~20% | ~15% | 很低 | 严重的计数和关系错误 |
| SD XL (微调后) | ~55% | ~45% | 中等 | 微调显著提升 |
| Math2Visual (模板渲染) | ~95% | ~90% | 高 | 模板方法最准确但缺乏灵活性 |

### 消融实验

| 配置 | 数量准确率 | 说明 |
|------|-----------|------|
| 结构化 prompt（完整） | 最高 | 包含所有视觉语言元素的描述 |
| 简化 prompt（仅题目文本） | 显著下降 | 缺少设计规范指导 |
| 微调 + 结构化 prompt | 最高（TTI类） | 两者结合效果最佳 |
| 微调 + 简化 prompt | 中等 | 微调有帮助但结构化描述更重要 |
| 不同运算类型对比 | - | 加法最易、除法最难 |

### 关键发现

- **现有 TTI 模型在教学可视化上严重不足**：即使是最强的 DALL-E 3，数量准确率也不到 40%，说明"数学关系的准确视觉表达"是当前 TTI 模型的盲区。
- **最常见的错误是数量错误和关系遗漏**：模型倾向于生成大致正确的场景但忽略精确计数，以及省略运算关系的视觉标记（如分组线、箭头）。
- **微调能显著改善但不能完全解决问题**：微调后模型学会了一些教学可视化的模式，但在复杂运算（如多步题、除法题）上仍然困难，表明这需要更深层的结构化推理能力。
- **结构化 prompt 比自然语言 prompt 效果好得多**：使用视觉语言元素组成的结构化 prompt 比直接用题目文本做 prompt 效果提升显著，说明"如何描述需要什么样的图"比"给模型看题目让它自己画"更重要。

## 亮点与洞察

- **将教师领域知识形式化为视觉语言**这一研究思路非常有价值——跨领域的形式化知识表示可以迁移到其他专业领域的自动可视化生成（如化学实验步骤图、物理受力分析图）。
- **评估框架的设计**比生成方法本身更有长期价值——定义了"数学教学可视化质量"的多维度结构化评估标准，为后续研究提供了统一对比基础。
- **数据集虽小（1,903）但标注质量高**，且每张图片都有结构化元素标注，支持细粒度的error analysis，这是一个可扩展的标注范式。

## 局限与展望

- **数据集规模偏小**：1,903 张图片对于微调大型 TTI 模型来说可能不够，效果天花板较低。
- **仅覆盖基础四则运算**：对于更复杂的数学概念（分数、面积、概率等），视觉语言和设计空间需要大幅扩展。
- **教学有效性的评估不够充分**：缺少在真实课堂环境中测试生成图片对学生学习效果的影响，目前只有专家评分。
- **模板渲染 vs 生成模型的 gap 很大**：说明目前依靠扩散模型自动生成教学图片还不够成熟，可能需要"模板+生成"的混合方案。

## 相关工作与启发

- **vs 通用 TTI (DALL-E / SD)**: 通用模型追求视觉质量但不关注语义精确性（如计数），本文揭示了这一根本性gap。
- **vs 数学可视化工具 (GeoGebra 等)**: 传统数学软件需要手动操作，Math2Visual 自动化程度更高但灵活性受限于模板库。
- **vs MWP 求解研究**: MWP 研究主要关注从文本到等式的解析，本文关注从文本到图片的生成，是一个被忽视但很重要的方向。

## 评分

- 新颖性: ⭐⭐⭐⭐ 教学可视化生成是新颖的交叉领域问题，形式化设计空间的方法有创意
- 实验充分度: ⭐⭐⭐ 多模型对比和微调实验完整，但数据集偏小，缺少真实教学场景验证
- 写作质量: ⭐⭐⭐⭐ 问题动机清晰，视觉语言定义过程描述详细，图表丰富
- 价值: ⭐⭐⭐⭐ 开创了教育AI中数学可视化生成的评估基准，对教育科技领域有实际应用潜力
