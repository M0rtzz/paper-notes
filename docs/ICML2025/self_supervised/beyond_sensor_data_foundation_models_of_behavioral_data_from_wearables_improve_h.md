# Beyond Sensor Data: Foundation Models of Behavioral Data from Wearables Improve Health Predictions

**会议**: ICML 2025  
**arXiv**: [2507.00191](https://arxiv.org/abs/2507.00191)  
**代码**: 无（Apple 内部研究）  
**领域**: 自监督学习  
**关键词**: 可穿戴设备, 健康预测, 基础模型, 行为数据, 不规则时序

## 一句话总结
在 162K 个体的 25 亿小时可穿戴行为数据上训练基础模型（WBM），系统优化不规则采样行为数据的 tokenizer 和架构，在 57 项健康检测任务上展示强大表现，特别在行为驱动任务（如睡眠预测）上表现突出。

## 研究背景与动机

1. **领域现状**：可穿戴设备基础模型主要聚焦低层传感器数据（PPG/ECG/加速度计），忽视了更高层的行为数据（活动量、睡眠指标、步态特征等）。
2. **现有痛点**：行为数据更对应人类行为时间尺度（天/周），比秒级传感器数据更适合健康状态检测，但面临不规则采样、缺失值、特征间采样率不同等挑战。
3. **核心矛盾**：大量无标签行为数据 vs 少量有标签健康任务——天然适合基础模型范式，但行为数据的特殊性需要定制化设计。
4. **本文要解决什么**：为可穿戴行为数据构建首个专用基础模型。
5. **切入角度**：系统探索 tokenizer（如何处理不规则采样）和架构（Transformer vs 其他）的设计选择。
6. **核心 idea**：行为数据 ≠ 传感器数据，需要专门的基础模型设计。

## 方法详解

### 整体框架
1. 数据：Apple Heart and Movement Study，162K 参与者，15B+ 小时级测量
2. 设计：系统探索 tokenizer + 架构组合
3. 预训练：自监督（掩码预测）
4. 评估：57 项健康检测任务上的线性探针/微调

### 关键设计

1. **行为数据 Tokenizer**:
   - 做什么：将不规则采样的多变量行为时序转化为 token 序列
   - 核心思路：测试了 patch embedding、per-feature tokenization、时间桶聚合等策略
   - 设计动机：不规则采样使标准方法（等间隔分 patch）失效

2. **架构选择**:
   - 做什么：对比 Transformer、Mamba、时间卷积等架构在行为数据上的表现
   - 发现：Transformer + 合适的位置编码（处理不规则时间间隔）效果最佳

### 损失函数 / 训练策略
- 掩码自监督预训练
- 57 项下游任务的零样本/少样本评估

## 实验关键数据

### 主实验

| 方法 | 57 任务平均 AUC | 睡眠预测 | 怀孕检测 |
|------|---------------|---------|---------|
| 手工特征 + XGBoost | 0.72 | 0.78 | 0.71 |
| PPG 基础模型 | 0.74 | 0.69 | 0.68 |
| **WBM** | **0.79** | **0.86** | **0.81** |
| WBM + PPG 组合 | **0.82** | **0.88** | **0.83** |

### 关键发现
- WBM 在行为驱动任务（睡眠、活动）上大幅领先 PPG 模型
- WBM 与 PPG 模型互补——组合后进一步提升
- 不规则采样的专门处理对性能至关重要

## 亮点与洞察
- **行为数据 vs 传感器数据**的区分有重要意义——不同时间尺度的信息互补
- 162K 人、25 亿小时的数据规模令人印象深刻
- "为数据定制模型"而非"让数据适配模型"的思路值得推广

## 局限性 / 可改进方向
- Apple 专有数据，可复现性受限
- 隐私敏感数据的基础模型共享困难
- 仅在 Apple Watch 数据上验证
