# Position: Uncertainty Quantification Needs Reassessment for Large-Language Model Agents

**会议**: ICML2025
**arXiv**: [2505.22655](https://arxiv.org/abs/2505.22655)
**代码**: 无
**领域**: llm_agent
**关键词**: 不确定性量化, 认知不确定性, 偶然不确定性, LLM Agent, 人机交互

## 一句话总结
这篇 position paper 挑战了传统"偶然 vs 认知"不确定性二分法在 LLM Agent 场景中的适用性，指出两类不确定性的定义本身相互矛盾，并提出三个面向 Agent 交互的新研究方向：欠规范不确定性、交互式学习、以及丰富的输出不确定性表达。

## 研究背景与动机

### LLM 幻觉不可完全消除
最新研究已经证明，LLM 的生成特性决定了幻觉无法被彻底根除。
因此不确定性量化成为提升 LLM 可信度的关键手段。

### 传统不确定性二分法的问题
经典机器学习将不确定性分为：
- **认知不确定性（epistemic）**：可通过更多数据/训练减少的
- **偶然不确定性（aleatoric）**：数据本身的固有噪声，不可减少

但论文通过文献综述发现，不同学派对这两个概念的定义**相互矛盾**：

| 学派 | 对认知不确定性的定义 |
|------|-------------------|
| 模型计数派 | 有多少模型与数据一致 |
| 分歧派 | 不同可能模型对输出分歧有多大 |
| 密度派 | 远离训练数据分布 = 高认知不确定性 |
| 残差派 | 总不确定性减去偶然不确定性 |

论文用一个简单的伯努利分类例子证明：不同定义可以对同一情况得出"最大"和"接近最小"认知不确定性的完全相反结论。

### 为什么在 LLM Agent 场景中更成问题
在多轮对话中：
- 最初看似偶然的不确定性（如欠规范问题）可以通过追问变成可减少的
- 最初看似认知的不确定性（如缺乏知识）在检索后仍无法消除时变成偶然的
- "可减少"与"不可减少"的边界是动态的

## 方法详解（三个新研究方向）

### 方向 1：欠规范不确定性（Underspecification Uncertainty）
在人与 LLM Agent 的交互中，用户初始提问往往缺乏足够信息。
这不是简单的"输入噪声"，而是**任务本身尚未完全定义**。
传统 UQ 框架假设任务是明确的，因此无法处理这种情况。

### 方向 2：交互式学习（Interactive Learning）
Agent 可以主动提问来消解欠规范不确定性：
- 检测到信息不足时进行澄清提问
- 使用 RAG 检索补充知识
- 逐步收窄不确定性范围

关键洞察：Agent 的"动作空间"包含了"提问"这个选项，这是传统 UQ 没有的。

### 方向 3：输出不确定性表达（Output Uncertainty）
LLM Agent 可以用比数字更丰富的方式表达不确定性：
- 解释为什么不确定
- 列出多个可能选项及其权衡
- 说明什么信息可以减少不确定性

这需要超越"输出一个概率值"的传统范式。

## 实验关键数据

### 本文为纯理论/立场论文，无实验部分

论文的"实验"体现在：

| 分析类型 | 核心发现 |
|----------|---------|
| 伯努利分类反例 | 不同 UQ 定义对同一情况结论相反 |
| 文献矛盾梳理 | 7 个主流定义间存在系统性冲突 |
| LLM Agent 场景分析 | 传统二分法在多轮交互中失去意义 |

### 反面论述（Counter-Position）

| 场景 | 传统 UQ 是否仍有用 |
|------|-------------------|
| 结构化分类任务 | ✓ 有用 |
| 单次预测（如 calibration） | ✓ 有用 |
| 多轮对话中的开放式问答 | ✗ 不适用 |
| Agent 的工具/API 调用决策 | ✗ 不适用 |

## 亮点与洞察

1. 勇于挑战经典。
   不确定性的"认知 vs 偶然"二分法已被使用数十年，
   本文通过具体反例和文献矛盾有力地动摇了其普适性。

2. 对 LLM Agent 场景的适配分析非常精准。
   在交互式、多轮、开放式的 Agent 场景中，
   不确定性的边界确实是动态的，传统工具力不从心。

3. 三个新方向都有很强的实际指导意义。
   特别是"交互式学习"和"丰富的不确定性表达"已经在工程实践中有对应需求。

4. Counter-position 的自我审视体现了学术严谨。
   明确划定传统 UQ 仍有效的场景，不做过度否定。

5. 本文的观点对 Agent 系统设计有直接影响：
   Agent 应该内建"何时该追问"的决策模块。

## 局限性 / 可改进方向

1. 纯 position paper，缺乏量化实验的支撑。
2. 三个新方向只是概念性描述，缺少具体方法论。
3. 对"欠规范不确定性"的形式化还不够——如何精确定义"任务尚未完全明确"？
4. 没有讨论计算成本：交互式消解不确定性需要更多推理轮次。
5. 对 Agent 安全场景中不确定性的角色讨论不足。

## 相关工作与启发

- 与传统 UQ（calibration、Bayesian DL）的关系：本文不否定它们，但指出其适用边界。
- 与 Reflexion/Self-Refine 等 Agent 反思方法的关系：它们隐含地处理了某些不确定性但未显式化。
- 启发后续工作：
  1. 开发面向 Agent 的原生 UQ 方法（非数字，而是结构化表达）。
  2. 将"何时追问"建模为强化学习问题。
  3. 在 benchmark 中加入欠规范任务来评估 Agent 的元认知能力。
