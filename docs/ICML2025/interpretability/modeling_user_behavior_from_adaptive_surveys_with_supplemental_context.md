# LANTERN: Modeling User Behavior from Adaptive Surveys with Supplemental Context

**会议**: ICML2025  
**arXiv**: [2507.20919](https://arxiv.org/abs/2507.20919)  
**领域**: others  
**关键词**: 用户行为建模, 调查数据, 后期融合, 交叉注意力, 多模态

## 一句话总结
提出LANTERN——模块化的用户行为建模架构，通过选择性门控+残差连接+后期融合交叉注意力将调查数据作为主信号、外部上下文作为补充，在多标签调查响应预测中超越纯调查基线。

## 研究背景与动机

### 调查数据的固有限制
调查疲劳/不完整回答/长度限制使单独调查数据不足以完整捕捉用户行为。

### 补充数据的噪声
人口统计/参与度/交易记录等被动收集的信号量大但噪声高、可能有偏。

### LANTERN的设计哲学
调查数据是主信号（结构化/高保真），外部信号是补充——只有在相关时才融入。

## 方法详解

### 后期融合(Late Fusion)
调查嵌入作为Query查询外部嵌入——只学习行为相关的交互而非所有可能的交互。

### 选择性门控+残差
保持调查信号的主导地位——如果外部信息不相关，模型可以忽略它。

### 模块化设计
调查编码器和补充编码器可独立替换——支持新数据源的灵活集成。
只有约50M参数——轻量可部署。

## 实验关键数据

### 多标签预测

| 方法 | 调查响应预测 F1 |
|------|-------------|
| 纯调查模型 | 基线 |
| 简单拼接 | 轻微提升 |
| **LANTERN** | **显著提升** |

### 调查主导性验证

| 配置 | 性能 |
|------|------|
| 仅调查 | 基线 |
| 仅外部 | 远低于基线 |
| **调查+外部(LANTERN)** | **最高** |

### 稀有/频繁属性分析

| 属性类型 | LANTERN改善 |
|---------|-----------|
| 频繁属性 | 中等 |
| **稀有属性** | **更大** |

### 关键发现
1. 后期融合比早期融合更适合调查-补充数据的融合
2. 选择性门控有效防止了噪声补充数据的干扰
3. 对稀有属性的改善更大——补充数据弥补了调查覆盖不足
4. 50M参数使模型可直接部署
5. 调查主导性验证确认了设计哲学的正确性

## 亮点与洞察

1. "调查主导+补充辅助"的设计哲学在工业界非常实用。
2. 选择性门控+后期融合+残差的三层保护确保了调查信号不被覆盖。
3. 模块化设计支持新数据源的快速集成。
4. 对稀有属性的改善体现了补充数据的真正价值。
5. 50M轻量级可立即部署到生产环境。

## 局限性 / 可改进方向

1. 仅在单一调查数据集上验证。
2. 补充数据的特征工程需要领域知识。
3. 多标签预测的类别不平衡处理未深入讨论。
4. 与LLM-based用户建模的对比缺失。
5. 隐私方面：调查+补充数据的融合可能引发隐私关切。

## 相关工作与启发

- 与DMT/ZEUS的区别：LANTERN的调查主导+选择性融合vs简单拼接。
- 与MultiTab方法的关系：结构化表格数据的特殊处理。
- 启发：主信号+补充信号的融合范式可推广到其他多源数据场景。

## 补充

### 调查主导性的重要性
纯外部信号模型远低于纯调查模型，证实了调查数据作为主信号的必要性。

### 50M参数的部署价值
内存占用低，推理快，可直接部署到生产系统。调查编码器和补充编码器都是轻量级。

### 稀有属性的改善机制
稀有属性在调查中样本少，补充信号的被动特征可以弥补。后期融合确保只从相关补充信息中学习。
