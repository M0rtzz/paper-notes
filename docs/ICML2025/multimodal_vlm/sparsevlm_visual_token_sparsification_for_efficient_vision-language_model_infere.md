# SparseVLM: Visual Token Sparsification for Efficient VLM Inference

**会议**: ICML2025  
**arXiv**: [2410.04417](https://arxiv.org/abs/2410.04417)  
**代码**: [GitHub - SparseVLM](https://github.com/Gumpest/SparseVLMs)  
**领域**: multimodal_vlm  
**关键词**: VLM加速, 视觉token稀疏化, 无训练, 文本引导, token回收

## 一句话总结
提出SparseVLM——无训练的文本引导视觉token优化：用自注意力矩阵中文本token对视觉token的评分来确定重要性，自适应确定每层稀疏化比例，并用token回收压缩被剪token为紧凑表示。

## 研究背景与动机

### VLM推理效率瓶颈
视觉token数量多但信息稀疏——大量冗余计算。

### 已有方法需要额外训练
大多数视觉token剪枝方法需要训练额外的网络来判断重要性。

### SparseVLM的创新
完全无训练——利用模型自身的注意力分布来判断token重要性。

## 方法详解

### 文本引导评分
视觉token补充文本推理→选择相关文本token→用文本-视觉自注意力权重评估视觉token重要性。

### 自适应层级稀疏化
基于排名的策略：每层根据重要性分布自动确定稀疏化比例，而非固定比例。

### Token回收
不是简单丢弃低重要性token，而是将它们压缩成更紧凑的表示保留信息。

## 实验关键数据

### 图像理解任务

| 方法 | 准确率保持 | 加速比 | 需要训练 |
|------|----------|--------|---------|
| FastV | 中等 | 中等 | 是 |
| **SparseVLM** | **更高** | **更高** | **否** |

### 视频理解任务

| 方法 | 效果 | 说明 |
|------|------|------|
| 均匀采帧 | 基线 | 损失时序信息 |
| **SparseVLM** | **更好** | 跨帧自适应稀疏 |

### 关键发现
1. 文本引导比纯视觉评分更准确（文本知道"在找什么"）
2. 自适应比例优于固定比例（不同层的重要性分布不同）
3. Token回收有效保留了被剪token的信息
4. 在图像和视频任务上都有效

## 亮点与洞察

1. 无训练=即插即用到任何VLM。
2. 文本引导评分直觉优雅——"文本告诉你哪些视觉信息重要"。
3. Token回收比简单丢弃更合理。
4. 在多种VLM架构上一致有效。
5. 对长视频理解有直接加速价值。

## 局限性 / 可改进方向
1. 自注意力矩阵的提取有额外计算开销。
2. 对极高稀疏率(>90%)的效果可能下降。
3. 与KV缓存压缩的结合未探讨。

## 补充分析

### 与FastV的核心差异
FastV训练专门的重要性预测器，SparseVLM利用模型已有的注意力——后者零额外训练且不引入新的分布偏差。

### 文本引导的数学直觉
如果文本query“图中有什么动物”，注意力中“动物”相关token高度关注的视觉token就是重要的，其余可稀疏化。

### 对视频VLM的特殊价值
视频包含大量帧→视觉token数爆炸。SparseVLM的自适应稀疏化可跨帧智能分配保留预算。

### Rank-based策略的优势
vs固定比例：不同层的token重要性分布不同。Rank-based根据实际分布自动调整，而非一刀切。

### 文本引导的数学直觉
如果文本query"这张图里有什么动物"，注意力矩阵中"动物"相关文本token高度关注的视觉token就是重要的，其余可以稀疏化。
