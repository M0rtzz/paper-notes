# Context is Key: A Benchmark for Forecasting with Essential Textual Information

**会议**: ICML 2025  
**arXiv**: [2410.18959](https://arxiv.org/abs/2410.18959)  
**代码**: https://servicenow.github.io/context-is-key-forecasting/v0/  
**领域**: 时间序列  
**关键词**: 多模态预测, 文本上下文, 时间序列基准, LLM预测, 基础模型

## 一句话总结
提出 CiK 基准——将数值时间序列与精心设计的文本上下文配对，要求模型必须理解文本才能成功预测，评估统计模型、时序基础模型和 LLM 预测器，发现简单的 LLM 提示方法在需要上下文信息的任务上表现最优。

## 研究背景与动机

1. **领域现状**：时间序列预测传统上仅依赖历史数值数据，但人类预测者经常依赖额外信息（背景知识、约束条件等），可通过自然语言高效传达。
2. **现有痛点**：尽管 LLM 预测器兴起，其整合文本信息的能力仍是开放问题——现有基准不要求理解文本上下文，无法评估这一能力。
3. **核心矛盾**：预测模型越来越强大，但缺乏需要多模态理解的标准化评估。
4. **本文要解决什么**：创建一个必须结合文本上下文才能解决的预测基准。
5. **切入角度**：设计多种类型的文本上下文——背景知识、约束、因果事件、领域规则等——每种都是正确预测的必要条件。
6. **核心 idea**：如Example——光伏发电量的预测需要知道"这是太阳能板"才能推断夜间为零。

## 方法详解

### 整体框架
CiK 基准包含多类任务，每类配有不同类型的文本上下文：
1. 背景知识型：如"太阳能发电在夜间为零"
2. 约束型：如"销售上限为 X"
3. 因果事件型：如"假期导致交通模式变化"
4. 统计信息型：如"历史峰值出现在 X 时"

### 关键设计

1. **上下文类型分类法**:
   - 做什么：系统化地定义文本上下文的种类
   - 设计动机：确保基准覆盖实际场景中的各种信息类型

2. **LLM 提示预测方法**:
   - 做什么：直接将数值序列和文本上下文作为 prompt 输入 LLM
   - 核心思路：将数值格式化为文本表格 + 上下文描述 → LLM 直接输出预测值
   - 发现：GPT-4o 在有上下文时显著优于纯数值方法

### 损失函数 / 训练策略
- 基准设计，无训练，评估现有模型

## 实验关键数据

### 主实验

| 方法 | 无上下文 | 有上下文 | 差异 |
|------|---------|---------|------|
| ARIMA | 0.45 | 0.45 | 0% (不能用上下文) |
| TimesFM | 0.38 | 0.38 | 0% (不能用上下文) |
| GPT-4o (数值only) | 0.42 | - | - |
| **GPT-4o (数值+上下文)** | - | **0.28** | **-33%** |

### 关键发现
- 没有上下文时 LLM 预测不比统计方法好
- 有上下文时 LLM 显著胜出——文本理解是关键差异化能力
- 时序基础模型（TimesFM等）无法利用文本上下文，这是重大局限

## 亮点与洞察
- **"上下文是关键"**——直接点名了时序基础模型的盲区
- 光伏发电的例子极其直观：不知道是太阳能板，就无法预测夜间为零
- 为多模态时序预测提供了首个系统化基准

## 局限性 / 可改进方向
- 上下文目前手工设计，规模有限
- LLM 的数值理解能力仍有局限
- 未测试端到端训练的多模态模型
