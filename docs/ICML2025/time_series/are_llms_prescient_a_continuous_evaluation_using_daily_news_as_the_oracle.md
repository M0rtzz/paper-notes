# Are LLMs Prescient? A Continuous Evaluation using Daily News as the Oracle

**会议**: ICML 2025  
**arXiv**: [2411.08324](https://arxiv.org/abs/2411.08324)  
**代码**: https://agenticlearning.ai/daily-oracle  
**领域**: 时间序列  
**关键词**: LLM评估, 持续评估, 新闻预测, 时间泛化, 知识过时

## 一句话总结
提出 Daily Oracle 基准——从每日新闻自动生成 QA 对来持续评估 LLM 的时间泛化和预测能力，揭示 LLM 性能随预训练数据过时而持续退化（True/False 题平均下降 21.55%），且 RAG 也无法完全挽救。

## 研究背景与动机

**领域现状**：现有 LLM 基准是静态的，易过时且有数据泄露风险，无法追踪模型能力随时间的变化。
**现有痛点**：静态基准缺乏时间维度——无法回答"LLM 的预测能力随知识截止日期过后如何变化？"
**核心矛盾**：世界在变，但基准不变。
**本文要解决什么**：设计始终保持相关性的持续评估方法。
**切入角度**：未来事件预测——每天从新闻生成预测性 QA，永远不会"过时"。
**核心 idea**：用新闻预测作为 LLM 时间泛化的探针，持续追踪性能退化曲线。

## 方法详解

### 整体框架
1. 每日自动抓取新闻文章
2. 用 LLM 生成 True/False 和 Multiple Choice 预测性问题
3. 第二天用实际新闻结果作为答案（oracle）
4. 在时间轴上连续评估各 LLM

### 关键设计

1. **自动 QA 生成流水线**:

    - 做什么：从新闻自动生成预测性问题-答案对
    - 核心思路：提取关键事件→生成预测性问题→用后续新闻验证答案
    - 设计动机：手工标注无法持续，自动化是持续评估的前提

2. **时间退化分析**:

    - 做什么：按时间轴追踪 LLM 预测准确率
    - 核心思路：2020.1-2024.12 的月度准确率曲线，识别知识截止日期前后的变化
    - 设计动机：量化知识过时的影响

### 损失函数 / 训练策略
- 纯评估方法，无训练

## 实验关键数据

### 主实验

| 指标 | 2020.1 (知识内) | 2024.12 (知识外) | 退化幅度 |
|------|---------------|-----------------|---------|
| True/False | ~75% | ~53% | -21.55% |
| Multiple Choice | ~62% | ~51% | -11.33% |

### 消融实验
| 配置 | 效果 | 说明 |
|------|------|------|
| 闭卷（无 RAG） | 退化严重 | 完全依赖预训练知识 |
| 开卷（RAG + 近期新闻） | 退化缓慢但仍存在 | RAG 帮助但无法完全补偿 |
| 不同模型（GPT-4/Claude） | 退化模式相似 | 知识过时是通用问题 |

### 关键发现
- 退化是平滑且持续的——不是在知识截止日期突然断崖式下降
- 即使给了最新新闻（RAG），模型仍然表现退化——推理能力本身似乎不足
- 不同类别的退化速度不同（政治 > 科技 > 文化）

## 亮点与洞察
- **"永不过时"的基准**构造思路值得推广到其他领域
- 退化的平滑性出人意料——暗示世界知识在模型中是渐进过时的
- RAG 不是万能药——即使有最新信息，模型仍需理解时间上下文

## 局限性 / 可改进方向
- QA 生成质量依赖 LLM，可能引入偏差
- 新闻领域特定，其他领域的时间泛化待探索
- 未区分"知识过时"和"推理能力不足"
