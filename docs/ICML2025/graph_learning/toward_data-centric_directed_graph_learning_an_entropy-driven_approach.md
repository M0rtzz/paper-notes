# Toward Data-centric Directed Graph Learning: An Entropy-driven Approach

**会议**: ICML2025  
**arXiv**: [2505.00983](https://arxiv.org/abs/2505.00983)  
**代码**: 待确认  
**领域**: graph_learning  
**关键词**: 有向图学习, 数据中心ML, 知识蒸馏, 信息熵, 层级编码

## 一句话总结

提出 EDEN（熵驱动有向图知识蒸馏），从数据中心视角利用层级知识树和互信息量化揭示有向图中拓扑与节点属性的潜在关联，作为即插即用模块增强任意 DiGNN 性能。

## 研究背景与动机

- **有向图**比无向图更强表达但现有 DiGNN 未充分利用数据知识
- **数据级局限**导致模型级次优：未挖掘有向边与节点特征/标签的关联
- **数据中心 ML**的趋势：改善数据表示比仅改进模型更根本

## 方法详解

### 层级知识树（HKT）构建

- 利用有向结构度量（入度/出度/核心度等）构建粗粒度树
- 节点按拓扑相似性聚类

### 互信息精细化

- 量化节点特征/标签的互信息
- 精细化 HKT 中的知识流，实现数据中心知识蒸馏

### 作为即插即用模块

- 可与任意 (Di)GNN 结合，额外参数极少
- 自然扩展到无向场景（退化为对称拓扑度量）
- 训练时作为辅助监督，推理时无额外开销

### 层级编码理论

- 提出结构化数据的层级编码理论，指导数据中心 ML
- 不同粒度的编码对应不同信息量
- 信息熵指导最优层级选择

### 总损失设计

$$\mathcal{L} = \mathcal{L}_{task} + \lambda \mathcal{L}_{KD}$$

其中 $\mathcal{L}_{KD}$ 基于 HKT 知识流与模型预测的 KL 散度

## 实验关键数据

### 14 个数据集，4 种任务

| 任务类型 | EDEN 改善幅度 |
|---|---|
| 节点分类（同配图） | +1-3% |
| 节点分类（异配图） | +2-5% |
| 链接预测 | +1-4% |
| 图分类 | +2-3% |

### 接入不同 DiGNN 的增益

| 基线 | + EDEN |
|---|---|
| DiGCN | 显著提升 |
| MagNet | 显著提升 |
| 标准 GCN | 同样有效 |

### 消融

- 移除 HKT → 退化为普通 KD，效果降低
- 移除互信息精细化 → 知识流不精确
- 不同拓扑度量选择对结果影响较小（鲁棒）

## 亮点与洞察

1. **数据中心视角**：从改善数据表示入手提升模型，比仅优化模型更根本
2. 熵驱动的知识流量化有理论支撑
3. 模块化设计适用广泛，对有向和无向图均有效
4. 无需修改原始模型架构

## 局限性 / 可改进方向

- HKT 构建依赖预定义拓扑度量的选择
- 大规模图的互信息计算开销可能较大
- 层级编码理论的更严格数学形式化尚待完善
