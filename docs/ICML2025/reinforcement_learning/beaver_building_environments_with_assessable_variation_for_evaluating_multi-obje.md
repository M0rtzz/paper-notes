# BEAVER: Building Environments with Assessable Variation for Evaluating Multi-Objective Reinforcement Learning

**会议**: ICML 2025  
**arXiv**: [2507.07769](https://arxiv.org/abs/2507.07769)  
**代码**: https://github.com/chennnnnyize/BEAVER  
**领域**: 强化学习  
**关键词**: 多目标RL, 建筑能源管理, 泛化评估, 上下文MDP, 基准环境

## 一句话总结
提出 BEAVER 基准——一个支持多目标和泛化评估的建筑能源管理 RL 环境，将问题形式化为多目标上下文 MDP（MOC-MDP），评估现有 MORL 方法在热对流差异和气候变化下的泛化能力。

## 研究背景与动机

**领域现状**：基于 RL 的建筑能源管理取得进展，但可扩展性和跨环境泛化仍是开放问题。
**现有痛点**：现有 MORL 研究假设单一静态环境，未考虑建筑动力学变化和多目标（舒适度 vs 能耗 vs 碳排放）的交互。
**核心矛盾**：不同建筑的热动力学差异大→在一栋楼训练的策略难以迁移到另一栋。
**本文要解决什么**：标准化评估 MORL 在建筑控制中的泛化能力。
**切入角度**：将建筑 RL 环境参数化——热对流系数和气候区域作为上下文变量，系统化构建变化的环境集。
**核心 idea**：MOC-MDP 框架 + 自动化环境构建 + 标准化评估。

## 方法详解

### 整体框架
1. 基于物理原理的建筑 RL 环境（EnergyPlus）
2. 参数化上下文：热对流动力学 + 气候区域
3. 多目标奖励：舒适度 + 能源成本
4. MORL 算法基准测试

### 关键设计

1. **MOC-MDP 形式化**:

    - 做什么：将多目标+跨环境泛化统一建模
    - 核心思路：上下文 $c$（热对流参数+气候区域）影响状态转移函数 $P_c$，偏好 $\omega$ 影响奖励组合
    - 设计动机：分离"环境变化"和"目标偏好"两个维度

2. **参数化环境构建**:

    - 做什么：自动生成不同热动力学和气候条件的建筑环境
    - 核心思路：调整对流系数→改变热惯性→影响 HVAC 控制难度
    - 设计动机：使泛化评估可控且可复现

### 损失函数 / 训练策略
- 评估现有 MORL 方法（GPI-PD、MORL/D 等）
- 多指标评估：Pareto 前沿覆盖、偏好适应性

## 实验关键数据

### 主实验
| MORL 方法 | 同环境 Pareto 覆盖 | 跨环境泛化 | 退化程度 |
|----------|------------------|-----------|---------|
| GPI-PD | 0.82 | 0.61 | -26% |
| MORL/D | 0.79 | 0.55 | -30% |
| 线性加权 | 0.71 | 0.58 | -18% |

### 关键发现
- 所有 MORL 方法在跨环境泛化时性能显著退化
- 上下文信息（热对流参数）对策略迁移至关重要
- 气候变化比热对流变化对泛化的影响更大

## 亮点与洞察
- 填补了建筑 MORL 基准的空白
- MOC-MDP 形式化优雅且通用

## 局限性 / 可改进方向
- 仅单区域建筑，复杂建筑待扩展
- MORL 方法列表可更全面
