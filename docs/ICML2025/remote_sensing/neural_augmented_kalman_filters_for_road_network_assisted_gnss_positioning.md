# Neural Augmented Kalman Filters for Road Network Assisted GNSS Positioning

**会议**: ICML2025  
**arXiv**: [2507.00654](https://arxiv.org/abs/2507.00654)  
**代码**: 待确认  
**领域**: remote_sensing  
**关键词**: GNSS定位, 卡尔曼滤波, 道路网络, 时序图神经网络, 城市定位

## 一句话总结
首个用深度学习将道路网络信息集成到GNSS卡尔曼滤波中的方法——训练时序图神经网络(TGNN)预测正确道路段及其不确定性，作为KF的量测更新，在城市场景中定位误差降低29%。

## 研究背景与动机

### 城市GNSS定位的困境
城市峡谷中多径和非视线误差导致定位误差可达20米。即使用3D模型辅助也难以低于10米。车道级定位（2-10米）需求与现实能力之间存在鸿沟。

### 道路网络的免费资源
OpenStreetMap等开源平台提供全球道路网络数据——这是一个低成本的辅助定位信息源。但道路观测是多模态高斯的，直接集成到KF中不straightforward。

### 已有方法的局限
- 基于信念理论的方法灵活性有限
- HMM+Viterbi仅考虑历史不看未来
- 都不能端到端学习最优道路选择策略

## 方法详解

### 整体架构
1. GNSS量测 → 标准KF预测步
2. TGNN接收KF状态+道路图 → 预测最佳道路段+不确定性
3. 道路段观测 → KF量测更新步

### TGNN设计
- 输入：以当前GNSS位置为中心的道路子图
- 图结构：节点=道路段，边=道路连接
- 时序：LSTM处理轨迹历史
- 输出：每个道路段的"正确概率" + 标准差预测头

### 标准差预测头
- 预测道路观测的不确定性
- 端到端优化使KF更新的幅度自适应
- 高不确定性时道路信息权重降低→KF更依赖GNSS

## 实验关键数据

### 定位误差对比（真实世界数据）

| 方法 | 中位误差(m) | P95误差(m) | 改善 |
|------|-----------|-----------|------|
| GNSS-only KF | 12.3 | 28.5 | 基线 |
| KF + Viterbi | 10.5 | 24.1 | 14.6% |
| **KF + TGNN (本文)** | **8.7** | **20.2** | **29%** |

### 消融实验

| 配置 | 中位误差(m) |
|------|-----------|
| KF + TGNN (无标准差头) | 9.8 |
| KF + TGNN (无LSTM) | 10.1 |
| KF + TGNN (完整) | **8.7** |

### 关键发现
1. 端到端学习的道路选择显著优于启发式方法
2. 标准差预测头对挑战性场景（十字路口/平行道路）尤其关键
3. LSTM的时序信息帮助在歧义区域做正确决策
4. 在"简单"场景中TGNN和Viterbi效果相当，在"困难"场景中差距拉大
5. 使用开源道路网络即可运行，无需额外传感器

## 亮点与洞察

1. 首个深度学习+道路网络+GNSS KF的端到端方法。
2. 标准差预测头优雅地解决了"何时信任道路信息"的难题。
3. 使用开源数据(OSM)使方法可复现且全球适用。
4. KF框架保持了实时性和物理可解释性。
5. 对自动驾驶/配送等应用有直接实用价值。

## 局限性 / 可改进方向

1. 依赖道路网络的准确性和完整性。
2. 仅在汽车场景验证，行人/自行车场景未测试。
3. 室内或多层道路（高架桥）的处理未讨论。
4. TGNN的推理延迟对实时性的影响未量化。
5. 3D道路信息（高程）的利用可进一步提升。

## 相关工作与启发

- 与Neural KF(Revach et al.)的区别：本文增加道路网络作为额外量测源。
- 与Map-matching的区别：Map-matching是后处理，本文是在线实时集成。
- 启发：TGNN+KF范式可推广到其他地理信息辅助定位场景。

## 补充分析

### 标准差预测头的工作机制
在十字路口或平行道路场景中，道路选择不确定，标准差预测头输出大值，KF更方侧重GNSS量测。在直道上不确定性低，道路信息权重提高。

### 与Viterbi算法的对比
Viterbi仅用历史信息做路径决策，TGNN通过LSTM结合历史+图结构做更全面的决策，在复杂场景中优势明显。

### 全球适用性
仅需OpenStreetMap数据即可运行，不依赖商业地图数据，具备全球部署潜力。
