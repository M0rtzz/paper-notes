# Language Models over Canonical Byte-Pair Encodings

**会议**: ICML 2025
**arXiv**: [2506.07956](https://arxiv.org/abs/2506.07956)
**代码**: 无
**领域**: LLM / NLP
**关键词**: BPE tokenization, canonical encoding, language modeling, probability mass, token-level LM

## 一句话总结
揭示基于 BPE 的语言模型会给指数多个"非规范"编码分配非零概率导致浪费，提出条件化（推理时约束）和构造化（新模型参数化）两种方法强制规范性，改善 held-out 似然。

## 研究背景与动机
1. **领域现状**: 现代 LM 将字符串概率表示为 BPE token 序列的分布，BPE 分词器是确定性的。
2. **现有痛点**: 对每个字符串，BPE 只产生一种规范编码，但模型给指数多个非规范编码分配非零概率——这些永远不会出现在训练数据中。
3. **核心矛盾**: 非规范编码的概率既是错误的（训练数据中不存在）又是浪费的（分流了合理输出的概率质量）。这是可避免的错误。
4. **本文解决什么**: 确保 token 级 LM 只给规范编码正概率。
5. **切入角度**: 分析 BPE 规范性的形式语言性质，用 FSA 表达约束。
6. **核心 idea**: 通过条件化（推理时 FSA 约束）和构造化（模型级保证）修复问题。

## 方法详解

### 整体框架
两种互补方法：(1) Canonicality by Conditioning: 推理时约束；(2) Canonicality by Construction: 模型参数化保证。

### 关键设计
1. **规范性 FSA**: BPE 的规范编码条件可用有限状态自动机表达。在解码时，只允许使当前序列仍在规范路径上的 token。设计动机：形式语言工具天然适合处理确定性分词器的约束。

2. **条件化方法**: 推理时使用 FSA 过滤非规范 token，类似 constrained decoding。无需额外训练。

3. **构造化方法**: 将 FSA 嵌入输出层，使非规范序列概率恒为零。需要训练但保证更强。

### 损失函数 / 训练策略
- 条件化：无训练需求
- 构造化：标准交叉熵 loss，架构含规范性约束

## 实验关键数据
### 主实验
| 模型 | 方法 | 困惑度变化 | 改善 |
|------|------|---------|------|
| GPT-2 Small | 原始 | 基线 | - |
| GPT-2 Small | 条件化 | 改善 | 概率重分配 |
| GPT-2 Medium | 条件化 | 改善 | 类似趋势 |
| 定制模型 | 构造化 | 最佳 | 训练时保证规范性 |

### 消融实验
| 配置 | 效果 | 说明 |
|------|------|------|
| 校正 vs 不校正 | 似然改善 | 浪费确实存在 |
| 大 vs 小词表 | 大词表更大改善 | 词表越大非规范编码越多 |
| 不同语言/语料 | 普遍改善 | 问题不限于特定语言 |

### 关键发现
- 非规范编码问题确实损害 held-out 似然，修复后全面改善
- 词表越大浪费越严重
- 条件化方法免训练即可获益

## 亮点与洞察
- 揭示了 BPE + 自回归 LM 组合中长期被忽视的概率漏洞
- FSA 的应用优雅地解决了规范性约束实现
- 两种互补方案适用于不同场景

## 局限性 / 可改进方向
- 条件化方法增加推理开销
- 构造化方法需重新训练，对大模型不实用
- 未量化对下游任务的实际影响

## 相关工作与启发
- 与 subword regularization/BPE dropout 方向不同
- 分词器的选择影响概率分布正确性
