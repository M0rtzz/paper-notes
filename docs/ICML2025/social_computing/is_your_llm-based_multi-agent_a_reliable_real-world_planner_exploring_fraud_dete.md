# Is Your LLM-Based Multi-Agent a Reliable Real-World Planner? Exploring Fraud Detection in Travel Planning

**会议**: ICML 2025 (Workshop MAS)  
**arXiv**: [2505.16557](https://arxiv.org/abs/2505.16557)  
**代码**: 无  
**领域**: LLM / NLP  
**关键词**: multi-agent planning, fraud detection, travel planning, LLM reliability, evaluation

## 一句话总结
提出 WandaPlan 评估环境，在旅行规划场景中注入三种递进式欺诈内容，揭示现有 LLM 多智能体规划系统在应对欺诈时的严重漏洞，并提出反欺诈 agent 作为缓解方案。

## 研究背景与动机
1. **领域现状**: LLM 多智能体规划系统正在快速发展，能自主协作完成复杂任务。
2. **现有痛点**: 这些系统依赖外部数据源（评论网站、社交媒体），易受虚假信息污染。现有评估只关注任务效率。
3. **核心矛盾**: 效率导向的框架越"高效"地接受虚假信息，结果越差。
4. **本文解决什么**: 系统评估多智能体规划系统在欺诈场景下的脆弱性。
5. **切入角度**: 构建真实世界模拟 + 欺诈注入的评估环境，设计三种递进欺诈场景。
6. **核心 idea**: 现有框架"重效率轻真实"，需引入反欺诈 agent。

## 方法详解

### 整体框架
WandaPlan 包含：模拟真实数据环境、三种欺诈注入方案、多维评估指标、反欺诈 agent。

### 关键设计
1. **三种欺诈场景**: (a) Misinformation Fraud: 单源虚假信息；(b) Team-Coordinated Multi-Person Fraud: 多号协调刷单；(c) Level-Escalating Multi-Round Fraud: 根据系统反馈逐步升级策略。设计动机：反映真实世界欺诈的演化规律。

2. **反欺诈 Agent**: 即插即用模块，负责交叉验证信息来源、检测评论异常模式、标记可疑数据。

3. **评估指标**: 不仅评估规划质量（路线合理性、预算符合度），还评估是否被欺诈影响。

### 损失函数 / 训练策略
评估性工作，核心在于 benchmark 设计和反欺诈 agent 提出。

## 实验关键数据

### 主实验
| 规划框架 | 无欺诈 | Misinformation | Multi-Person | Multi-Round |
|----------|--------|----------------|-------------|-------------|
| 框架 A | 较高 | 下降 | 显著下降 | 严重下降 |
| 框架 B | 较高 | 下降 | 显著下降 | 严重下降 |
| + 反欺诈 Agent | 较高 | 小幅下降 | 中等下降 | 中等下降 |

### 消融实验
| 配置 | 欺诈识别率 | 规划质量 | 说明 |
|------|---------|---------|------|
| 完整反欺诈 | 最高 | 最高 | 交叉验证+异常检测 |
| 无交叉验证 | 降低 | 降低 | 单源验证不足 |
| 无异常检测 | 降低 | 降低 | 无法识别协调欺诈 |

### 关键发现
- 所有框架在面对欺诈时性能显著下降
- 多人协调欺诈比单人更难检测
- 反欺诈 Agent 有效缓解但不能完全消除影响

## 亮点与洞察
- 填补了多智能体规划可靠性评估的空白
- 递进欺诈设计反映真实场景
- 揭示"重效率轻安全"隐患

## 局限性 / 可改进方向
- 仅关注旅行规划，其他领域适用性待验证
- 欺诈模式是预定义的，AI 生成的适应性欺诈未考虑
- Workshop paper 规模受限

## 相关工作与启发
- 与 TravelPlanner 等 benchmark 互补
- AI agent 安全性评估应成为标配
