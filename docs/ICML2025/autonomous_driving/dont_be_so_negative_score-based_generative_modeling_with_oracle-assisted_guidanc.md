# Don't be so negative! Score-based Generative Modeling with Oracle-assisted Guidance

**会议**: ICML2025  
**arXiv**: [2307.16463](https://arxiv.org/abs/2307.16463)  
**代码**: 待确认  
**领域**: autonomous_driving  
**关键词**: 扩散模型, 分类器引导, 约束生成, 碰撞避免, 负样本

## 一句话总结

提出 Gen-neG，利用 oracle 标记的负样本迭代训练分类器引导扩散模型，将生成分布从无效区域引导至正支撑域，应用于自动驾驶碰撞避免和安全人体运动生成。

## 研究背景与动机

- 约束域中扩散模型生成违规样本（碰撞/越界）
- 拒绝采样在实时场景代价高（违规率30-50%需大量并行采样器）
- Oracle（碰撞检测等）在实践中容易获得
- 发现classifier guidance隐含条件被忽略

## 方法详解

### 问题设定

给定训练数据集中的"好"样本 + oracle 函数 $O(x) \in \{\text{valid}, \text{invalid}\}$，目标是减少生成模型在无效区域的概率质量

### 迭代 Gen-neG 算法
1. 训练基线扩散模型 $p_\theta$ → 从中生成合成样本
2. oracle 标注每个样本为正/负
3. 在**当前扩散模型的噪声边际分布**上训练分类器 $c_\phi$
4. 用 $c_\phi$ 进行 classifier guidance 改进模型 → 得到 $p_{\theta,1}$
5. 从 $p_{\theta,1}$ 再生成 → oracle 标注 → 训练新分类器 → 重复
6. 经过 $K$ 轮得到逐步改进的模型

### 核心发现（本文最重要贡献）

分类器必须在当前扩散模型分布的噪声版本上训练，否则引导方向错误 → 违规率不降反升

### 蒸馏加速

可将 $K$ 轮分类器蒸馏为单个分类器，推理时仅需一个分类器

### 引导公式

修改后的 reverse SDE：$\tilde{s}(x_t,t) = s_\theta(x_t,t) + w \cdot \nabla_{x_t}\log c_\phi(x_t,t)$

其中 $w$ 控制引导强度

## 实验关键数据

### 自动驾驶车辆摆放（碰撞+越界避免）

| 方法 | 碰撞率↓ | 越界率↓ | 有效样本密度↑ |
|---|---|---|---|
| baseline DM | 较高 | 较高 | 低 |
| + 拒绝采样 | 0（但计算贵） | 0 | 低 |
| **Gen-neG 1轮** | 降低50%+ | 降低50%+ | 中 |
| **Gen-neG 多轮** | **大幅降低** | **大幅降低** | **高** |

### 人体运动生成（地面穿透避免）

| 指标 | baseline | Gen-neG |
|---|---|---|
| 穿透率 | 较高 | **显著减少** |
| FID | 基准 | **保持或改善** |
| 多样性 | 基准 | **保持** |

### 消融实验

- 迭代轮数增加→违规率持续降低，收敛约3-5轮
- 蒸馏版本 vs 迭代版本：性能可比但推理更高效
- 分类器训练分布不匹配→引导完全失效（验证核心发现）
- 引导强度参数 $w$ 的敏感性分析：过大导致模式坍塌

### 定量分析

- 每单位面积的碰撞数从基线显著下降（详见可视化图1）
- 验证集上对数似然保持稳定→未损害生成质量

## 亮点与洞察

1. 揭示classifier guidance必须在当前噪声分布上训练——此前被忽略的必要条件
2. Oracle作为免费约束信号——碰撞检测等物理约束天然可用
3. 迭代改进逐步收紧正支撑域，每轮分类器针对当前分布的残余违规训练
4. 蒸馏压缩多轮分类器为单一模型，实用性强

## 局限性 / 可改进方向

- 迭代多分类器增加训练时间
- Oracle需可微或可近似
