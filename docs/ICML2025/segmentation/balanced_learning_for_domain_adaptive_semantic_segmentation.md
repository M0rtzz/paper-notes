# Balanced Learning for Domain Adaptive Semantic Segmentation

**会议**: ICML 2025  
**arXiv**: [2512.06886](https://arxiv.org/abs/2512.06886)  
**代码**: https://github.com/Woof6/BLDA  
**领域**: 语义分割  
**关键词**: 域自适应, 语义分割, 类别不平衡, logit校准, 自训练

## 一句话总结
提出 BLDA——通过分析网络预测 logit 的分布来评估和缓解域自适应语义分割中的类别偏差，用共享锚点分布对齐各类 logit 分布，并在自训练中在线校正 logit 生成无偏伪标签。

## 研究背景与动机

**领域现状**：无监督域自适应（UDA）语义分割通过自训练技术从源域向目标域迁移知识，已成为主流范式。

**现有痛点**：自训练难以平衡学习各类别——类别不平衡和域间分布偏移共同导致某些类过度预测（如道路）而另一些类预测不足（如骑行者）。传统重加权/重采样方法假设测试分布已知，在 UDA 中不成立。

**核心矛盾**：域间的数据和标签分布偏移导致类别偏差，但目标域的先验类别分布未知。

**本文要解决什么**：无需先验知识即可评估和缓解类别偏差。

**切入角度**：分析网络预测的 logit 分布——过度预测的类 logit 值偏大，预测不足的类偏小。

**核心 idea**：用共享锚点分布对齐各类 logit→后处理校准 + 在线训练时 logit 修正。

## 方法详解

### 整体框架
1. 分析各类 logit 正/负分布的差异来量化偏差程度
2. 后处理：用 CDF 映射将各类 logit 对齐到共享锚点分布
3. 在线校正：用高斯混合模型在线估计 logit 分布，将修正项嵌入损失函数
4. 辅助损失：利用 CDF 衡量各类样本的辨别难度作为域共享结构知识

### 关键设计

1. **基于 logit 分布的偏差评估**:

    - 功能：直接从预测 logit 分布量化各类的过/欠预测程度
    - 核心思路：计算各类正 logit 和负 logit 的均值/方差差异→偏差程度与 logit 分布偏移高度相关
    - 设计动机：不需要目标域标签分布先验，通过模型预测自身评估偏差

2. **锚点分布对齐（后处理）**:

    - 功能：将各类 logit 分布映射到统一的锚点分布
    - 核心思路：设定共享正/负锚点分布→基于 CDF 映射校正各类 logit
    - 设计动机：直接在推理时纠偏，无需重新训练

3. **在线 logit 修正（训练时）**:

    - 功能：在自训练时生成无偏伪标签
    - 核心思路：用高斯混合模型在线拟合各类 logit 分布→计算修正项嵌入交叉熵损失
    - 设计动机：后处理仅纠正推理，训练时也需要无偏信号

### 损失函数 / 训练策略
- 标准自训练损失 + logit 修正项 + CDF 辅助损失
- 即插即用，可集成到任何自训练 UDA 方法

## 实验关键数据

### 主实验
GTA5→Cityscapes UDA：

| 方法 | mIoU ↑ | rider IoU ↑ | bicycle IoU ↑ |
|------|--------|------------|---------------|
| DAFormer | 68.3 | 48.5 | 54.2 |
| DAFormer + re-weight | 68.8 | 49.1 | 55.8 |
| **DAFormer + BLDA** | **70.4** | **56.3** | **62.1** |

### 消融实验
| 配置 | mIoU | 说明 |
|------|------|------|
| 基线（DAFormer） | 68.3 | 类别偏差严重 |
| + 后处理校准 | 69.2 | 推理时纠偏 |
| + 在线 logit 修正 | 69.8 | 训练时纠偏 |
| + CDF 辅助损失 | **70.4** | 完整方法 |

### 关键发现
- 欠预测类（rider +7.8%, bicycle +7.9%）提升最显著
- logit 分布偏差与类别偏差的排序高度一致（r > 0.9）
- 与多种 UDA 方法（DAFormer、HRDA、MIC）集成均有效

## 亮点与洞察
- **从 logit 分布角度看类别偏差**视角新颖——无需标签分布先验，从模型自身预测诊断问题
- 即插即用的设计使方法实用性极高
- CDF 作为域共享结构知识的利用是巧妙的附加贡献

## 局限性 / 可改进方向
- 锚点分布的选择是超参数
- 高斯混合假设可能不适合所有场景
- 仅在城市场景分割上验证
