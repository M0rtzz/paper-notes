# Expert Evaluation of LLM World Models: A High-Tc Superconductivity Case Study

**会议**: ICML 2025  
**arXiv**: [2511.03782](https://arxiv.org/abs/2511.03782)  
**代码**: 无  
**领域**: LLM 评估 / 科学推理  
**关键词**: LLM世界模型评估, 检索增强生成(RAG), 高温超导, 专家评测, 科学文献问答

## 一句话总结

以高温超导（HTS）领域为案例，构建了专家级数据集（1,726篇论文 + 67道专家问题），系统评估6种LLM系统的科学文献理解能力，发现基于精选文献的RAG系统在事实完整性和证据支持方面显著优于通用闭源模型。

## 研究背景与动机

高温超导（High-Tc Superconductivity）自1986年发现以来，积累了海量实验文献，涵盖ARPES、STM、中子散射、NMR、光学电导率等数十种实验手段。该领域面临的核心矛盾是：

1. **文献量巨大**：数千篇论文分散记录了四十年的实验数据，新入行研究者几乎不可能全面消化
2. **观点多元冲突**：不同理论框架对同一实验现象给出相互矛盾的解释
3. **多探针综合**：一个有价值的科学回答需要综合多种实验探针的结果
4. **时间维度复杂**：早期实验的重要性可能被后来的发现修正甚至推翻

理想状态下，年轻研究者希望拥有一个"随时可用的专家顾问团"——能可靠、全面地回答复杂科学问题。本文评估**当前LLM系统距离这一理想有多远**。

## 方法详解

### 整体框架

本文的方法论由三个核心模块组成：

```
专家精选文献库 (1,726篇)  →  6种LLM系统回答67道专家问题  →  专家盲评打分
```

整体流程如图1所示：首先由领域专家精选超导文献，然后构建专家级问题集，之后让不同LLM系统回答这些问题，最后由12位顶级物理学家以盲评方式对回答进行多维度评分。

### 关键设计

#### 1. 文献数据库构建

- **来源**：从15篇综述文章的参考文献出发，收集形成3,279篇论文的初始集合
- **综述覆盖**：涵盖 Varma(2020)、Proust(2019)、Lee(2006) 等经典综述
- **时效补充**：由于最新综述发表于2020年，额外补充28篇近年实验论文
- **理论/实验分类**：使用LLM对每篇论文的标题和摘要进行分类，采用L3Score方法计算置信度；最终从3,279篇中筛选出**1,726篇实验论文**作为主要数据源
- **元数据管理**：使用Zotero进行论文元数据存储

#### 2. 专家问题集设计（67道）

问题由12位领域专家设计，覆盖高温超导的几乎所有核心物理概念：

| 物理概念类别 | 涉及的典型实验技术 | 问题特点 |
|---|---|---|
| 量子临界点 (QCP) | 比热、磁化率、输运 | 需综合多种实验证据 |
| 载流子密度与掺杂依赖性 | Hall效应、ARPES | 需理解材料族系差异 |
| 配对对称性 | 相位敏感实验、SQUID | 需区分d波/s波证据 |
| 正常态对称性破缺 | STM、中子散射 | 需呈现多元理论观点 |
| 条纹序 (spin vs charge) | 中子散射、X射线 | 需权衡驱动力争议 |
| 涡旋性质 | STM、μSR | 需结合材料特异性 |
| 自旋涨落非公度性 | 中子散射、NMR | 需跨材料族系比较 |
| 费米面重构 | ARPES、量子振荡 | 需融合多探针结果 |

每道问题的设计原则：
- 涉及**多个理论概念**和**多种实验探针**的综合推理
- 对于学术界意见不一致的问题，要求能呈现多元视角
- 评估系统不仅答对，还需要引用正确的**实验证据**

#### 3. 被评估的六种LLM系统

本文测试了两类共6个系统：

**通用闭源模型（4个）**—— 基于训练数据 + 网络搜索：

| 系统编号 | 系统名称 | 知识来源 | 图像检索能力 |
|---|---|---|---|
| System-1 | ChatGPT-4o | 训练数据 + 网络搜索 | 无 |
| System-2 | Perplexity | 训练数据 + 网络搜索 | 有（网络图片） |
| System-3 | Claude 3.5 | 训练数据 + 网络搜索 | 无 |
| System-4 | Gemini Advanced Pro 1.5 | 训练数据 + 网络搜索 | 无 |

**基于精选文献的RAG系统（2个）**—— 基于1,726篇精选论文：

| 系统编号 | 系统名称 | 知识来源 | 图像检索能力 |
|---|---|---|---|
| System-5 | NotebookLM | 精选文献库 | 无（不稳定） |
| System-6 | 自研RAG系统 | 精选文献库 | 有（论文图表） |

#### 4. 自研RAG系统设计（System-6）

这是本文的核心技术贡献，系统架构包含三个关键环节：

**索引构建**：
- 使用PDFFigures解析PDF，分离文本和图表（图、表及其标题）
- 文本分块后使用Gecko文本嵌入模型建立文本索引
- 图像使用多模态嵌入模型（ALIGN），将图像嵌入与标题文本嵌入取均值作为特征向量
- 文本索引和图像索引分别构建

**检索与生成**：
- 给定查询，先从文本索引检索相关段落
- 使用Gemini 1.5 Flash基于检索到的文本段落生成连贯回答
- 生成过程要求包含引用来源

**图像检索**：
- 同时从图像索引中检索与查询相关的图表
- 将最相关的实验数据可视化图表附在回答中
- 这使得回答不仅有文字依据，还有**可视化实验证据**

#### 5. NotebookLM提示工程（System-5）

由于NotebookLM默认面向普通用户，作者进行了以下提示调整：
- 要求使用"适合技术受众的语言"，假设读者"拥有物理学博士学位"
- 优先引用"有实验结果的文献"而非理论文献
- 提供"主要不同观点的摘要"
- 优先使用"数值结果作为各观点的证据"
- 附带超导材料常用缩写对照表（如 LSCO: La₂₋ₓSrₓCuO₄）

### 评估策略（五维评分体系）

本文的评估对应专家评分的五个维度：

1. **视角平衡性 (Balanced Perspective)**：在学术界存在分歧时，是否呈现多元观点
2. **事实完整性 (Factually Comprehensive)**：是否涵盖所有已知实验事实
3. **简洁性 (Succinctness)**：回答是否简明扼要、不冗余
4. **证据支持 (Supported by Evidence)**：是否基于可靠实验证据并正确引用文献
5. **图像相关性 (Relevance of Images)**：检索的图表是否支撑论点（仅适用于System-2和System-6）

评估采用**盲评**方式：每位专家不知道回答来自哪个系统，独立打分。

## 实验关键数据

### 主实验

| 评估维度 | 通用闭源模型（最佳） | RAG系统（最佳） | 优势方 |
|---|---|---|---|
| 视角平衡性 | ChatGPT/Claude 中等 | NotebookLM/Custom 更优 | RAG |
| 事实完整性 | 中等水平 | 显著更高 | RAG大幅领先 |
| 简洁性 | 较好 | 略逊（回答更长） | 闭源略优 |
| 证据支持 | 弱（引用常不可靠） | 强（精确引用文献） | RAG显著领先 |
| 图像相关性 | Perplexity有限 | Custom系统优秀 | RAG领先 |

核心结论：**两个使用精选文献的RAG系统（System-5、System-6）在关键指标上全面超越四个通用闭源模型**，特别是在事实完整性和证据支持方面优势明显。

### 消融实验

| 配置 | 关键表现差异 | 说明 |
|---|---|---|
| 通用模型 vs RAG | RAG在completeness上显著优于通用模型 | 精选文献限定知识边界，减少幻觉 |
| 纯文本RAG vs 多模态RAG | System-6在图像证据上具有独特优势 | 实验科学依赖数据可视化 |
| 有网络搜索 vs 无网络搜索 | 网络搜索并不保证准确性 | 网络来源可靠性不如同行评审论文 |
| 有提示工程 vs 默认提示 | 调整后提示显著提升回答质量 | 对NotebookLM效果尤为明显 |

### 关键发现

1. **精选文献的价值**：将LLM知识源限定在专家筛选文献上比让模型自由搜索互联网效果更好，减少了不可靠信息源的干扰
2. **多模态检索的必要性**：实验科学的回答需要数据可视化支持，纯文字回答即使正确也缺乏说服力；System-6能检索论文原始图表是关键优势
3. **所有系统的共同短板**：即使最佳系统也难以全面覆盖复杂问题的所有方面——对于涉及10个方面的问题，最好的系统也仅覆盖一小部分
4. **引用可靠性差距**：通用闭源模型经常生成不存在的引用或引用不匹配的文献，RAG系统引用准确性显著更高
5. **简洁性的权衡**：RAG系统由于检索到更多相关信息，回答更长，简洁性上不如通用模型

## 亮点与洞察

1. **评估方法学的独特性**：本文最大贡献不是技术创新，而是方法论——用**真正的领域顶级专家**（来自Cornell、Harvard、Stanford、MIT等）以盲评方式评估LLM，评估严谨性远超自动化benchmark
2. **问题集的设计哲学**：67道问题不是简单事实问答，而是需要综合多种实验手段、权衡不同理论框架的深层推理题，这才是评估"世界模型"的正确方式
3. **RAG并非万能**：即使在精选文献上做RAG，系统仍无法展现专家级的"判断力"——区分哪些早期实验仍有价值、哪些结论已被后续工作修正
4. **对科学AI的启示**：要让AI成为合格的"科学助手"，需要的不仅是更好的模型，还需要**领域专家精心策划的知识库**和**多模态证据呈现能力**

## 局限性 / 可改进方向

1. **单一领域验证**：仅在高温超导领域测试，结论是否迁移到其他科学领域尚不确定
2. **评估成本高昂**：需要顶级专家大量时间进行盲评，难以规模化
3. **文献库静态性**：1,726篇论文截止到近年，无法持续跟踪最新进展
4. **缺乏交互式评估**：仅测试单轮问答，未评估多轮追问等交互场景
5. **理论文献排除**：仅使用实验论文，可能错失理论框架对综合理解的贡献
6. **模型版本快速更新**：测试时的模型版本已有更新，结果可能随模型迭代变化

## 相关工作与启发

- **科学RAG**：验证了高度专业化科学领域中RAG的必要性，启发后续工作对其他领域构建专家策划知识库
- **LLM评估**：与MMLU、SciQ等自动benchmark不同，强调了专家人工评估在科学领域的不可替代性
- **多模态科学AI**：图像检索能力对实验科学至关重要，可探索更先进的科学图表理解模型
- **知识策划 (Curation)**：精选文献 > 全网搜索，对构建领域专用AI系统有重要指导意义
