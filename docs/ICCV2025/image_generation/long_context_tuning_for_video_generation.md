# Long Context Tuning for Video Generation

**会议**: ICCV 2025  
**arXiv**: [2503.10589](https://arxiv.org/abs/2503.10589)  
**代码**: 暂无（项目页面可视化）  
**领域**: 视频生成 / 扩散模型  
**关键词**: 场景级视频生成, 长上下文, 多镜头一致性, 交错位置编码, 异步噪声

## 一句话总结

本文提出 Long Context Tuning (LCT)，将预训练单镜头视频扩散模型的上下文窗口扩展到场景级别，通过交错 3D RoPE、异步噪声策略和上下文因果注意力，实现跨镜头视觉/动态一致性的多镜头场景生成，并展现出零样本组合生成等涌现能力。

## 研究背景与动机

1. **单镜头到场景级的鸿沟**：当前视频生成模型（SoRA、Kling、Gen3）已能生成分钟级单镜头视频，但真实影视内容由多镜头场景组成（如泰坦尼克号的经典场景由 4 个不同镜位的镜头构成）。生成这样的场景需要跨镜头的视觉一致性（人物外观、背景、光线）和时间一致性（动作连贯、镜头运动）。

2. **现有方法局限**：
   - **外观条件生成**：将角色 ID、背景作为条件输入，但难以保持抽象元素（光线、色调）的一致性
   - **关键帧方法**：先生成一致关键帧再逐镜头 I2V 动画，但独立生成各镜头无法保证时间一致性，且稀疏关键帧条件化效果有限

3. **核心思路**：直接在预训练单镜头模型上扩展上下文窗口，让模型从场景级数据中学习跨镜头一致性，而非依赖额外条件网络。

## 方法详解

### 整体框架

LCT 在预训练单镜头 DiT (3B 参数, MMDiT 架构) 基础上进行三项关键设计：

### 关键设计一：交错 3D RoPE

单镜头模型中，文本 token 先于视频 token 沿空间对角线排列。多镜头扩展时：

- **保持单镜头内相对位置不变**：每个镜头内的文本-视频 token 相对位置关系保持原模型设置，继承预训练的文本-视觉对齐能力
- **跨镜头使用不同绝对位置**：各镜头按顺序追加，形成交错的"[text]-[video]-[text]-[video]-..."序列
- 全局 prompt 用虚拟视频 token 配对，作为普通文本-视频对处理

### 关键设计二：异步噪声策略

将条件输入与扩散样本统一化：
- 训练时为每个镜头独立采样扩散时间步 $t_i$，而非所有镜头使用相同 $t$
- 低噪声的镜头自然成为高噪声镜头的外观参考，建立动态跨镜头依赖
- 推理时可灵活配置：设置部分镜头为低噪声作为视觉条件，或同步所有时间步进行联合生成

### 关键设计三：上下文因果注意力

LCT 训练后的双向注意力模型可进一步微调为上下文因果注意力：
- 镜头内保持双向注意力
- 跨镜头使用因果掩码（只关注前序上下文）
- 自回归生成时可使用 KV-cache，历史镜头的 K/V 无需重复计算，大幅降低计算开销

### 数据构建

- 两层 prompt 结构：全局 prompt（角色、环境、故事概述）+ 逐镜头 prompt（事件、动作）
- 约 50 万场景样本（平均 5 镜头），通过场景边界检测和镜头切割获取
- 额外约 100 万单镜头视频通过事件变化分割为"伪多镜头"数据
- 多镜头数据在镜头 prompt 前添加 "[SHOT CUT]" 标记

### 损失函数

每个镜头按 Rectified Flow 独立计算：

$$\mathcal{L} = \mathbb{E}_{t, z_0, \epsilon}\|v_\Theta(z_t, t, c_{text}) - (\epsilon - z_0)\|_2^2$$

逐镜头损失取平均后反向传播。

## 实验

### 定量对比

| 方法 | Visual Aesthetic↑ | Quality↑ | Consistency↑ | Text↑ | User Study↑ |
|------|-------------------|----------|--------------|-------|-------------|
| VideoStudio | 61.68 | 73.13 | 95.25 | 28.00 | 2.14 |
| StoryDiffusion+Kling | 60.40 | 74.04 | 96.57 | 27.33 | 2.50 |
| IC-LoRA+Kling | 57.88 | 69.07 | 96.27 | 27.90 | 1.57 |
| **Ours** | **60.79** | 67.44 | 95.65 | **30.14** | **3.79** |

### 消融实验

| 配置 | 效果 |
|------|------|
| 无交错 RoPE | 模型无法区分不同镜头 token 归属，一致性下降 |
| 同步 vs 异步时间步 | 异步策略显著提升跨镜头条件化能力 |
| 因果注意力微调 | KV-cache 加速显著，生成质量接近双向模型 |

### 关键发现

- 用户研究中 LCT 平均排名 3.79/5，远超所有基线（最高 2.50）
- 文本对齐能力显著优于基线（30.14 vs 最高 28.00），归因于逐镜头独立 prompt
- **涌现能力**：组合生成（给定角色+环境图自动融合）、交互式镜头扩展（有/无机位切换）
- 生成的 20 镜头、3 分钟视频保持了一致的视觉和语义连贯性

## 亮点与洞察

1. **极简架构改动**：不引入额外参数，仅修改注意力范围和位置编码
2. **涌现能力**：组合生成和交互式镜头延伸未经显式训练即可实现
3. **实用的生成策略**：历史池 + 相关性检索（而非简单自回归），避免误差累积
4. **与 LLM 长上下文训练的思想类似**，将其成功迁移到视频扩散模型

## 局限性

- 训练需要 128 张 H800 GPU，训练 135K 迭代，计算成本巨大
- 当前分辨率限于 480×480 等效面积，距 1080p 生产需求仍有差距
- 跨镜头一致性依赖场景级数据质量，自动标注可能引入噪声
