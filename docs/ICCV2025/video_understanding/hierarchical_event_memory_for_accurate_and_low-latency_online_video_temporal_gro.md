# Hierarchical Event Memory for Accurate and Low-latency Online Video Temporal Grounding

**会议**: ICCV 2025  
**arXiv**: [2508.04546](https://arxiv.org/abs/2508.04546)  
**代码**: [https://github.com/minghangz/OnVTG](https://github.com/minghangz/OnVTG)  
**领域**: 视频理解  
**关键词**: 在线视频时序定位, 层级事件记忆, 事件提议, 未来预测, 低延迟

## 一句话总结

针对在线视频时序定位（OnVTG）任务，提出层级事件记忆机制存储不同时间尺度的历史事件信息，结合基于段树结构的事件提议和未来预测分支，在TACoS、ActivityNet Captions和MAD三大数据集上实现了SOTA的定位精度和低延迟预测。

## 研究背景与动机

在线视频时序定位（OnVTG）要求模型在视频流中实时定位与文本查询相关的事件，只能观察到当前时刻之前的帧。现有方法存在两个核心问题：

**缺乏有效的事件建模**：现有方法（如Gan et al.）采用帧级记忆存储最近的帧特征，预测每帧是否为目标事件的起止时间。但目标事件持续时间差异很大（如"推门"很短，"吹萨克斯"很长），帧级预测方法无法有效建模不同持续时间的事件

**无法保留长期历史信息**：固定大小的帧级记忆在长时间相似帧出现时会被冗余内容填满，有价值的事件信息可能被移除。例如查询"那个人再次吹萨克斯"需要知道他第一次吹萨克斯的时间，而这可能发生在视频开头

## 方法详解

### 整体框架

流式视频输入到预训练视觉编码器提取帧特征，在短期窗口内使用段树结构构建多尺度事件提议，利用层级事件记忆提供长期历史上下文来精化当前事件提议。提议特征与文本查询特征通过Transformer decoder融合后，分类器判断各提议是否为正样本，正样本进一步回归时序边界。未来预测分支预测目标事件是否即将开始并回归起始时间。

### 关键设计

1. **层级事件构建（Hierarchical Event Construction）**：使用段树结构在短期窗口 $[t-L_s, t]$ 内生成多尺度事件提议。第1尺度的提议特征通过1D卷积从帧特征获得：$P^1 = \{p_i^1\}_{i=t-L_s+1}^{t}$。更高尺度通过合并相邻低尺度提议得到：$p_i^{j+1} = \text{MLP}([p_{2i}^j; p_{2i-1}^j])$。第 $j$ 尺度的提议持续时间为 $2^j$。当持续时间超过短期窗口长度时，使用记忆中最近的历史事件参与合并计算。

2. **层级事件记忆与更新**：

    - **动态记忆大小配置**：总记忆大小为 $K$，各尺度的大小按正样本频率动态分配：$K_i = 1 + (K-L)\frac{w_i}{\sum_{j=1}^{L} w_j}$，确保每个尺度至少1个位置，正样本频率高的尺度分配更多空间
    - **自适应记忆更新**：当新事件加入导致某尺度超出容量时，计算相邻事件的余弦相似度，如果存在相似度大于阈值 $\delta$ 的相邻对则通过平均池化合并，否则按先进先出移除最旧的事件。这避免了冗余事件占满记忆

3. **记忆驱动的事件精化**：使用Transformer编码器层将历史记忆融入当前事件提议：$P^j = E(P^j; M)$。小尺度记忆提供细粒度近期信息，大尺度记忆保留长期粗粒度信息。

4. **未来预测分支（Future Prediction Branch）**：解决仅依赖事件提议导致的起始时间延迟问题（只有当事件快结束时模型才能获得完整提议）。在时刻 $t$，用MLP预测目标事件是否在未来窗口 $[t+a, t+b]$ 内开始（概率 $c_t^f$），并回归起始时间偏移 $o_t^f$。这使得模型在事件即将开始时就能预测起始时间。

### 损失函数 / 训练策略

总损失：$\mathcal{L} = \mathcal{L}_{cls} + \mathcal{L}_{reg} + \mathcal{L}_{future}$

- **分类损失**：Focal Loss监督事件提议的正负样本判断
- **回归损失**：DIoU Loss监督正样本提议的时序边界回归
- **未来预测损失**：Focal Loss监督是否即将发生 + L1回归起始时间偏移

超参数：$a=-4, b=4, L=8$（8个尺度），短期窗口长度8，记忆总大小64。使用AdamW优化器，学习率在不同数据集上分别为0.002/0.0001/0.001。

### 关键发现

- **事件记忆是最关键的组件**：引入事件记忆后R0.5@1从34.88提升到41.61（+6.73%），因为它能构建不同持续时间的提议并获取长期历史信息
- 动态大小配置额外贡献+2.36%，自适应更新贡献+0.22%
- 未来预测分支在降低起始时间延迟方面效果显著（从22.38s降到-1.28s，即提前预测），但会牺牲约6.75%的R0.5@1精度——这是精度-延迟的tradeoff
- w/o FP vs Gan et al.：TACoS上R0.5@1提升14.45%，MAD上R0.3@5提升11.05%，提升巨大
- 推理速度614.5 FPS（MAD数据集），满足实时要求，仅比基线慢17%但性能是其3.3倍
- 层级记忆理论上可以用 $K$ 大小的记忆获取 $2^K$ 时间跨度的事件信息，远超帧级记忆的 $K$ 帧

## 亮点与洞察

- **层级事件记忆的设计精巧**：小尺度记忆存储近期细粒度信息，大尺度记忆存储长期粗粒度信息，自然地满足了不同持续时间目标事件的定位需求
- **段树结构**高效地生成多尺度事件提议，避免了传统方法中提议数量随视频长度二次增长的问题
- **未来预测分支**提供了灵活的精度-延迟权衡，用户可以根据应用需求选择是否启用
- 动态记忆大小配置是数据驱动的设计——按正样本频率分配记忆容量，确保有限记忆被有效利用

## 局限与展望

- 未来预测分支启用后性能下降明显（TACoS上R0.5@1降低6.75%），低延迟和高精度难以兼得
- 使用C3D/CLIP等预提取特征作为输入，未考虑端到端训练可能带来的提升
- 自适应记忆更新中的相似度阈值 $\delta$ 是超参数，不同数据集可能需要不同设置
- 仅处理单个查询的定位，未考虑多查询同时定位的场景
- MAD数据集上绝对性能仍然较低（R0.5@5仅11.07%），长电影场景的定位仍然非常具有挑战性

## 相关工作与启发

- Gan et al.是唯一的OnVTG前序工作，本文从帧级预测升级到事件级预测是关键性进步
- OAT和HAT等在线动作定位方法也使用提议，但使用帧级记忆且未考虑起始时间延迟
- 层级记忆的思路可推广到其他长视频理解任务（如视频问答、视频摘要）
- 未来预测的思想类似于自动驾驶中的轨迹预测——在事件发生前预估其起始
