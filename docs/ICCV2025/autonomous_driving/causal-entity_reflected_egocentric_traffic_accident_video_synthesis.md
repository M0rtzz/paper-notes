# Causal-Entity Reflected Egocentric Traffic Accident Video Synthesis

**会议**: ICCV2025  
**arXiv**: [2506.23263](https://arxiv.org/abs/2506.23263)  
**代码**: [lotvsmmau.net/Causal-VidSyn](http://lotvsmmau.net/Causal-VidSyn)  
**领域**: autonomous_driving  
**关键词**: 事故视频合成, 视频扩散模型, 因果实体定位, 驾驶员注视, 自动驾驶安全

## 一句话总结

提出 Causal-VidSyn 扩散模型，通过事故原因问答（ArA）和驾驶员注视引导的因果 token 选择（CTS&CTG）机制，实现对自车视角交通事故视频中因果实体的精确定位与生成，同时构建了最大规模驾驶员注视数据集 Drive-Gaze（154 万帧）。

## 研究背景与动机

- 全自动驾驶（FSD）技术的事故场景数据极其稀缺，严重制约了安全性研究
- 现有视频扩散模型（如 CogVideoX、Abductive-OAVD）面向通用视频生成，无法准确生成涉及因果实体的事故视频
- 核心难点：事故场景中因果实体（引发事故的目标）通常很小且场景变化剧烈，难以在自车视角中精确识别
- 关键洞察：准确识别事故参与者并捕获其事故相关行为是成功生成事故视频的关键
- 本文利用两种信息线索：① 事故原因-碰撞描述（语义理解事故参与者及行为）；② 驾驶员注视点（提供对事故区域的直接视觉注意力）

## 方法详解

### 整体框架

Causal-VidSyn 从两个层面实现因果实体感知的视频合成：
1. **扩散规则层面（Stage-1）**：正反时序扩散对比学习（RPFD）
2. **知识层面（Stage-2）**：因果 token 选择（CTS）与因果 token 定位（CTG）模块注入 3D-Unet 骨干

### Drive-Gaze 数据集

- 数据来源：MM-AU 数据集的 11,727 个事故视频
- 规模：9,727 个事故视频，154 万帧驾驶员注视数据
- 收集方式：10 名受试者使用 Tobii Pro Fusion（250Hz）桌面式眼动仪
- 帧分辨率：1280×720，最终注视图由高斯滤波（50×50 像素核）卷积生成
- 每个长视频约 10 分钟，为避免疲劳每小时只标注一个长视频

### 正反时序扩散对比学习（RPFD，Stage-1）

- 核心思想：事故预防文本提示与反向时序帧（事故消散过程）对应
- 反向扩散可视为对正向文本和视觉提示的反事实干预
- 对比两条扩散路径的噪声表示学习：
  - 正向路径：事故原因-碰撞提示 + 正向时序帧
  - 反向路径：事故预防建议提示 + 反向时序帧
- 损失函数包含正反向 MSE 损失和负相似度损失（逆余弦相似度, λ=0.2）

### 因果 Token 选择（CTS）

- 在 3D-Unet 每层 TA（时序注意力）后注入 CTS 模块
- **门控注视融合**：将驾驶员注视 token 与视觉 token 通过 Gumbel-Softmax 门控融合
  - 注视 token 仅使用 CLIP 的位置编码（PE）层提取，避免稀疏注视在深层出现零值问题
- **跨层采样适配器**：使用双线性插值 + Conv2D(1×1) 统一不同层的尺度
- **因果倾向 token 选择**：通过 MLP + softmax 计算每个 token 的重要性分数，选取 top-d（每帧 1/4 的 token）作为因果倾向 token
- CTS 可理解为逐层"拉出"因果 token 的过程，注视图增强了拉力

### 因果 Token 定位（CTG）

- 在 3D-Unet 最后一层，通过事故原因问答（ArA）头实现因果 token 的最终定位
- ArA 设计为多选 VideoQA 任务：1 个正确答案 + 4 个干扰答案
- 对背景 token 进行随机遮蔽干预（1/4 token 加噪）
- 使用交叉注意力（CA）对齐正确/干扰原因与因果/背景 token
- 损失：因果 token 交叉熵 + 背景 token 交叉熵（均匀分布）+ KLD 干预差异

### 训练与推理

- **三阶段渐进训练**：Stage-0（正向扩散 10K 步）→ Stage-1（对比扩散 10K 步）→ Stage-2（+CTS&CTG 10K 步）
- Stage-2 总损失：$\mathcal{L}_{ST2} = \mathcal{L}_{MSE} + 0.3 \mathcal{L}_{ArA}$
- **推理时 CTS 和 CTG 即插即用可移除**，仅需视频或文本输入，使用 DDIM 调度器解码
- 支持 V2V（视频编辑、正常→事故扩散）和 T2V（文本→事故视频）两种模式

## 实验关键数据

### 评测任务与数据集

| 任务 | 数据集 | 功能 |
|------|--------|------|
| N2A（正常→事故） | BDD-A (2000 clips) | 从正常驾驶视频生成事故视频 |
| AEdit（事故编辑） | DADA-2000 (3000 clips) | 微小文本修改反映因果实体变化 |
| T2V（文本→视频） | DADA-2000 (500 clips) | 从文本生成事故视频 |

### 主要定量结果（N2A 任务）

| 方法 | CLIPs↑ | FVD↓ | TempC↑ |
|------|--------|------|--------|
| A-OAVD | 25.8 | 6378.9 | 0.992 |
| LAMP | 25.4 | 6208.2 | 0.991 |
| CogV-X* | 25.6 | 9075.6 | 0.992 |
| **Causal-VidSyn** | **26.5** | **6192.3** | 0.994 |

### AEdit 任务

| 方法 | CLIPs↑ | FVD↓ | Afd↑(%) |
|------|--------|------|---------|
| A-OAVD | 26.9 | 5358.2 | 49.4 |
| LAMP | 26.1 | 6191.6 | 34.7 |
| **Causal-VidSyn** | **28.7** | **5352.9** | **55.4** |

### CTS&CTG 可移植性

将 CTS&CTG 模块嫁接到其他 SOTA 方法后：
- CogV-X-T 在 AEdit 的 CLIPs 提升 +2.9
- Latte-T 在 T2V 的 CLIPs 提升 +0.7
- 证明了模块的即插即用特性

## 优势与局限

**优势**：
- 首次在视频扩散中实现因果实体感知的交通事故视频生成
- CTS&CTG 即插即用，可移植到 DiT 和 UNet 类模型
- 构建了最大规模的驾驶事故注视数据集 Drive-Gaze
- 训练时使用注视数据，推理时无需注视输入

**局限**：
- 生成分辨率较低（224×224）
- 在特定因果实体编辑类别间 CLIPs 增益不一致
- 依赖事故文本描述的质量

## 个人思考

- 将因果推理引入视频扩散是非常有价值的方向，尤其对自动驾驶安全测试场景
- RPFD 的正反向时序对比学习是巧妙的设计，利用了事故的时间不对称性
- CTS 的"逐层拉出因果 token"的直觉解释很清晰，注视图引导的 token 选择为扩散模型提供了类人的注意力先验
- 训练时有注视，推理时移除的设计使方法在部署时更加实用
