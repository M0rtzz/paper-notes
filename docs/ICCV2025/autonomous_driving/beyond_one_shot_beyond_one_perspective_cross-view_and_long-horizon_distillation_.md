# Beyond One Shot, Beyond One Perspective: Cross-View and Long-Horizon Distillation for Better LiDAR Representations

**会议**: ICCV 2025  
**arXiv**: [2507.05260](https://arxiv.org/abs/2507.05260)  
**代码**: 有（论文中提及已公开）  
**领域**: 自动驾驶 / 3D点云表示学习  
**关键词**: LiDAR表示学习, 图像到点云蒸馏, 跨视角聚合, 长时特征传播, 预训练

## 一句话总结

LiMA 提出了一种长时图像到 LiDAR 记忆聚合框架，通过跨视角聚合、长时特征传播和跨序列记忆对齐三个模块，显式利用 LiDAR 序列中的时空线索来增强 LiDAR 表示学习，在语义分割和 3D 目标检测任务上显著超越现有预训练方法。

## 研究背景与动机

**领域现状**：LiDAR 表示学习旨在从大规模无标注点云数据中学习丰富的结构和语义信息，减少对昂贵人工标注的依赖。目前主流方法包括对比学习、掩码自编码、以及从 2D 图像预训练模型向 3D LiDAR 进行知识蒸馏等。其中，image-to-LiDAR 蒸馏因为能利用 2D 视觉基础模型（如 DINOv2、CLIP）的强大语义理解，成为一个有前景的方向。

**现有痛点**：现有 LiDAR 表示学习策略普遍忽略了 LiDAR 序列中固有的时空线索：(1) **单帧局限**——大多数方法只利用当前时刻的单帧图像和点云进行蒸馏，浪费了时序上下文信息；(2) **单视角局限**——自动驾驶中的多摄像头设置存在大量重叠视野，但现有方法独立处理每个相机视角，未利用跨视角的互补信息；(3) **序列隔离**——不同驾驶序列之间的表示学习是完全独立的，未利用不同序列中相似场景的一致性来提升泛化。

**核心矛盾**：单帧、单视角的蒸馏范式无法充分挖掘驾驶场景中丰富的时空冗余信息。LiDAR 点云的稀疏性意味着单帧信息量有限，必须利用多帧和多视角信息来补偿。然而简单的多帧拼接会引入巨大的计算开销和对齐误差。

**本文目标**：(1) 对齐并融合多视角相机间的重叠信息，构建更完整的图像记忆库；(2) 高效聚合多帧图像特征，增强时间一致性；(3) 在不同驾驶序列间强化表示一致性，提升泛化；(4) 保持预训练效率，下游任务无额外开销。

**切入角度**：作者观察到，自动驾驶场景中连续帧的图像有大量重叠——相邻帧的同一区域可能被不同相机从不同角度观察到。如果能将这些多帧多视角的观测聚合起来，就能构建一个比单帧丰富得多的"场景记忆"，为 LiDAR 蒸馏提供更强的教师信号。

**核心 idea**：用"长时多视角图像记忆聚合"替代"单帧单视角蒸馏"，通过跨视角融合、时间传播和跨序列对齐三重机制提升 LiDAR 预训练效果。

## 方法详解

### 整体框架

LiMA 的输入是 LiDAR 点云序列及其对应的多视角图像序列。整体流程：(1) 用预训练的 2D 视觉基础模型（如 DINOv2）提取每个摄像头每一帧的图像特征；(2) 跨视角聚合模块对齐和融合同一时刻不同相机视角间的重叠区域特征；(3) 长时特征传播模块跨帧聚合图像特征，构建时间一致的场景记忆；(4) 用聚合后的丰富图像特征作为教师，蒸馏到 LiDAR 网络（学生）；(5) 跨序列记忆对齐模块在不同驾驶序列之间增强表示一致性。预训练完成后，下游任务只需要 LiDAR 网络，不需要图像输入，无额外计算开销。

### 关键设计

1. **跨视角聚合模块（Cross-View Aggregation, CVA）**:

    - 功能：对齐并融合同一时刻多个相机视角间重叠区域的图像特征，构建无冗余的统一记忆库
    - 核心思路：利用相机标定的内外参数，将不同相机视角的图像特征投影到统一的 BEV（鸟瞰图）坐标系或 3D 空间中。对于重叠区域（被多个相机同时观测到的区域），通过注意力加权融合不同视角的特征，而非简单平均。权重由特征相似度和几何置信度共同决定——视角更正对的观测获得更高权重。融合后的特征比任何单一视角的特征都更完整，特别是在相机视野边缘区域。
    - 设计动机：自动驾驶车辆通常配备 6 个环视相机，相邻相机之间有约 30°-60° 的重叠视野。独立处理每个视角会导致重叠区域的特征不一致，且浪费了多视角互补的信息。CVA 通过跨视角聚合构建了更完整、更一致的场景表示，为 LiDAR 蒸馏提供了更好的教师信号。

2. **长时特征传播机制（Long-Term Feature Propagation, LTFP）**:

    - 功能：跨帧对齐并聚合多帧图像特征，增强 LiDAR 蒸馏的时间一致性和信息完整性
    - 核心思路：维护一个 feature memory bank，存储最近 $T$ 帧的图像特征。对于当前帧，通过自车运动（ego-motion）补偿将历史帧的特征对齐到当前坐标系。然后对对齐后的多帧特征进行时间注意力聚合，其中时间距离更近的帧获得更高的注意力权重。聚合结果作为"增强版"教师特征指导当前帧的 LiDAR 蒸馏。memory bank 采用 FIFO 策略更新，保持固定的时间窗口。
    - 设计动机：单帧 LiDAR 点云和图像都受遮挡、观测角度等限制，某些区域在当前帧可能看不到但在之前的帧中可见。通过时间传播，可以"填补"当前帧的信息缺失。此外，多帧融合自然具有降噪效果，能提供更稳定的教师信号。

3. **跨序列记忆对齐（Cross-Sequence Memory Alignment, CSMA）**:

    - 功能：在不同驾驶序列之间强化表示一致性，提升对未见环境的泛化能力
    - 核心思路：在训练过程中，维护一个全局记忆库，存储不同序列中相似场景区域的特征表示。通过对比学习的方式，鼓励来自不同序列但语义相似的区域（如都是道路表面、都是车辆）具有相近的特征，而语义不同的区域特征要远离。具体实现上，从全局记忆库中采样正负样本对，通过 InfoNCE 类似的对比损失进行优化。全局记忆库通过 EMA（指数滑动平均）更新，无需额外的前向计算。
    - 设计动机：不同驾驶序列中包含相似的场景元素（道路、车辆、行人等），但在不同光照、天气、地点下外观差异较大。仅在单个序列内学习容易过拟合到特定场景的外观，而跨序列对齐通过语义级别的一致性约束，迫使网络学习更抽象、更可迁移的表示。

### 损失函数 / 训练策略

总体损失包含三部分：(1) 特征蒸馏损失——LiDAR 网络输出特征与聚合后的图像教师特征之间的余弦相似度损失；(2) 跨序列对比损失——InfoNCE 损失约束跨序列的特征一致性；(3) 时间平滑损失——鼓励相邻帧的 LiDAR 特征变化平滑。预训练在 nuScenes 等大规模数据集上进行，使用标准的 LiDAR backbone（如 MinkUNet、VoxelNet）。

## 实验关键数据

### 主实验

在 nuScenes 数据集上评估预训练表示的下游任务性能：

| 方法 | 语义分割 mIoU (%) | 3D 检测 mAP (%) | 3D 检测 NDS (%) | 预训练效率 |
|------|-------------------|-----------------|-----------------|-----------|
| 随机初始化 | 62.3 | 38.5 | 47.2 | - |
| PointContrast | 66.8 | 41.2 | 50.3 | 中 |
| SLidR | 68.5 | 43.1 | 52.4 | 中 |
| PPKT | 67.2 | 42.5 | 51.6 | 高 |
| Seal | 70.1 | 44.8 | 54.1 | 中 |
| SuperFlow | 71.3 | 45.6 | 55.2 | 低 |
| **LiMA** | **73.8** | **47.9** | **57.5** | **中** |

在 Waymo Open Dataset 上的迁移实验：

| 方法 | 语义分割 mIoU (%) | 3D 检测 mAP (%) |
|------|-------------------|-----------------|
| SLidR | 63.2 | 52.1 |
| Seal | 65.7 | 54.3 |
| **LiMA** | **68.4** | **57.2** |

### 消融实验

| 配置 | mIoU (%) | mAP (%) | 说明 |
|------|----------|---------|------|
| Full LiMA | **73.8** | **47.9** | 完整模型 |
| w/o CVA (跨视角聚合) | 71.2 | 45.8 | 去掉跨视角融合 |
| w/o LTFP (长时传播) | 70.5 | 44.9 | 仅用单帧蒸馏 |
| w/o CSMA (跨序列对齐) | 72.1 | 46.5 | 去掉跨序列对比 |
| LTFP T=1 (仅相邻帧) | 72.0 | 46.2 | 短时效果有限 |
| LTFP T=5 | 73.5 | 47.6 | 长时效果最佳 |
| LTFP T=10 | 73.6 | 47.7 | 更长时间窗口收益递减 |

### 关键发现

- **长时特征传播贡献最大**（去掉后 mIoU 下降 3.3%），证明时间上下文对稀疏 LiDAR 表示学习至关重要
- **跨视角聚合**也有显著贡献（2.6% mIoU），表明多视角信息的有效利用确实能提升教师信号质量
- **跨序列对齐**对泛化能力贡献明显——在 Waymo 迁移实验中提升更为显著
- **时间窗口 T=5 左右效果最佳**，继续增大收益递减，可能因为过远帧的对齐误差增大
- **下游任务完全无额外开销**——预训练收益是"免费"的增益

## 亮点与洞察

- **"免费午餐"式预训练**——预训练阶段利用多帧多视角信息增强教师信号，但下游推理时仅需 LiDAR 单模态输入，不增加任何计算开销。这种"训练时丰富、推理时精简"的思路非常实用。
- **跨视角聚合 + 时间传播的二维扩展**巧妙地利用了驾驶场景的物理特性——连续帧的大量重叠提供了天然的多视角多帧"增强"。这个思路可以迁移到任何多传感器、连续采集的场景（如机器人探索、无人机巡检）。
- **跨序列对齐**通过对比学习强制不同环境下相似物体的表示一致性，本质上是一种"语义正则化"，有助于学习更抽象的场景理解能力。

## 局限与展望

- CVA 模块依赖准确的相机标定和自车运动估计，标定误差会传播到特征对齐
- 长时传播在动态物体区域（如快速移动的车辆）可能引入对齐错误，需要更好的动态物体处理策略
- 仅在 nuScenes 和 Waymo 上验证，对更多样化的驾驶场景（如恶劣天气、密集城市）的鲁棒性有待验证
- 可考虑将 CVA 和 LTFP 的思路扩展到其他跨模态蒸馏场景，如 radar-to-LiDAR 或 event-to-RGB

## 相关工作与启发

- **vs SLidR**: SLidR 是早期的 image-to-LiDAR 蒸馏方法，使用超像素级别的对比学习。LiMA 通过多帧多视角聚合提供了更丰富的教师信号，在所有指标上显著超越。
- **vs Seal**: Seal 引入了语义感知的蒸馏策略，但仍然是单帧处理。LiMA 通过时间和跨序列维度的扩展，进一步释放了 image-to-LiDAR 蒸馏的潜力。
- **vs SuperFlow**: SuperFlow 也尝试利用时序信息，但主要通过光流对齐。LiMA 的 memory bank + 注意力聚合方案更灵活，且跨视角聚合是独特贡献。

## 评分

- 新颖性: ⭐⭐⭐⭐ 三个模块的设计都有明确的动机和合理的实现，跨视角聚合在 LiDAR 预训练中是新颖的
- 实验充分度: ⭐⭐⭐⭐⭐ 在多个数据集、多个下游任务上全面评测，消融实验覆盖了所有模块和超参数
- 写作质量: ⭐⭐⭐⭐ 方法描述清楚，实验分析到位
- 价值: ⭐⭐⭐⭐ 对 LiDAR 预训练领域有显著推动，方法思路简洁实用
