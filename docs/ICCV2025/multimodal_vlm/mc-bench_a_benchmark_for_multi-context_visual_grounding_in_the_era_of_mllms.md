# MC-Bench: A Benchmark for Multi-Context Visual Grounding in the Era of MLLMs

**会议**: ICCV 2025  
**arXiv**: [2410.12332](https://arxiv.org/abs/2410.12332)  
**代码**: [github.com/xuyunqiu/MC-Bench](https://github.com/xuyunqiu/MC-Bench)  
**领域**: 多模态VLM  
**关键词**: 多上下文视觉定位, 多图推理, 实例级评估, MLLM基准, 跨图理解

## 一句话总结

提出多上下文视觉定位（Multi-Context Visual Grounding）这一新任务和 MC-Bench 基准——包含 2000 个人工标注样本、3 种文本描述风格、20 项实用技能，评估了 20+ 个 MLLM 和基础模型，揭示现有模型与人类之间存在显著性能差距（人类 AP50=41.3% vs. 最优端到端模型 AP50=30.7%），并提供了一个 GPT-4o + G-DINO 的 agentic 基线（AP50=36.2%）。

## 研究背景与动机

### 问题定义

多上下文视觉定位是一个全新的任务：给定多张图像和一条开放式文本描述，模型需要在多张图像中定位文本所指的所有实例（以边界框形式输出）。这要求模型同时具备跨图推理、开放式语言理解和精确实例定位三种能力。

### 已有方法的不足

1. **现有 MLLM 聚焦单图任务**：大多数具有区域定位能力的 MLLM（如 Shikra、Ferret、Kosmos-2）仅能处理单张图像，忽略了跨图上下文
2. **多图基准缺乏实例级标注**：现有多图 MLLM 基准（如 MMBench、MMMU、BLINK、MileBench）均为图像级任务（如 VQA），没有实例级的边界框标注
3. **文本描述过于简单**：传统视觉定位任务（如 REC、OVD）使用类别名称或简单指代短语，不支持复杂推理、比较和常识性描述
4. **视频模型不适合非连续多图**：为视频理解设计的多帧模型（如 VideoLLM）依赖帧间连续性，无法处理空间或语义相关但非时序连续的多图场景

### 核心动机

**关键洞察**：真实世界中的图像不是孤立的，而是通过空间、时间或语义上下文相互关联。多图实例级智能在自动驾驶（多相机角度理解行人）、安防监控（跨摄像头追踪目标）、通用 AI 助手（图表分析、GUI 代理）等场景中至关重要。然而，目前没有任何基准系统评估 MLLM 在多图实例级任务上的能力——这是一个被严重忽视的研究方向。

## 方法详解

### 整体框架

MC-Bench 包含三个核心贡献：
1. **任务定义**：多上下文视觉定位——从多图+开放文本中定位实例
2. **数据集构建**：2000 个人工标注样本，覆盖 3 种文本风格和 20 项技能
3. **系统性评估**：20+ 模型评测 + agentic 基线 + 微调基线 + 人类评估

### 关键设计

#### 1. **任务形式化：多上下文视觉定位**

- **做什么**：定义一个统一的多图实例级视觉语言任务
- **核心思路**：
  - **输入**：多张图像（当前设定为图像对）+ 一条开放式文本描述
  - **输出**：所有被文本描述的实例的边界框，包括正确的分组信息
  - **三种文本风格**：
    - **指代式（Referring）**：通过类别、属性或位置信息直接或间接描述实例（346 个）
    - **比较式（Comparison）**：通过跨图比较（数量、颜色、大小等）定位实例（810 个）
    - **推理式（Reasoning）**：需要常识知识或多跳推理才能定位（844 个）
  - **一对多匹配**：一条描述可能对应多组不同类别的实例
  - **包含负样本**：288 个样本中文本描述不指代图中任何实例，模型应拒绝预测

- **设计动机**：指代式测试基本的跨图感知能力，比较式测试跨图对比分析能力，推理式测试深层语义推理能力——三种风格递进式地评估 MLLM 的多模态理解深度

#### 2. **数据集构建流程**

- **做什么**：构建高质量、高多样性的评估数据集
- **核心思路**：
  - **多源图像采集**：从 10+ 数据集（MS-COCO、BLINK、Mantis-Eval、DocVQA 等）和网络爬取，收集 3345 张覆盖自然图像、图表、文档、艺术品、科学图示等多领域的图像
  - **文本标注**：标注员选择语义相关的图像对，撰写连接两图的开放式文本描述
  - **实例标注**：另一组标注员为正样本中被描述的实例绘制边界框
  - **循环审核**：文本标注员审核边界框标注的一致性，不一致的样本重新标注
  - 使用 Label Studio 构建在线标注平台
  - **统计**：3200 个边界框，文本长度 2-24 词（均值 7.2），实例数 1-17 个

- **设计动机**：评估基准的质量直接决定其价值。多源图像确保领域多样性，人工标注确保金标准质量，循环审核确保文本-实例的一致性

#### 3. **评估指标设计**

- **做什么**：设计同时评估图像级和实例级性能的指标体系
- **核心思路**：
  - **图像级（Acc）**：模型是否正确判断哪些图像包含目标实例（不考虑具体位置）
  - **实例级（AP50）**：模型是否准确定位目标实例（IoU≥0.5）
  - 对于多组实例的样本，使用匈牙利算法做最优组匹配
  - 分别报告 referring、comparison、reasoning 三种风格的结果

- **设计动机**：图像级指标评估"理解"能力（哪张图有目标），实例级指标评估"定位"能力（目标在哪里），两者互补

#### 4. **Agentic 基线：GPT-4o + G-DINO**

- **做什么**：构建一个分而治之的 agent 系统作为上界参考
- **核心思路**：
  - GPT-4o 作为推理代理：(1) 分析多图输入，判断哪些图包含目标；(2) 为每个目标实例生成简洁的指代表达
  - G-DINO 作为定位工具：用 GPT 生成的指代表达在对应图像中检测目标
- **设计动机**：端到端模型在多图理解和精确定位两方面都有不足；通过组合推理专长（GPT-4o）和定位专长（G-DINO），建立一个更强的基线

### 损失函数 / 训练策略

MC-Bench 本身是评估基准，不涉及训练。微调基线使用方案：
- 基础模型：Qwen2-VL-7B
- 数据：50K+ 多上下文指令微调样本（部分来自现有数据集，部分合成）
- 方法：LoRA 微调
- 指标：Acc（图像级）和 AP50（实例级）

## 实验关键数据

### 主实验

**各类模型在 MC-Bench 上的表现**：

| 模型类型 | 代表模型 | Acc↑ | AP50↑ |
|----------|---------|------|-------|
| 闭源通用 MLLM | GPT-4o | 78.3 | 2.8 |
| 闭源通用 MLLM | Gemini-1.5 Pro | 62.5 | 28.2 |
| 开源通用 MLLM | Qwen2-VL-72B (seq) | 71.4 | 30.7 |
| 开源通用 MLLM | Qwen2-VL-7B (seq) | 54.9 | 19.1 |
| 开源专家 MLLM | CogVLM-Grounding-17B | 48.5 | 17.5 |
| 开源专家 MLLM | Ferret-13B | 44.8 | 12.9 |
| 基础模型（无 LLM） | G-DINO-B | 30.8 | 15.0 |
| **Agentic 基线** | **GPT-4o + G-DINO** | **77.9** | **36.2** |
| 微调基线 | Qwen2-VL-7B + IT | 57.7 | 22.6 |
| **人类** | **—** | **92.3** | **41.3** |

**三种文本风格的细分结果（Qwen2-VL-72B seq）**：

| 风格 | Acc | AP50 |
|------|-----|------|
| Referring | 61.6 | 33.7 |
| Comparison | 79.1 | 33.2 |
| Reasoning | 68.0 | 27.0 |

### 消融实验

**模型规模效应（Qwen2-VL）**：

| 模型大小 | Acc | AP50 | 说明 |
|---------|-----|------|------|
| 7B (seq) | 54.9 | 19.1 | — |
| 72B (seq) | 71.4 | 30.7 | 模型越大表现越好，符合 scaling law |

**图像输入方式**：

| 输入方式 | Acc (Qwen2-VL-7B) | AP50 | 说明 |
|---------|-------------------|------|------|
| 序列输入 | 54.9 | 19.1 | 保持多图独立性 |
| 拼接输入 | 51.4 | 17.8 | 丢失图像边界信息 |

**负样本拒绝能力**：大多数模型面对负样本时仍产生预测（Gemini 最优：平均 0.42 个预测/负样本 vs. 人类 0.19）

**实例分组能力**：多数模型仅预测 1 组实例，无法正确分组多组目标

### 关键发现

1. **GPT-4o 理解强但定位极弱**：Acc 最高（78.3%）但 AP50 仅 2.8%，说明闭源 API 模型的边界框输出能力极差
2. **专家 MLLM 反而不如通用 MLLM**：针对单图定位训练的专家模型在多图场景中泛化差，17B CogVLM 仅相当于 7B Qwen-VL
3. **Agentic 基线轻松超越所有端到端模型**：GPT-4o + G-DINO 的 AP50（36.2%）远超最好的端到端模型（30.7%），显示组合方案的优势
4. **人类与模型差距显著**：人类 Acc 92.3% vs. 最优模型 78.3%，AP50 41.3% vs. 36.2%，尤其在推理式描述上差距更大
5. **多实例分组是最大瓶颈**：去掉分组要求后，几乎所有模型的 AP50 显著提升，说明模型能定位但不能分组
6. **小目标检测困难**：MLLM 在中小目标上性能急剧下降，而 agentic 基线借助 G-DINO 在小目标上优势明显

## 亮点与洞察

1. **填补重要空白**：首个多图实例级MLLM基准，弥补了现有多图基准仅有图像级标注的不足
2. **任务设计的递进性**：referring→comparison→reasoning 的三种风格从感知到推理递进评估
3. **负样本设计**：要求模型"说不"的能力是一个被忽视但重要的维度
4. **agentic 基线的启示**：推理和定位的分工合作远优于端到端模型，提示了多图 MLLM 的发展方向
5. **分组问题的发现**：揭示了一个之前未被注意的 MLLM 弱点——无法将多个检测结果按语义分组

## 局限性 / 可改进方向

1. **规模较小**：仅 2000 个样本，部分技能类别的样本数不到 50 个，统计显著性可能受限
2. **仅限图像对**：当前设定为两张图像，更多图像的扩展评估尚未进行
3. **标注主观性**：开放式文本描述的标注存在主观性，尽管有循环审核，但不同标注员可能对同一图像对写出不同描述
4. **未评估最新模型**：由于论文提交时限，部分最新模型（如 GPT-4.5、Claude 等）未纳入评估
5. **缺乏训练方案探索**：微调基线相对简单（LoRA + 50K 数据），更系统的多上下文训练策略有待研究

## 相关工作与启发

- 与 BLINK/MileBench 的区别：这些基准评估图像级多图理解（如选择题），MC-Bench 要求实例级定位
- 与 RefCOCO 等定位基准的区别：传统定位基准是单图+简单描述，MC-Bench 是多图+开放式描述
- 启发：多图实例级理解可能需要全新的模型架构——当前 MLLM 将多张图像简单拼接或序列输入的方式不足以建立有效的跨图关联

- **新颖性**: ⭐⭐⭐⭐⭐ — 首创多上下文视觉定位任务，填补了重要研究空白
- **实验充分度**: ⭐⭐⭐⭐⭐ — 20+ 模型全面评测，含人类评估、agentic 基线和微调基线，多维度分析深入
- **写作质量**: ⭐⭐⭐⭐ — 结构清晰，分析详尽，图表丰富
