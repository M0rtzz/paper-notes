# VIKI-R: Coordinating Embodied Multi-Agent Cooperation via Reinforcement Learning

**会议**: NeurIPS 2025 (Datasets and Benchmarks Track)  
**arXiv**: [2506.09049](https://arxiv.org/abs/2506.09049)  
**代码**: 有（Project Page）  
**领域**: 强化学习 / 具身多智能体  
**关键词**: 多智能体合作, 视觉-语言模型, 具身智能, 层次基准, 强化学习微调

## 一句话总结

提出 VIKI-Bench（首个面向具身多智能体合作的层次基准）和 VIKI-R（基于 VLM + RL 的两阶段合作框架），在智能体激活、任务规划和轨迹感知三个层级上显著超越基线。

## 研究背景与动机

在动态环境中协调多个具身智能体合作是 AI 核心挑战之一。现有工作的不足：

**缺乏统一基准**: 没有专门针对具身多智能体合作的标准化评测
**LLM 为主**: 现有方法主要依赖 LLM 进行规划，对视觉输入的利用不足
**单一体态**: 大多数方法仅支持单一类型的机器人，缺乏异构体态支持
**缺少细粒度评估**: 现有评测通常只看最终成功率，缺乏过程评估

## 方法详解

### 整体框架

**VIKI-Bench 基准**:
- 三层结构：智能体激活 → 任务规划 → 轨迹感知
- 多样化机器人体态（移动机器人、机械臂、人形机器人等）
- 多视角视觉观测
- 结构化监督信号

**VIKI-R 框架**:
1. 阶段一：CoT 标注 SFT（Chain-of-Thought 标注的监督微调）
2. 阶段二：多层级奖励的强化学习

### 关键设计

1. **VIKI-Bench 三层结构**:

    - **L1 - 智能体激活 (Agent Activation)**: 判断哪些智能体应被激活
    - **L2 - 任务规划 (Task Planning)**: 为激活的智能体分配子任务和执行顺序
    - **L3 - 轨迹感知 (Trajectory Perception)**: 评估执行轨迹的正确性

2. **CoT 标注微调**:

    - 为每个合作场景生成详细的推理链标注
    - 包含视觉观察 → 智能体角色分析 → 任务分解 → 协作策略
    - 使用 SFT 将这些推理能力注入预训练 VLM

3. **多层级强化学习**:

    - 三个层级各有独立的奖励信号
    - L1 奖励：智能体选择的正确性
    - L2 奖励：子任务分配的合理性
    - L3 奖励：轨迹执行的效率和正确性
    - 综合奖励促进全局最优合作策略

### 损失函数 / 训练策略

$$\mathcal{L} = \mathcal{L}_{\text{SFT}} + \alpha \cdot r_{\text{L1}} + \beta \cdot r_{\text{L2}} + \gamma \cdot r_{\text{L3}}$$

RL 阶段使用 PPO 算法优化。

## 实验关键数据

### 主实验（VIKI-Bench 各层级）

| 方法 | L1 Acc ↑ | L2 Acc ↑ | L3 Acc ↑ | Overall ↑ |
|------|---------|---------|---------|----------|
| GPT-4V (zero-shot) | 52.3 | 38.5 | 31.2 | 40.7 |
| Qwen-VL-Chat | 45.8 | 32.1 | 25.8 | 34.6 |
| LLaVA-1.5-13B | 48.2 | 35.6 | 28.5 | 37.4 |
| CoT Prompting (GPT-4V) | 58.5 | 45.2 | 36.8 | 46.8 |
| CogAgent | 55.2 | 42.8 | 34.5 | 44.2 |
| **VIKI-R (SFT only)** | 68.5 | 55.8 | 48.2 | 57.5 |
| **VIKI-R (SFT + RL)** | **75.2** | **63.5** | **55.8** | **64.8** |

### 异构体态实验

| 体态组合 | GPT-4V ↑ | LLaVA ↑ | VIKI-R ↑ |
|---------|---------|--------|---------|
| 2×移动机器人 | 48.5 | 42.3 | **72.5** |
| 移动+机械臂 | 42.1 | 35.8 | **68.2** |
| 3种混合体态 | 35.8 | 28.5 | **62.5** |
| 人形+移动 | 38.2 | 32.1 | **65.8** |

### 消融实验

| 设置 | L1 | L2 | L3 | Overall |
|------|-----|-----|-----|---------|
| VIKI-R 完整 | 75.2 | 63.5 | 55.8 | 64.8 |
| 去掉 RL (仅 SFT) | 68.5 | 55.8 | 48.2 | 57.5 |
| 去掉 CoT 标注 | 62.3 | 48.5 | 42.1 | 51.0 |
| 单层级奖励 | 70.8 | 58.2 | 50.5 | 59.8 |
| 去掉多视角输入 | 71.5 | 58.8 | 48.5 | 59.6 |

### 关键发现

1. RL 阶段在 SFT 基础上进一步提升 7.3%（64.8 vs 57.5），证明 RL 对合作策略的优化有效
2. 异构体态组合越复杂，VIKI-R 相比基线的优势越大
3. 涌现现象：RL 训练后出现了"组合式合作模式"，不同智能体自动学到互补策略
4. 多层级奖励比单一奖励有效，说明分层评估对合作学习至关重要

## 亮点与洞察

- **首个层次基准**: VIKI-Bench 填补了具身多智能体合作评测的空白
- **视觉驱动合作**: 从视觉观测直接推理合作策略，比纯文本规划更实际
- **涌现行为**: RL 训练促进了组合式合作模式的涌现，令人兴奋
- **D&B Track**: 作为 Datasets and Benchmarks Track 论文，基准的价值可能比方法更长远

## 局限性 / 可改进方向

1. 目前仅在仿真环境中验证，真实机器人部署的 sim-to-real gap 有待解决
2. 智能体数量限制在 2-4 个，更大规模的合作场景未涉及
3. CoT 标注的生成依赖人工，规模化成本高
4. VLM 推理速度可能无法满足实时合作需求
