# Scaling Can Lead to Compositional Generalization

**会议**: NeurIPS 2025  
**arXiv**: [2507.07207](https://arxiv.org/abs/2507.07207)  
**代码**: [GitHub](https://github.com/smonsays/scale-compositionality)  
**领域**: Image Generation / Compositional Generalization  
**关键词**: 组合泛化, 缩放定律, 线性可解码性, MLP, 扩散模型

## 一句话总结

证明标准 MLP 通过简单地扩大数据量和模型规模即可实现组合泛化（compositional generalization），无需显式的模块化架构设计，并发现组合泛化成功时任务成分可从隐藏激活中线性解码。

## 研究背景与动机

组合泛化——理解和生成已知成分的全新组合——被认为是智能的核心能力。长期以来，研究界对神经网络能否真正实现组合泛化争论不休。一方面，大规模模型展现了令人印象深刻的组合能力；另一方面，即使最强大的模型仍存在频繁的组合失败案例。

许多工作主张需要显式赋予网络组合结构（模块化架构、图网络等）才能实现组合性。但本文提出一个根本性问题：**简单地缩放标准神经网络是否就足以实现组合泛化？**

## 方法详解

### 整体框架

本文通过 Hyperteacher 任务族系统研究组合泛化。一个组合任务族由 M 个模块中选取 K 个组合而成，产生 O(M^K) 个指数级任务。标准 MLP 在子集任务上训练，在未见过的组合任务上评估泛化能力。

### 关键设计

1. **组合任务族形式化 (Definition 2.1)**: 定义了组合任务族的严格数学框架。组合算子 C 必须满足：(i) 单射性——每种成分组合产生不同任务；(ii) 低复杂度——实现 C 的最短程序长度随 K 亚指数增长。这排除了纯上下文敏感的退化情况。

2. **Hyperteacher 作为通用组合任务族**: 用线性超网络将 K 个权重矩阵求和，参数化单隐层 ReLU 网络。这既是一个通用的组合任务类（神经网络的灵活性覆盖了广泛行为），也允许精确控制实验条件。

3. **组合泛化的理论保证 (Theorem 3.1)**: 证明 ReLU MLP 可用 O(1/√ε + M) 个神经元逼近 Hyperteacher 任务族到任意精度 ε。关键点在于泛化所需神经元数**线性**于模块数 M，而记忆所有任务需要指数级容量。因此模型规模增大时，泛化解的简洁性将越来越占优势。

4. **任务成分的线性可解码性 (Theorem 4.1)**: 证明对于任何在大部分任务上成功的模型，存在解码器可从输入恢复任务成分。实验中发现更强的结论：成功组合泛化的模型中，成分可从隐层激活中**线性**解码。

### 训练策略

使用标准 MLP（ReLU 激活），输入为数据点 x 与任务编码 φ(z, r) 的拼接。训练在子集任务上进行（多 epoch 或单 epoch），评估在保留的组合任务上。

## 实验关键数据

### 主实验

| 数据集/配置 | 指标 | 条件 | 结果 |
|-------------|------|------|------|
| Hyperteacher (M=16, K=3) | 组合泛化 R² | Identity 编码 | 1.00 |
| Hyperteacher (M=16, K=3) | 组合泛化 R² | Language 编码 | 1.00 |
| Hyperteacher (M=16, K=3) | 组合泛化 R² | Few-shot 编码 | 0.97 |
| Hyperteacher (M=16, K=3) | 组合泛化 R² | Invertible NN 编码 | 0.95 |
| 图像生成模型 | 生成成功率 vs 可解码性 | 多任务族 | 强正相关 |

### 消融实验

| 配置 | 指标 | 说明 |
|------|------|------|
| 训练任务数 vs 总任务数 | 达到 R²>0.95 的阈值 | 所需训练任务数**亚指数增长** |
| 模型宽度扩大 | 组合泛化 R² | 单调提升（所有任务编码） |
| 模型深度增加 | 组合泛化 R² | 单调提升 |
| 违反组合支持 | 组合泛化 R² | 显著下降 |
| 违反连通支持 | 组合泛化 R² | 显著下降 |
| 少数模块低频出现 | 组合泛化 R² | 明显下降（不平衡的真正原因是欠表示） |

### 关键发现

- 实现组合泛化所需的训练任务数随总任务数亚指数增长，符合 Definition 2.2 的要求。
- 即使身份编码（成分直接可见），不能组合泛化的网络也会在深层丢失成分信息——拥有解纠缠表示本身不足以保证组合泛化。
- 线性可解码性与组合泛化之间存在极强相关性（R² 高达 0.95+），在图像生成模型上也验证了这一关系。
- 训练分布需要满足"组合支持"和"连通支持"条件，分布覆盖不足会导致失败。

## 亮点与洞察

- **颠覆性结论**：标准 MLP 无需特殊模块化设计即可组合泛化，挑战了"组合性需要符号机制"的观点。
- **理论-实验一致**：Theorem 3.1 给出的线性复杂度泛化解与实验中规模扩大带来的改善完美对应。
- **与图像生成的联系**：将线性可解码性指标应用到 Stable Diffusion 等模型，发现它与生成成功率正相关——提供了诊断组合失败的工具。
- **对生物学的启示**：任何保信息映射（不只是语言）都足以实现组合泛化，可能解释了缺乏复杂语言的动物如何实现组合泛化。

## 局限性 / 可改进方向

- SGD 保证找到泛化解（而非记忆解）的理论条件尚未明确。
- 仅考虑了任务被完全指定的情况，未处理模糊/不完整的任务描述。
- 合成数据上的缩放实验需要已知的生成过程，真实数据上如何设计覆盖充分的训练分布仍是开放问题。
- 语言编码未显示出优于随机正交投影的优势，但实际中语言可能通过其他机制（如组合结构本身帮助学习）发挥作用。

## 相关工作与启发

- 与 Fodor & Pylyshyn (1988) 的经典批评形成对话：连接主义模型确实可以实现系统性组合。
- 与缩放定律文献（Kaplan et al., Brown et al.）一致：规模带来新能力。
- 为理解大模型的组合失败提供了可操作的诊断工具（线性可解码性）。

## 方法论细节补充

- Hyperteacher 的具体实现：M 组权重矩阵 Θ_m ∈ R^{I×H}，组合算子 C(z) = Ω·ReLU(Σ z_k·Θ_k·x)，输出读出为共享投影 Ω。
- 任务编码包括6种：Identity（直接成分向量）、Orthogonal（随机正交投影）、Language（语言指令）、Invertible NN（可逆神经网络）、Interval Shuffle（非线性映射）、Few-shot（示例数据点）。
- 组合泛化度量：在保留任务上的 R² 值，要求任务编码是信息保持（injective）的。
- 训练分布必须满足"组合支持"（每个模块至少出现在某些训练任务中）和"连通支持"（没有子集模块仅单独出现），否则理论上无法保证泛化。
- Compositional Preference 任务族作为补充验证：需要在网格世界中学习具有组合奖励函数的最优策略。
