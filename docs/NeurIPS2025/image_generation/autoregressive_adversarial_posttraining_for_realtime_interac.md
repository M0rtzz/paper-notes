# Autoregressive Adversarial Post-Training for Real-Time Interactive Video Generation

**会议**: NeurIPS 2025  
**arXiv**: [2506.09350](https://arxiv.org/abs/2506.09350)  
**代码**: [https://seaweed-apt.com/2](https://seaweed-apt.com/2)  
**作者**: Shanchuan Lin, Ceyuan Yang, Hao He, Jianwen Jiang, Yuxi Ren, Xin Xia, Yang Zhao, Xuefeng Xiao, Lu Jiang（ByteDance Seed）  
**领域**: 视频生成 / 图像生成  
**关键词**: 视频生成, 实时, 对抗训练, 自回归, 一步生成, interactive, streaming, KV cache, student-forcing

## 一句话总结

提出 AAPT（Autoregressive Adversarial Post-Training），通过三阶段后训练流程（扩散适配→一致性蒸馏→对抗训练）将预训练的 8B 视频扩散 Transformer 转化为单步自回归实时视频生成器，利用 student-forcing 训练策略有效抑制误差累积，在单张 H100 上以 24fps/736×416 流式生成长达一分钟（1440帧）的视频，同时支持姿态控制和相机控制等实时交互应用。

## 研究背景与动机

**大规模视频扩散模型的计算瓶颈**：当前主流视频生成模型（如 Sora、CogVideoX、Hunyuan Video、Wan2.1 等）基于扩散 Transformer（DiT）架构，生成质量已达到非常高的水平。然而，这些模型需要数十步去噪迭代才能获得高质量输出，生成几秒钟的高分辨率视频可能需要数分钟的计算时间，这彻底阻碍了它们在实时和交互式应用中的落地。

**现有加速方案的局限性**：围绕视频生成加速，学术界已探索了多条路径，但每条路径都存在显著不足。步数蒸馏方法（如 progressive distillation、consistency distillation）可以减少采样步数至 4-8 步，但质量下降明显且不天然支持自回归和交互式生成。早期 GAN 方法虽然天生支持单步生成，但受限于模型规模，生成的视频在分辨率、时长和质量上远不及现代扩散模型。基于 token 的自回归方法（如 VideoPoet）继承了 LLM 的 KV cache 优势，但逐 token 解码速度太慢，难以达到实时帧率。

**Diffusion Forcing 的折中方案**：近期的 diffusion forcing 方法（如 CausVid、MAGI-1、SkyReel-V2）通过为不同帧分配渐进的噪声水平来实现因果流式解码，并结合步数蒸馏进一步加速。其中 CausVid 蒸馏至 4 步，在单张 H100 上仅能达到 640×352/9.4fps 且延迟 1.30 秒；MAGI-1 蒸馏至 8 步但需要 24 张 H100 才能在 1280×720 达到实时。更关键的问题在于，这些方法仍然受限于固定时长训练窗口（通常 5 秒），生成长视频时需要反复重启和重新处理重叠上下文帧，造成额外的等待时间和质量损失。

**交互式视频生成的三大挑战**：真正的交互式视频生成需要同时解决三个核心挑战：（1）实时吞吐率——至少 24fps 以实现流畅的视觉体验；（2）低延迟——用户操作后应在毫秒级别看到响应；（3）长时间因果生成——能够持续数十秒甚至数分钟地生成连贯视频。现有方法最多只能部分满足其中一到两项。

**关键洞察：对抗训练是自回归视频生成的天然范式**。作者观察到，对抗训练（adversarial training）为自回归生成提供了一个独特的优势组合：首先，GAN 的判别器不需要成对的 ground-truth 监督（不像蒸馏方法需要教师模型的多步输出作为目标），这使得可以用生成器自身的输出（student-forcing）来训练长视频生成器，即便长时间的单镜头训练数据极其稀缺；其次，对抗训练天然适配单步生成，允许架构设计完全围绕 1NFE 优化，充分利用 KV cache；第三，判别器可以在分段的层面评估长视频的每个段落是否"像真实视频"，从而绕过了长视频训练数据不足的限制。

## 方法详解

### 整体框架

AAPT 的总体目标是将一个预训练的双向注意力视频扩散 Transformer（8B 参数，36 个 Transformer 块）转化为一个单步、逐帧、因果自回归的视频生成器。整个转化过程分为三个顺序执行的阶段：

**阶段一：扩散适配（Diffusion Adaptation）**。首先将预训练模型的双向全注意力替换为块因果注意力（block causal attention）——文本 token 仅互相注意，视觉 token 可以注意文本 token 以及当前帧和之前帧的视觉 token。同时修改模型输入：除了原有的噪声输入和条件输入外，增加一个"回收通道"（recycled input），通过通道拼接的方式接收上一自回归步骤生成的帧（第一步时为用户提供的首帧）。此阶段使用 teacher-forcing 训练，即提供 ground-truth 帧作为回收输入。使用 flow-matching 参数化（$x_t = (1-t) \cdot x_0 + t \cdot \epsilon$，$t \sim \mathcal{U}(0,1)$），通过 shifting 函数 $\text{shift}(t,s) = \frac{s \times t}{1 + (s-1) \times t}$（$s=24$）调整时间步分布。训练分三阶段课程学习：先在 736×416/5秒视频上训练 20k 迭代（batch 256），再混合 1280×720 训练 6k 迭代（batch 128），最后将 736×416 时长延长至 15 秒训练 4k 迭代（batch 32）。

**阶段二：一致性蒸馏（Consistency Distillation）**。在 32 个固定步上执行一致性蒸馏，为对抗训练提供更好的单步生成初始化。遵循 improved consistency distillation 技术，不使用指数移动平均作为一致性目标。不使用 classifier-free guidance（CFG），因为发现 CFG 在自回归生成中会引入伪影。此阶段仍使用 teacher-forcing，训练 5k 迭代。蒸馏后的模型输出较为模糊，但为下一阶段提供了关键的初始化。

**阶段三：对抗训练（Adversarial Training）**。这是 AAPT 的核心阶段。生成器从一致性蒸馏权重初始化，判别器从扩散适配权重初始化（总共 16B 参数参与训练）。生成器在 student-forcing 模式下自回归运行——仅接收首帧作为输入，后续帧全部使用自身生成的输出作为回收输入。判别器以并行方式一次性评估所有生成帧，为每一帧产生判别 logit，实现多时长判别（multi-duration discrimination）。使用 R3GAN 的 relativistic pairing loss，配合近似 R1 和 R2 正则化来稳定训练。

**推理流程**：推理时，模型在每个自回归步骤中仅处理当前帧的 token 并缓存 KV，生成下一个潜在帧（对应 4 个视频帧），然后将结果通过因果 VAE 解码器流式输出给用户。用户可以在任意时刻注入新的控制信号（文本、姿态、相机参数），模型在下一步立即响应。视觉 token 最多注意 $N=30$ 个过去的潜在帧（5 秒），同时始终注意文本 token 和首帧，以限制注意力计算和 KV cache 的增长。

### 关键设计

**设计一：Student-Forcing 对抗训练**。这是本文最核心的技术贡献。在对抗训练阶段，生成器严格按照推理时的行为运行——以 KV cache 自回归方式逐帧生成，每一步的输入都来自自身上一步的输出（而非 ground-truth）。这种 student-forcing 训练使得生成器学会从自身"不完美"的历史输出中继续生成合理的后续帧，从根本上解决了 teacher-forcing 训练中训练-推理分布不匹配导致的误差累积问题。实验表明，使用 teacher-forcing 对抗训练的模型在推理时仅生成几帧就出现严重的内容漂移和崩塌，而 student-forcing 模型可以稳定生成长达一分钟的连贯视频。关键实现细节：（1）将回收帧的输入从梯度图中 detach 以提高训练稳定性；（2）允许梯度通过 KV cache 流回以更新所有参数；（3）生成器使用 FSDP ZERO 2（避免反复聚合参数），判别器使用 ZERO 3（节省显存）。

**设计二：高效的单步自回归架构**。相比于 diffusion forcing 的单步变体，AAPT 的架构在效率上具有 2× 的优势。diffusion forcing 在使用 KV cache 时，每个自回归步骤需要处理两帧的计算量——当前需要去噪的帧和作为条件的上一帧（因为上一帧在不同时间步的 KV 不同）。而 AAPT 的设计将上一帧通过通道拼接作为输入回收，每步仅需处理一帧的 token 量。此外，由于是纯单步生成（1NFE），不需要保留多步扩散相关的组件（如自注意力中的 noise conditioning），使架构更加简洁高效。RoPE 位置编码沿空间维度动态拉伸以适应不同分辨率，沿时间维度使用固定间隔以支持任意长度的生成。

**设计三：长视频对抗训练策略**。训练长视频生成器面临数据稀缺问题——大多数训练数据集中单镜头视频的平均时长仅 8 秒，30-60 秒的长镜头极其罕见。AAPT 利用对抗训练的无配对监督特性巧妙地解决了这个问题：让生成器产生长视频（如 60 秒），将其分割为短段（如 10 秒）并保留 1 秒重叠，然后让判别器分段评估每个短段是否符合真实视频的分布。训练数据中的真实视频仍然是短视频。为了适配 GPU 显存，生成器也逐段生成，每段之间复用 detach 后的 KV cache，梯度在每段评估后累积反传。这种技术可以扩展到训练任意长度的生成器，代价是训练时间的线性增长。对抗训练分三步推进：首先在 5-10 秒视频上训练 500 次生成器更新（lr=3e-6, batch=256）；然后启用长视频训练，扩展一次至最长 19 秒（10 + (10-1)），训练 500 次更新；最后扩展 5 次至最长 55 秒（10 + 5×(10-1)），降低 batch 至 64、提高 lr 至 1e-5。

### 损失函数与优化

对抗训练采用 R3GAN 的 relativistic pairing loss，具体形式为：

$$\mathcal{L}_{RpGAN}(x_0, \epsilon) = f(D(G(\epsilon, c), c) - D(x_0, c))$$

其中 $f_G(x) = -\log(1+e^{-x})$ 用于生成器更新，$f_D(x) = -\log(1+e^{x})$ 用于判别器更新，$c$ 为文本和交互条件。作者发现 relativistic loss 比标准 non-saturating loss 更稳定。

R1 和 R2 正则化通过近似方法实现：

$$\mathcal{L}_{aR1} = \lambda \|D(x_0, c) - D(\mathcal{N}(x_0, \sigma\mathbf{I}), c)\|_2^2$$

$$\mathcal{L}_{aR2} = \lambda \|D(G(\epsilon, c), c) - D(\mathcal{N}(G(\epsilon, c), \sigma\mathbf{I}), c)\|_2^2$$

其中 $\sigma=0.1$，$\lambda=1000$。这种近似方法避免了显式计算 Jacobian 的昂贵开销。

判别器接收随机采样的时间步 $t \sim \mathcal{U}(0,1)$（不经过 shifting），将帧输入替代原始噪声输入通道（沿用 APT 的设计）。判别器使用 RMSProp 优化器（$\alpha=0.9$）。

此外，作者训练了一个轻量级 VAE 解码器来满足实时预算：将原始 VAE 解码器的每个分辨率尺度的残差块从 3 个减少到 2 个，通道数从 [128, 256, 512, 512] 减少到 [64, 128, 256, 512]，获得近 3 倍的解码加速，且无可见的质量下降。

### 关键发现

1. **Teacher-forcing 在对抗训练中彻底失败**：使用 teacher-forcing 对抗训练的模型在推理时几帧之内就出现严重的内容漂移和崩塌。作者推测 LLM 可以使用 teacher-forcing 是因为离散 codebook 编码的词汇对轻微不准确性不敏感，而视频模型预测整个帧的连续潜在值，轻微的不准确性会不断累积。

2. **帧回收输入至关重要**：将回收输入遮蔽为零张量（仅保留首帧）的消融实验表明，没有回收输入的模型无法生成大幅度运动，且运动变得不连贯。

3. **对抗训练提升视觉质量**：AAPT 的帧质量和成像质量均优于扩散基线模型，与 APT 在图像域的发现一致——对抗训练可以作为生成质量的"增强器"，而非仅仅是加速工具。

4. **CFG 在自回归生成中有害**：在一致性蒸馏阶段省略 classifier-free guidance，因为 CFG 会在自回归设定中引入伪影。在扩散基线的长视频生成中，降低 CFG scale 或使用 rescale 技术也无法缓解曝光问题。

## 亮点与洞察

- **速度里程碑**：单张 H100 上 8B 模型 24fps/736×416 实时流式生成是一个标志性成果，延迟仅 0.16 秒，首次真正实现了大模型级别的实时交互式视频生成。这比最优的 diffusion forcing 方法 CausVid 快 2.6 倍，且使用更大的模型和更高的分辨率。

- **对抗训练+自回归的范式创新**：本文首次将对抗后训练扩展到自回归视频生成场景，证明了对抗训练不仅是加速工具，更是一个天然适合自回归生成的训练范式——student-forcing 解决了误差累积，无配对监督特性绕过了长视频数据稀缺问题。

- **工程深度**：论文在系统工程层面做了大量工作——256 张 H100 训练约 7 天，轻量级 VAE 解码器 3x 加速（残差块从 3 减至 2，通道数从 [128,256,512,512] 减至 [64,128,256,512]），FSDP 混合策略（ZERO 2/3 切换），Flash Attention 3 实现块因果注意力，RoPE 修改以支持任意长度生成，Ulysses 上下文并行策略等。这些工程细节对于复现和后续工作至关重要。

- **Student-forcing 的普适性启示**：teacher-forcing vs student-forcing 的对比分析对所有连续值自回归生成模型都有重要参考价值——当输出空间是连续的（而非离散 token），student-forcing 可能是避免推理时分布漂移的必要条件。

## 局限性

- **计算资源门槛**：虽然实现了单卡 H100 实时推理，但 H100 本身就是最高端的 GPU，普通消费者和研究者难以获取。1280×720 分辨率需要 8 张 H100，进一步限制了可及性。训练需要 256 张 H100 约 7 天。

- **一致性维持困难**：模型在长视频中可能出现主体和场景的一致性问题。生成器使用基础滑动窗口机制，判别器的分段评估无法强制长程一致性。一旦单步生成出现缺陷，该缺陷可能在场景中保持很长时间（因为判别器同时也在强制时间一致性）。

- **单步生成质量上限**：1NFE 的质量仍然无法达到多步扩散的极限质量。从实验数据看，AAPT 的时间质量（89.69）低于扩散基线（90.40），动态程度（47.08）也低于基线（52.52），说明运动的丰富度有所牺牲。

- **超长视频退化**：作者测试了零样本五分钟生成，模型仍然可以生成内容但伴随明显的伪影。当前的 60 秒训练上限仍然制约了更长视频的质量。

- **判别器架构的局限**：判别器与生成器使用相同的 8B 架构，对抗训练总参数量 16B，训练效率受限。判别器的分段评估策略虽然绕过了长视频监督问题，但无法捕捉跨段的长程一致性约束。可能的改进方向包括在判别器中加入身份嵌入（identity embeddings）来强制跨段一致性。

- **分辨率和帧率限制**：当前的实时性能仅在特定分辨率和帧率配置下实现。736×416 虽然对移动端和部分应用足够，但与现代高端视频模型支持的 1080p 或 4K 分辨率仍有差距。8 卡才能达到 720p 实时的要求限制了广泛部署。

- **交互控制的精度**：虽然模型支持姿态和相机控制，但从定量指标看仍落后于专门针对这些任务优化的 SOTA 方法（如 OmniHuman-1 在姿态精度上的优势）。实时性约束可能限制了模型在精细控制任务上的极致表现。

## 相关工作

- **vs APT（Adversarial Post-Training）**：AAPT 直接扩展自同一团队的 APT 方法。APT 在非自回归设定下实现了 2 秒/1280×720/24fps 的单步文本到视频生成，但最多支持 49 帧且不支持流式和交互。AAPT 将其推广到自回归场景，引入 student-forcing 和长视频训练，将时长从 2 秒扩展到 60 秒并实现实时交互。

- **vs CausVid**：CausVid 将双向视频扩散模型转换为因果 diffusion forcing 生成器并蒸馏至 4 步，是此前单卡流式生成的最强方法。但其 4 步推理、每次输出 16 帧的chunk design 导致延迟（1.30s）和帧率（9.4fps）远不如 AAPT。CausVid 生成长视频仍需反复重启和重处理重叠帧。

- **vs MAGI-1 / SkyReel-V2**：两者均为从头训练的 diffusion forcing 大模型。MAGI-1（24B）蒸馏至 8 步需要 24 张 H100 仅达 3.43fps；SkyReel-V2（14B）60 步仅 0.89fps。两者都需要通过 extension 技术生成长视频，且在定性对比中 20-30 秒后出现严重退化。

- **vs 世界模型（GameNGen、Oasis、Genie-2）**：这些方法追求类似的实时交互视频生成目标，但通常使用较小模型（几百万参数）在低分辨率（~320×240）运行。AAPT 基于 8B 参数的强大视频扩散基础模型，在更高分辨率和质量上实现实时生成。

- **vs Token 自回归方法（VideoPoet、Emu3）**：基于离散 token 的视频自回归方法使用 next-token prediction，KV cache 高效但逐 token 解码速度慢，且需要将视频量化为离散 codebook，可能丢失视觉细节。AAPT 折中地每步生成一整帧的所有 token（而非单个 token），同时用对抗目标取代 next-token prediction loss，针对高速生成进行了优化。这种设计兼顾了 LLM 风格 KV cache 的效率优势和扩散模型的连续潜在空间表示优势。

- **vs Consistency Models / DMD**：一致性模型和分布匹配蒸馏可以实现单步或少步生成，但它们通常在非自回归设定下操作，难以直接应用于流式长视频生成。AAPT 将一致性蒸馏仅作为对抗训练的初始化阶段，最终的单步生成能力来自对抗训练，这使得 student-forcing 训练成为可能。

## 评分

- 新颖性: ⭐⭐⭐⭐⭐ 首次将对抗后训练扩展到自回归视频生成，student-forcing 对抗训练和长视频分段判别策略均为原创性贡献
- 实验充分度: ⭐⭐⭐⭐⭐ VBench-I2V 短/长视频定量评测、姿态控制和相机控制两个交互任务、速度延迟对比、长视频训练消融、student/teacher-forcing 消融、帧回收消融，覆盖全面
- 写作质量: ⭐⭐⭐⭐ 动机清晰，技术描述系统完整，附录提供了充分的实现细节
- 价值: ⭐⭐⭐⭐⭐ 里程碑式工作，首次实现大模型级别的实时交互式视频流式生成，对游戏、VR、数字人等应用有重大推动意义
