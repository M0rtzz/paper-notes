# MGE-LDM: Joint Latent Diffusion for Simultaneous Music Generation and Source Extraction

**会议**: NeurIPS 2025  
**arXiv**: [2505.23305](https://arxiv.org/abs/2505.23305)  
**代码**: 无（提供了音频样例页面）  
**领域**: Image Generation / Audio Generation  
**关键词**: 音乐生成, 源分离, 潜在扩散模型, 多轨建模, CLAP

## 一句话总结

提出 MGE-LDM，首个在统一的潜在扩散框架中同时实现音乐混合生成、部分生成（源补全）和文本驱动任意源提取的模型，通过联合建模混合-子混合-源三元组并利用扩散修复（inpainting）实现各任务。

## 研究背景与动机

音乐生成模型虽已取得显著进展，但大多只能生成单一混合波形，缺乏对独立乐器轨道的控制能力。要获取独立轨道需要额外的源分离技术。

近期工作（MSDM、GMSDI、MSG-LD）尝试在统一扩散框架中联合建模多轨，但存在两个关键限制：

**依赖预定义乐器类别**：只能处理固定的乐器集合（如贝斯、鼓、吉他、钢琴）

**线性混合假设**：假设混合是各源的线性叠加——在波形域成立，但在潜在扩散模型的非线性编解码器结构下不适用

这些限制使得现有方法无法处理任意乐器的混合生成和分离。

## 方法详解

### 整体框架

MGE-LDM 将三个潜在变量——混合 z^(m)、子混合 z^(u)、源 z^(s)——联合编码到潜在空间，训练一个三轨扩散模型。推理时通过条件修复（inpainting）实现不同任务：混合生成、源补全、源提取。

### 关键设计

1. **三轨联合潜在表示**: 给定多轨音频 {x_i}，随机采样一个轨道作为源 x^(s)，其余为子混合 x^(u)，混合为 x^(m)。三者分别经预训练 VAE 编码器映射到潜在空间。这种设计天然支持可变数量的乐器轨道，且不依赖乐器类别标签。

2. **Track-aware 自适应时间步 (Section 3.4)**: 不同于传统扩散修复对所有区域使用统一噪声调度，本文为每个轨道分配独立的时间步条件。训练时在四种模式间随机切换：

    - (τ, τ, τ)：标准联合去噪
    - (0, τ, τ)：混合已知，去噪子混合和源
    - (τ, 0, τ)：子混合已知，去噪混合和源
    - (τ, τ, 0)：源已知，去噪混合和子混合
   
   这使模型学到条件评分函数，推理时可精确地仅生成缺失轨道。

3. **CLAP 条件与文本驱动提取**: 使用预训练 CLAP 模型的音频分支提取条件向量（训练时），文本分支提取提示嵌入（推理时）。通过音频-文本嵌入的随机线性插值弥合模态差距。源提取被形式化为条件修复问题——给定混合潜在表示和文本提示，生成目标源。

4. **数据集无关训练范式**: 消除了对固定乐器标签的依赖。Slakh2100 提供干净的独立轨，MUSDB18 和 MoisesDB 包含"other"等聚合标签。三轨公式将聚合标签视为子混合，使所有数据集统一参与训练。

### 损失函数 / 训练策略

采用 v-objective 训练，使用余弦噪声调度。使用 GroupNorm（3 组对应三轨道）替代 LayerNorm。支持 Classifier-Free Guidance (CFG)，每个 c^(k) 以概率 p 独立 dropout。

## 实验关键数据

### 主实验

| 任务/数据集 | 指标 | MGE-LDM (T1) | MSG-LD | MSDM |
|-------------|------|---------------|--------|------|
| 混合生成 (S_A) | FAD ↓ | **0.47** | 1.38 | 4.21 |
| 混合生成 (S_Full) | FAD ↓ | 1.79 | **1.55** | 6.04 |
| 混合生成 (M_u) | FAD ↓ | 6.34 | 4.61 | 7.92 |
| 全数据集训练 (T4, M_u) | FAD ↓ | **2.78** | - | - |
| 全数据集训练 (T4, M_o) | FAD ↓ | **1.47** | - | - |

### 消融实验

| 配置 | 指标 | 说明 |
|------|------|------|
| 仅 S_A 训练 (T1) | FAD 0.47 (S_A) | 同域最优，跨域泛化有限 |
| 全 Slakh (T2) | FAD 0.63 (S_Full) | 多样性训练提升跨乐器泛化 |
| 仅真实录音 (T3) | FAD 2.87/1.59 (M_u/M_o) | 跨域到合成数据也表现尚可 |
| 全数据集 (T4) | 最优整体 | 更广训练分布带来最佳整体性能 |
| 文本条件引导生成 | 显著改善 FAD | 约束生成乐器集合，减少分布不匹配 |

### 关键发现

- 三轨潜在空间建模有效避免了线性混合假设的限制，且支持类别无关的源操作。
- Track-aware 自适应时间步策略是性能的关键——使模型不仅学习联合分布，还学习各种条件分布。
- 训练数据的多样性（跨数据集）直接转化为更好的跨域泛化，但可能牺牲单域最优性。
- 文本条件在引导生成匹配特定乐器集方面非常有效，缓解了"生成内容过于多样"的问题。

## 亮点与洞察

- **统一性**：首次在单一框架中同时实现生成、补全和提取三个音乐任务，且不依赖预定义乐器类别。
- **潜在空间修复的巧妙利用**：将源分离重新定义为条件生成问题，绕过了潜在空间中非线性编解码器带来的线性混合限制。
- **数据集聚合能力**：将"other"等模糊标签自然整合为子混合，解决了异质多轨数据集的统一训练难题。

## 局限性 / 可改进方向

- 潜在空间的非线性编解码可能引入重建误差，影响分离质量。
- 需要预训练 VAE 和 CLAP 模型，增加了系统复杂度。
- 迭代式部分生成（逐个音源补全）可能累积误差。
- 源提取依赖 CLAP 文本-音频对齐质量，对不常见乐器可能效果受限。
- 未与端到端判别式分离方法（如 Bandit）在分离质量上深入对比。

## 相关工作与启发

- MSDM 开创了统一扩散框架处理合成/分解的思路，本文在类别无关和潜在空间两个方向上显著推进。
- TD-Paint 的逐区域时间步技术被创造性地迁移到多轨音频建模。
- 为交互式音乐制作提供了新工具——用自然语言控制音源的添加和提取。
