# V-CECE: Visual Counterfactual Explanations via Conceptual Edits

**会议**: NeurIPS 2025
**arXiv**: [2509.16567](https://arxiv.org/abs/2509.16567)
**代码**: [项目页面](https://nickspanos55.github.io/vcece)
**领域**: 可解释 AI, 反事实解释, 扩散模型
**关键词**: 反事实解释, 概念编辑, 黑盒, 知识图谱, 扩散模型

## 一句话总结
提出 V-CECE，首个系统性解决人类语义理解与神经网络推理差异的视觉反事实解释框架，通过知识图谱保证编辑最优性，利用扩散模型执行概念级编辑，无需训练即可生成人类可理解的反事实图像。

## 研究背景与动机
- 现有反事实图像生成方法存在问题：编辑分散不可解释、过度依赖训练来引导生成
- 白盒方法需要访问分类器梯度，无法解释商业闭源模型
- 现有语义反事实方法假设分类器以人类语义级别推理，却未提供验证
- 关键问题：人类和神经网络分类器理解语义的方式是否存在差异？这种差异如何影响反事实解释？

## 方法详解

### 整体框架
两个核心阶段：
1. **解释阶段**：利用知识图谱计算最优语义编辑集合
2. **生成阶段**：利用扩散模型执行概念编辑并验证分类翻转

### 关键设计
1. **最优编辑保证**（基于 WordNet 知识图谱）：
   - 三种编辑操作：插入 I、删除 D、替换 S
   - 替换代价 = 知识图谱上两概念间最短路径
   - 建立二部图匹配问题，用匈牙利算法求解最小代价编辑集 E
   - 时间复杂度 O(mn log n)

2. **编辑排序策略**（三种方案）：
   - **Local Edits**：LVLM 逐步观察当前图像选择下一个编辑
   - **Global Edits**：统计跨所有图像的编辑重要性分数排序
   - **Local-Global**：使用局部编辑集但按全局重要性排序

3. **编辑执行**：
   - 使用冻结的 Stable Diffusion v1.5 Inpainting 模型
   - GroundingDINO + SAM 生成概念掩码
   - Claude 3.5 Sonnet 处理放置位置和背景补全的常识推理
   - 迭代编辑直到分类标签翻转或编辑集耗尽

### 损失函数 / 训练策略
- 完全即插即用，零训练
- 不对扩散模型进行微调，避免引入数据集偏差
- 40 步采样，DPM++ 2M SDE 采样器

## 实验关键数据

### 主实验（BDD100K 数据集）

| 方法 | FID↓ | SR↑ | 访问级别 | 训练 |
|------|------|------|---------|------|
| STEEX | 58.8 | 99.5% | 白盒 | 数天 |
| ACE ℓ₁ | 1.02 | 99.9% | 白盒 | 数天 |
| TIME | 51.5 | 81.8% | 黑盒 | 数小时 |
| V-CECE (DenseNet) | 81.90 | 88.9% | 黑盒 | N/A |
| V-CECE (Claude 3.5) | **45.22** | **97.8%** | 黑盒 | N/A |
| V-CECE (Claude 3.7) | 45.83 | 98.35% | 黑盒 | N/A |

### 人类与模型的语义差异

| 分类器类型 | 平均编辑次数 |E| | 成功率 SR |
|-----------|------------|---------|
| DenseNet (CNN) | 4.77-5.37 | 84.8-88.9% |
| ConvNeXt (CNN) | 4.91-5.86 | 81.4-95.4% |
| Claude 3.5 Sonnet (LVLM) | **2.65-3.10** | **96.6-97.8%** |

### 关键发现
- CNN 分类器需要更多语义编辑才能翻转标签，且图像质量显著下降 → 表明 CNN 不以人类语义推理
- LVLM 分类器仅需 2-3 步编辑即可翻转，图像质量保持较好 → LVLM 的推理更接近人类语义
- 这一差异揭示了"语义解释鸿沟"：试图用人类语义解释 CNN 可能产生误导性结论
- V-CECE 在使用 LVLM 分类器时甚至超过了需要训练的 TIME 方法

## 亮点与洞察
- 首次系统性地探讨了人类与分类器在语义理解上的"解释鸿沟"
- 最优编辑集的理论保证（匈牙利算法）为反事实解释提供了坚实基础
- 完全黑盒 + 零训练 + 即插即用的设计具有极强实用性
- 揭示了一个重要发现：用人类语义解释 CNN 分类器可能根本不合理

## 局限性 / 可改进方向
- 依赖 WordNet 知识图谱，对于不在 WordNet 中的概念无法处理
- SD v1.5 的生成质量限制了编辑图像的真实感
- 目标检测（GroundingDINO）的准确性直接影响编辑质量
- 仅在 BDD100K 和 Visual Genome 上验证，其他领域有待探索
- 当图像中概念边界模糊时，插入/删除操作可能引入伪影

## 相关工作与启发
- 将反事实解释分解为"分类器是否语义推理"和"最小编辑是什么"两个独立问题的思路新颖
- 利用 LVLM 进行编辑排序和放置推理展示了大模型在 XAI 中的潜力
- 反事实解释方法的评价标准应考虑分类器类型对结果的影响
- 知识图谱（WordNet）在结构化语义距离计算中的应用为编辑最优性提供了基础
- Local-Global 混合排序策略兼顾了场景特定性和分类器全局偏差
- 研究发现 LVLM 分类器与 CNN 在语义理解上的本质差异对 XAI 领域有深远影响
