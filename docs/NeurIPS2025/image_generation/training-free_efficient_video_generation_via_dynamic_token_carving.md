# Training-Free Efficient Video Generation via Dynamic Token Carving (Jenga)

**会议**: NeurIPS 2025  
**arXiv**: [2505.16864](https://arxiv.org/abs/2505.16864)  
**代码**: https://github.com/dvlab-research/Jenga (有)  
**领域**: 视频生成  
**关键词**: Video DiT, 动态注意力剪裁, 渐进分辨率, 空间填充曲线, 推理加速

## 一句话总结

提出 Jenga，一种即插即用的视频扩散 Transformer 推理加速管线，通过 3D 空间填充曲线驱动的动态注意力剪裁和渐进分辨率生成的协同设计，在多个 Video DiT 模型上实现 6-9 倍加速且几乎无质量损失。

## 研究背景与动机

视频扩散 Transformer（Video DiT）模型在生成质量上表现出色，但推理速度极慢，难以实际部署。两个核心瓶颈制约着效率：

- **自注意力二次复杂度**：视频 token 数量 = 帧数 × 空间分辨率，一段短视频可能有数万个 token，自注意力计算量随 token 数量平方增长
- **多步扩散过程**：扩散模型需要数十步迭代去噪，每步都必须进行完整的 Transformer 前向传播

现有加速方法如 token 合并（TokenMerge）、注意力广播（PAB）等取得了一定加速效果，但加速比通常有限（2-4×）。本文从扩散去噪过程的特性出发，提出两个关键观察：

1. **早期步骤不需要高分辨率**：去噪初期主要确定粗粒度结构（整体布局、主体位置），低分辨率潜表示即可胜任
2. **后期步骤不需要密集注意力**：去噪后期主要进行局部细节精化，全局注意力中大量远距离 token 交互是冗余的

这两个观察为两种互补的加速策略提供了理论基础。

## 方法详解

### 整体框架

Jenga 是一个纯推理时的加速管线，不修改模型参数。包含两个核心组件，在去噪过程的不同阶段协同工作：

- 前半段：使用渐进分辨率——从低分辨率潜表示开始去噪，逐步上采样到目标分辨率
- 后半段：使用动态注意力剪裁——限制每个 token 仅与局部相关 token 交互

### 关键设计

1. **基于 3D 空间填充曲线的块级注意力**：将视频的时空 token 排列视为 3D 网格（时间 × 高 × 宽），利用 3D Hilbert 曲线等空间填充曲线将其线性化排序。空间填充曲线的关键性质是保持空间局部性——曲线上相邻的点在 3D 空间中也大致相邻。之后将排序后的 token 划分为固定大小的块，每个 token 仅与同块内的 token 进行注意力计算。这种设计既保留了时空局部性（块内 token 在空间和时间上相近），又自然引入了跨帧交互（Hilbert 曲线在时间维度上的折叠使不同帧的空间邻近 token 可能落在同一块中）。

2. **动态稀疏度调度**：注意力的稀疏程度随去噪步骤动态变化。早期步骤使用较大的注意力块（更接近全局注意力），因为粗粒度结构需要全局信息；后期步骤使用更小的块（更高稀疏度），因为细节精化主要是局部操作。稀疏度随去噪进程线性或分段增加，避免任何步骤的突然切换导致的质量损失。

3. **渐进分辨率生成**：将扩散去噪过程分为多个阶段。前 N 步在低分辨率潜空间进行去噪（通过下采样 latent），随后通过插值或学习到的上采样操作逐步提升分辨率，最后几步在全分辨率下精细化。低分辨率去噪不仅减少了每步的 token 数量（直接降低注意力计算量），还减小了潜表示的尺寸，加速所有层的计算。

### 损失函数 / 训练策略

Jenga 完全不需要训练——它仅修改推理时的注意力计算方式和潜表示分辨率调度，可直接应用在任何预训练 Video DiT 模型上。这是其最大的实用优势之一。

## 实验关键数据

### 主实验（VBench 基准）

在 CogVideoX-5B 上的对比：

| 方法 | VBench 总分 ↑ | 加速比 ↑ | 需要训练 |
|------|-------------|---------|---------|
| CogVideoX（基线） | 82.14 | 1.0× | — |
| TokenMerge | 81.53 | 2.1× | 否 |
| PAB | 81.72 | 2.8× | 否 |
| FastVideo | 81.89 | 3.2× | 是 |
| Delta-DiT | 81.95 | 3.5× | 否 |
| **Jenga** | **82.13** | **8.83×** | **否** |

Jenga 的加速比是第二名 Delta-DiT 的 2.5 倍，同时 VBench 分数几乎无损（-0.01%），在所有 training-free 方法中独领风骚。

### 跨模型泛化

| 基座模型 | 加速比 | VBench 变化 |
|---------|-------|-----------|
| CogVideoX-5B | 8.83× | -0.01% |
| Open-Sora | 6.92× | -0.15% |
| Latte | 6.09× | -0.23% |
| HunyuanVideo | 8.16× | -0.08% |

无需任何调整即可迁移到不同 Video DiT 架构，加速比稳定在 6-9×，展现了极强的通用性。

### 消融实验

| 配置 | VBench | 加速比 |
|------|--------|-------|
| 完整 Jenga | 82.13 | 8.83× |
| 仅动态注意力剪裁 | 82.08 | 4.12× |
| 仅渐进分辨率 | 82.11 | 3.85× |
| 固定稀疏度（不动态） | 81.65 | 7.21× |
| 随机 token 选择（非曲线） | 80.32 | 8.56× |

### 关键发现

- 两个组件的加速效果是近似相乘的关系（4.12× × 3.85× ≈ 8.83× 中的大部分），说明它们作用在不同维度上
- 3D 空间填充曲线对比随机 token 选择有巨大优势（82.13 vs 80.32），证明保持时空局部性对生成质量至关重要
- 动态调度 vs 固定稀疏度的对比（82.13 vs 81.65）表明，让稀疏度匹配去噪阶段的需求十分关键
- 推理时间从分钟级降至秒级（如 CogVideoX 从 342s 降至 38.7s），使交互式视频生成成为可能

## 亮点与洞察

- **数学工具的妙用**：将 Hilbert 曲线等空间填充曲线引入 3D 视频 token 排序，保持时空局部性的同时天然实现跨帧交互——这一思路远优于简单的时空分块或随机稀疏化
- **两个互补加速维度的协同**：分辨率加速（减少 token 总数）和注意力加速（减少每个 token 的交互范围）在不同去噪阶段各司其职，加速效果近乎相乘
- **极高实用价值**：无需训练、即插即用、有开源代码，任何使用 Video DiT 的团队都可以直接受益
- **加速比遥遥领先**：8.83× 在 training-free 视频生成加速领域极为突出，且质量损失可忽略

## 局限性 / 可改进方向

1. 对于需要强全局一致性的长视频（如长距离物体运动、场景切换），后期步骤的局部注意力可能造成时空不一致
2. 渐进分辨率策略的调度参数（何时上采样、上采样倍率）需要针对不同模型和视频长度调优
3. 主要在 VBench 上评估质量，该基准侧重短视频，在长视频（>100帧）上的效果未知
4. 空间填充曲线的选择（Hilbert vs Z-order vs 其他）对结果的影响未深入消融
5. 理论分析相对薄弱，对"为什么早期不需要高分辨率"和"为什么后期不需要全局注意力"主要依赖实验观察

## 相关工作与启发

- **TokenMerge**：通过合并相似 token 减少序列长度，思路不同但可与 Jenga 的块级注意力互补
- **PAB（Progressive Attention Broadcasting）**：利用相邻去噪步之间注意力图的相似性进行广播，是另一正交的加速维度
- **Delta-DiT**：利用相邻步的特征差异进行增量计算，与 Jenga 的动态稀疏度调度思路有一定互补性
- **FlashAttention**：底层注意力计算的 IO 优化，可与 Jenga 的块级稀疏注意力叠加使用
- **启发**：渐进分辨率的思路可能推广到图像生成（前几步低分辨率后面高分辨率）、3D 生成等其他迭代生成场景

## 评分

⭐⭐⭐⭐⭐ (4.5/5)

理由：加速效果极其出色（8.83×），质量损失极小（0.01%），无需训练且跨模型通用。空间填充曲线的引入优雅地解决了 3D token 稀疏化的核心难题。扣分点在于理论分析深度不足，以及长视频场景的验证不充分。
