# Poly-Guard: Massive Multi-Domain Safety Policy-Grounded Guardrail Dataset

**会议**: NeurIPS 2025 (Dataset & Benchmark)  
**arXiv**: [2506.19054](https://arxiv.org/abs/2506.19054)  
**代码**: 有  
**领域**: AI Safety / 内容安全  
**关键词**: 安全护栏, 多领域安全, 策略对齐, 对抗攻击, 过拒绝

## 一句话总结

提出 Poly-Guard，首个大规模多领域安全策略驱动的护栏数据集，覆盖8个安全关键领域，基于真实行业安全指南构建风险分类体系，并对19个先进护栏模型进行系统性评估。

## 研究背景与动机

LLM 在各行业的广泛应用带来了内容安全的紧迫需求。现有护栏基准存在三个关键缺陷：

**缺乏标准化政策基础**：现有基准基于临时性风险分类，与真实行业安全规范脱节
**忽视领域特定风险**：同一风险类别在不同领域（如金融、法律、医疗）有不同含义和严重程度
**良性数据不足**：缺乏高质量的挑战性良性数据，导致模型过度拒绝

Poly-Guard 旨在填补这些空白，提供第一个与行业安全策略对齐的大规模护栏评估数据集。

## 方法详解

### 整体框架

Poly-Guard 的构建遵循五大设计原则：
1. 广泛的领域覆盖
2. 基于真实政策的风险构建
3. 多样化的交互格式
4. 高质量良性数据筛选
5. 对抗攻击增强

### 关键设计

**领域覆盖（8个安全关键领域）**：
- 金融 (Finance)
- 法律 (Law)
- 代码生成 (CodeGen)
- 医疗 (Healthcare)
- 教育 (Education)
- 政府 (Government)
- 通用 (General)
- 内容创作 (Content Creation)

**策略驱动的风险分类构建**：
- 每个领域基于真实行业安全指南和法规构建风险分类
- 例如金融领域参考 SEC、FINRA 等监管机构的指引
- 法律领域参考律师职业道德规范
- 这确保了风险分类的实际操作意义

**多样化交互格式**：
- 陈述句 (Declarative statements)
- 问题 (Questions)
- 指令 (Instructions)
- 多轮对话 (Multi-turn conversations)

**去毒化提示的良性数据**：
- 使用 detoxification prompting 技术生成与有害内容表面相似但实际无害的数据
- 专门用于挑战模型的过度拒绝行为
- 确保模型不会因过于保守而拒绝合理请求

**攻击增强实例**：
- 模拟对抗输入：越狱提示、角色扮演、编码混淆等
- 设计用于绕过安全护栏的高级攻击手法
- 测试模型在对抗条件下的鲁棒性

### 损失函数 / 训练策略

作为基准数据集，Poly-Guard 本身不涉及训练。评估协议包括：
- 二分类评估：安全/不安全
- F1 分数作为主要指标
- 分领域和分风险类别的细粒度评估
- 对抗攻击下的鲁棒性评估

## 实验关键数据

### 主实验

19个护栏模型的跨领域F1分数：

| 模型类别 | 代表模型 | 平均F1 | 领域方差 | 对抗鲁棒性 |
|---------|---------|-------|---------|----------|
| 商业API | OpenAI Mod | 中等 | 高 | 中等 |
| 开源大模型 | Llama Guard 3 | 较高 | 中等 | 较弱 |
| 专用护栏 | WildGuard | 较高 | 中等 | 中等 |
| 小型模型 | DistilBERT-based | 低 | 极高 | 弱 |

分领域性能差异：

| 领域 | 最佳模型F1 | 最差模型F1 | F1差距 | 主要失败点 |
|------|----------|----------|-------|----------|
| 通用 | ~0.85 | ~0.45 | 0.40 | 微妙有害内容 |
| 金融 | ~0.75 | ~0.30 | 0.45 | 领域特定风险 |
| 法律 | ~0.70 | ~0.25 | 0.45 | 法律专业语境 |
| 代码生成 | ~0.65 | ~0.20 | 0.45 | 恶意代码检测 |
| 医疗 | ~0.72 | ~0.28 | 0.44 | 医疗误导信息 |

### 消融实验

| 分析维度 | 关键发现 |
|---------|---------|
| 模型演化 | 新模型覆盖更多风险类别，但常见类别性能可能下降 |
| 过拒绝率 | 部分模型过拒绝率 >20%，尤其在良性边界样本上 |
| 对抗鲁棒性 | 所有模型对优化的对抗攻击仍然脆弱 |
| 领域迁移 | 通用领域训练的模型在专业领域表现显著下降 |

### 关键发现

1. **所有模型的F1分数差异大**：在不同风险类别间表现不一致，暴露出领域覆盖限制
2. **模型演化的矛盾**：随着模型迭代，安全风险覆盖变广，但在常见风险类别上的性能可能反而下降
3. **对抗攻击脆弱性**：所有模型在优化的对抗攻击下仍然容易被绕过
4. **过拒绝问题普遍**：多数模型在良性但敏感话题上存在过度拒绝
5. **领域特异性不可忽视**：同一概念在不同领域的安全含义需要不同的判断标准

## 亮点与洞察

1. **首个政策对齐基准**：与行业安全规范直接关联，确保实际操作意义
2. **系统性去毒化**：良性数据的构建方法新颖且有效
3. **多维度评估框架**：覆盖领域、风险类别、交互格式、对抗条件
4. **实用洞察丰富**：对现有模型的发现直接服务于护栏系统改进

## 局限性 / 可改进方向

1. 8个领域仍不够全面，可扩展至更多行业（如军事、自动驾驶）
2. 安全政策具有时效性和地区差异，需要持续更新
3. 主要关注英语内容，多语言安全评估是重要扩展方向
4. 多模态内容（图像、视频中的安全问题）未涉及
5. 安全标注的一致性和主观性问题需要更多讨论

## 相关工作与启发

- **ToxiGen/SafetyBench**：通用安全基准，缺乏领域特异性
- **Llama Guard**：Meta 的护栏模型系列
- **WildGuard**：开放域护栏模型
- 启发：安全评估需要从通用走向领域定制，政策对齐是关键方向
