# Connecting the Dots: A Machine Learning Ready Dataset for Ionospheric Forecasting Models

**会议**: NeurIPS 2025 (Workshop on ML for Physical Sciences)  
**arXiv**: [2511.15743](https://arxiv.org/abs/2511.15743)  
**代码**: [GitHub](https://github.com/FrontierDevelopmentLab/2025-HL-Ionosphere)  
**领域**: 时间序列 / 空间天气 / 遥感  
**关键词**: 电离层预测, TEC全球预报, 多源异构数据对齐, 空间天气ML数据集, 时空预测

## 一句话总结

作为2025 NASA Heliolab的成果，本文构建了首个全面的ML-ready电离层预测数据集，将太阳动力学观测站（SDO）极紫外辐照度嵌入、太阳风参数、行星际磁场、地磁活动指数、JPL稠密TEC全球电离层图、Madrigal稀疏TEC、太阳通量指数以及轨道力学参数等7大类异构数据源统一对齐到一致的时间-空间结构中，并在此基础上训练了包括LSTM、球面神经算子（SFNO）和GraphCast在内的多种时空预测架构，实现了对全球垂直总电子含量（vTEC）在安静和地磁活跃条件下长达12小时的自回归预测，超越了持续性基线。

## 研究背景与动机

**领域现状**：现代社会高度依赖一系列复杂的技术基础设施——全球导航卫星系统（GNSS）为数十亿设备提供精确定位服务，低轨道（LEO）卫星星座（如Starlink）承载着越来越多的通信任务，航空网络依赖GNSS进行精密进近导航，电力电网需要防范地磁感应电流的冲击。这些基础设施都极易受到太阳活动引发的空间天气扰动的影响。太阳耀斑、日冕物质抛射（CME）和高能粒子事件不仅对太空运营构成威胁，还能触发地球有效扰动（geo-effective disturbances），直接影响地面生活。电离层作为距地面约60-1000公里高度的电离大气层，是太阳-地球耦合链条中的关键环节——太阳辐射和粒子能量通过磁层传递到电离层-热层系统，导致电子密度的剧烈变化，这些变化直接影响穿过电离层的无线电信号的传播。用于量化这种影响的核心物理量是总电子含量（Total Electron Content, TEC），它衡量从地面到卫星的视线路径上单位面积柱上的自由电子总数，单位为TECU（1 TECU = $10^{16}$ 电子/m²）。准确预测TEC的时空分布对于GNSS定位误差校正、无线电通信链路预算以及太空态势感知至关重要。

**现有痛点**：过去几十年中，监测近地空间环境的观测能力显著增强。NASA的SDO卫星持续提供高分辨率太阳极紫外（EUV）观测，ACE和Wind卫星在日地L1点测量太阳风和行星际磁场参数，全球数百个GNSS地面站提供广泛的TEC测量，而地磁台站网络则记录着Kp、AE、SYM-H等地磁活动指数。然而，这些丰富的多源数据面临着一个根本性的困境：它们分散在不同的数据平台上（NASA GSFC的OMNIWeb、NOAA SWPC、JPL、MIT Haystack Observatory的Madrigal数据库等），采用着截然不同的数据格式和时间分辨率（从SDO的15秒到Kp指数的3小时再到太阳通量的日值），空间覆盖和网格体系也各不相同。更关键的是，这些数据产品在设计之初就不是为机器学习工作流量身打造的——它们的缺失值编码方式不统一（例如OMNI数据集对不同通道使用不同的哨兵值来标记缺失），时间戳格式各异，需要大量的领域知识和预处理工作才能转化为模型可以直接使用的训练数据。这种状况严重阻碍了ML模型的开发和系统性比较。

**核心矛盾**：多源异构观测数据的科学丰富性与机器学习管线对统一、清洁、即插即用数据的工程需求之间存在根本性的鸿沟。太阳活动到电离层响应是一条复杂的因果链——太阳EUV辐射直接电离上层大气，太阳风携带的行星际磁场通过磁重联将能量注入磁层和电离层，而地磁指数则反映了这一能量注入的强度。要构建能够捕捉这一完整因果链的预测模型，就必须将这些跨越不同时间尺度和空间域的数据融合到一个连贯的框架中。然而目前缺乏这样的标准化ML-ready数据集，使得研究者不得不各自为战地进行繁琐的数据预处理，导致模型之间缺乏可比性，也限制了社区整体的研究进展。

**本文目标** 本文的目标可以分解为三个具体的子问题：（1）如何将来自7个以上数据源、涵盖多种物理量、具有不同时间分辨率和空间覆盖的异构数据系统性地对齐到统一的时间-空间结构中？（2）如何处理多源数据中普遍存在的缺失值、数据间隙和非标准编码问题，使最终产品干净可靠？（3）如何在此数据集上建立有意义的ML基准测试，涵盖不同地磁条件并防止数据泄漏？

**切入角度**：作者团队是2025年NASA前沿发展实验室（Frontier Development Lab, FDL）Heliolab项目的成员，这是NASA、Trillium Technologies、Google Cloud、NVIDIA和Pasteur Labs联合支持的研究项目。这一背景使他们能够直接访问所有相关数据源，并得到来自JPL、NASA戈达德等机构的领域专家支持。他们从一个关键观察出发：尽管电离层建模社区拥有丰富的多模态观测数据，但将这些数据"连接起来"（Connecting the Dots）形成一个完整的太阳-地球因果链数据产品，是释放数据驱动方法潜力的必要前提。

**核心 idea**：通过构建一个模块化、时空对齐的ML-ready数据管线，将太阳-太阳风-磁层-电离层因果链上的7类异构数据源统一到单一数据结构中，为电离层预测的ML基准测试提供标准化基础设施。

## 方法详解

### 整体框架

本文的方法框架是一个端到端的数据集构建和基准测试管线。输入端是来自7个不同数据平台的原始异构数据流，包括卫星遥感、地基网络、在线数据库等多种来源；中间经过数据采集、清洗、缺失值处理、时间对齐和空间网格映射等步骤；输出端是一个统一的、按时间索引的模块化数据产品，存储在Google Cloud Bucket上并配套开源的PyTorch数据加载代码。在此基础上，团队训练了名为IonCast的一系列时空预测模型（LSTM基线、SFNO、GraphCast），以JPL的稠密TEC全球电离层图（GIM）作为预测目标，在15分钟的时间分辨率下进行全球vTEC的自回归预报，预报时效达到12小时。

整个管线的设计理念是"模块化"和"可复现"：每个数据源的处理逻辑是独立的模块，用户可以根据需要选择性地使用部分数据源；所有的处理参数（如缺失值填充的最大回溯时间）都可以由用户自定义；完整的代码在GitHub上公开，确保结果可复现。

### 关键设计

1. **多源异构数据的系统性整合**:

    - 功能：将7大类数据源的原始数据流统一到一个按时间索引、在纬度-经度网格上对齐的模块化数据结构中
    - 核心思路：数据整合遵循太阳-地球因果链的物理逻辑——从驱动端（太阳辐射、太阳风）到响应端（地磁活动、电离层TEC）依次整合。具体而言，7类数据源包括：（a）**NASA/GSFC OMNI2太阳风和磁场数据**——通过OMNIWeb获取，包含地磁指数（AU、AL、AE反映极光电急流，SYM-H反映环电流，ASY-D反映部分环电流不对称性）和太阳风/IMF参数（行星际磁场分量 $B_x$、$B_y$、$B_z$ 以及太阳风速度分量 $v_x$、$v_y$、$v_z$），原始时间分辨率为1分钟。（b）**NOAA SWPC/GFZ地磁指数**——提供3小时分辨率的Kp和Ap指数，Kp是一个全球性的准对数地磁活动指标，Ap是其线性化版本。（c）**JPL Dense TEC GIM**——NASA喷气推进实验室基于全球GNSS站网的日常业务处理生成的全球电离层图，空间分辨率为1°×1°的纬度-经度网格，时间分辨率15分钟，单位TECU。这是模型的核心预测目标。（d）**Madrigal Sparse TEC**——由MIT Haystack天文台托管的GNSS接收器衍生的TEC图，同样映射到1°×1°网格但更稀疏，时间分辨率5分钟。（e）**SDO-FM EUV辐照度嵌入**——利用SDO太阳动力学观测站的全日面极紫外观测数据，通过新颖的变分自编码器（NVAE）方法压缩为低维嵌入向量，原始时间分辨率仅15秒，为数据集中最快的数据流。（f）**太阳通量指数**——包括F10.7（10.7厘米太阳射电通量）、S10.7、M10.7、Y10.7等多波长太阳通量以及JB08热层加热率dSt/dt，日分辨率。（g）**轨道力学参数**——根据太阳和月球相对于地球的几何位置计算的派生特征，包括太阳/月球天顶角、亚太阳/亚月球点的经纬度、太阳/月球对足点位置以及日-地和月-地距离等。这些几何特征对于捕捉TEC的日变化和潮汐调制至关重要。此外还包括**准偶极坐标**（Quasi-Dipole Coordinates），这是将地球磁力线投影到地理参考系的年度映射表，有助于高纬度地区的电离层建模。所有数据的时间范围统一对齐到SDO基础模型的覆盖期2010年5月13日至2024年8月1日。
    - 设计动机：电离层的物理驱动力不是单一的，而是来自完整的太阳-太阳风-磁层-电离层耦合链。只使用单一数据源（如之前大多数ML工作仅使用TEC历史数据进行自回归预测）无法捕捉外部驱动因子的变化。本数据集的价值在于将因果链上的所有关键环节的数据整合到一个可以直接"插入"ML训练管线的标准化格式中，使研究者可以系统性地探索不同数据组合对预测性能的影响。

2. **缺失值处理与时间对齐策略**:

    - 功能：系统性地处理多源数据中普遍存在的缺失值、数据间隙和非标准编码，并将不同时间分辨率的数据流对齐到统一的时间步长
    - 核心思路：缺失值处理采用三级策略。第一级是**标准化缺失值编码**：不同数据源使用不同的约定来标记缺失值（例如OMNI数据集对不同物理通道使用不同的哨兵数值，这些值看似有效数据实则表示缺失），统一将所有缺失值表示为NaN。第二级是**大间隙数据列剔除**：如果某个特征通道存在多年的持续数据缺失，则直接从数据集中移除该列，避免引入过多噪声。第三级是**小间隙前向填充**：对于短暂的数据间断，使用最近的有效值进行前向填充（forward-filling）。关键参数是"最大回溯时间"（maximum rewind time），对大多数数据流设定为等于其原始采样间隔（如3小时数据的回溯时间为3小时），以确保只填充微小中断。唯一的例外是OMNI数据，其回溯时间设为50分钟（原始分辨率为1分钟），以适应该数据源较频繁的短暂中断。如果某个间隙超过了回溯时间阈值，则直接跳过对应的时间戳，不进行填充，以避免传播陈旧数据。这一前向填充逻辑同时也被用作将所有特征重采样到标准时间步长（如15分钟）的简单插值策略。
    - 设计动机：缺失值处理是多源数据融合中最容易被忽视却最关键的环节之一。如果不仔细处理，模型可能学到虚假的模式（如将哨兵值当作物理有效值），或者因为大面积NaN导致训练不稳定。前向填充策略的选择是一个务实的折中——它足够简单、物理上合理（假设短时间内物理量变化不大），且不引入未来信息。最大回溯时间参数的设置为用户提供了灵活性，可以根据具体应用场景调整数据质量和完整性之间的平衡。

3. **地磁风暴事件目录与数据分割**:

    - 功能：构建基于物理的地磁风暴事件分类目录（MESTICI量表），用于指导训练/验证/测试集的划分，防止数据泄漏
    - 核心思路：基于Kp指数时间序列，使用简单阈值方法将整个14年（2010-2024）的时间范围划分为不同的子区间。每个子区间被赋予一个事件标识符，由两部分组成：**NOAA G-level**（基于Kp值的地磁扰动等级，从G0-Calm $K_p<5$ 到G5-Extreme $K_p \ge 9$）和**持续时间** $\ell$（以小时为单位）。例如，事件ID "G2H6"表示一个达到G2级别（中度风暴，$6 \le K_p < 7$）且持续至少6小时的地磁事件。NOAA的G-level分级为：G0（$K_p<5$，安静）、G1（$5 \le K_p < 6$，轻微风暴）、G2（$6 \le K_p < 7$，中度风暴）、G3（$7 \le K_p < 8$，强风暴）、G4（$8 \le K_p < 9$，严重风暴）和G5（$K_p \ge 9$，极端风暴）。这一事件目录的核心用途是确保在训练/验证/测试划分时，同一个地磁风暴事件不会被拆分到不同的数据子集中——这是电离层预测领域一个特别容易发生的数据泄漏形式，因为一次大的地磁风暴可能持续数天，其前数据和后恢复期在物理上高度相关。
    - 设计动机：在传统的随机时间分割方案下，模型可能因为在训练集中"见过"了某次风暴的恢复阶段而能在测试集中准确预测同一风暴的高峰阶段——这种性能是虚假的，不具备真正的泛化能力。基于事件的分割策略确保了模型在评估时面对的是它从未见过的完整地磁事件，从而更真实地反映了模型在业务预报场景中的表现。事件持续时间的加入使分类更加精细，有助于研究者针对不同强度和持续时间的风暴事件分析模型表现。

### 损失函数 / 训练策略

本文的核心贡献在于数据集而非模型，因此对训练策略的描述相对简洁。IonCast系列模型均在15分钟时间步长的对齐数据上训练，以JPL Dense TEC GIM作为预测目标。训练采用自回归方式，模型在每个时间步预测下一个时间步的全球TEC图（1°×1°网格），然后将预测结果作为下一步的输入进行迭代预报，预报时效可达12小时（即48个时间步的自回归展开）。PyTorch数据集类支持用户指定训练/验证/测试的时间范围，并提供针对每个数据源的归一化方案。数据分割基于地磁风暴事件目录进行，确保同一事件不被跨集分割。基线模型为持续性预报（persistence forecast，即假设TEC保持不变），所有IonCast模型均需超越该基线。

### 消融实验

由于本文定位为数据集论文，消融实验的详细结果在配套的IonCast模型论文中。但从数据集设计角度，以下方面构成了隐含的消融维度：

| 数据组合配置 | 预期效果 | 说明 |
|------------|---------|------|
| 仅历史TEC（自回归） | 基本性能 | 无外部驱动信息 |
| TEC + 太阳风/IMF | 性能提升 | 捕捉太阳风驱动的电离层扰动 |
| TEC + 地磁指数 | 性能提升 | 反映磁层-电离层耦合 |
| TEC + SDO EUV嵌入 | 性能提升 | 捕捉太阳EUV辐射的直接电离效应 |
| 全部数据源 | 最佳性能 | 完整因果链信息 |
| 安静条件（Kp<5） | 较好预测 | 物理过程更规律 |
| 活跃条件（Kp≥5） | 预测退化 | 非线性耦合增强，物理过程更复杂 |

### 关键发现

- **地磁活跃期的预测挑战更大**：在Kp≥5的地磁风暴期间，电离层的变化更加剧烈和非线性，TEC预测的误差显著增大。这与物理预期一致——风暴期间高纬度粒子沉降、对流电场穿透到赤道区域等过程引入了额外的复杂性。然而，多源驱动数据的输入（特别是太阳风参数和地磁指数）有助于在风暴期间改善预测。
- **数据集的模块化设计支持灵活的实验**：研究者可以方便地选择不同的数据源组合，用于系统性地研究哪些物理驱动因子对不同条件下的TEC预测最有价值。这是此前使用非标准化数据难以实现的。
- **事件目录对防止数据泄漏至关重要**：基于Kp阈值的MESTICI事件分类方案提供了合理的训练/测试分割策略，确保模型评估结果反映的是真实的泛化能力，而非对同一风暴事件不同阶段的记忆。
- **安卓手机GNSS众包数据的潜力**：数据集中包含来自Madrigal的稀疏TEC数据，其中一部分来源于安卓手机的GNSS接收器。尽管单个手机的测量噪声较大，但海量设备的众包效应可以显著增加TEC观测的空间覆盖密度，特别是在传统GNSS站网稀疏的海洋和发展中国家地区。
- **SDO基础模型嵌入为整合卫星图像提供了高效路径**：直接使用SDO全日面EUV图像会引入极高的数据维度，而通过SDO基础模型（SDO-FM）将其压缩为紧凑的嵌入向量，在保留关键太阳活动信息的同时大幅降低了计算负担。
- **14年的数据跨度覆盖了超过一个完整的太阳活动周期**（第24周期的下降段和第25周期的上升段），这对于训练能够应对不同太阳活动水平的鲁棒模型至关重要。

## 亮点与洞察

- **太阳-地球全因果链的数据整合思路**：这是本文最核心的贡献。以往的电离层ML工作大多只使用TEC历史数据做自回归预测，或者至多加入一两个驱动指标。本文系统性地将从太阳表面到电离层响应的完整因果链上的所有关键观测量整合到一起，为社区提供了一个可以直接使用的"一站式"数据产品。这种"数据即基础设施"的思路正是ML for Science领域目前最需要的——许多科学领域并不缺观测数据，而是缺少将异构数据转化为ML-ready格式的系统性工程。

- **基于物理的数据分割策略（MESTICI事件目录）**：这个设计非常巧妙。在气象和空间天气预报中，数据泄漏是一个经常被忽视的问题——时间上相邻的数据点在物理上高度相关，随机分割几乎必然导致信息泄漏。通过将数据按地磁事件的自然边界来分割，确保了训练集和测试集之间的物理独立性。这一策略可以推广到其他具有事件驱动特征的时间序列预测任务中。

- **前向填充机制中"最大回溯时间"参数的引入**：这是一个工程上极其务实的设计。不同数据源的自相关时间尺度不同——1分钟分辨率的太阳风数据在几分钟内可能发生剧烈变化（特别是在行星际激波到来时），而日分辨率的太阳通量在一天内变化甚微。通过为每个数据源设定不同的最大回溯时间，系统在保持数据完整性的同时避免了不适当的外推。

- **安卓手机众包GNSS数据的创新性引入**：虽然论文中对此着墨不多，但将消费级手机的GNSS测量纳入电离层数据集是一个极具前瞻性的思路。随着支持原始GNSS观测的手机数量持续增长，这一数据源有潜力从根本上解决传统电离层观测的空间覆盖问题，特别是在GNSS基础设施薄弱的地区。

- **SDO-FM嵌入作为太阳观测数据的高效压缩手段**：直接将太阳全日面EUV图像（高分辨率2D数据）纳入电离层预测管线会带来巨大的计算和存储开销。使用预训练的太阳基础模型（SDO-FM）生成低维嵌入是一种优雅的解决方案——它既保留了太阳活动的关键物理信息，又避免了维度灾难。这一思路可以推广到其他需要整合高维遥感数据的时间序列预测任务中。

## 局限与展望

- **作为Workshop论文，基准测试的深度有限**：虽然提到了LSTM、SFNO和GraphCast三种架构，但具体的实验设计、超参数选择、训练细节和详细的数值比较都推迟到了配套的IonCast论文中。数据集论文本身缺乏对不同数据源组合的系统性消融实验，无法定量回答"哪些数据源对TEC预测贡献最大"这一核心问题。

- **空间分辨率受限于GNSS站网分布**：JPL Dense TEC GIM虽然覆盖全球，但其1°×1°的网格分辨率是通过对稀疏站点数据进行插值得到的，在海洋和南半球高纬度地区的数据质量较差。Madrigal Sparse TEC更是直接反映了GNSS站网的不均匀分布。这意味着模型在这些区域的预测性能可能不可靠，但数据集本身无法解决这一观测基础设施层面的问题。

- **缺失值处理策略偏简单**：前向填充虽然实用，但在太阳风参数快速变化的场景下（如行星际激波到来时参数可能在1-2分钟内跳变数倍），50分钟的最大回溯时间可能导致填充的值与真实值差距很大。更高级的插值方法（如基于物理约束的插值或基于神经网络的数据填补）可能提供更好的数据质量。

- **未包含热层参数和中性大气模型**：电离层的状态不仅受太阳和地磁驱动，还与热层的中性大气密度、温度和组成密切相关（通过碰撞电离和复合过程耦合）。数据集中缺乏经验热层模型（如NRLMSISE-00或JB2008）的输出参数，限制了模型捕捉热层-电离层耦合效应的能力。

- **预测目标仅限于垂直TEC（vTEC）**：vTEC是路径积分量，丢失了电离层电子密度的垂直结构信息。对于某些应用（如精密单点定位PPP或无线电掩星反演），需要电子密度剖面的预测，而非仅仅是柱积分值。未来可以考虑引入电离层探测仪（ionosonde）数据和GPS无线电掩星（RO）数据来支持三维电离层建模。

- **时间覆盖跨度虽达14年但仅涵盖约1.5个太阳周期**：第24太阳周期（约2008-2019年）是近一个世纪以来最弱的太阳周期之一，而第25周期（自2019年起）的活动水平显著更高。这种太阳活动水平的不均衡可能导致模型在强太阳活动条件下的训练数据不足。延伸时间覆盖到更早的太阳周期（如通过整合非SDO来源的老数据）会增加数据的多样性。

- **可扩展的改进方向**：包括引入电离层闪烁（scintillation）数据以支持GNSS信号质量预测、整合COSMIC-2星座的GPS RO数据以提供三维电子密度信息、以及利用即将发射的ESA Vigil L5卫星的侧视太阳观测来改善CME和太阳风预报的前置时间。

## 相关工作与启发

- **vs JPL GIM (Mannucci et al., 1998; Martire et al., 2024)**：JPL的全球电离层图是本数据集的核心预测目标之一。JPL GIM本身已经是一个经过复杂处理的数据产品（基于Kalman滤波和球谐分析从稀疏GNSS观测中估计全球TEC分布），但它的输出格式（IONEX文件）不是为ML工作流设计的。本数据集的价值在于将GIM TEC与其他驱动数据源对齐，提供标准化的PyTorch接口。

- **vs 传统物理模型 (IRI, NeQuick, SAMI3)**：国际参考电离层（IRI）和NeQuick等经验/半经验模型使用参数化的物理公式来描述电离层，而SAMI3等第一性原理模型则数值求解电离层的连续性方程和动量方程。这些模型在捕捉电离层的平均行为方面表现不错，但在地磁活跃期的预测上存在困难。ML方法有潜力通过从大量观测数据中学习来弥补物理模型的不足，而本数据集正是为此提供基础。

- **vs SDO基础模型 (Walsh et al., 2024)**：SDO-FM是本数据集的上游依赖之一。SDO-FM使用NVAE架构将SDO的AIA和EVE仪器数据压缩为紧凑的嵌入表示，旨在为各种下游太阳物理任务提供特征。本数据集展示了SDO-FM嵌入在电离层预测这一全新下游任务中的应用可能性。

- **vs GraphCast (Lam et al., 2023) 和 FourCastNet (Bonev et al., 2025)**：这些是气象预报领域的先进ML架构，本文将其思路迁移到空间天气领域。GraphCast使用图神经网络架构处理球面上的网格数据，SFNO（FourCastNet的后续工作）使用球面傅里叶变换来高效处理全球尺度的物理场。将这些架构应用于电离层预测是一个自然的扩展，因为TEC的全球分布同样是球面上的物理场。

- **vs 现有电离层ML工作**：此前的大量工作通常只使用单一数据源（如仅使用TEC历史数据或仅使用Kp/Dst指数），数据预处理方式各不统一，缺乏标准化的评估协议。本数据集通过提供统一的数据产品和基准测试框架，使得不同模型之间的系统性比较成为可能。

- **启发**：本文体现了ML for Science领域一个重要趋势——"数据集即论文"。在许多科学领域，高质量的、ML-ready的数据集的价值可能不亚于模型创新本身，因为它降低了入门门槛、促进了可比性、加速了社区整体的研究进展。对于其他具有多源异构观测数据的地球科学领域（如海洋学、地震学、大气化学），本文提供了一个可以参考的数据集构建范式：识别完整的物理因果链 → 收集因果链各环节的数据 → 统一时空对齐 → 构建事件目录防止泄漏 → 提供开源工具和基准模型。

## 评分

- 新颖性: ⭐⭐⭐⭐ 首个将太阳-地球因果链上多源异构数据系统性整合为ML-ready格式的电离层数据集，MESTICI事件目录的防泄漏设计有实际意义
- 实验充分度: ⭐⭐⭐ Workshop论文体量限制明显，基准测试的详细数值推迟到配套论文，缺乏数据源组合的消融实验
- 写作质量: ⭐⭐⭐⭐ 数据集描述系统性强，Table 1对数据源的总结清晰高效，整体逻辑流畅
- 价值: ⭐⭐⭐⭐ 对空间天气ML社区有重要的基础设施价值，为电离层数字孪生的构建奠定了数据基础，开源的数据和代码降低了入门门槛
