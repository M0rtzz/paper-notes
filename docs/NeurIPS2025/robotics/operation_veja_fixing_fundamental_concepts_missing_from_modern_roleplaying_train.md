# Operation Veja: Fixing Fundamental Concepts Missing from Modern Roleplaying Training Paradigms

**会议**: NeurIPS 2025
**arXiv**: [2601.06039](https://arxiv.org/abs/2601.06039)
**代码**: 有 (https://github.com/HyouinKyoumaIRL/Operation-Veja)
**领域**: NLP / 角色扮演 AI / 对话系统
**关键词**: 角色扮演, VEJA 框架, 数据策化, LLM 角色一致性, 合成数据

## 一句话总结

提出 VEJA（Values-Experiences-Judgments-Abilities）框架，通过结构化角色定义指导人工数据策化，解决现有角色扮演模型缺乏内在价值冲突推理和角色深度的系统性问题。

## 研究背景与动机

现代角色扮演（Roleplaying）模型虽然越来越复杂，但始终无法捕捉可信、引人入胜的角色本质。作者认为这种失败源于训练范式忽视了角色内在世界的动态交互——特别是价值冲突推理（value-conflicted reasoning）。

人类的互动不是检索"正确"回复的过程，而是不断在冲突价值观之间协商——例如礼貌与效率、好奇心与实用性、脆弱性与专业性之间的平衡。当前模型缺乏这种内在审慎过程，导致角色表现浅薄、可预测且缺乏说服力。

作者以《命运石之门》中的牧�的红莉栖（Makise Kurisu）为例：没有现有技术能复现她"智识好奇心与社交防卫性的冲突"这一核心特质，模型的回复仅仅是反应性的，从不具有真正的探究性或内在冲突驱动性。

## 方法详解

### 整体框架

VEJA 框架定义了角色深度的四个核心支柱，形成因果链：

**经验 → 价值观 → 判断 → 能力表达**

1. **Values（价值观）**：驱动角色目标和欲望的高层动机，是行动的"为什么"
2. **Experiences（经历）**：塑造价值观和判断的具体过往事件，是世界观的证据基础
3. **Judgments（判断）**：角色对世界的具体观点、信念和启发式规则，是价值观经由经历过滤后的输出
4. **Abilities（能力）**：角色的技能、知识和能力范围，定义了角色权威领域并影响其词汇和兴趣

### 关键设计：现有范式的系统性批判

作者深入分析了四种主流方法的根本局限：

**1. RAG 方法 — 缩放问题**
- 人类价值系统是组合爆炸式的，无法穷举
- 当引入情境修饰词（如"这个人很有魅力，所以我暂时放松专业判断"）时，复杂度超出检索能力

**2. 基于事实的价值注入 — 去语境化问题**
- 模型学会将价值观表示为简单孤立的公式
- 角色文件写"对陌生人开放"→模型过度依赖这单一特质，忽视社交暗示

**3. 文学对话学习 — 隐式上下文诅咒**
- 文学角色的对话背后推理比表面文字更复杂，但训练数据只能暗示审慎过程
- 文学作品通过叙述而非对话传递经历背景，模型无法学会将过去事件与当前行为关联

**4. 合成数据 — 鸡生蛋问题**
- 用大模型生成价值冲突对话，但生成器本身不具备这种能力
- 形成递归质量天花板：GPT-4 被要求生成冲突价值角色对话时，必然偏向某一价值

### 数据策化流程

以牧瀬红莉栖为案例：

- **基线（合成）数据集**：定义 10 天故事时间线 + 20 个用户，使用 Gemini Pro 2.5 生成
- **VEJA 策化（人工）数据集**：15 位人类写手获得相同时间线 + VEJA 结构化角色档案，根据指导写对话

### 评估方法

使用 LLM-as-judge（Gemini 2.5 Flash）进行盲测 A/B 对比：
- 随机配对两组对话，打乱顺序后匿名呈现
- 评估哪段对话更具角色一致性
- 进行 100 次成对比较

## 实验关键数据

### 主实验：LLM-as-Judge 偏好结果（N=100）

| 评估维度 | VEJA 策化（人工）胜出 | 合成基线胜出 | 平局 |
|---------|---------------------|-------------|------|
| 角色一致性偏好 | **43** | 28 | 29 |

排除平局后，VEJA 的胜率 = 43/(43+28) = **60.6%**

### 评审理由分析

| 案例 | 赢家 | 评审关键理由 |
|------|------|------------|
| Test #1 | VEJA（人工） | 基线对话让角色说中文（不符合角色设定）；VEJA 对话展现了锋利机智、被伪装的好奇心、对陌生人的怀疑 |
| Test #2 | VEJA（人工） | 基线对话完全违背角色——出现粗俗场景；VEJA 对话捕捉了智识好奇心、怀疑主义和微妙机智 |
| Test #4 | 合成基线 | 基线对话围绕数据、AI 模型展开，更贴合角色专业兴趣；VEJA 对话涉及个人生活话题，不太可能引起角色兴趣 |

### 关键发现

1. **VEJA 策化数据在角色一致性上显著优于合成数据**，评审频繁指出 VEJA 数据展现了更好的叙事连续性、更细腻的反应、角色性格与对话之间更清晰的联系
2. **合成数据的常见问题**：通用化、脱离角色、过度依赖单一特质
3. **VEJA 不能完全消除质量差异**：仍有 28 次合成数据胜出，说明人类写手质量参差不齐
4. **人工 vs 机器的混淆变量存在**，但作者论点是 VEJA 框架是人类写手实现高一致性的必要工具

## 亮点与洞察

- **四大范式的系统性批判非常深刻**：对 RAG、事实注入、文学学习、合成数据的失败模式分析透彻，每个都指出了根本性的结构问题而非工程问题
- **VEJA 的因果链设计**：经历→价值观→判断→能力，形成连贯的内在逻辑，类似斯坦尼斯拉夫斯基的表演理论
- **"递归质量天花板"概念精辟**：用大模型生成训练数据的根本矛盾——生成器本身不具备要学习的能力
- **以具体角色（牧瀬红莉栖）为案例**：让抽象框架有了具体的、可理解的锚点

## 局限性 / 可改进方向

- **数据集规模极小**：仅以单一角色原型（牧瀬红莉栖）为对象，15 位写手的 VEJA 数据 vs 20 个用户的合成数据
- **人工策化不可扩展**：论文承认手工过程劳动密集，扩展性存疑
- **评估依赖 LLM-as-judge**：评审本身有偏差（如位置偏差、风格偏差），且以 LLM 评估来挑战合成数据存在讽刺性
- **人工 vs 机器混淆变量**：胜出的是人工写的数据，无法完全剥离 VEJA 框架本身的贡献
- **缺少消融实验**：未分析 V、E、J、A 四个组件各自的贡献
- **未进行模型微调实验**：只比较了数据质量，未展示在 VEJA 数据上训练的模型表现如何

## 相关工作与启发

- **RoleLLM**：通过性格保真度基准评估角色扮演，但其事实化的评估框架可能导致去语境化问题
- **COSER**：协调 LLM 扮演已有角色，试图捕捉内在想法和动机，但训练数据（对话文本）只能暗示推理过程
- **ChatHaruhi**：从文学作品中复活动漫角色，面临隐式上下文和经历传递的挑战
- **Generative Agents**：展示了复杂智能体系统但底层角色缺乏深度推理
- **启发**：角色扮演 AI 的核心挑战不是能力问题，而是数据构建范式问题；需要从"模仿人格"转向"模拟人格"

## 评分

- **新颖性**: ⭐⭐⭐⭐ — VEJA 框架的概念贡献有原创性，对现有方法的批判有深度
- **有效性**: ⭐⭐⭐ — 实验规模太小，混淆变量未解决，结果有说服力但不够充分
- **可复现性**: ⭐⭐⭐⭐ — 数据集开源，方法描述清晰
- **影响力**: ⭐⭐⭐ — 对角色扮演 AI 社区的思路转变有启发，但需要更大规模验证
