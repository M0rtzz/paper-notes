# NeSyPr: Neurosymbolic Proceduralization For Efficient Embodied Reasoning

**会议**: NeurIPS 2025  
**arXiv**: [2510.19429](https://arxiv.org/abs/2510.19429)  
**代码**: 无  
**领域**: 具身智能 / 神经符号推理  
**关键词**: 神经符号推理, 具身智能, 知识编译, 过程化表示, 语言模型

## 一句话总结

NeSyPr 提出了一种神经符号过程化框架，将符号规划器生成的任务计划转化为可组合的过程化表示，嵌入到语言模型的推理过程中，从而在资源受限的环境中实现高效的具身推理。

## 研究背景与动机

在动态环境中部署语言模型（LM）执行具身任务面临三大挑战：

**延迟问题**：大规模推理引擎（如 GPT-4 级别模型）的在线调用延迟高，难以满足实时性要求

**连接性限制**：物理系统可能处于网络不稳定的环境中，无法持续访问云端服务

**资源约束**：边缘设备的计算资源有限，无法运行大规模模型或符号规划器

现有方法要么依赖大规模 LLM 进行多步推理（计算昂贵），要么依赖外部符号规划器（需在线访问），都不适合延迟敏感和资源受限的场景。人类认知科学中的"知识编译"概念——将陈述性知识转化为过程性知识——启发了本文的方法设计。

## 方法详解

### 整体框架

NeSyPr 包含三个核心阶段：

1. **符号计划生成**：利用符号工具（如 PDDL 规划器）基于其陈述性知识生成任务特定的计划
2. **过程化表示转换**：将显式符号计划转化为可组合的过程化表示，编码计划中的隐式产生式规则
3. **LM 推理集成**：将组合后的过程化表示无缝集成到 LM 的推理过程中

### 关键设计

**过程化表示（Procedural Representation）**：
- 从符号计划中提取产生式规则（production rules），即"条件-动作"对
- 将这些规则编码为向量表示，可以与 LM 的隐藏状态进行组合
- 支持模块化组合：不同子任务的过程化表示可以灵活组合

**知识编译机制**：
- 类比人类将"知道什么"（陈述性知识）转化为"知道怎么做"（过程性知识）的认知过程
- 将多步符号推理抽象为单步 LM 推理
- 编译后的模型不再需要外部符号指导

**可组合性设计**：
- 过程化表示是模块化的，可以跨任务复用
- 新任务可以通过组合已有的过程化模块来处理
- 支持动态环境中的自适应推理

### 训练策略

- 使用符号规划器生成多样化的任务计划作为训练数据
- 通过知识蒸馏将符号推理能力转移到紧凑的 LM 中
- 采用渐进式训练：先学习简单任务的过程化表示，再组合处理复杂任务

## 实验关键数据

### 主实验

在三个具身推理基准上的表现：

| 方法 | PDDLGym (成功率%) | VirtualHome (成功率%) | ALFWorld (成功率%) | 模型大小 |
|------|-------------------|----------------------|-------------------|---------|
| GPT-4 (CoT) | 82.5 | 71.3 | 68.7 | ~1.8T |
| LLaMA-70B | 75.8 | 63.2 | 61.4 | 70B |
| Symbolic Planner | 91.2 | 58.6 | 52.3 | - |
| ReAct | 70.3 | 65.8 | 63.5 | 70B |
| NeSyPr (7B) | 88.7 | 74.6 | 72.1 | 7B |
| NeSyPr (3B) | 85.2 | 70.9 | 68.4 | 3B |

### 效率对比

| 方法 | 平均推理时间 (s) | 内存使用 (GB) | 是否需要在线访问 |
|------|-----------------|-------------|----------------|
| GPT-4 (CoT) | 12.3 | - | 是 |
| Symbolic Planner | 2.1 | 4.2 | 是（规划器） |
| ReAct (70B) | 8.7 | 140 | 否 |
| NeSyPr (7B) | 0.8 | 14 | 否 |
| NeSyPr (3B) | 0.4 | 6 | 否 |

### 消融实验

| 变体 | PDDLGym | VirtualHome | ALFWorld |
|------|---------|-------------|---------|
| NeSyPr (完整) | 88.7 | 74.6 | 72.1 |
| 去除过程化表示 | 72.4 | 61.3 | 58.9 |
| 去除可组合性 | 81.5 | 68.7 | 65.2 |
| 去除符号计划引导 | 76.8 | 64.5 | 62.7 |

### 关键发现

1. NeSyPr 用 7B 参数模型就超越了 70B 模型和 GPT-4 的推理表现
2. 推理速度比 GPT-4 快约 15 倍，比 ReAct 快约 10 倍
3. 过程化表示是性能提升的最关键组件，去除后性能下降约 16%
4. 在不需要在线访问的情况下，仍能接近甚至超越符号规划器的表现

## 亮点与洞察

- **范式创新**：将认知科学中的知识编译概念引入具身 AI，提供了一种新的知识表示和推理范式
- **效率突破**：用紧凑模型实现了接近大规模模型的推理能力，适合边缘部署
- **可组合性**：过程化表示的模块化设计使得系统具有良好的泛化和扩展能力
- **实用价值**：解决了实际部署中的延迟和资源限制问题

## 局限性 / 可改进方向

1. 依赖符号规划器生成初始计划，限制了适用的任务类型
2. 当前评估环境相对结构化，在更开放的真实世界场景中的表现有待验证
3. 过程化表示的生成需要领域特定的 PDDL 建模
4. 未探讨持续学习和在线更新过程化知识的能力

## 相关工作与启发

- **知识编译理论**（Anderson, 1982）→ 认知科学中陈述性到过程性知识的转化启发了核心设计
- **ReAct**（Yao et al., 2023）→ LLM + 工具使用的范式，但需要大模型在线推理
- **SayCan**（Ahn et al., 2022）→ 语言模型与机器人动作的结合，但依赖大规模模型
- **神经符号 AI** 的最新进展为将符号推理与神经网络结合提供了理论基础
