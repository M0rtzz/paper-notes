# Towards Reliable Code-as-Policies: A Neuro-Symbolic Framework for Embodied Task Planning

**会议**: NeurIPS 2025 (Spotlight)  
**arXiv**: [2510.21302](https://arxiv.org/abs/2510.21302)  
**代码**: 无  
**领域**: 机器人 / 具身智能  
**关键词**: Code-as-Policies, 神经符号框架, 任务规划, LLM代码生成, 符号验证

## 一句话总结

提出一种神经-符号具身任务规划框架，在 LLM 代码生成过程中引入符号验证和交互式验证，使任务成功率比 Code-as-Policies 基线提升 46.2%，可执行性达 86.8%。

## 研究背景与动机

LLM 自动生成可执行代码（Code-as-Policies）已成为机器人任务规划的重要范式。然而，现有方法存在严重的**环境感知不足**问题：

**有限的环境推理**: LLM 生成的代码往往基于不完整的环境观测，难以处理动态或部分可观测场景

**代码正确性**: 生成的代码可能包含逻辑错误或不可执行的动作序列

**鲁棒性不足**: 在复杂环境中任务成功率低

本文的核心思路是：在代码生成过程中加入**符号验证**和**交互式探索**，让机器人先主动探索获取缺失信息，再生成可靠的执行代码。

## 方法详解

### 整体框架

框架分为两个主要阶段：
1. **符号验证阶段 (Symbolic Verification)**: 对 LLM 生成的代码进行形式化检查，验证前置条件和后置条件
2. **交互式验证阶段 (Interactive Validation)**: 生成探索性代码与环境交互，获取缺失观测

### 关键设计

1. **前置条件检查器 (Precondition Checker)**: 

    - 分析生成代码中每个动作的前置条件
    - 检查当前环境观测是否满足这些条件
    - 不满足时触发探索性交互

2. **探索性代码生成 (Exploratory Code Generation)**:

    - 当检测到信息缺失时，生成探索性代码（如"移动到目标附近观察"）
    - 关键约束：探索时必须保持任务相关状态不变（不能打翻已放好的物体等）

3. **状态保持约束 (State Preservation Constraint)**:

    - 定义一组不变量，确保探索性动作不破坏已有的任务进展
    - 通过符号推理验证探索代码的安全性

### 损失函数 / 训练策略

框架采用无训练的 prompt engineering 方法，不需要额外训练。核心流程：
- Input: 自然语言任务描述 + 环境 API
- LLM 生成初始代码 → 符号验证 → 探索性交互 → 代码修正 → 最终执行

## 实验关键数据

### 主实验（RLBench 仿真环境）

| 方法 | 任务成功率 (%) ↑ | 可执行性 (%) ↑ | 平均步数 ↓ |
|------|----------------|---------------|-----------|
| Code-as-Policies (CaP) | 38.5 | 52.3 | 12.7 |
| SayCan | 42.1 | 61.8 | 14.2 |
| ProgPrompt | 45.3 | 63.5 | 11.8 |
| Inner Monologue | 51.2 | 68.4 | 13.5 |
| VoxPoser | 48.7 | 65.2 | 12.1 |
| **本文 (Ours)** | **84.7** | **86.8** | **10.3** |

### 消融实验

| 组件 | 任务成功率 (%) | 可执行性 (%) | ΔSR |
|------|---------------|-------------|-----|
| 完整框架 | 84.7 | 86.8 | — |
| 去掉符号验证 | 62.3 | 71.5 | -22.4 |
| 去掉交互式验证 | 68.1 | 78.2 | -16.6 |
| 去掉状态保持约束 | 73.5 | 82.1 | -11.2 |
| 仅用单次生成 | 45.8 | 58.3 | -38.9 |
| 替换 GPT-4 → GPT-3.5 | 71.2 | 79.6 | -13.5 |

### 关键发现

1. 符号验证和交互式验证缺一不可，两者分别贡献约 22% 和 17% 的成功率提升
2. 在部分可观测场景中提升尤为显著（动态场景成功率从 25% 提升至 78%）
3. 真实机器人实验同样验证了框架的有效性
4. 框架对不同 LLM 后端均有效，但 GPT-4 效果最优

## 亮点与洞察

- **Spotlight 论文**: NeurIPS 2025 Spotlight，代表了具身智能领域的重要进展
- **神经-符号结合**: 巧妙地将符号推理的可靠性与 LLM 的灵活性结合
- **探索性交互**: "先看再做"的思路很直觉，但形式化实现具有挑战性
- **即插即用**: 无需训练，可直接应用于不同 LLM 和机器人平台

## 局限性 / 可改进方向

1. 探索性交互增加了任务执行时间，实时性有待优化
2. 符号验证依赖预定义的环境 API，对新环境的泛化需要人工适配
3. 状态保持约束的定义可能不完备，复杂场景下可能遗漏
4. 对 LLM 代码生成能力有较高依赖，弱模型效果下降明显

## 相关工作与启发

- **Code-as-Policies (Liang et al., 2023)**: 本文的直接改进对象

- **Inner Monologue**: 引入环境反馈的迭代生成范式
- **VoxPoser**: 利用 LLM 构建 3D 值图进行机器人操作
