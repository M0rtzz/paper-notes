# Learning to Condition: A Neural Heuristic for Scalable MPE Inference

**会议**: NeurIPS 2025  
**arXiv**: [2509.25217](https://arxiv.org/abs/2509.25217)  
**代码**: 无  
**领域**: 概率图模型 / 组合优化  
**关键词**: MPE推理, 概率图模型, 神经启发式, 分支定界, 条件化策略

## 一句话总结

提出 Learning to Condition (L2C)，用注意力神经网络学习对变量-值对进行评分，指导概率图模型中 MPE 推理的条件化决策，在保持解质量的同时大幅缩减搜索空间。

## 研究背景与动机

最可能解释(MPE)推理是概率图模型(PGM)中的基本推理任务：给定观测证据，找到未观测变量的最可能赋值。该问题是 NP-hard 的，随着模型复杂度增长变得极其困难。

经典的精确方法如 AND/OR 搜索和整数线性规划(ILP)能保证最优性，但对大规模实例计算代价极高。近似方法虽然可扩展，但常常牺牲解的质量。**条件化(Conditioning)** 是一种经典策略——固定部分变量的值来简化问题结构，但其效果高度依赖于选择哪些变量、赋什么值、以什么顺序进行。

核心挑战在于：错误的条件化决策可能永久排除真正的最优解，而过于保守则无法获得计算收益。现有方法依赖手工设计的启发式或基于结构的度量，泛化能力有限。本文提出用神经网络从求解器的搜索轨迹中学习条件化策略。

## 方法详解

### 整体框架

L2C 包含三个阶段：(1) 数据收集——从 MPE 求解器的搜索行为中提取监督信号；(2) 训练注意力网络——学习对变量-值对评分；(3) 推理时集成——作为条件化预处理或 B&B 求解器内部的启发式。

### 关键设计

1. **数据收集流水线**: 给定 PGM，随机采样赋值并划分为查询集和证据集，调用 MPE oracle 求解。对每个实例，随机选取子集变量逐一固定值后重新求解，记录求解时间、节点数等统计量。optimality 标签来自 oracle 解中变量取值是否一致；simplification 标签通过求解器统计量的逆排序分布得到。采样参数 $c_{max}$ 控制每个实例评估的变量数量以平衡计算开销。

2. **注意力网络架构**: 每个变量-值对用嵌入向量表示，二值变量有 $2 \times |X|$ 大小的嵌入表。额外的状态嵌入区分观测/未观测变量。未观测变量的嵌入通过多头注意力以证据变量嵌入为 key/value 进行上下文化。经过共享编码器(全连接+ReLU+残差+dropout)后，分别送入两个预测头：
   - **最优性头(Optimality Head)**: 两层 MLP + sigmoid，估计该赋值出现在 MPE 解中的概率
   - **简化头(Simplification Head)**: 两层 MLP，输出未归一化分数，经 softmax 预测赋值的简化效用排名

3. **推理时策略**: 提供两种使用模式：
   - **贪心条件化**: 迭代筛选满足最优性阈值 $\tau$ 的赋值，选简化分数最高的加入证据
   - **束搜索**: 维护 $W$ 个候选序列，每步扩展并按分数剪枝，探索多条条件化路径
   - **B&B 集成**: 将神经网络分数作为分支选择和节点选择启发式嵌入 SCIP 求解器

### 损失函数 / 训练策略

联合多任务损失：$\mathcal{L} = \lambda_{opt} \cdot \mathcal{L}_{opt} + \lambda_{rank} \cdot \mathcal{L}_{rank}$

- **最优性损失**: 二元交叉熵，对每个变量-值对独立计算，支持多最优解的监督
- **排序损失**: list-ranking 交叉熵，鼓励模型优先选择能更大幅降低推理代价的赋值
- 仅对有监督信号的变量子集计算排序损失，其余变量用掩码屏蔽
- 训练使用 Adam 优化器，学习率 $8 \times 10^{-4}$，指数衰减 0.97，batch size 128

## 实验关键数据

### 主实验

在 14 个高树宽二值 PGM（90~1444 变量）上评估，每个模型生成 12000 训练 / 1000 测试样本。

| 方法 | 优于无条件化 oracle 的配置数(12中) | 决策时间(秒) | 特点 |
|------|------|------|------|
| L2C-Rank | **大多数配置优于 oracle** | 0.002~0.025 | 持续稳定 |
| L2C-Opt | 多数配置优于 oracle | 0.001~0.018 | 稳定 |
| Strong Branching | 少数配置有提升 | 1.2~20.9 | 大网络超时 |
| Graph Heuristic | 偶有提升 | 0.001~0.107 | 随网络增大变慢 |

### 消融实验 / AOBB oracle 结果

| 配置 | 关键指标 | 说明 |
|------|---------|------|
| L2C-Rank + SCIP (25% 条件化) | LL gap 持续为负(绿色) | 条件化越深，改善越明显 |
| L2C + AOBB | 近最优解 + 大幅节点缩减 | 搜索空间显著缩小 |
| Graph/SB + AOBB | 条件化越深，解质量下降越多 | 决策质量不足 |
| L2C 作为 B&B 启发式 | 多数 heatmap 为绿色 | 优于 SCIP 默认策略 |

### 关键发现

- L2C-Rank 在贪心条件化下，随条件化深度增加，LL gap 持续变负，说明方法具有良好的可扩展性
- 在 AOBB oracle 下，L2C 在大幅减少搜索节点(y轴)的同时将解质量损失控制在接近零(x轴靠近原点)
- 决策速度方面，L2C 在最大网络(1440变量)上仅需约 0.025 秒，而 Strong Branching 在 BN30/BN32 上超时
- 作为 B&B 内部启发式时，L2C 同样在相同时间预算下找到更高质量的解

## 亮点与洞察

- **核心洞察**: 解的模糊性调节条件化风险——若变量在所有最优解中取值相同，固定它非常有效但也有风险；若取值分散，条件化更宽容
- 将传统的静态变量选择问题转化为可学习的、实例自适应的决策过程
- 双头设计巧妙平衡了"保留最优解"和"简化推理"两个可能冲突的目标
- 数据收集策略高效：不需要枚举所有 MPE 解，仅用一次 oracle 求解加随机采样即可获得监督信号

## 局限性 / 可改进方向

- 目前仅在二值变量 PGM 上验证，多值离散变量的扩展有待探索
- 大规模模型(百万级变量)的可扩展性未验证
- 监督信号依赖 oracle 质量，当 oracle 本身受限时标签可能有噪声
- 未探索更丰富的求解器信号(如 branch-and-cut 决策)
- 学习变量排序以开发新型神经界限策略是未来方向

## 相关工作与启发

- 与 MIP 中的强分支(strong branching)启发式类似，但用神经网络替代昂贵的逐一评估
- 受统计物理中 decimation 策略启发，但通过学习实现自适应选择
- Gasse et al. (2019) 用 GCN 学习 MIP 分支决策的工作是重要前驱
- 对其他组合优化问题中"学习搜索启发式"的研究有借鉴意义
