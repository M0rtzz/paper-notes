# Adaptive Cooperative Transmission Design for Ultra-Reliable Low-Latency Communications via Deep Reinforcement Learning

**会议**: NeurIPS 2025  
**arXiv**: [2511.02216](https://arxiv.org/abs/2511.02216)  
**代码**: 无  
**领域**: Agent / 无线通信  
**关键词**: URLLC, 两跳中继通信, 深度强化学习, 5G NR, 自适应传输

## 一句话总结

提出 DRL-CoLA 双智能体深度 Q 网络算法，让两跳中继通信系统中的源节点和中继节点分别以分布式方式学习自适应配置 5G NR 传输参数（numerology、mini-slot、MCS），在仅使用本地 CSI 的条件下，实现端到端严格时延约束内的近最优丢包率。

## 研究背景与动机

**领域现状**：下一代无线通信需要支持远程手术、自动驾驶、实时 VR/AR 等任务关键应用，这些场景要求 URLLC（ultra-reliable low-latency communication），丢包率低至 $10^{-5} \sim 10^{-7}$，端到端时延仅几毫秒。协作通信通过中继节点增强可靠性，是提升 URLLC 性能的主流方案之一。

**现有痛点**：现有两跳传输方案大多采用一次性传输（one-shot），任何一跳的解码错误直接导致传输失败；且依赖两跳全局 CSI 进行最优资源分配，获取全局 CSI 的开销与 URLLC 的紧凑时延预算矛盾。ARQ 重传能提高可靠性，但会不可避免地增加延迟。此外，已有工作仅分别优化 AMC 或 numerology 等单一参数，未联合利用 5G NR 的全部灵活性。

**核心矛盾**：可靠性与低延迟之间存在根本的 trade-off——重传提升可靠性但消耗时延预算；高速调制减少传输时间但增大误码率。在两跳场景下此矛盾更加尖锐，因为两跳共享同一个端到端时延预算。

**本文要解决什么？** (1) 在每次（重）传输中联合优化 numerology $\mu$、mini-slot 大小 $N_{\rm sym}$ 和 MCS $I_{\rm MCS}$ 三组参数；(2) 仅基于本地 CSI 和 ARQ 反馈实现分布式决策，避免全局 CSI 开销；(3) 在 finite blocklength 条件下最大化端到端成功投递概率。

**切入角度**：将两跳传输建模为马尔可夫决策过程 (MDP)，每跳节点作为独立智能体，从本地观测学习时延感知的传输策略。利用 delay outage rate (DOR) 作为奖励信号，让源节点在不知第二跳具体传输结果的条件下也能合理分配时延预算。

**核心idea一句话**：用双智能体 DQN 从本地 CSI + ARQ 反馈中学习分布式时延感知传输策略，替代需要全局 CSI 的传统优化方法。

## 方法详解

### 整体框架

系统由源节点 S、半双工中继 R、目标 D 组成的两跳 decode-and-forward 中继架构。S→R 为第一跳, R→D 为第二跳，共享端到端时延预算 $T_{\rm th}$。每次传输尝试中，当前活跃节点从 $\{(\mu, N_{\rm sym}, I_{\rm MCS})\}$ 的离散动作空间中选择传输参数，传输后根据 ARQ 反馈决定重传或切换到下一跳。整个过程建模为 MDP，S 和 R 分别训练各自的 DQN。

### 关键设计

1. **分布式 MDP 建模**:

    - 功能：将两跳传输过程形式化为可用 RL 求解的序列决策问题
    - 核心思路：每个智能体 $i$ 的状态由本地 SNR $\gamma_i$、下一跳平均 SNR $\bar{\gamma}_{i+1}$、包大小 $H$ 和剩余时延预算 $\tau_n$ 组成（维度仅为 4）。动作为 $(\mu, N_{\rm sym}, I_{\rm MCS})$ 三元组。把终止分为 Success 和 Failure 两个吸收态，解码错误概率作为转移概率的核心
    - 设计动机：状态仅包含本地信息，避免全局 CSI 交换的过高开销；引入 $\bar{\gamma}_{i+1}$ 使第一跳能大致评估为第二跳留多少时延预算合适

2. **基于 DOR 的奖励设计**:

    - 功能：在第一跳无法观测第二跳传输结果的条件下，给出时延感知的反馈信号
    - 核心思路：成功时奖励 $\mathcal{R} = 1 - \mathcal{P}_{\rm DOR}(\bar{\gamma}_{i+1}, \tau_{n+1})$，即用剩余时延预算 $\tau_{n+1}$ 计算下一跳的 delay outage rate；失败（时延耗尽）给 $-1$；非终止的无效重传给 $-0.1$
    - 设计动机：DOR 可从平均 SNR 解析计算，无需实际观测下一跳。奖励随剩余时延单调递增，自然激励智能体为后续跳保留充足预算

3. **双智能体 DQN 训练**:

    - 功能：让 S 和 R 各自学习最优传输策略 $\pi_i: s \mapsto a$
    - 核心思路：采用标准 DQN（主网络 + 目标网络 + 经验回放），两个智能体独立训练但通过共享的端到端时延约束间接耦合。每个智能体用 $\epsilon$-greedy 策略探索离散动作空间（$5 \times 4 \times 15 = 300$ 个动作），用 MSE 损失训练 Q 网络
    - 设计动机：DQN 天然适合离散动作空间；双目标网络提供训练稳定性；与 A2C/PPO 等策略梯度方法相比，在该离散控制任务中收敛更快、稳态奖励更高

### 损失函数 / 训练策略

损失为标准 DQN 的时序差分 MSE 损失：$\mathcal{L}_i(\boldsymbol{\theta}_i) = \mathbb{E}_{e_n \sim \mathcal{M}_i}[(y_n^{(i)} - Q_i(s_n^{(i)}, a_n^{(i)}; \boldsymbol{\theta}_i))^2]$，其中目标值 $y_n^{(i)} = \mathcal{R}_{n+1}^{(i)} + \gamma \max_{a'} Q_i(s_{n+1}^{(i)}, a'; \boldsymbol{\theta}_i^-)$。目标网络每 $E'$ 个 episode 同步一次。

## 实验关键数据

### 主实验

| 对比方案 | 需全局CSI | 丢包率（vs 最优） | 说明 |
|----------|-----------|-------------------|------|
| One-shot optimal | 是 | 最优下界 | 用全局CSI最优分配符号数 |
| DRL-CoLA | 否 | 近最优 | 仅用本地CSI+ARQ |

DRL-CoLA 在所有测试时延区间下均接近需要全局 CSI 的 one-shot 最优方案的丢包率性能。

### 消融实验

| 配置 | 收敛速度 | 稳态累积奖励 | 说明 |
|------|----------|--------------|------|
| DRL-CoLA (DQN) | 最快 | 最高 | 适合离散动作空间 |
| DRL-CoLA (A2C) | 较慢 | 较低 | 策略梯度方法在此不占优 |
| DRL-CoLA (PPO) | 中等 | 中等 | 优于A2C但不及DQN |

### 关键发现

- **中继位置影响**：丢包率关于 S-R 距离 $d_1$ 呈 V 形曲线，$d_1 = d_2$（对称放置）时最低；但 $d_1 > d_2$ 时丢包率低于 $d_1 < d_2$，因为第二跳面临更紧凑的时延预算，靠近 D 的中继能更好地利用剩余时延
- **DQN vs 策略梯度**：在 300 维离散动作空间下，DQN 的值函数逼近比策略梯度更高效，收敛更快且稳态性能更好
- **分布式决策的有效性**：即使无全局 CSI，仅利用本地 SNR + 下一跳平均 SNR + 剩余时延预算，RL 策略就能逼近全局最优

## 亮点与洞察

- **DOR 作为跨跳奖励桥梁**是本文最巧妙的设计——它用解析公式将"为下一跳留足时延"的隐式目标显式化到奖励信号中，无需两跳间额外通信。这种思路可迁移到任何多阶段资源共享场景
- **问题建模的工程直觉**：把 5G NR 的三组可调参数打包为 RL 动作空间，把 finite blocklength 的解码错误概率融入转移概率，整体建模非常自然
- 仅 4 维状态 + 300 维动作的轻量级 MDP，说明在硬件资源受限的节点上也可部署

## 局限性 / 可改进方向

- **信道模型过于理想**：仅考虑 Rayleigh 平坦衰落、单载波，未涉及频率选择性衰落、多用户干扰等实际场景
- **无多用户/多中继扩展**：仅限两跳单用户场景，多中继选择、多用户复用的扩展未讨论
- **静态平均 SNR 假设**：$\bar{\gamma}_{i+1}$ 作为先验知识输入，未考虑其变化场景
- **缺少与最新 multi-agent RL 方法的对比**：如 QMIX、MAPPO 等在通信场景中的应用

## 相关工作与启发

- **vs Saatchi et al. (2023)**: 他们在单跳 point-to-point 场景下联合优化 numerology + mini-slot + MCS，本文将其扩展到两跳中继场景，增加了跨跳时延分配和分布式决策的难度
- **vs 传统 one-shot 两跳优化**: 传统方法需全局 CSI 且无重传，本文用 RL + ARQ 在不获取全局 CSI 的前提下实现近似性能
- 论文展示了 RL 在通信底层物理层优化中的实用性，但领域与主流 AI 距离较远

## 评分

- 新颖性: ⭐⭐⭐ 将已有单跳 RL 传输设计扩展到两跳场景，DOR 奖励设计有巧思，但整体框架（DQN + MDP）较常规
- 实验充分度: ⭐⭐⭐ 与最优方案和其他 RL 算法对比合理，但实验场景单一、无真实硬件验证
- 写作质量: ⭐⭐⭐⭐ 问题建模与算法推导清晰
- 价值: ⭐⭐⭐ 通信领域的扎实工作，但对更广泛 AI 社区的启发有限
