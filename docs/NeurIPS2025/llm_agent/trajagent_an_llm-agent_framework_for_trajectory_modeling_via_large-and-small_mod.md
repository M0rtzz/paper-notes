# TrajAgent: An LLM-Agent Framework for Trajectory Modeling via Large-and-Small Model Collaboration

**会议**: NeurIPS 2025  
**arXiv**: [2410.20445](https://arxiv.org/abs/2410.20445)  
**代码**: https://github.com/tsinghua-fib-lab/TrajAgent  
**领域**: LLM Agent  
**关键词**: 轨迹建模, 自动化机器学习, 大小模型协作, 多任务框架

## 一句话总结
首个 LLM 代理框架自动处理轨迹建模全流程，通过 UniEnv 统一接口和协作学习双层优化（LLM 推理 + 小模型训练），性能相比基线最高提升 69.91%。

## 研究背景与动机

1. **领域现状**：轨迹建模涵盖表示、分类、预测、恢复、生成等 5 大类，已有 18 种专用模型。

2. **现有痛点**：
   - 每个任务需要专家手动选模型、调参、处理数据
   - 不同任务的数据格式、评估指标各异，难以统一处理

3. **核心矛盾**：轨迹建模任务多样且破碎，缺乏统一框架。

4. **切入角度**：用 LLM 作为编排器自动选模型 + 小模型执行具体任务，UniEnv 统一接口。

5. **核心idea一句话**：主智能体工作流（理解→规划→执行→总结）+ 协作学习（代理端推理 + 模型端训练）。

## 方法详解

### 整体框架
TrajAgent 包含三个核心组件：(1) UniEnv 统一环境；(2) 主智能体工作流（任务理解→规划→执行→总结）；(3) 协作学习（LLM 推理 + 小模型训练的双层优化）。

### 关键设计

1. **UniEnv 统一环境**:
   - 做什么：统一 18 种模型、5 类任务的数据格式和评估接口
   - 核心思路：对轨迹数据进行标准化解析、编码和评估

2. **协作学习双层优化**:
   - 代理端（LLM）：维护长短期记忆，Think-Action 循环推理，选择历史最佳轨迹加入 prompt
   - 模型端（小模型）：数据增强（几何感知增强、多种操作符）+ 参数优化

## 实验关键数据

### 主实验

| 任务 | Baseline | TrajAgent | 提升 |
|------|---------|-----------|------|
| 轨迹分类 | - | - | +2.38% |
| 轨迹预测 | - | - | +23.45% |
| 轨迹生成 | - | - | +69.91% |

### 协作学习消融

| 配置 | 轨迹预测 | 轨迹生成 |
|------|---------|--------|
| 完整 TrajAgent | 最优 | 最优 |
| 无代理端推理 | 下降 | 下降 |
| 无数据增强 | 下降 | 明显下降 |

### 关键发现
- 轨迹生成提升最显著（69.91%），可能因为多样性和数据增强对生成任务影响最大
- LLM 推理 + 小模型训练的双层优化互相补强

## 亮点与洞察
- **首个统一轨迹建模框架**：将 18 种模型和 5 类任务统一到一个框架下
- **大小模型协作的范例**：LLM 做编排+推理，小模型做执行+训练

## 局限性 / 可改进方向
- 依赖 LLM 能力，开源模型在某些任务上推理不稳定
- 仅在轨迹领域验证，泛化到其他时空任务需进一步工作

## 相关工作与启发
- **vs AutoML**：传统 AutoML 解决超参搜索，TrajAgent 解决全流程自动化
- **vs HuggingGPT**：HuggingGPT 是通用 LLM-模型协作，TrajAgent 是垂直领域深化
