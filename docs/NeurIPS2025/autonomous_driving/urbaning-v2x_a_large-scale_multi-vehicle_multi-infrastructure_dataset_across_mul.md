# UrbanIng-V2X: A Large-Scale Multi-Vehicle Multi-Infrastructure Dataset Across Multiple Intersections for Cooperative Perception

**会议**: NeurIPS 2025  
**arXiv**: [2510.23478](https://arxiv.org/abs/2510.23478)  
**代码**: [GitHub](https://github.com/thi-ad/UrbanIng-V2X)  
**领域**: 协同感知, 自动驾驶, 数据集  
**关键词**: V2X, 协同感知, 多交叉路口, LiDAR, 3D 目标检测

## 一句话总结
提出首个跨多交叉路口的真实世界车路协同感知数据集 UrbanIng-V2X，包含 3 个城市路口、2 辆联网车辆、多达 3 个基础设施传感器杆，共 34 个序列、712k 标注实例和 13 个目标类别。

## 研究背景与动机
- 单智能体感知系统受限于视场和遮挡，难以在复杂城市路口场景中可靠工作
- 现有真实 V2X 数据集存在明显局限：DAIR-V2X 仅单车、TUMTraf-V2X 仅单路口、V2X-Real 无多路口覆盖
- 缺少同时包含多车辆、多基础设施、多交叉路口的综合数据集，限制了算法泛化能力评估
- 相似路口布局可能导致过拟合和虚高性能，需要跨路口验证

## 方法详解

### 整体框架
- 数据采集环境：德国 Ingolstadt 市 3 个城市交叉路口（High-Definition Testfield）
- 每个场景包含 2 辆车和最多 3 个基础设施传感器杆协同工作
- 34 个 20 秒序列，覆盖白天、阴天、暮光、夜间等多种照明条件

### 关键设计
1. **传感器配置**：

    - 每辆车：6 个 FHD RGB 相机（360° 覆盖）+ 1 个 128 线 LiDAR + 高精度 IMU + RTK GPS（1cm 精度）
    - 每个杆：1-3 个 VGA 热红外相机 + 1 个 64 线 LiDAR + 1 个 32 线盲区 LiDAR
    - 总计：12 个车载 RGB 相机、2 个车载 LiDAR、17 个基础设施热红外相机、12 个基础设施 LiDAR

2. **时空同步**：

    - UTC 时钟作为统一时间参考
    - 车辆端：IMU 通过 GPS 同步 UTC，PTP 协议同步相机和 LiDAR
    - LiDAR 相位锁定，确保旋转周期同步对齐
    - 基础设施热红外相机最大时间偏差 16.6ms

3. **标定与融合**：

    - 锥形反射标定靶 + RTK GPS + 手动标注的多步骤外参标定
    - 逐点运动补偿处理 LiDAR 旋转扫描引起的运动模糊
    - 最大空间误差估计 0.7m（50 km/h 目标速度下）

4. **数据划分策略**：

    - EIS（Equal Intersection Split）：序列级划分，每个路口均匀分配到训练/验证/测试
    - SIS（Separate Intersection Split）：留一路口测试，评估跨路口泛化能力

### 损失函数 / 训练策略
- 使用 PointPillars 作为骨干网络
- 检测范围：x 方向 ±100m，y 方向 ±40m
- 仅考虑融合点云中至少 5 个 LiDAR 点的目标
- 评估指标：mAP@0.3 和 mAP@0.5

## 实验关键数据

### 数据集对比

| 属性 | V2V4Real | DAIR-V2X | TUMTraf-V2X | V2X-Real | **UrbanIng-V2X** |
|------|---------|---------|------------|---------|----------------|
| 交叉路口数 | 0 | 28 | 1 | 1 | **3** |
| 车辆数 | 2 | 1 | 1 | 2 | **2** |
| RGB 图像 | 40k | 39k | 5k | 171k | **81.6k** |
| 热红外图像 | 0 | 0 | 0 | 0 | **38.8k** |
| 3D 标注框 | 240k | 464k | 29k | 1.2M | **712k** |
| 类别数 | 5 | 10 | 8 | 10 | **13** |
| 数字孪生 | 否 | 否 | 否 | 否 | **是** |

### 主实验（SIS_1/2vs3 划分，跨路口评估）

| 融合方法 | mAP@0.3 (Vehicle) | mAP@0.5 (Vehicle) |
|---------|-------------------|-------------------|
| 无融合 | 较低基线 | 较低基线 |
| 晚期融合 | 最弱协同性能 | 最弱协同性能 |
| F-Cooper | 中等 | 中等 |
| V2X-ViT | 中等偏高 | 中等偏高 |
| AttFuse | **最优** | **最优** |

### 关键发现
- 中间融合方法整体优于无融合、晚期融合和早期融合
- AttFuse 取得最佳性能，至少领先其他方法 0.6 mAP@0.5
- 车辆类检测性能最好；行人类最具挑战（目标最小）
- 两轮车和重型车辆因类内尺寸变化大而挑战较高
- 不同路口的场景复杂度差异显著：路口2最密集、路口3最稀疏

## 亮点与洞察
- 首个同时具备多车辆、多基础设施、多路口、热红外、数字孪生的真实世界 V2X 数据集
- SIS（跨路口划分）策略有效揭示了过拟合问题，比传统帧级划分更能评估泛化能力
- 热红外相机的引入拓展了多模态感知的研究范围（尤其夜间场景）
- 完整的开发者工具包（OpenCOOD/nuScenes 格式转换器）降低了使用门槛

## 局限性 / 可改进方向
- 仅 34 个序列的数据规模相对不大，扩展性有待提升
- 3 个路口的多样性仍然有限（均位于同一城市）
- 基础设施热红外相机分辨率较低（VGA），在密集场景中效果受限
- 未包含天气变化（如雨雪）场景

## 相关工作与启发
- 跨路口泛化评估的理念对自动驾驶数据集设计具有重要参考价值
- 多种传感器同步方案（PTP/NTP/GPS 时间锁定 + LiDAR 相位锁定）可供其他项目参考
- 数据集配套数字孪生（CARLA 模型）的做法值得推广
