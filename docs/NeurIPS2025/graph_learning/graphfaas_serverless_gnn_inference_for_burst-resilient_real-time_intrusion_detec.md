# GraphFaaS: Serverless GNN Inference for Burst-Resilient, Real-Time Intrusion Detection

**会议**: NeurIPS 2025  
**arXiv**: [2511.10554](https://arxiv.org/abs/2511.10554)  
**代码**: 无  
**领域**: 图神经网络 / 系统  
**关键词**: GNN 推理, Serverless, 入侵检测, 突发负载, 图分区

## 一句话总结

提出 GraphFaaS——基于 Serverless 架构的 GNN 推理系统，通过时间局部性图构建、频率过滤、特征长度感知节点嵌入和贪心 best-fit 图分区算法，在突发负载下实现平均检测延迟降低 85%（14.16s→2.1s）和变异系数降低 64%，同时保持检测准确率不变。

## 研究背景与动机

基于来源图（provenance graph）的入侵检测是图机器学习在网络安全中的重要应用：系统审计日志被建模为来源图（节点=系统实体如进程/文件/网络，边=事件如读写/创建），GNN 学习正常行为模式以检测异常。

然而，现有 GNN 推理架构面临两个关键挑战：

1. **延迟稳定性**：入侵检测要求持续低延迟——推理时间过长可能错过攻击的响应窗口，造成不可逆后果
2. **突发负载**：系统活动（尤其是恶意活动）高度不规则，工作负载存在突发性激增

传统静态配置的 GNN 推理架构无法同时满足这两个需求。Serverless（函数即服务）计算天然适合处理突发负载——按需分配资源、自动扩缩、无空闲开销。但将 GNN 推理适配到 Serverless 环境需要解决图结构数据的特殊挑战。

## 方法详解

### 整体框架

GraphFaaS 由三个核心组件组成：(1) 图构建——利用时间局部性和频率过滤构建增量子图；(2) Serverless 节点嵌入——并行化节点特征转换；(3) Serverless GNN 推理——基于贪心图分区的并行消息传递。

### 关键设计

1. **时间局部性图构建与频率过滤**:
    - 功能：避免每次检测间隔都重建整个来源图，只处理变化的部分
    - 核心思路：保留活跃节点 2K-hop 邻域（K 为 GNN 层数）；通过频率过滤移除训练数据中高频出现的边和节点（常见模式通常无害），只保留罕见模式
    - 设计动机：来源图大部分结构在检测间隔之间不变，增量处理避免冗余计算

2. **特征长度感知的 Serverless 节点嵌入**:
    - 功能：将节点属性（进程名、文件路径、IP 地址等文本特征）转换为向量表示
    - 核心思路：由于节点属性是文本串，嵌入时间与串长度相关，因此按特征长度分组——短串合并到一个执行单元，长串单独处理，确保每个单元在时间阈值内完成
    - 设计动机：Serverless 函数有执行时间限制，不均匀的节点属性长度需要自适应调度

3. **贪心 Best-Fit 图分区算法**:
    - 功能：将大图分区为满足大小限制的子图以并行推理
    - 核心思路：维护每个 cluster 的当前节点集和边数，对新节点计算加入各 cluster 的增量边数 $\Delta_k$，选择最紧密适配的 cluster（best-fit），若超出容量则创建新 cluster
    - 设计动机：需要平衡子图大小（控制延迟）和子图数量（减少计算浪费），同时利用重叠邻域合并减少分区数

### 超级节点处理

当某个节点的 K-hop 邻域本身就超过大小限制时（依赖爆炸问题），触发**垂直扩展**——为该 Serverless 函数实例分配更多计算资源（CPU、内存），而非产生更多实例。

## 实验关键数据

### 主实验

| 指标 | GraphFaaS | Baseline (Flash) | 改进幅度 |
|------|-----------|-----------------|---------|
| 平均检测延迟 | **2.10 秒** | 14.16 秒 | **-85%** |
| 标准差 | 1.09 | 4498.92 | 大幅降低 |
| 变异系数 (CV) | **0.52** | 1.46 | **-64%** |
| 检测准确率 | 与 Flash 相同 | - | 无损失 |
| 最大延迟 | < 10 秒 | 远超 10 秒 | 更稳定 |

### 实验设置

- 平台：OpenFaaS（开源 Serverless 平台）
- 数据集：DARPA TC Engagement 3（11 天审计日志，4 个攻击活动）
- 基线：Flash（最先进的来源图入侵检测系统），使用 Docker 容器模拟静态配置

### 关键发现

- GraphFaaS 在突发流量期间也能保持低延迟（延迟峰值不超过 10 秒），而 baseline 出现严重延迟暴增
- 检测准确率完全保持——底层检测模型不变，仅推理架构不同
- Serverless 的按需扩缩特性天然适配入侵检测的不规则工作负载

## 亮点与洞察

- **首个 Serverless GNN 推理系统**：将 Serverless 计算的弹性优势与 GNN 推理需求巧妙结合
- **延迟 85% 降低且准确率无损**：实现了系统效率与检测效果的双赢
- **面向真实安全场景**：低延迟和突发容忍力对入侵检测的及时响应至关重要

## 局限性 / 可改进方向

- 来源图的**依赖爆炸问题**仍是瓶颈——超级节点需要垂直扩展，limitando了水平扩展能力
- 仅在 DARPA TC 单一数据集上评估，需要在更多实际场景中验证
- GNN 层数固定，未探索在攻击期间动态调整层数以平衡检测范围和延迟
- Serverless 冷启动延迟可能在极端突发场景下成为额外开销
- 未与其他分布式 GNN 推理系统进行对比

## 相关工作与启发

- **Flash / Kairos / ORTHRUS**: 基于来源图的入侵检测系统，是本文的基线和对比对象
- **Dorylus**: 使用 Serverless 进行 GNN 训练（而非推理）
- **λGrapher**: 利用请求级图局部性优化 GNN serving
- **对安全系统设计的启示**: Serverless 架构可能成为安全分析系统的主流部署方式
