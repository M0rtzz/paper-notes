# Multimodal Bayesian Network for Robust Assessment of Casualties in Autonomous Triage

**会议**: NeurIPS 2025  
**arXiv**: [2512.18908](https://arxiv.org/abs/2512.18908)  
**代码**: 无  
**领域**: 医学AI/概率图模型/自主分诊  
**关键词**: 贝叶斯网络, 大规模伤亡事件, 多模态融合, 自主分诊, DARPA Triage Challenge

## 一句话总结

提出基于专家知识驱动的贝叶斯网络决策支持框架，融合多个计算机视觉模型的输出来评估伤亡人员状况，无需训练数据且支持不完整信息推断，在DARPA Triage Challenge中将分诊准确率从14%提升至53%，诊断覆盖率从31%提升至95%。

## 研究背景与动机

大规模伤亡事件（MCIs）如恐怖袭击、武装冲突或大规模自然灾害可能在数分钟内产生数百甚至上千名伤亡人员，迅速压垮应急系统：

**人力短缺**：医护人员不足，小团队需在危险条件下同时对多名患者进行分诊

**传统手动分诊方法的失败**：面对大量伤亡和复杂伤情时会失灵

**自主感知系统的局限**：
   - **算法孤立**：各生命体征独立处理，不考虑临床相互依赖性
   - **上下文盲区**：无法区分心率升高是疼痛反应、失血还是焦虑
   - **噪声脆弱性**：在灾难场景的嘈杂、部分数据条件下表现不稳定

**缺乏MCI标准数据集**：数据驱动方法难以获得训练数据

## 方法详解

### 整体框架

系统架构基于ROS2框架：
- 机器人平台搭载高分辨率RGB相机、麦克风、雷达、激光雷达和热成像相机
- 独立的感知算法节点分别评估特定生理体征
- 贝叶斯网络(BN)作为**状态估计器**，将各节点预测融合为概率化的伤亡评估
- 推断时信息流是双向的：观察子节点（症状）可更新父节点（病因）概率

### 关键设计

1. **专家知识驱动的BN构建**：

    - 不依赖训练数据，完全基于3位资深急诊医学专家的结构化知识
    - **迭代构建过程**：工程团队访谈→初始模型→专家验证→反馈修正
    - **条件概率表(CPT)参数化**：
      - 强因果关系（如下肢截肢→严重出血）：概率0.8–0.95
      - 中等相关（如头部创伤→异常言语警觉性）：概率0.4–0.6
      - 弱/稀有依赖：接近基线先验概率

2. **伤亡评估变量设计**：

    - **输入变量**（来自感知算法）：
      - 严重出血（Present/Absent）
      - 呼吸窘迫（Present/Absent）
      - 头部创伤（伤口/正常）
      - 躯干创伤
      - 下肢创伤（伤口/截肢/正常）
      - 上肢创伤
      - 眼部警觉性（睁开/关闭/NT）
      - 言语警觉性（正常/异常/缺失/NT）
      - 运动警觉性
    - BN通过因果依赖推断未直接观测的生理状态

3. **鲁棒推断机制**：

    - 当视觉节点输出分类标签时，BN将其作为硬证据即时更新未观测变量概率
    - 单个输入模块故障不会导致系统崩溃——BN对缺失变量进行边际化
    - 这种**优雅退降**对实战部署至关重要

4. **模块化和技术无关性**：

    - 各感知估计器作为独立"黑盒"模块运行
    - BN框架不需要了解底层感知技术的内部架构
    - 可轻松集成新算法或传感模态

### 评分标准

基于DARPA Triage Challenge评分框架，每位伤亡最高12分：
- 严重出血/呼吸窘迫：黄金窗口内正确=4分，正确=2分
- 创伤评估（头/躯干/下肢/上肢）：全部正确=2分，≥2项正确=1分
- 警觉性评估（眼/言语/运动）：全部正确=2分，≥2项正确=1分

### 关键发现

1. **BN的最大价值在于"补盲"**：在感知模块完全无法输出的情况下（如头部创伤从0跳到15次正确评估），BN通过因果推断弥补了缺失
2. **系统可靠性提升最为显著**：从31%到95%，意味着BN几乎为每位伤亡都提供了评估
3. **呼吸窘迫评估改善最大**：正确数从5增至16，BN成功从其他可观测信号推断呼吸状况
4. **准确率和覆盖率同时提升**：不是简单地"多猜"而是"猜得更多且更准"
5. **实际竞赛成绩**：Team Chiron在11支队伍中获得第4名

## 亮点与洞察

- **极其实用的方法论**：不需要训练数据，仅靠专家知识和概率推理即可大幅提升系统性能
- **优雅退降设计**：单个传感器/算法失败不会导致全系统崩溃，对野外部署至关重要
- **双向推理能力**：不仅从原因推果，也能从观察到的症状反推病因
- **超轻量级**：模型<100MB，推理时间<1ms（甚至在Raspberry Pi 3上）
- **模块化架构**：可即插即用新的感知算法，无需修改核心架构

## 局限性 / 可改进方向

- CPT参数基于启发式设定，虽然有医学专家指导，但具体概率值的最优性未经验证
- 仅在2个场景20个伤亡案例上评估，样本量较小
- 第三个场景（飞机坠毁）因硬件故障未能提供数据
- 感知模块作为黑盒输入，不输出置信度分数（未来可整合）
- 当前仅处理分类输入作为硬证据，未利用软证据（概率分布输入）
- 未与其他混合架构进行直接对比

## 相关工作与启发

- 贝叶斯网络在医学领域有悠久历史（腹痛分诊、儿科哮喘检测等）
- 与数据驱动的深度学习方法相比，知识驱动方法在数据稀缺的高风险场景中更可靠
- 非接触式生命体征预测（心率从眼动估计、SpO2等）为输入节点提供了更多可能
- 对于所有需要在不确定环境下进行多模态融合决策的系统（如自动驾驶、灾害救援）都有启发
