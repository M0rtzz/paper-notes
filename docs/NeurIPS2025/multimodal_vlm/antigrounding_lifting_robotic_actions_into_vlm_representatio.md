# AntiGrounding: Lifting Robotic Actions into VLM Representation Space for Decision Making

**会议**: NeurIPS 2025  
**arXiv**: [2506.12374](https://arxiv.org/abs/2506.12374)  
**代码**: 无  
**领域**: 多模态VLM / 机器人操作  
**关键词**: VLM, 机器人操作, 反接地, MPC, 结构化VQA

## 一句话总结

提出 AntiGrounding 框架，反转传统指令接地管道——将候选机器人轨迹渲染到多视角图像中并用 VLM 进行结构化 VQA 评估，绕过压缩中间表征的信息瓶颈，实现复杂操作任务的零样本闭环控制。

## 研究背景与动机

- VLM 隐式编码了丰富的空间和语义先验，但当前方法通过**压缩中间表征**（符号技能序列、约束等）来桥接 VLM 与机器人控制
- 中间表征引入信息瓶颈：丢失精细空间/物理/几何细节和基于这些细节的扩展推理
- 例如，倒茶任务被简化为 `pour(teapot, cup)` 原语，丧失了连续倾斜调整和精确对齐等关键信息
- **核心问题**：如何直接利用 VLM 的内在表征空间进行细粒度动作评估？

## 方法详解

### 整体框架

AntiGrounding 遵循 Real2Sim2Real 管道，核心循环为：

1. **Real2Sim**：3D 重建（SPAR3D）+ 物理参数辨识（Scalable Real2Sim）+ 6D 位姿对齐（SAM-6D）
2. **VLM 引导的 MPC**：生成候选轨迹 → 多视角渲染 → VLM 结构化 VQA 评分 → 执行最优轨迹
3. **离线策略优化**：基于执行反馈自适应调整 VLM 评估策略

### 关键设计

1. **VLM 驱动的轨迹评估（结构化 VQA）**:
    - 功能：构建评估模板，包含安全性、任务对齐、效率、物理可行性等维度的子问题
    - 核心思路：多 VLM agent 集成（MoA 框架），每个 agent 从多个视角评估，聚合分数 $S_{j,t} = \frac{1}{M'}\sum_m \sum_v C'_{v,t}(\sum_k w_k \cdot s_{m,v,j,k,t})$
    - 设计动机：直接在 VLM 的原生表征空间评估视觉化轨迹，避免将 3D 信息压缩到低维表示

2. **自适应视角置信度与退火轨迹生成**:
    - 功能：视角置信度 $C_{v,t} = \frac{q_{\text{view}}}{1 + \lambda_C \cdot \sigma_{v,t}}$；采样半径按 $R_{t'} = R_{\min} + (R_0 - R_{\min}) e^{-\lambda_R t'}$ 指数衰减
    - 核心思路：高一致性视角获得更高权重；退火策略平衡探索与利用，子任务转换时重置
    - 设计动机：不同视角信息量随操作阶段变化，需要自适应权重；轨迹搜索需从粗到精

### 损失函数 / 训练策略

- 无需训练——AntiGrounding 是零样本框架
- 离线策略优化：累积执行数据，meta-VLM 回顾审查，迭代调整评估模板权重
- 刚性耦合旋转模块：处理 VLM 难以解释的 3D 旋转变换

## 实验关键数据

### 主实验（表格）

| 任务 | Code as Policies | Voxposer | Rekep | **AntiGrounding** |
|------|-----------------|----------|-------|-----------------|
| 圆形胶带堆叠 | 1/10 | 3/10 | 5/10 | **6/10** |
| 海绵堆叠 | 2/10 | 2/10 | 6/10 | **7/10** |
| 玩具迷宫导航 | 1/10 | 3/10 | 3/10 | **5/10** |
| 抽屉取物 | 1/10 | 4/10 | 4/10 | **6/10** |
| 倒水 | 0/10 | 0/10 | **6/10** | 5/10 |
| 拖鞋摆放 | 0/10 | 0/10 | 5/10 | **7/10** |

### 消融实验

- 去除多视角融合：成功率显著下降
- 去除自适应视角选择：不可靠视角干扰决策
- 去除离线策略优化：长期任务鲁棒性下降

### 关键发现

- 在精确放置和杂乱场景中显著优于基线，尤其需要精细空间推理的任务
- 倒水任务 Rekep 略优（6/10 vs 5/10），因倒水更依赖预定义约束
- 完全自动化的 Real2Sim2Real 管道减少了人工干预

## 亮点与洞察

- **逆向思维**：将动作"提升"到 VLM 空间而非"接地"到动作空间，避免信息瓶颈
- 将机器人决策转化为标准 VQA 任务，直接利用预训练知识
- 多视角融合隐式编码 3D 信息到 VLM 潜空间

## 局限性 / 可改进方向

- VLM 推理成本高，每 MPC 步需多次调用，实时性受限
- 成功率整体偏低（最高 7/10），距可靠部署有差距
- 未与端到端 VLA 方法（如 RT-2）进行对比
- 对 VLM 空间推理能力高度依赖

## 相关工作与启发

- Code as Policies、Voxposer、Rekep 依赖中间表征，AntiGrounding 绕过了这些
- 与 SayCan 等符号技能方法相比，无需预定义技能库
- "反接地"思想可推广到自动驾驶规划评估等场景
