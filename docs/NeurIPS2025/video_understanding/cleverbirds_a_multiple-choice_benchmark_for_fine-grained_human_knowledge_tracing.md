# CleverBirds: A Multiple-Choice Benchmark for Fine-grained Human Knowledge Tracing

**会议**: NeurIPS 2025  
**arXiv**: [2511.08512](https://arxiv.org/abs/2511.08512)  
**代码**: https://cleverbirds-benchmark.github.io  
**领域**: 视频理解  
**关键词**: 知识追踪, 细粒度识别, 基准数据集, 视觉学习, 人类认知建模, 鸟类识别

## 一句话总结

提出CleverBirds——迄今最大规模的视觉知识追踪基准，通过eBird公民科学平台收集了40000+参与者完成的1700万+多选鸟类物种识别问答（涵盖10000+物种），系统性评估了多种知识追踪与分类方法，揭示了细粒度视觉知识建模特别是预测学习者错误选择方面的核心挑战。

## 研究背景与动机

### 知识追踪的核心挑战

知识追踪（Knowledge Tracing, KT）旨在通过观察学习者与教学材料的交互过程，建模其知识状态并预测未来表现，是构建有效自动辅导系统的核心组件。早期方法基于隐变量概率模型建模学习者对知识的掌握程度，近年来深度学习方法虽能捕捉更复杂的关系，但需要大量训练数据。然而，现有知识追踪数据集主要集中在数学、编程和语言学习等少数领域，在视觉识别领域严重缺乏。

### 视觉知识追踪的空白

许多专业领域（如医学、艺术、生物学）要求学习者掌握视觉识别技能，这些任务本质上是分类问题——学习者需要学会不同视觉概念之间的决策边界。以鸟类识别为例，全球有超过11000种不同的鸟类物种，某些物种之间的区分需要极其精细的视觉判断。已有的视觉知识追踪数据集规模极小：[Crowston et al., 2019] 的引力波分类数据仅包含21个类别，[Lee et al.] 的三个数据集每个仅有6750次交互和不到1000张图像。这些数据集在规模、概念数量和交互深度上都远不能满足研究需要。

### CleverBirds的定位

CleverBirds填补了这一空白。与现有基于图像的知识追踪数据集相比，CleverBirds具备以下独特优势：

1. **概念空间极大**：10779种鸟类物种，远超现有任何KT数据集的概念数量
2. **参与者数量多且技能水平多样**：40000+真实志愿者，而非众包平台的短期参与者
3. **交互序列长**：参与者平均回答400个问题，超过50%的用户回答了100+个问题
4. **真实的学习动态**：可以观察到参与者随时间推移的明显进步

## 方法详解

### 数据集构建

#### 测验设计

CleverBirds数据源自eBird（康奈尔大学鸟类学实验室创建的公民科学项目）的在线鸟类物种识别测验。测验于2018年3月首次上线，初衷是鼓励用户为新上传的图像提供质量评分。具体流程如下：

1. **参数选择**：用户选择地理位置、年份中的时间、物种出现频率等参数来生成测验问题。例如选择"苏格兰爱丁堡，5月15日，常见物种"，则只会呈现该地区该时间段常见鸟类的问题
2. **每轮测验**：包含20道多选题，每道题展示一张鸟类图像和5个选项（4个物种名称 + "以上都不是"选项）
3. **答案选项设计**：为增加难度，候选答案从以正确物种为中心的分类学列表滑动窗口中抽取，确保干扰项与正确答案在分类学上相似
4. **反馈机制**：提交答案后，系统揭示正确物种，用户被要求对图像质量进行1-5分评分
5. **图像来源**：图像来自公民科学家上传到Macaulay Library的照片，其物种标签经过与预期地理分布的交叉验证，不一致的情况会由专家审核员审查

#### 图像质量控制

测验图像在最近5到365天内上传的图像中采样，包含一定比例的未评分图像和质量评分 ≥ 2.4（满分5分）的图像。这种设计一方面通过测验为未评分图像获取质量标注，另一方面保留足够多的高质量图像供用户学习。图像质量的评判标准包括清晰度、鸟类可见度、照片尺寸和水印等。

#### 数据过滤与划分

数据集基于2018年3月14日至2024年10月8日期间的所有在线测验。为防止对特定用户过拟合，按用户ID进行数据集划分：

| 数据集 | 用户数 | 交互量占比 |
|--------|--------|-----------|
| 训练集 | 28,100 | 70.6% |
| 验证集 | 6,021 | 14.6% |
| 测试集 | 6,023 | 14.8% |

由于图像版权限制，数据集不提供原始图像，而是为每张图像提供特征嵌入：
- **DINOv2**（ViT-B/14骨干）：对最后一层LayerNorm后的patch token进行平均池化（排除特殊token），得到14,747,840个特征向量
- **ResNet-50**（ImageNet预训练）：使用分类器前全局平均池化层的输出，得到14,753,114个特征向量

两者之间5274的差异源于DINOv2特征提取时间较晚，部分图像已被所有者删除。

#### 隐私保护

数据集对所有用户相关标识符以及测验、问题和图像资产的标识符进行了匿名化处理。测验位置使用H3地理空间索引在分辨率3下进行聚合，每个单元的平均面积约为12393 km²。该项目已获得爱丁堡大学信息学院伦理委员会批准（项目编号954242）。

### 数据集特征分析

CleverBirds的规模和复杂度在知识追踪领域前所未有。以下是其核心统计特征的详细分析：

**数据规模与构成**：

| 统计指标 | 数值 | 说明 |
|---------|------|------|
| 总交互数 | 17,859,392 | 远超现有任何视觉KT数据集 |
| 唯一图像-物种-选项组合 | 98% | 极少重复的题目组合 |
| 唯一物种-选项对 | 26% | 丰富的干扰项变化 |
| 图像唯一性 | 83% | 绝大多数图像从未被重复展示 |
| 物种数量 | 10,779 | 涵盖全球鸟类分类学的广泛范围 |
| 地理位置数 | 4,000+ | 跨越全球的测验选择地点 |
| 时间覆盖 | 全年52周 | 贯穿所有季节的完整时间分布 |
| DINOv2特征向量数 | 14,747,840 | ViT-B/14骨干提取 |
| ResNet-50特征向量数 | 14,753,114 | ImageNet预训练提取 |

这一数据集的独特之处在于其交互数据的高度非重复性——98%的交互涉及唯一的图像-物种-选项组合，这意味着模型无法简单记忆特定组合的答案模式，必须学会建模更深层次的认知过程。

#### 用户参与度分析

用户参与度的分布呈现出典型的长尾特征，但尾部的"重度用户"占比相当可观：

- 超过50%的40000+用户回答了100+个问题，这些用户提供了足够长的交互序列来分析学习动态
- 超过10%的用户回答了1000+个问题，构成了极其珍贵的深度学习轨迹数据
- 用户平均遇到138个不同物种，10%的用户遇到超过300种，覆盖了相当广泛的概念空间
- 用户准确率分布广泛，集中在60-70%左右，既有接近30%准确率的新手，也有超过90%的资深观鸟者
- 物种层面的平均准确率同样分布广泛，反映了不同物种在视觉上的可辨识程度差异巨大

这种多样化的参与模式为研究不同技能水平学习者的认知差异提供了理想条件——从刚接触鸟类识别的新手到积累了数千次交互经验的准专家级参与者。

#### 学习动态

数据清晰地展现了学习进步，这是CleverBirds区别于静态分类基准的关键特征：

- 在前20次接触某一物种的过程中，用户准确率平均提高了20%，这一提升在统计上高度显著
- 超过一半的用户在20个问题（一次完整测验）的滑动窗口内展示出可测量的进步
- 即使对于低质量图像（模糊、部分遮挡等），用户的平均准确率也在50%以上
- 学习曲线的形态因物种而异——常见且视觉特征鲜明的物种学习速度更快，而外观相似的物种对之间的区分能力提升则需要更长时间
- 反馈效果验证：研究表明，仅提供正确物种标签作为反馈就足以让学习者在细粒度分类任务中获得显著进步，这与先前关于"人类可以仅通过标签监督获得视觉专业知识"的研究结论一致

这些学习动态特征使得CleverBirds不仅仅是一个静态的预测基准，更是一个研究人类视觉技能习得过程的"实验室"。

#### 任务难度

数据集通过最常混淆的物种对来体现任务难度。例如：

- **大沙锥 vs 普通沙锥**：区分在于普通沙锥翅膀上特有的白色后缘
- **纹腹鹰 vs 库氏鹰**：纹腹鹰头部更小、尾部更方正、脚更小
- **松雀（淡色型）vs 松雀（普通型）**、**罗斯鹅 vs 雪雁**等

这些物种对之间的差异极其细微，需要训练有素的观察力才能辨别。更重要的是，测验中的图像可能包含部分遮挡、远距离拍摄或不常见角度的照片，进一步增加了难度。

### 问题形式化

#### 核心定义

在时间步 $t \in \{1, \dots, T\}$，学习者被呈现一张图像 $I_t$ 和一个有序候选答案列表 $\mathbf{c}_t = (c_{t,1}, \dots, c_{t,K-1}, \text{NOTA})$，其中包含 $K-1$ 个随机排列的候选物种和一个"以上都不是"（NOTA）选项。

图像通过固定视觉编码器表示为 $\mathbf{x}_t = f(I_t) \in \mathbb{R}^d$。学习者观察图像后，根据其内部状态选择一个回应 $r_t \in \mathbf{c}_t$，然后接收真实物种标签 $y_t$ 作为反馈。单次交互记为 $h_t = (\mathbf{x}_t, \mathbf{c}_t, y_t, r_t)$。

#### 学习者模型

学习者的回应由其不可观测的内部状态 $\theta_t$（总结了其积累的知识和记忆）以及输入图像和候选选项共同决定。假设每次交互后学习者状态更新，学习者的回应过程建模为：

$$r_t = \arg\max_{c \in \mathbf{c}_t} P(c \mid \mathbf{x}_t, \mathbf{c}_t, \theta_t)$$

其中 $r_t^{\text{bin}} = \mathbb{I}[y_t = r_t] \in \{0, 1\}$ 表示答案是否正确。

#### 近似模型

由于 $P(\cdot)$ 不可观测，通过跨学习者训练的共享参数模型 $\phi$ 来近似。$\phi$ 不包含学习者特定参数，学习者特定行为通过条件化每个个体的近期交互历史 $\mathcal{H}_t = (h_\tau)_{\tau=\max(1, t-W)}^{t-1}$ 和当前问题来实现。模型预测分类结果：

$$\hat{r}_t = \arg\max_{c \in \mathbf{c}_t} \phi(c \mid \mathbf{x}_t, \mathbf{c}_t, y_t, \mathcal{H}_t)$$

上下文可分为三类：
- **用户上下文（U）**：个人交互历史、历史表现和偏好
- **物种上下文（S）**：从训练集聚合的物种特征（如平均难度）
- **图像上下文（Img）**：提取的图像特征，隐式编码质量和歧义等信息

### 评估指标

**二分类任务**（预测正确/错误）：
- **Binary Macro Accuracy**：正确/不正确两个类别的宏平均准确率
- **Binary AP**：以少数类（错误）为正类的平均精度

**多选任务**（预测具体选择）：
- **Multiple Choice Accuracy**：在标签1-5上的准确率（5表示"以上都不是"）
- **Multiple Choice Incorrect Set Accuracy**：仅在参与者答错的问题子集上计算准确率

### 基线方法

#### 多选分类器

1. **启发式方法**：
    - **Always Correct**：假设全知学习者，总选正确答案
    - **Random**：随机猜测

2. **混淆先验分类器（Conf Prior）**：通过训练集中正确物种和干扰项之间的混淆概率来估计每个选项的概率，然后重新归一化形成有效分布。还有一个约束版本（Conf Prior Inc），限制始终预测错误选择

3. **MLP**：单层MLP，输入正确物种的250维嵌入以及可选的上下文（用户/物种/图像）。输出所有物种上的概率分布，然后掩码为仅包含呈现的5个选项。图像特征通过嵌入层转换到MLP输入维度

4. **知识追踪模型**：

   | 模型 | 架构 | 核心特点 |
   |------|------|---------|
   | simpleKT | 轻量基线 | 简单高效 |
   | KQN (Knowledge Query Networks) | 轻量基线 | 基于查询 |
   | DKT (Deep Knowledge Tracing) | LSTM | 隐状态建模历史 |
   | DKT+ | LSTM + 正则化 | DKT的改进版本 |
   | ATKT | LSTM + 对抗训练 | 对抗增强 |
   | SAKT | 自注意力 | 选择性关注相关历史 |
   | AKT | 注意力 + Rasch正则化 | Rasch模型启发的正则化 |
   | DKVMN | 键值记忆网络 | 外部化知识到键值记忆 |

5. **语言建模范式**：
    - **LM-Seq2seq**：微调T5风格编码器-解码器transformer，基于交互历史和当前问题生成正确物种token。训练时使用token级交叉熵损失，NOTA通过聚合可见选项之外的所有概率来处理
    - **LM-MCC**：微调BERT风格编码器transformer，为每个问题-选项对打分，使用二元标签（每个问题恰好一个正确答案）进行监督

   两个模型使用自定义分词器（仅包含11142个物种token加上填充、段分隔和类型标记等特殊token），每个问题占据恰好8个token。设 $W=50$（历史窗口长度），其他模型则使用完整历史。这种紧凑的token化设计避免了使用自然语言描述物种名称时可能产生的歧义问题。

#### 二分类分类器

- **Logistic Regression (LR)**
- **XGBoost (XG)**
- **Random Forest (RF)**
- **Average Species**（基于训练集中平均物种准确率的启发式方法）

各模型使用不同的上下文组合（U、S、U+S）。

## 实验关键数据

### 主实验——多选任务

| 模型 | 上下文 | 全数据集准确率 | 错误子集准确率 |
|------|--------|--------------|--------------|
| Always Correct | - | ~当做上界 | 0% |
| Random | - | ~20% | ~20% |
| Conf Prior | S | <9% | ~23% |
| Conf Prior Inc | S | ~10% | ~24% |
| MLP | U+S | ~70% | <25% |
| MLP | U+S+Img (DINOv2) | **~76%** | **~25%** |
| MLP | Img only | ~76% | ~11% |
| LM-MCC | U+S | ~70% | <25% |
| LM-Seq2seq | U+S | ~70% | <25% |
| AKT | U | ~70% | <25% |
| ATKT | U | ~70% | <25% |
| simpleKT | U | ~70% | <25% |

### 主实验——二分类任务  

| 模型 | 上下文 | 宏平均准确率 | 平均精度（AP） |
|------|--------|------------|--------------|
| Avg Species | S | <70% | <60% |
| RF | S | ~72% | ~70% |
| RF | U | ~78% | ~78% |
| RF | **U+S** | **~80%** | **>80%** |
| KT方法（AKT等） | U | ~54% | ~35% |

### 关键发现

#### 发现一：特征工程的上下文产生了强基线

在二分类任务中，专门训练的二分类模型（特别是随机森林 + 用户和物种上下文组合）表现最佳，宏平均准确率和AP均超过80%。在多选任务中，简单的单层MLP配合用户和物种上下文即可匹配容量更大的Transformer模型的表现。这一结果的意义在于：精心设计的特征工程可以极大地弥补模型复杂性的不足，在许多实际应用场景中，简单可解释的模型搭配好的特征，远比复杂的端到端模型更实用。

#### 发现二：用户上下文比物种上下文更有价值

比较不同上下文类型，模型在同时接收用户和物种上下文（U+S）时表现最优，仅使用用户上下文（U）的模型紧随其后。在二分类器中，包含用户上下文的模型在准确率和平均精度上始终领先。仅依赖物种上下文的模型虽然仍优于简单启发式方法，但用户特定上下文对强预测是必要的。这表明学习者的个体差异（包括历史准确率、学习轨迹、已接触的物种集合、擅长和薄弱的物种类别等）包含了比全局物种难度统计更丰富、更具预测力的信号。这一发现为智能辅导系统设计提供了直接指导——系统应优先跟踪和利用个体学习者的交互历史。

#### 发现三：多选训练的分类器在二分类任务上不如直接训练的二分类分类器

尽管简单概率模型容量较小，它们在二分类目标上直接训练后能达到80%+的AP和平均准确率。原因在于多选模型需要分配容量来学习错误回应的结构，而非仅仅判断正确与否。未来可以探索将二分类分类器的输出作为辅助信号或门控机制来增强多选模型。

#### 发现四：图像特征补充用户和物种上下文，尤其对错误选择的预测

带DINOv2图像特征的MLP（U+S+Img）在全数据集上准确率略高（~76%），但在错误答案子集上显著优于其他参数模型（~25%）。而移除用户和物种上下文的纯图像MLP在全数据集上表现类似，但在错误子集上仅有~11%。这证明图像特征对CleverBirds的知识追踪有帮助，特别是与正确的上下文结合时。

这一发现的深层含义是：图像本身携带了影响学习者出错模式的关键信息——某些图像的拍摄角度、光照条件或鸟类姿态会系统性地导致特定类型的错误。例如，侧面拍摄的普通沙锥图像可能更容易与大沙锥混淆，而正面拍摄则可能更容易正确识别。图像特征能够编码这些"容易出错的条件"，从而在与用户历史和物种统计信息结合时，提供更精确的错误预测。

#### 发现五：预测错误选择极具挑战性

这是本文最重要的发现之一。训练模型在全数据集上约70%的多选准确率看似不错，但在参与者选择错误答案的子集上，所有训练模型准确率不到25%——仅略高于20%的随机基线。最佳模型相对于纯基于错误假设的混淆先验仅有微弱提升，说明：

- 现有方法远未能近似学习者的内部知识状态
- 理解"为什么一个人会犯特定的错误"远比"他是否会答对"困难得多
- 这一子集上的高性能意味着模型能真正理解学习者的认知偏差

#### 发现六：知识追踪基线表现差异极小

所有测试的KT方法（AKT、ATKT、simpleKT、DKT、DKT+、SAKT、DKVMN、KQN等）表现相似，平均精度约0.35，平均准确率约54%。尽管架构多样（从简单RNN到注意力机制到外部记忆），但在CleverBirds上的表现远不如简单的特征工程分类器。这些模型可能利用了正类的捷径——在正类上准确率较高但负类表现差。

这一发现深刻暴露了现有KT方法的设计假设与大规模细粒度视觉场景之间的不匹配：大多数KT方法为数十到数百个概念的数据集设计，且强调概念级可解释性，这限制了模型在面对10000+概念空间时的灵活性。它们通常为每个概念维护独立的知识状态表示，当概念数量急剧增大时，参数空间爆炸而每个概念的训练样本相对稀疏，导致学习不充分。此外，这些模型通常假设概念之间相对独立，未能利用鸟类分类学中丰富的层次结构信息（如科、属、种之间的分类关系）。

### 与现有KT数据集的对比

| 数据集 | 领域 | 概念数 | 参与者数 | 总交互数 | 是否视觉 |
|--------|------|--------|---------|---------|---------|
| ASSISTments | 数学 | ~100 | ~4,000 | ~350K | 否 |
| EdNet | 综合教育 | ~300 | ~780K | ~130M | 否 |
| Gravity Spy | 引力波 | 21 | ~8,000 | ~1M | 部分 |
| 蝴蝶/视网膜/Greebles | 视觉 | 3-5 | ~150 | ~6.75K | 是 |
| **CleverBirds** | **鸟类识别** | **10,779** | **40,000+** | **17.8M** | **是** |

CleverBirds在概念数量上比现有最大的视觉KT数据集高出三个数量级，在交互数量上高出近四个数量级，代表了视觉知识追踪领域的巨大飞跃。

### 图像特征的预测能力

实验还评估了仅使用图像特征进行物种分类的性能（见附录Table A9），以验证预训练视觉特征对细粒度鸟类识别的有效性。DINOv2特征在该任务上表现优于ResNet-50，证实了利用这些特征来近似学习者视觉表征的合理性。

## 亮点与洞察

### 学术价值

- **填补关键空白**：构建了首个大规模视觉知识追踪基准，10779个概念远超现有任何KT数据集（此前最大的也仅约几百个概念）
- **暴露现有方法的不足**：系统性地揭示了当前知识追踪方法在大规模细粒度视觉场景中的局限性，特别是在预测学习者错误方面表现糟糕
- **数据集设计的精妙**：从公民科学项目中提取的真实学习数据比实验室环境收集的数据质量更高、规模更大、动态更丰富，参与者的自愿参与确保了高质量的交互

### 方法论启示

- **简单模型的生命力**：特征工程 + 简单分类器（随机森林）在二分类任务上大幅超越所有深度学习KT方法，这提示在数据特征足够好的情况下，模型复杂性并非关键
- **用户上下文的重要性**：个体交互历史的预测价值远大于全局物种统计，暗示知识追踪模型应更多关注个性化建模
- **视觉特征与认知建模的桥梁**：论文假设人类学习者和神经网络在同一任务上提取相似的图像概念，并用预训练CNN/ViT特征来近似人类视觉表征，这一假设在实验中得到了部分验证

### 应用前景

- 智能辅导系统中个性化推荐难度适当的练习题，根据学习者的知识盲点自适应调整教学内容
- 公民科学培训中自适应地识别和弥补学习者的知识盲点，提高公民科学数据的质量
- 推广到医学影像（如病理切片、X光片识别）、艺术品鉴赏（如画作风格、年代判断）等其他需要视觉专业知识的领域
- 为开发基于强化学习的自适应教学策略（如最优问题序列设计）提供大规模实验平台
- 研究人类视觉专业知识习得的认知科学——例如探索新手到专家过程中注意力模式的变化、概念间迁移学习的规律等

## 与现有工作的关键区别

本文与以往视觉知识追踪工作的核心差异体现在三个维度：

**规模维度**：CleverBirds的10779个概念和17.8M交互量比现有最大视觉KT数据集大3-4个数量级，首次使得在真正大规模概念空间上的知识追踪研究成为可能。

**任务维度**：不同于以往视觉KT数据集使用实验室控制的受试者和有限的图像集（如3类视网膜病变各仅数百张图像），CleverBirds使用来自真实用户自发参与的自然数据，图像来源于公民科学家的现场拍摄，包含了质量、角度、光照等方面的自然变异，更接近真实世界的视觉识别挑战。

**评估维度**：本文引入了"错误子集准确率"这一关键评估指标，将评估重心从"预测是否答对"转向"预测具体会犯什么错"，这是对学习者认知状态建模能力的更深层次检验。

## 关键设计

1. **大规模真实学习数据**：从eBird公民科学平台收集17M+多选交互数据，涵盖10779种鸟类和40000+参与者，数据自然蕴含了完整的学习动态（准确率随接触次数增长20%），使其成为迄今最大的视觉知识追踪基准。

2. **多层次评估框架**：将预测任务分解为二分类（正确/错误）和多选（具体选项）两个层级，使用宏平均准确率、AP、全数据集与错误子集准确率等互补指标，系统量化不同模型和上下文组合的预测能力，特别是通过"错误子集准确率"这一独特指标揭示了模型近似学习者内部知识状态的真正能力。

3. **三类上下文信号设计**：将可用信息系统化地划分为用户上下文（个体历史）、物种上下文（全局统计）和图像上下文（视觉特征），通过消融实验揭示了各类信号的贡献度差异（U > S, Img在错误预测中尤为关键），为后续研究指明了方向。

## 局限与展望

- **领域特异性**：数据集仅聚焦于鸟类物种识别，向医学影像、物体识别等其他领域的直接迁移能力有待验证
- **地理和选择偏差**：参与者来自eBird平台，已知偏向北半球发达国家用户（如北美和欧洲），导致热带地区和南半球物种的代表性不足，这可能影响模型在这些区域物种上的泛化能力
- **多选格式的局限**：多选格式与开放式识别场景不同，限制了知识追踪到有限选项集，影响了生态效度。尽管通过分类学相似干扰项的动态采样部分缓解了这一问题
- **反馈形式单一**：学习者仅获得正确物种标签作为反馈，缺少更丰富的差异化描述（如"注意翅膀后缘的白色条纹"），未来可探索更精细的反馈机制对知识追踪模型的影响
- **标签噪声**：虽然iNaturalist上鸟类识别的错误率仅3.3%，但公民科学数据中的标注错误仍可能对模型训练产生轻微影响。不过由于核心任务是预测用户回应而非真实物种标签，标签噪声的影响相对有限
- **现有方法的不足**：所有测试模型在预测错误选择上表现极差（<25%），说明近似学习者的内部知识状态仍然极具挑战性。未来应探索更长的时间上下文、物种间知识迁移、以及将二分类信号融入多选模型的方法来提升错误预测能力
- **图像表示的局限**：使用DINOv2/ResNet-50固定特征来近似人类视觉表征虽有理论支撑，但学习者可能关注不同的视觉线索（如新手关注颜色而专家关注形态结构），更精细的人类注意力建模可能带来提升
- **时间窗口约束**：Transformer模型仅使用 $W=50$ 的历史窗口，而一些重度用户有1000+次交互。探索如何高效利用完整交互历史（例如通过检索增强或记忆压缩机制）是未来重要方向
- **跨物种知识迁移未被充分挖掘**：现有方法未利用鸟类分类学的层次结构——对同一科或属下其他物种的掌握程度应能提供迁移信息。例如，能区分两种莺类的用户可能更容易区分其他外观相似的物种对。构建利用分类学层次结构的知识追踪模型是一个有前景的研究方向
- **缺乏主动学习/教学组件**：CleverBirds目前仅作为一个被动的评估基准——记录用户在随机呈现的问题上的表现。未来可以结合主动学习策略，研究最优问题呈现顺序（如使用间隔重复或课程学习策略）以最大化用户学习效率
