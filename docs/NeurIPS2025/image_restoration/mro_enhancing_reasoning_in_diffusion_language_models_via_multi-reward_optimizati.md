# MRO: Enhancing Reasoning in Diffusion Language Models via Multi-Reward Optimization

**会议**: NeurIPS 2025  
**arXiv**: [2510.21473](https://arxiv.org/abs/2510.21473)  
**代码**: 未公开  
**领域**: 扩散语言模型 / LLM推理  
**关键词**: 扩散语言模型, 多奖励优化, token相关性, 重要性采样, Reject Sampling, 强化学习

## 一句话总结

提出多奖励优化方法MRO，通过定义和优化扩散语言模型（DLM）去噪过程中的序列内token相关性和序列间token相关性，结合test-time scaling、reject sampling和强化学习三种策略，配合组步重要性采样降低奖励方差，在推理基准上显著提升DLM性能并实现采样加速。

## 研究背景与动机

### 领域现状

扩散语言模型（Diffusion Language Models, DLMs）是自回归LLM的新兴替代方案。DLM通过逐步去噪（denoising）生成文本：从全掩码序列出发，每步恢复部分token，多步后得到完整输出。相比自回归模型的逐token生成，DLM可以并行去噪、全局规划。

### 核心问题

DLM在**推理任务**上仍然落后于自回归LLM，尤其是当去噪步数减少时性能退化更加明显。作者分析发现根本原因在于：DLM在每个去噪步中**独立生成被掩码的token**，忽略了它们之间的相关性。

### 核心洞察

作者定义了两种关键的token相关性：

**序列内相关性（Intra-sequence correlation）**：同一序列内不同位置token之间的依赖关系

**序列间相关性（Inter-sequence correlation）**：不同去噪步骤之间生成的token序列之间的一致性

增强这两种相关性可以改善推理性能。传统DLM训练只优化单步去噪的交叉熵损失，无法捕获这些跨步跨位置的相关性。

## 方法详解

### 整体框架

DLM基础模型 → 定义多种奖励信号（捕获intra/inter相关性） → 三种优化策略（Test-time Scaling / Reject Sampling / RL） → 组步重要性采样缓解方差 → 推理性能提升 + 采样加速

### 关键设计

#### 1. 多奖励信号设计

**做什么**：设计多个奖励函数，分别度量去噪过程中token相关性的不同方面。

- **序列内奖励**：评估单个去噪步内生成的token组合是否形成连贯的局部结构
- **序列间奖励**：评估相邻去噪步之间的token生成是否保持一致性和渐进改进

**设计动机**：单一奖励信号难以同时捕获多维度的token相关性。多奖励架构允许分别优化不同方面，加权组合形成综合信号。

#### 2. 三种优化策略

**Test-time Scaling**：在推理阶段增加计算预算，生成多个候选序列并根据奖励选择最优，不修改模型权重。

**Reject Sampling**：从DLM采样大量序列，用奖励函数过滤保留高质量样本，作为优化训练数据微调DLM。

**强化学习（RL）**：直接将奖励信号作为RL的reward，通过策略梯度方法更新DLM参数，使模型倾向于生成高奖励序列。

#### 3. 组步重要性采样（Group Step Importance Sampling）

**做什么**：将多个连续去噪步分组，在组级别而非步级别计算重要性权重，降低奖励估计的方差。

**核心思路**：DLM的去噪轨迹很长（多步），逐步计重要性比导致方差爆炸。将步骤分组后，只需计算组级别的比率，显著降低采样效率问题。

**设计动机**：标准重要性采样在长轨迹上方差过大，是RL for DLM的关键瓶颈。组步策略在保持无偏性的同时大幅降低方差。

## 实验关键数据

### 推理基准性能

| 方法 | 推理基准表现 | 采样效率 |
|------|------------|---------|
| 基线DLM | 落后于等参数量AR模型 | 标准 |
| + Test-time Scaling | 明显提升 | 增加推理计算 |
| + Reject Sampling | 显著提升 | 需要大量采样 |
| + RL (MRO) | 最优推理性能 | 模型权重更新后直接提升 |

### 采样加速

MRO训练后的模型在**减少去噪步数**时性能退化更小，实现了高性能与高效采样的兼得。这是因为增强了token相关性后，每步去噪更加"有效"，减少了步数的冗余。

### 关键发现

- **token相关性是DLM推理瓶颈的根源**：独立token生成假设在推理任务中特别有害，因为推理需要严格的逻辑链
- **多奖励优于单奖励**：分别优化intra和inter相关性的多奖励方案优于单一综合奖励
- **组步重要性采样是关键**：没有该技术，RL训练的方差过大，难以稳定收敛

## 亮点与洞察

- **首次系统分析DLM推理短板的根因**：将问题归结为token相关性的缺失，而非简单的"模型不够大"或"训练数据不足"，提供了可操作的优化方向
- **三种策略的递进关系**：Test-time Scaling（不改模型）→ Reject Sampling（数据驱动微调）→ RL（直接奖励优化），形成从简到深的完整优化路径
- **DLM专属的RL技术**：组步重要性采样是为DLM的长去噪轨迹量身定制的，解决了将RL应用于DLM的核心技术障碍

## 局限性 / 可改进方向

- **仅有摘要级信息**：HTML全文不可用，方法和实验细节（具体奖励函数设计、消融数据、基准对比数字）无法完整获取

- **计算开销**：RL训练+多奖励采样的计算成本可能较高
- **与AR模型的差距**：MRO改善了DLM的推理能力，但DLM是否能在推理任务上追平甚至超越等规模的AR模型仍是开放问题

## 相关工作与启发

- **vs 自回归LLM推理优化（如CoT、ToT）**：AR模型通过prompt工程增强推理，DLM则需要从模型训练层面解决token独立性问题，是完全不同的技术路径
- **vs MDLM/SEDD等DLM基础工作**：这些工作聚焦DLM的基础训练和采样方法，MRO是在此基础上针对推理任务的专门优化
- **vs RLHF for AR模型**：RL应用于DLM面临独特挑战（长去噪轨迹→高方差），MRO的组步重要性采样是关键技术差异
