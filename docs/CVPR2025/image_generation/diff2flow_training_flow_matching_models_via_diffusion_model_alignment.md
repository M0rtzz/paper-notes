# Diff2Flow: Training Flow Matching Models via Diffusion Model Alignment

**会议**: CVPR 2025  
**arXiv**: [2506.02221](https://arxiv.org/abs/2506.02221)  
**代码**: https://github.com/CompVis/diff2flow  
**领域**: 扩散模型  
**关键词**: 流匹配, 扩散模型对齐, 时间步映射, LoRA微调, 两步生成

## 一句话总结
提出 Diff2Flow，通过系统化的时间步重缩放、插值对齐和速度推导，将预训练扩散模型无缝转化为流匹配模型，仅需 2.5K 步 LoRA 微调即可提升 T2I 质量，还能实现 2 步生成（reflow）而无需一致性蒸馏。

## 研究背景与动机

**领域现状**：扩散模型（DDPM/DDIM）和流匹配（Flow Matching）是两个主流生成框架。大量预训练权重存在于扩散模型中（如 SD1.5/2.1），但流匹配在理论和效率上有优势（直线路径、更少步数）。

**现有痛点**：从头训练流匹配模型需要巨大计算资源；直接将扩散权重当流匹配用不工作（时间步定义不同、插值不同、速度/噪声参数化不同）；现有扩散到 FM 的转换方法缺乏系统化框架。

**核心矛盾**：想要利用扩散预训练权重，但两种框架的数学定义不兼容。

**切入角度**：建立扩散和 FM 之间的精确数学映射——时间步重缩放、插值对齐、速度推导——使两者轨迹对齐后直接微调。

**核心idea一句话**：可逆时间步映射 $f_t$ + 插值缩放 $1/(\alpha_t + \sigma_t)$ + v-参数化速度推导，实现扩散→FM 的无缝转换，支持 LoRA 微调和 reflow 两步生成。

## 实验关键数据

| 配置 | FID↓ | CLIP↑ | 美学↑ |
|------|------|-------|------|
| SD1.5 (DDIM 25步) | 56.77 | 26.34 | 5.32 |
| **Diff2Flow (FM 25步)** | **52.80** | **26.54** | **5.99** |

2.5K 步即收敛。支持 Reflow 实现 2 步生成（无需一致性蒸馏）。解决扩散模型的非零终端 SNR 问题。LoRA 微调时 FM 微调失败但 Diff2Flow+LoRA 成功。
