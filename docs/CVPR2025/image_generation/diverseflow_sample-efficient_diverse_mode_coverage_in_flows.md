# DiverseFlow: Sample-Efficient Diverse Mode Coverage in Flows

**会议**: CVPR 2025  
**arXiv**: [2504.07894](https://arxiv.org/abs/2504.07894)  
**代码**: 无（未提及）  
**领域**: 扩散模型 / 图像生成  
**关键词**: 流匹配, 多样性, 行列式点过程, 模式覆盖, 推理时优化

## 一句话总结
提出 DiverseFlow，一种训练免费的推理时方法，通过行列式点过程（DPP）在 ODE 求解过程中耦合多个样本使其互相排斥，在固定采样预算下显著提升流匹配模型的模式覆盖多样性（如从 5.64 个模式提升到 7.11 个）。

## 研究背景与动机

**领域现状**：流匹配/扩散模型的采样本质上是独立的——每个样本独立从噪声演化到图像。当采样预算有限时（如只生成 10 张），IID 采样容易重复覆盖高概率模式而错过低概率模式。

**现有痛点**：(1) IID 采样在有限预算下无法保证模式覆盖——多义 prompt（如"A famous boxer"既指拳击手也指犬种）可能只覆盖一种含义；(2) 现有多样性方法（如 Particle Guidance）在图像空间强行排斥导致伪影；(3) 没有理论保证的方式来在流匹配推理时提升多样性。

**核心矛盾**：需要样本间互相排斥以覆盖不同模式，但排斥力过强会破坏单个样本的质量（偏离数据流形）。

**本文要解决什么？** 在不重训练的情况下，让固定数量的样本尽可能多地覆盖不同模式。

**切入角度**：用 DPP（行列式点过程，源自量子物理的多样性概率模型）在 ODE 求解的每步计算样本间排斥梯度，修改流速度使样本远离彼此。质量约束防止样本偏离有效分布。

**核心idea一句话**：在 ODE 每步用 DPP 对数似然梯度修改流速度使样本互斥，配合质量约束在多样性和保真度间平衡。

## 方法详解

### 关键设计

1. **DPP 耦合采样**：K 个样本在每个 ODE 步骤中通过 DPP 核的梯度耦合。DPP 的行列式定义使其自然偏好距离远的样本集合，梯度指向增加样本间距离的方向

2. **质量约束**：限制修改后的速度不能偏离原始流速度太远（$\gamma(t)$ 控制排斥力强度），防止样本跑出数据流形

3. **特征空间距离**：使用 ViT 特征空间（而非像素空间）计算样本间距离，确保语义级别的多样性

## 实验关键数据

| 设置 | 方法 | 模式发现数↑ |
|------|------|----------|
| 合成2D (10模式, K=10) | IID | 5.64 |
| | Particle Guidance | 8.8 |
| | **DiverseFlow** | **7.11 (MB-OT)** / **10 (对称GMM)** |

ImageNet-256 (LFM, CFG=2): Recall 0.41→**0.46**，Precision 基本保持（0.77→0.76）。多义 prompt 能覆盖不同语义（拳击手+犬种）。

### 关键发现
- DPP 比简单排斥（Particle Guidance）在图像质量上更好——PG 产生伪影而 DiverseFlow 不会
- 仅用确定性 ODE 采样（不适用 SDE）
- Recall 提升 12%（0.41→0.46）几乎不损 Precision
