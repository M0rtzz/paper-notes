# Not Just Text: Uncovering Vision Modality Typographic Threats in Image Generation Models

**会议**: CVPR 2025  
**arXiv**: [2412.05538](https://arxiv.org/abs/2412.05538)  
**代码**: 无  
**领域**: AI安全 / 扩散模型  
**关键词**: 排版攻击, 视觉模态安全, 图像生成, 对抗攻击, 安全防御评估

## 一句话总结
本文揭示了图像生成模型在视觉模态上存在的"排版攻击"漏洞——攻击者可以通过在输入图像中嵌入文本来操纵生成结果，并系统评估了现有防御方法对此类视觉模态威胁的无效性，提出了 VMT-IGMs 数据集作为评估基准。

## 研究背景与动机

**领域现状**：当前的图像生成模型（Text-to-Image、Image-to-Image）已能轻松生成高质量、高逼真度的图像。针对这些模型的安全风险，研究社区提出了大量防护策略，但这些策略几乎全部聚焦于语言模态——即过滤或审查文本输入中的不当内容。

**现有痛点**：在实际应用中，特别是涉及真实图像编辑的任务（如 Image-to-Image），攻击者可能通过修改输入图像来操控生成结果。这种视觉模态的威胁具有更高的安全风险，因为它可以轻易侵犯图像所有者的权利，例如将某人的照片修改为不当内容。然而，现有的安全研究几乎忽略了这个攻击面。

**核心矛盾**：防御策略过度依赖语言模态的安全检查（如文本分类器、prompt 过滤），但图像生成模型同时接受视觉输入，而这个输入通道几乎没有安全防护。攻击者可以完全绕过文本层面的防御，直接通过视觉通道注入恶意指令。

**本文目标**：(1) 系统性地揭示图像生成模型在视觉模态上的脆弱性；(2) 评估现有防御方法对视觉模态攻击的有效性；(3) 提供标准化数据集和基准来推动这一方向的研究。

**切入角度**：NLP 领域的"排版攻击"（typographic attack）已被证明可以欺骗 CLIP 等视觉-语言模型——在图像上叠加文本可以改变模型对图像的理解。作者将这种攻击思路迁移到图像生成领域，探索在输入图像中嵌入排版内容是否能操纵生成模型的输出。

**核心 idea**：利用排版攻击（在输入图像中添加文本覆盖层）来揭示图像生成模型中视觉模态的安全漏洞，并系统评估现有防御方案的盲区。

## 方法详解

### 整体框架
研究分为三个阶段：(1) 攻击构建——设计针对图像生成模型视觉输入通道的排版攻击方法；(2) 漏洞评估——在多种主流图像生成模型上测试攻击效果；(3) 防御评估——测试现有安全方法对视觉模态攻击的防护能力。最终构建 VMT-IGMs 数据集供社区研究使用。

### 关键设计

1. **排版攻击方法（Typographic Attack on Vision Modality）**:

    - 功能：在不修改文本 prompt 的情况下，通过修改输入图像来操纵生成模型的输出
    - 核心思路：在输入图像上叠加特定的文本内容（如不当词汇、指令性文本），利用图像生成模型内部的视觉-语言对齐能力（如 CLIP 编码器），使模型在处理带有嵌入文本的图像时，被文本内容引导生成不符合原始 prompt 的输出。攻击过程不需要对模型权重的访问（黑盒攻击），只需要能够修改输入图像。排版的位置、大小、字体、颜色等参数都可以调整以优化攻击效果。
    - 设计动机：现代图像生成模型广泛使用 CLIP 等视觉-语言模型作为条件编码器，这些模型对图像中的文本内容非常敏感。排版攻击利用的正是这种跨模态对齐能力的脆弱性——模型无法区分"正常图像内容"和"恶意嵌入的文本"。

2. **多模型系统性漏洞评估**:

    - 功能：全面评估排版攻击在不同图像生成模型和任务上的效果
    - 核心思路：选取多种代表性的图像生成模型，涵盖 Text-to-Image（如 Stable Diffusion）和 Image-to-Image（如 InstructPix2Pix、ControlNet 等）任务。对每个模型，测试不同类型的排版内容（不当内容、身份替换、风格修改等）和不同的攻击参数。使用自动化指标（如 CLIP 相似度变化、内容检测器分数）和人工评估来量化攻击成功率。
    - 设计动机：不同模型架构和训练方式可能对排版攻击有不同的敏感度，系统性评估可以揭示哪些设计选择使模型更加脆弱。

3. **VMT-IGMs 数据集与防御评估**:

    - 功能：提供标准化基准数据集，并揭示现有防御策略的不足
    - 核心思路：构建 Vision Modal Threats in Image Generation Models（VMT-IGMs）数据集，包含精心设计的攻击样本和对应的生成结果。在此数据集上评估现有的多种安全防护方法，包括输入过滤、输出检测、safety filter 等。实验结果表明，现有的防御手段在面对视觉模态攻击时表现不佳。
    - 设计动机：安全研究需要标准化的评估基准。VMT-IGMs 填补了视觉模态安全评估数据集的空白，帮助社区开发更全面的防御策略。

### 损失函数 / 训练策略
本文是分析性工作，不涉及模型训练。排版攻击是无梯度的黑盒方法，通过在图像表面叠加文本来实现，不需要对目标模型进行任何优化。

## 实验关键数据

### 主实验（攻击成功率）

| 模型 | 任务类型 | 攻击成功率 | 生成质量影响 |
|------|---------|-----------|------------|
| Stable Diffusion (I2I) | 图像编辑 | 高 | 显著偏离原始intent |
| InstructPix2Pix | 指令编辑 | 高 | 生成内容受排版控制 |
| ControlNet | 条件生成 | 中-高 | 视觉条件被覆盖 |
| DALL-E 系列 | T2I+视觉条件 | 中 | 部分场景有效 |

### 防御评估

| 防御方法 | 文本攻击防护 | 视觉攻击防护 | 说明 |
|---------|-----------|------------|------|
| Prompt 过滤 | 有效 | 无效 | 完全旁路视觉通道 |
| Safety Filter (输出) | 部分有效 | 有限 | 仅检测明显不当内容 |
| NSFW 检测器 | 有效 | 有限 | 对隐性操纵无效 |
| 输入图像审查 | 不适用 | 有限 | 排版可被优化为不易检测 |

### 关键发现
- 几乎所有主流图像生成模型都对视觉模态的排版攻击存在脆弱性，包括已部署安全机制的商业模型
- 现有防御方法中，专门针对文本 prompt 的过滤对视觉攻击完全无效——攻击者甚至不需要使用任何文本 prompt
- 基于输出的安全检测器对排版攻击也效果有限，因为攻击可以产生看似正常但语义已被操纵的输出
- 图像中嵌入的文本大小和位置对攻击效果有显著影响，较大且居中的文本攻击成功率更高

## 亮点与洞察
- **安全盲区的系统揭示**：本文的核心贡献不在于提出新方法，而在于系统性地暴露了一个被忽视但重要的安全问题——视觉通道成为安全防御的"后门"。这对实际部署的图像生成服务有直接的安全意义
- **攻击的简单性令人警醒**：排版攻击不需要任何梯度计算或模型内部访问，仅在图像上叠加文字就能成功，这意味着攻击门槛极低、实际威胁极大
- **VMT-IGMs 作为标准基准**：填补了视觉模态安全评估的空白，为后续防御研究提供了可比较的基准

## 局限与展望
- 论文主要是揭示问题和评估现有防御，但未提出有效的防御方案
- 排版攻击的范围可能有限——在某些应用场景中（如纯 T2I 无图像输入），视觉模态攻击不适用
- 未深入分析为什么某些模型更容易受到排版攻击——是 CLIP 的问题还是生成模型本身的问题
- 未来工作可以探索：(1) 训练对排版鲁棒的视觉编码器；(2) 输入图像的 OCR 预处理和文本擦除；(3) 多模态联合安全检查机制

## 相关工作与启发
- **vs 文本对抗攻击**: 传统的对抗 prompt 攻击在文本层面操纵模型，已有较多防御方案。本文揭示视觉层面的攻击是一个全新且未被防御的攻击面
- **vs CLIP 排版攻击**: 先前研究已证明排版可以欺骗 CLIP 的图像分类。本文将这一发现推广到图像生成模型，展示了更严重的实际安全风险
- **vs 图像对抗扰动**: 传统的对抗扰动（如 FGSM）需要梯度访问且扰动不可见。排版攻击虽然可见但更加简单，且对黑盒模型同样有效

## 评分
- 新颖性: ⭐⭐⭐⭐ 首次系统研究图像生成模型的视觉模态安全威胁，角度新颖
- 实验充分度: ⭐⭐⭐⭐ 覆盖多种模型和防御方法，VMT-IGMs 数据集是重要贡献
- 写作质量: ⭐⭐⭐⭐ 问题阐述清晰，安全威胁分析到位
- 价值: ⭐⭐⭐⭐ 对图像生成系统的安全部署有直接指导意义，但缺乏防御方案略显不足
