# TraF-Align: Trajectory-aware Feature Alignment for Asynchronous Multi-agent Perception

**会议**: CVPR 2025  
**arXiv**: 待确认  
**代码**: 待确认  
**领域**: 协同感知  
**关键词**: cooperative perception, asynchronous fusion, feature alignment, trajectory prediction, V2V communication

## 一句话总结

提出TraF-Align框架，通过预测特征级目标轨迹来学习特征的流动路径，生成沿时间排列的采样点将当前时刻query引导至相关历史特征，解决异步多智能体感知中的时空失配问题。

## 研究背景与动机

**领域现状**: 协同感知（Cooperative Perception）通过多个智能体（车辆/路侧设备）共享感知信息来扩展单车感知范围，已成为自动驾驶的重要方向。现有方法假设各智能体数据在时间上同步，通过空间变换对齐不同视角的特征后融合。

**现有痛点**: (1) 真实场景中智能体间存在不可避免的通信延迟（几十到几百毫秒），导致接收到的其他车辆特征已过时；(2) 延迟引起空间和语义层面的双重失配——移动目标的位置和外观都发生了变化；(3) 现有补偿方法要么简单用GPS/位姿做刚性变换（忽略场景动态），要么用光流/稀疏注意力做点对点对齐（缺乏对目标运动连续性的建模）。

**核心矛盾**: 异步条件下特征的空间位置随时间连续变化，但现有方法没有显式建模特征在时间维度上的流动路径——它们只考虑两个时间点之间的jump，而非连续轨迹。

**本文要解决什么？** 在异步多智能体感知中，如何利用历史观测预测特征的连续运动轨迹，从而在任意延迟下精确对齐ego当前query到协作车辆的历史特征。

**切入角度**: 将特征对齐问题转化为特征级轨迹预测问题——从过去观测中学习目标特征的运动轨迹，沿轨迹生成时间有序的采样点来引导跨时间的attention。

**核心idea一句话**: 通过预测特征级目标轨迹而非简单的空间变换来实现异步特征对齐，让attention从当前时刻query沿预测轨迹回溯到相关历史特征。

## 方法详解

### 整体框架

TraF-Align的核心流程：(1) 每个智能体独立提取BEV特征；(2) 协作车辆发送其历史BEV特征到ego车辆；(3) TraF-Align利用收到的多帧历史特征预测特征级目标轨迹，生成从历史到当前时刻的连续采样位置；(4) 以ego的当前时刻特征作为query，沿预测轨迹的采样点从历史特征中聚合信息；(5) 融合对齐后的协作特征与ego自身特征完成检测。

### 关键设计

1. **特征级轨迹预测模块**: 不同于直接对image-level做运动估计，TraF-Align在BEV特征空间上预测每个目标的运动轨迹。利用过去多帧协作车辆的BEV特征，通过时序建模（如temporal encoder）捕捉目标运动趋势，外推到ego当前时刻。输出是一组时间有序的采样点坐标，描述了特征从过去到现在的流动路径。这种设计显式建模了运动的连续性，而非仅做两帧之间的对齐。

2. **轨迹引导的跨时间Attention**: 基于预测的特征轨迹，在每个采样时间点生成采样位置，构建trajectory-aware deformable attention。当前时刻的ego query不再盲目搜索全局特征，而是沿着预测轨迹精准定位到历史特征中目标曾经所在的位置。这种沿轨迹的attention路径兼顾了空间对齐（目标在哪）和语义一致性（目标怎么变化），有效减少了因延迟导致的错误关注。

3. **多智能体异步融合策略**: 框架设计可处理不同协作车辆具有不同延迟的情况，通过per-agent轨迹预测独立处理每个协作源的时间偏移，最终在ego坐标系下统一融合。这种per-agent设计比全局统一补偿更灵活，能适应实际部署中不同车辆有不同通信延迟的场景。

## 实验关键数据

### 关键发现

- 在**V2V4Real**真实世界协同感知数据集上验证，证明方法在真实通信延迟和传感器噪声下有效
- 在**DAIR-V2X-Seq**车路协同序列数据集上评估，展示了对路侧-车端异步感知的适用性
- 与现有异步补偿方法（如SyncNet、基于光流的方法）相比，TraF-Align在不同延迟程度下都取得了更好的检测精度
- 轨迹预测模块带来的增益在大延迟场景下更为显著——延迟越大，简单空间变换的失效越严重，轨迹建模的优势越明显
- 消融实验表明轨迹引导attention的每个组件都有贡献，验证了显式轨迹建模的必要性

## 亮点与洞察

- 将异步对齐问题重新formulate为特征级轨迹预测，视角新颖——不是修补两个快照之间的偏差，而是建模连续运动
- 沿轨迹的attention设计在概念上非常直觉：既然目标沿轨迹移动，那么查找其历史特征也应该沿轨迹回溯
- 方法在真实数据集V2V4Real上验证，而非仅限于仿真，增强了实用价值
- 框架设计对不同延迟量级和多智能体场景具有良好适应性

## 局限性 / 可改进方向

- 轨迹预测依赖历史多帧特征的可用性，在通信极不稳定（帧丢失严重）的场景下可能退化
- 特征级轨迹预测增加了计算开销，需要评估在边缘设备上的实时性
- 对于突然出现的新目标（无历史观测），轨迹预测无法启动，仍需退化到传统对齐方法
- 目前验证的下游任务以3D目标检测为主，在语义分割、运动预测等任务上的泛化性待验证
- 非线性运动（急转弯、紧急刹车）场景中轨迹预测精度可能不足，可考虑引入不确定性估计
