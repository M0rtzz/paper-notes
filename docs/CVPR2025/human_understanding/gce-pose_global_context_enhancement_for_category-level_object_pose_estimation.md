# GCE-Pose: Global Context Enhancement for Category-Level Object Pose Estimation

**会议**: CVPR 2025  
**arXiv**: [2502.04293](https://arxiv.org/abs/2502.04293)  
**代码**: https://colin-de.github.io/GCE-Pose/  
**领域**: 3D视觉 / 人体理解  
**关键词**: 类别级位姿估计, 全局上下文, 语义形状重建, 深度线性形状模型, 特征融合

## 一句话总结

GCE-Pose 提出一种"先补全再聚合"的策略，通过语义形状重建（SSR）模块将部分观测补全为完整的几何+语义 3D 表示，再通过全局上下文增强（GCE）特征融合模块将全局信息注入局部关键点特征，在 HouseCat6D 和 NOCS-REAL275 上显著超越现有方法。

## 研究背景与动机

**领域现状**：类别级物体位姿估计旨在对同一类别的未见实例预测 6D 位姿和尺寸，主流方法是估计 NOCS（归一化物体坐标空间）坐标后用 Umeyama 算法求解位姿。近期方法如 AG-Pose 利用关键点检测和 DINOv2 特征取得了不错效果。

**现有痛点**：由于类别级估计是 model-free 的（推理时只有单张 RGB-D 输入,无需 CAD 模型），深度传感器捕获的部分观测不可避免地面临遮挡和不完整几何的问题。现有方法仅从部分观测中提取特征，缺乏对完整物体的全局上下文理解。

**核心矛盾**：部分观测提供的信息有限——一个被严重遮挡的杯子可能只看到把手部分，仅从这些有限信息很难准确建立与 NOCS 空间的对应关系。虽然有些方法引入了几何形状先验，但忽略了语义上下文（如物体各部分的功能性语义特征）。

**本文要解决什么？** 如何在仅有部分 RGB-D 观测的情况下，获取既包含几何信息又包含语义信息的完整物体全局表示，并有效融合到局部特征中。

**切入角度**：观察到几何先验和语义先验是互补的——几何先验提供物体的完整形状信息，语义先验（来自 DINOv2）提供物体各部分的功能性理解。将两者结合可以在遮挡和形状变化下提供鲁棒的全局上下文。

**核心idea一句话**：通过深度线性语义形状模型将部分点云补全为带语义特征的完整 3D 形状，再用交叉注意力将全局上下文融入局部关键点特征。

## 方法详解

### 整体框架

GCE-Pose 由四个模块组成：(A) 从 RGB-D 输入提取点云和图像特征，通过关键点检测器选取最具判别力的关键点及其特征；(B) SSR 模块利用类别形状先验将部分点云补全，并附上类别级语义原型；(C) GCE 模块通过交叉注意力将全局语义形状信息融合到关键点特征中；(D) 融合后的特征用于预测关键点的 NOCS 坐标，进而回归位姿和尺寸。

### 关键设计

1. **语义形状重建 (SSR) 模块**:

    - 功能：将部分观测的点云补全为带语义信息的完整 3D 形状
    - 核心思路：采用深度线性语义形状模型（DLSSM），每个类别 $k$ 维护一个几何原型 $c^k$ 和变形基向量 $v^k$，以及一个语义原型 $c^k_{sem}$（由 DINOv2 特征构建）。给定部分点云，通过变形网络 $\mathcal{D}^k$ 和缩放网络 $\mathcal{S}^k$ 预测实例特定的形状参数 $a^k$ 和缩放 $s^k$，最终重建形状为 $\mathbf{U}_k = (s^k \odot (c^k + \sum_i a_i^k v_i^k), c^k_{sem})$。训练分两阶段：先用完整点云学习形状空间，再用部分点云学习从部分到完整的映射
    - 设计动机：线性形状模型参数化紧凑高效，同时语义特征随几何变形自然传播，无需额外的语义对齐步骤

2. **语义原型构建**:

    - 功能：为每个类别构建带有部件语义信息的 3D 原型
    - 核心思路：对每个物体实例从多个虚拟视角渲染 RGB 图像，用 DINOv2 提取 2D 语义特征图，通过深度图和相机参数投影到 3D 空间得到密集语义点云，再通过 KNN 聚合到形状原型的各个点上。跨实例取平均得到类别级语义原型 $c^k_{sem} = \frac{1}{N}\sum_{i=1}^N \mathbf{F}_{instance}(P_i^k)$
    - 设计动机：DINOv2 的零样本语义特征对形状变化具有鲁棒性，类别级平均能捕捉共性语义结构

3. **全局上下文增强 (GCE) 特征融合**:

    - 功能：将全局语义形状信息有效融合到局部关键点特征中
    - 核心思路：使用可学习的位置编码分别为局部关键点（相机空间坐标）和全局重建（归一化物体空间坐标）生成位置 token，与各自特征拼接后通过共享的 MLP 投影到统一空间。然后以关键点特征为 query、全局特征为 key/value 进行交叉注意力：$\mathbf{F}_{gce} = \mathbf{F}_{kpt} + \text{CrossAttn}(\mathbf{F}''_{kpt}, \mathbf{F}''_{global})$
    - 设计动机：部分观测（相机空间）和全局重建（物体空间）存在坐标系域间差距，通过可学习的位置编码来弥合这一gap

### 损失函数 / 训练策略

总损失由五项组成：$\mathcal{L}_{all} = \lambda_1 \mathcal{L}_{ocd} + \lambda_2 \mathcal{L}_{div} + \lambda_3 \mathcal{L}_{rec} + \lambda_4 \mathcal{L}_{nocs} + \lambda_5 \mathcal{L}_{pose}$。其中 $\mathcal{L}_{ocd}$ 和 $\mathcal{L}_{div}$ 约束关键点质量（贴合物体表面且分散），$\mathcal{L}_{rec}$ 是形状重建的 Chamfer 距离，$\mathcal{L}_{nocs}$ 是 NOCS 坐标的 Smooth L1 损失，$\mathcal{L}_{pose}$ 包含旋转、平移和尺寸的直接回归损失。在单张 4090 GPU 上训练 150 个 epoch。

## 实验关键数据

### 主实验

| 数据集 | 方法 | 5°2cm | 10°5cm | IoU75 |
|--------|------|-------|--------|-------|
| HouseCat6D | AG-Pose (DINO) | 21.3 | 54.3 | 53.0 |
| HouseCat6D | SecondPose | 11.0 | 35.7 | - |
| HouseCat6D | **GCE-Pose** | **24.8** | **58.4** | **60.6** |
| NOCS-REAL275 | AG-Pose (DINO) | 57.0 | 84.7 | 80.1 |
| NOCS-REAL275 | GS-Pose | - | 60.1 | 63.2 |
| NOCS-REAL275 | **GCE-Pose** | **57.0** | **86.3** | **79.8** |

### 消融实验

| 配置 | 5°2cm | 5°5cm | 说明 |
|------|-------|-------|------|
| Baseline (AG-Pose DINO) | 21.3 | 22.1 | 无全局先验 |
| + Instance Geo. | 22.2 | 23.7 | 仅几何重建 +4% |
| + Categorical Sem. | 22.7 | 24.3 | 仅语义原型 +7% |
| + Mean shape + Sem. | 23.4 | 24.2 | 均值形状 +10% |
| **Full GCE-Pose** | **24.8** | **25.7** | 完整模型 **+16%** |

### 关键发现

- 几何先验和语义先验的贡献是互补的：单独使用几何提升 4%，语义提升 7%，联合使用提升 16%，存在明显的协同效应
- 实例级几何重建比均值形状更重要：可以捕捉实例内形状变化
- GCE 融合模块中使用 key token（而非 value token）效果更好，因为 key 初始化了局部-全局之间的注意力权重
- SSR 模块对噪声和遮挡场景表现出良好的鲁棒性，语义特征在形状变化下保持一致

## 亮点与洞察

- **"先补全再聚合"策略**：不是直接从部分观测提特征，而是先重建完整形状再提取全局特征。这种策略让模型在严重遮挡下也能获得对物体完整形状的理解。该思路可迁移到任何需要处理部分观测的 3D 理解任务
- **语义随几何自然传播**：线性形状模型的巧妙之处在于语义原型固定不变，只需做几何变形就能得到实例特定的语义形状表示，无需额外的语义对齐或匹配步骤
- **位置编码桥接域间差距**：部分观测和全局重建处于不同坐标系，用可学习位置编码自动学习坐标变换关系，简洁有效

## 局限性 / 可改进方向

- 依赖 ground truth 分割 mask 作为输入，实际应用中分割质量会影响性能
- 每个类别需要单独训练形状模型（变形网络和缩放网络），跨类别泛化性有限
- 线性形状模型对大形变类别（如衣物、柔性物体）表达能力可能不足
- 未在透明或反光物体上验证，这些物体的深度数据质量差

## 相关工作与启发

- **vs AG-Pose**: AG-Pose 仅从部分观测提取关键点特征，GCE-Pose 额外引入全局上下文作为补充信息，在严格指标上提升 16%
- **vs GS-Pose**: GS-Pose 选择单个参考实例做语义匹配，对类内形状变化和部分观测噪声敏感；GCE-Pose 学习跨实例的变形先验，更鲁棒
- **vs SPD/RePoNet**: 早期方法只引入几何形状先验，GCE-Pose 进一步引入语义先验，实现了几何+语义的全局上下文增强
