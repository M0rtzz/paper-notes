# LotusFilter: Fast Diverse Nearest Neighbor Search via a Learned Cutoff Table

**会议**: CVPR 2025  
**arXiv**: [2506.04790](https://arxiv.org/abs/2506.04790)  
**代码**: https://github.com/matsui528/lotf (有)  
**领域**: 向量检索 / 多样化搜索  
**关键词**: 多样化最近邻搜索、学习数据结构、截取表、RAG、后处理

## 一句话总结
提出LotusFilter，通过预计算截取表和贪心集合过滤，以O(S+KL)线性复杂度实现多样化近邻搜索，比传统方法快50-100倍，内存仅1/40。

## 研究背景与动机

**领域现状**：ANNS是RAG/推荐系统基石，但返回的K个结果往往高度相似缺乏多样性。

**现有痛点**：传统多样化(DNNS)方法需O(DS²K²)复杂度，需加载原始向量，无法与现代ANNS(HNSW/IVF)高效集成。

**核心洞察**：可以离线预计算所有向量的"邻近关系"（截取表），推理时仅用简单集合删除操作完成多样化。

## 方法详解

### 整体框架
预处理：为每个向量$x_n$计算 $\mathcal{L}_n = \{i : \|x_n - x_i\|^2 < \varepsilon\}$ → 查询时：ANNS获取S个候选 → 贪心弹出最相似者加入结果集 → 删除其邻近向量 → 重复直到K个结果。

### 关键设计

1. **截取表构造**：每个向量存储距离小于ε的邻居ID集合，O(NT)预计算，N=900k仅需1分钟
2. **OrderedSet数据结构**：数组+哈希表混合实现，Pop O(L)、Remove O(1)
3. **超参数学习**：Bracketing一维搜索自动优化ε，平衡proximity和diversity

### 损失函数 / 训练策略
目标：$f(\mathcal{K}) = \frac{1-\lambda}{K}\sum_{k}\|q-x_k\|^2 - \lambda\min_{i,j}\|x_i-x_j\|^2$

## 实验关键数据

### 主实验：OpenAI数据集(N=900K, D=1536)

| 方法 | 综合f↓ | 总时间(ms) | 内存(GB) |
|------|--------|-----------|----------|
| 纯ANNS | 0.855 | 0.855 | - |
| k-means聚类 | 0.941 | 7.88 | 4.42×10¹⁰ |
| GMM | 0.977 | 14.4 | 4.42×10¹⁰ |
| **LotusFilter** | **1.00** | **1.03** | **1.14×10⁹** |

### 关键发现
- 1.03ms总时间（GMM的1/14），内存仅传统方法1/40
- 预处理N=900k仅54秒，截取表136MB

## 亮点与洞察
- 简洁性：复杂多样化问题用集合删除解决
- 纯后处理设计，与任何ANNS黑盒兼容
- 直接应用于RAG系统消除重复检索

## 局限性 / 可改进方向
- 单一全局ε对不同查询/数据区域可能非最优
- 预处理需提前确定ε
- 低维度无优势
