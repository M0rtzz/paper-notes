# Event Fields: Capturing Light Fields at High Speed, Resolution, and Dynamic Range

**会议**: CVPR 2025  
**arXiv**: [2412.06191](https://arxiv.org/abs/2412.06191)  
**代码**: 有（开源模拟器、数据和处理代码）  
**领域**: 3D视觉  
**关键词**: 事件相机, 光场, 高速成像, 空间复用, 时间复用

## 一句话总结
本文提出 Event Fields——一种利用事件相机捕获高速、高分辨率、高动态范围光场的新范式，设计了万花筒（空间复用，捕获时间导数）和振镜（时间复用，捕获角度导数）两种互补光学方案，实现了 250fps 百万像素动态场景重聚焦和 100Hz 实时深度估计等前所未有的能力。

## 研究背景与动机

1. **领域现状**：事件相机因其高时间分辨率（μs 级）、高动态范围（140dB）、低带宽和异步工作模式，在高速应用中得到广泛使用，包括光流、目标跟踪、HDR 成像等。另一方面，光场成像通过捕获角度辐射信息实现后处理重聚焦和深度估计，但传统光场相机受限于帧率和动态范围。

2. **现有痛点**：事件相机只捕获时间维度的亮度变化，丢失了角度信息。传统光场相机（Lytro、Raytrix）帧率低、动态范围有限，无法处理高速场景。现有事件相机+光场的尝试要么限于静态场景（LCOS 方案，响应时间 22ms），要么牺牲空间分辨率（微透镜阵列方案）。

3. **核心矛盾**：事件相机擅长高速时间感知但缺少角度维度，光场擅长角度感知但缺少高速能力。如何在一台事件相机上同时捕获时间和角度信息？

4. **本文要解决什么？** 将角度信息编码到事件相机已有的空间或时间维度中，从而用单台事件相机捕获完整的光场——即 "Event Field"。

5. **切入角度**：光场在角度维度通常很平滑（冗余），因此角度导数 $\partial L / \partial \omega$ 是稀疏的——这与事件相机天然捕获稀疏变化的特性完美契合。

6. **核心idea一句话**：通过光学设计将角度信息复用到空间维度（万花筒）或时间维度（振镜），使事件相机从捕获"时间导数"扩展为捕获"时空-角度导数"，实现高速光场。

## 方法详解

### 整体框架
Event Fields 的数学框架建立在全光函数 $L(\vec{x}, \vec{\omega}, \lambda, t)$ 的导数之上。标准事件相机只测量时间导数 $\partial B / \partial t$，本文通过两种复用策略将角度维度编码进来：空间复用测量多角度的时间导数 $\partial B(\mathbf{x_s}, \omega, t_k) / \partial t$，时间复用直接测量角度导数 $\partial B(\mathbf{x}, \omega) / \partial \omega$。在此基础上，通过现有事件处理算法（E2VID、TimeLens）重建高速光场视频，进而实现重聚焦、深度估计等应用。

### 关键设计

1. **万花筒空间复用（Kaleidoscope for Spatial Multiplexing）**:

    - 功能：将多角度光场信息编码到空间维度，捕获各角度的时间导数。
    - 核心思路：在主镜头后放置矩形万花筒（四面镜），主镜头在万花筒入口成像，经多次反射后在事件相机传感器上形成 3×3 个翻转图像，每个对应不同虚拟相机位置。映射函数为 $\mathbf{x_s} = \mathbf{x} \bmod (\mathbf{r/n})$，其中 $\mathbf{n}$ 为角度视图数，$\mathbf{r}$ 为传感器分辨率。事件相机测量的是 $\partial B / \partial t (\mathbf{x_s}, \omega, t_k) \approx p_k C / \Delta t_k$。额外搭配分光镜和 RGB 相机可实现彩色高速光场（传感器融合）。
    - 设计动机：万花筒无需移动部件，结构简单可靠。用空间分辨率换角度分辨率（降低 $n$ 倍），但对动态场景天然适用（不受运动模糊影响，因为每个角度独立积累事件）。缺点是无法捕获静态场景（无时间变化时无事件触发）。

2. **振镜时间复用（Galvanometer for Temporal Multiplexing）**:

    - 功能：将角度信息编码到时间维度，直接捕获角度导数。
    - 核心思路：在事件相机光路中放置双轴振镜，沿周期曲线 $\mathcal{C}(t)$（如圆形 Lissajous 曲线）以 250Hz 高速扫描，虚拟旋转相机。对静态场景，事件相机测量的是角度导数 $\partial B / \partial \omega \cdot \partial \mathcal{C}(t) / \partial t |_{t=t_k} \approx p_k C / \Delta t_k$。这意味着事件直接编码了光场的角度梯度信息。使用 E2VID 以 10000fps 重建视频，在 250Hz 扫描下得到 250fps × 40 视图的光场。
    - 设计动机：不牺牲空间分辨率，能捕获静态场景（因为角度变化本身触发事件），直接获得角度导数是更高效的光场表示。代价是运动物体会有运动模糊（对象运动与振镜运动叠加），且带宽需求更高。

3. **物理仿真器（Blender 插件）**:

    - 功能：为两种方案提供可控的公平对比和系统评估。
    - 核心思路：基于 Blender 开发了一个渲染 Event Fields 的插件，通过渲染全光函数→光学传播计算亮度→逐像素追踪亮度变化→阈值触发事件的流程，模拟空间复用和时间复用两种设计。支持 GPU 加速和 GUI 交互。
    - 设计动机：实际系统中两种方案的光学参数（焦距等）不同，难以在相同条件下公平对比。模拟器可以精确控制场景和设计参数，系统评估两种方案的优劣。

### 损失函数 / 训练策略
本文是一个计算成像系统，不涉及神经网络训练。核心算法包括：（1）使用预训练 E2VID 从事件流重建高速视频；（2）使用 TimeLens 进行事件-RGB 融合的帧插值；（3）基于光场理论的后处理重聚焦（光场 shear 和积分）；（4）基于 depth-from-focus 的深度估计。

## 实验关键数据

### 主实验（两种方案对比）

| 特性 | 万花筒（空间复用） | 振镜（时间复用） |
|------|------------------|----------------|
| 捕获导数类型 | 时间导数 | 角度+时间导数 |
| 静态场景捕获 | ✗ | ✓ |
| 动态场景重聚焦 | ✓（无运动模糊） | ✓（受运动速度影响） |
| 彩色支持 | ✓（分光镜+RGB） | ✗ |
| 空间分辨率 | 降低 n 倍 | 全分辨率 |
| 带宽需求 | 低 | 高 |
| 重聚焦质量 | 离散跳跃感 | 平滑连续 |

### 应用演示

| 应用 | 方案 | 关键性能 |
|------|------|---------|
| SloMoRF 慢动作重聚焦 | 万花筒+RGB 融合 | 720fps 彩色光场，三焦面动态重聚焦 |
| 高速重聚焦 | 振镜 | 250fps × 40 视图，百万像素分辨率 |
| 即时真实深度估计 | 振镜 | 100Hz 实时深度测量 |
| HDR 光场 | 万花筒 | 捕获 140dB 动态范围（vs RGB 60dB） |

### 关键发现
- 万花筒方案对动态场景更鲁棒（不受运动模糊影响），但无法处理静态场景（无事件触发），且空间分辨率降低 9 倍（3×3 视图）。
- 振镜方案可以捕获静态场景且不牺牲空间分辨率，但运动物体速度增加时重聚焦质量下降（实验中 8× 速度出现明显模糊）。
- 事件相机在振镜扫描频率超过 250Hz 时达到读出带宽极限，开始随机丢失事件——但反直觉地，旋转风扇上的文字在高扫描频率下反而更清晰（运动模糊减少）。
- HDR 实验中，事件光场成功同时捕获了明亮 LED 和暗区纹理，而 RGB 相机无论长或短曝光都无法兼顾。

## 亮点与洞察
- **"Event Fields"概念本身**是最大贡献——将事件相机从"时间导数传感器"重新定义为"时空-角度导数传感器"，开辟了全新的计算成像范式。光场的角度稀疏性恰好匹配事件相机的稀疏感知特性，这是一个非常优美的观察。
- **两种互补方案的系统对比**非常有价值——万花筒适合高速动态场景，振镜适合需要高空间分辨率或静态场景的情况，为实际应用提供了清晰的选择指南。
- **SloMoRF（慢动作重聚焦）**演示令人印象深刻——在水花中追踪扔入的木块，同时实现 720fps 和动态重聚焦，这是传统相机完全无法实现的。
- 仿真器的开源对社区有长远价值，可以加速 Event Fields 领域的算法研究。

## 局限性 / 可改进方向
- 当前使用预训练的 E2VID 和 TimeLens，这些模型针对普通事件流优化，对 Event Fields 的特殊数据分布可能不是最优的。专门为 Event Fields 训练的重建模型会显著提升质量。
- 振镜方案对运动物体的模糊问题可以通过自适应扫描（运动区域快扫、静态区域慢扫）缓解，使用 MEMS 阵列可以实现空间变化的扫描速度。
- 实验中只展示了宏观场景，微观场景（数字全息、光场显微镜）可能是更有价值的应用方向（已有 EventLFM 初步探索）。
- 两种方案各有局限，可以考虑混合设计——同时使用空间和时间复用捕获更完整的 Event Fields。

## 相关工作与启发
- **vs Habuchi et al.（LCOS+事件）**: 他们用 LCOS 在光圈面编码角度，但 LCOS 响应时间 22ms 限制了动态场景。振镜方案的响应速度快几个数量级。
- **vs Guo et al.（微透镜阵列+事件）**: MLA 方案功能上等价于万花筒的空间复用，但万花筒实现更简单且适用于宏观场景。
- **vs He et al.（旋转楔形镜+事件）**: 他们用旋转光学器件实现全方向边缘检测，与振镜方案有类似思路，但 Event Fields 首次将其扩展为完整的光场捕获框架。
