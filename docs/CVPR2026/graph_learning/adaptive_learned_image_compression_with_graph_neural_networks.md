# GLIC: Adaptive Learned Image Compression with Graph Neural Networks

**会议**: CVPR 2026  
**arXiv**: [2603.25316](https://arxiv.org/abs/2603.25316)  
**代码**: https://github.com/UnoC-727/GLIC (有)  
**领域**: 图学习 / 图像压缩  
**关键词**: 学习图像压缩, GNN, 自适应连接, 双尺度图, 复杂度感知度分配

## 一句话总结
提出GLIC，首个基于GNN的端到端学习图像压缩框架，通过双尺度图（局部密集+全局稀疏）实现灵活感受野，复杂度感知的自适应度分配让每个像素根据内容复杂度决定邻居连接数量，在Kodak/Tecnick/CLIC上相对VTM-9.1实现BD-rate减少19-22%。

## 研究背景与动机

**领域现状**：端到端学习图像压缩(LIC)已超越传统编解码器(VTM)。主流方法基于CNN(标准卷积)或Transformer(窗口注意力)构建非线性变换网络。

**现有痛点**：(1) 标准CNN的 $k \times k$ 卷积固定连接每个像素到 $k^2$ 个邻居——感受野固定、连接模式静态；(2) 窗口注意力同样在预设块内操作；(3) 两者都无法自适应地连接空间距离远但特征相关的像素；(4) 自然图像冗余高度非均匀——平滑区域高冗余、纹理/边缘低冗余——静态连接无法适配。

**核心矛盾**：高效压缩需要建模局部和全局冗余，但CNN/Transformer的固定连接模式在"连哪里"和"连多少"两个维度都不灵活。

**切入角度**：GNN的动态、数据依赖连接天然适合捕捉非欧几何关系和空间变化的冗余模式。

**核心idea**：双尺度图决定"where to connect"（局部稠密+全局稀疏），自适应度分配决定"how much to connect"（基于内容复杂度），两者解耦实现内容自适应的冗余建模。

## 方法详解

### 整体框架
标准VAE-based LIC pipeline。编码器 $g_a$ 和解码器 $g_s$ 中的非线性变换用**Graph-based Feature Aggregation (GFA)** block替代CNN/Transformer。前期阶段用轻量Conv Block，后两阶段用GFA block。熵模型采用超先验结构。

### 关键设计

1. **双尺度候选采样（灵活感受野）**:

    - **局部密集图**：$L \times L$ 固定邻域内所有像素作为候选→保留精细局部结构
    - **全局稀疏图**：以stride $\Omega_h, \Omega_w$ 进行网格采样，每个节点获取 $G \times G$ 个全局候选→提供远程上下文
    - 候选集 $\mathcal{C}_i = \mathcal{C}_i^{loc} \cup \mathcal{C}_i^{glo}$
    - 设计动机：局部捕获精细结构，全局捕获远程冗余。相比全局注意力 $O(N^2)$，双尺度采样保持近线性复杂度

2. **复杂度感知度分配（自适应连接密度）**:

    - **RMS-Gradient复杂度评分**：用Sobel算子计算每个像素的梯度幅值，跨通道RMS聚合
    $s(u,v) = \sqrt{\frac{1}{C}\sum_{c=1}^C |\nabla F_c(u,v)|^2}$
    - **从全局预算到逐节点配额**：目标平均度 $\bar{d}$ → 总边预算 $B = N\bar{d}$ → 按复杂度比例分配 $q_i = B \cdot \frac{s_i}{\sum s_k}$ → 目标度 $d_i^* = \text{clip}(\text{round}(q_i), 1, n_i)$
    - 设计动机：高梯度区域(边缘/纹理)需要更多邻居聚合信息来消除冗余；平滑区域少量邻居足够→避免过度连接

3. **图构建与聚合**:

    - 每个节点在候选集中按余弦相似度二分搜索阈值→选择 $d_i^*$ 个邻居→有向邻接矩阵
    - 边条件化消息传递更新特征（GFA-Local和GFA-Global顺序执行）
    - 设计动机：先密集采样确定"where"，再复杂度评分确定"how much"，最后相似度选择确定"which"——三步解耦的设计清晰高效

### 损失函数 / 训练策略
标准率失真目标：$\mathcal{L} = \mathcal{R}(\hat{y}) + \mathcal{R}(\hat{z}) + \lambda \mathcal{D}(x, \hat{x})$，其中 $\mathcal{R}$ 是比特率，$\mathcal{D}$ 是失真(MSE或MS-SSIM)，$\lambda$ 平衡率失真。

### 消融实验

| 配置 | BD-rate | 说明 |
|------|---------|------|
| 仅局部图 | -15% | 缺乏远程冗余建模 |
| 仅全局图 | -14% | 缺乏精细结构 |
| 双尺度图 | -18% | 联合显著提升 |
| 固定均匀度 | -17% | 不区分复杂度 |
| **完整GLIC** | **-21.69%** | 复杂度感知度分配的额外增益 |

### 关键发现
- 双尺度图 vs 单尺度图：联合使用比任何单一尺度都显著更好——局部和全局冗余需要同时建模
- 自适应度分配 vs 固定均匀度：额外带来3-4%的BD-rate改进——内容复杂度确实影响最优连接密度
- RMS-Gradient比简单平均梯度作为复杂度度量更有效——对大值更敏感，符合能量压缩现象
- GLIC在高码率和低码率下都保持优势——不依赖特定码率段

## 亮点与洞察
- **GNN在LIC中的范式创新**：将固定连接的CNN/Transformer彻底替换为数据驱动的动态图连接，这是LIC架构的根本性创新。图像像素作为图节点+非欧关系建模是压缩领域的新范式
- **"where+how much"的解耦设计**：双尺度采样(where)和复杂度评分(how much)分别处理连接范围和连接密度——问题分解优雅且各自可独立优化
- **复杂度感知的物理直觉**：高梯度≈低冗余≈更多邻居→更好压缩。这个设计具有强物理直觉——平滑区域信息量少不需要密集建模
- **无padding的效率优势**：序列化分组的GNN避免了窗口注意力中空体素的padding开销——对稀疏特征图尤其有利

## 局限性 / 可改进方向
- GNN的图构建（二分搜索阈值）引入额外计算开销——对实时编码场景可能是瓶颈
- 当前GFA block仅在变换网络的后两阶段使用——全阶段GNN是否可行需要探索
- 解码端的GNN需要与编码端一致的图结构——如何在不传输图结构的前提下重建是开放问题
- 与VTM-9.1的比较是学术标准，商业编解码器(VVC, AV1)的对比待补充

## 相关工作与启发
- **vs CNN-based LIC (Cheng et al., TIC)**: CNN固定感受野→对远程冗余建模不足。GLIC的全局稀疏图解决了这一问题
- **vs Transformer-based LIC (STF, T-CA)**: 窗口注意力仍受限于预设块大小。GLIC的全局采样不受窗口限制
- **vs 形变卷积LIC**: 形变卷积的偏移范围有限，GLIC的图连接无距离限制
- **vs 先前GNN压缩方法**: 先前方法受限于全局二次复杂度或局部感受野，GLIC通过双尺度+自适应度平衡了两者
