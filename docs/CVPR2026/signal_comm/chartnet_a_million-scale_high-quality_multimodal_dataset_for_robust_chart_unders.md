# ChartNet: A Million-Scale, High-Quality Multimodal Dataset for Robust Chart Understanding

**会议**: CVPR 2026  
**arXiv**: [2603.27064](https://arxiv.org/abs/2603.27064)  
**代码**: [HuggingFace](https://huggingface.co/datasets/ibm-granite/ChartNet) (有)  
**领域**: Signal/Communication (多模态文档理解)  
**关键词**: 图表理解, 大规模数据集, 代码引导生成, VLM, 多模态推理

## 一句话总结

发布 ChartNet——150 万规模的高质量多模态图表数据集，通过代码引导合成管线生成包含图像-代码-数据表-文本-推理QA 的对齐五元组，在图表理解和推理任务上显著提升 VLM 性能，小模型微调后超越 GPT-4o。

## 研究背景与动机

1. **领域现状**：图表是传达定量信息的基础媒介，理解图表需要视觉、数值和语言能力的整合。现有 VLM 在复杂图表推理上仍有不足，尤其是开源模型明显落后于闭源系统。
2. **现有痛点**：(a) 现有数据集规模小、类别窄、模态不完整——多数仅关注单一任务（QA或摘要），缺少绘图代码、grounding标注、推理链等关键模态；(b) 最常用的 ChartQA 仅含 14K 图表、3 种类型，且偏向简单数据提取，现代 VLM 已性能饱和。
3. **核心矛盾**：训练前沿多模态模型需要大规模、高质量、多模态对齐的监督数据，但图表领域的数据严重不足。
4. **本文要解决什么**：构建百万级规模的多模态图表数据集，统一覆盖图表理解的完整谱系。
5. **切入角度**：图表本质上是**可编程生成**的——绘图代码作为结构化中间表示,可驱动大规模自动合成和增强。
6. **核心 idea**：代码引导的图表生成管线——从种子图表重建代码，迭代增强生成多样化图表，并从代码推导数据表、描述和推理 QA。

## 方法详解

### 整体框架

五阶段管线：(1) 图表到代码重建（VLM 从种子图像生成绘图代码）；(2) 代码引导增强（LLM 迭代改写代码）；(3) 图表渲染（执行代码生成图像）；(4) 质量过滤（VLM 检测渲染缺陷）；(5) 属性生成（从代码+图像提取数据表、描述等）。

### 关键设计

1. **代码引导数据生成管线**：
   - 种子集：150K 张来自 TinyChart 的图表图像
   - 重建：使用 pixtral-large 将图像转为 Python 绘图代码
   - 增强：使用 gpt-oss-120b 迭代改写代码，变化图表类型/样式/数据，每张种子图可生成任意数量变体
   - 渲染：执行代码，77% 执行成功率
   - 过滤：VLM 检测视觉缺陷（文本重叠、标签裁切等），36.5% 被剔除
   
   设计动机：传统方法在图像空间增强，无法保证数据准确性和多样性。代码空间增强天然保证数据-图像一致性，且可产生丰富的类型/风格变化。

2. **推理 QA 生成（CoT Reasoning）**：基于 Vision-R1 框架，生成四步伪 CoT（Summary→Caption→Reasoning→Conclusion），通过模态桥接让纯语言模型 gpt-oss-120b 生成详细推理链，包裹在 `<think>` 和 `<answer>` 标签中。
   设计动机：复杂图表问题需要多步推理，显式推理轨迹提供了可验证的中间步骤监督。

3. **专项子集**：
   - **人工标注子集**：96,643 张经严格人工验证
   - **真实世界图表**：30K 张来自世界银行、Pew Research 等权威来源
   - **Grounding QA**：从绘图代码提取几何感知标注，生成带边界框的 QA
   - **Safety 数据**：7K 训练 + 600 测试的安全对齐偏好对
   
   设计动机：合成数据提供规模和多样性，但需要真实数据和人工标注提供质量锚点。

### 数据集规格

- 150 万多模态五元组（图像、代码、CSV、文本、推理 QA）
- 24 种图表类型、6 个绘图库
- 约 168 小时在 100+ A100/H100 GPU 上生成

## 实验关键数据

### 主实验 — 微调前后对比

| 模型 | 代码执行率 | 数据提取 | 摘要 | QA推理 |
|------|----------|---------|------|--------|
| Granite-2B | 63.4 → **90.4** | 53.8 → **70.3** | 64.0 → **83.9** | 59.9 → **65.0** |
| LLaVA-7B | 45.3 → **83.9** | 17.0 → **58.8** | 51.2 → **80.3** | 55.1 → **70.3** |
| SmolVLM-256M | N/A → **14.9** | 22.0 → **36.4** | 26.6 → **60.0** | 55.0 → **60.8** |

### 与大模型和GPT-4o对比

| 指标 | GPT-4o | Granite-2B+ChartNet | LLaVA-7B+ChartNet |
|------|--------|---------------------|-------------------|
| 代码执行率 | 95.9 | 90.4 | 83.9 |
| 数据提取 | 46.7 | **70.3** | **58.8** |
| 摘要 | 77.1 | **83.9** | **80.3** |
| QA推理 | 61.1 | 65.0 | **70.3** |

### 关键发现

1. **小模型超越大模型**：2B 模型微调后在数据提取（70.3 vs 46.7）和摘要（83.9 vs 77.1）上超越 GPT-4o，7B 微调后在 QA 推理（70.3 vs 61.1）上超越所有开源和闭源模型。
2. **无到有的能力赋予**：256M 超紧凑模型原本无代码生成能力，微调后获得 14.9% 执行率。
3. **规模无关的增益**：从 256M 到 7B 所有尺度模型都一致获得显著提升。
4. 泛化验证：在真实世界基准 ChartCap（摘要）和 ChartMimic-v2（代码生成）上同样获得大幅提升。

## 亮点与洞察

- 代码引导的合成管线是核心方法论创新——图表天然可编程这一洞察巧妙地将数据生成问题转化为代码增强问题。
- 数据集的完整性令人印象深刻：图像-代码-CSV-文本-推理QA 五元组 + grounding + safety。
- "图表领域中，扩大模型规模远不如提供高质量代码对齐多模态监督有效"——这一结论对图表理解社区有重要指导意义。
- 质量过滤管线（人工+VLM 联合）确保了高质量下界。

## 局限性 / 可改进方向

- 合成数据可能仍存在分布偏差，与真实世界图表的样式差异需进一步弥合。
- 150K 种子图来自 TinyChart 单一来源，种子多样性可进一步提升。
- 当前仅评估 SFT，未探索 RLHF 或 DPO（虽然已准备了 safety 偏好数据）。
- 3D 图表、仪表盘等复杂可视化类型覆盖有限。

## 相关工作与启发

- 与 UniChart (611K) 和 MMC (600K) 相比，ChartNet 在规模(1.5M)、类型多样性(24种)和模态完整性上全面领先。
- 代码引导生成思路可推广至其他可编程视觉内容（如 LaTeX 公式、流程图、科学图示）。
- 为多模态文档理解社区提供了重要基础设施。
