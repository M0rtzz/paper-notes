# SAP: Segment Any 4K Panorama

**会议**: CVPR 2026  
**arXiv**: [2603.12759](https://arxiv.org/abs/2603.12759)  
**代码**: [项目页](https://lutao2021.github.io/SAP_Page/)  
**领域**: 图像分割 / 全景图像  
**关键词**: 全景分割, SAM2, 等距柱状投影, 拓扑-记忆对齐, 4K分辨率  

## 一句话总结

将全景分割转化为拓扑-记忆对齐问题，通过固定轨迹透视扫描将ERP全景图变为伪视频序列，完美复用SAM2流式记忆，零样本4K全景分割提升+17.2 mIoU。

## 研究背景与动机

**4K全景图像在VR/AR/机器人中广泛使用**，但ERP格式的严重几何畸变（极区拉伸、左右边界不连续）使标准分割模型性能大幅下降。SAM系列只支持 $1024^2$ 输入，4K全景需压缩到 $1024\times512$ 再填充，丢失细节且浪费计算。

**OmniSAM等方法尝试利用SAM2的视频分割能力**，但其滑动窗口直接在ERP上操作——违反了SAM2记忆机制的核心假设：相邻帧应对应平滑相机运动和重叠视觉内容。ERP上的滑动窗口打破了物理视角连续性，导致记忆传播不稳定。

**核心洞察**：畸变和接缝伪影是表面症状，深层问题是球面全景违反了流式记忆模型的拓扑假设。解决方案不是修改模型，而是将输入重新序列化以匹配模型的时序先验——"改变输入而非改变模型"。

## 方法详解

### 整体框架

ERP全景图 + prompt点 → 固定轨迹透视投影生成重叠透视伪视频 → SAM2流式处理利用记忆传播保持一致性 → 反投影融合得到ERP分割结果。

### 关键设计

1. **列优先锯齿扫描轨迹**:

    - 功能：将ERP全景图序列化为物理上连续的透视视图序列
    - 核心思路：沿经度从左到右扫描，每列内沿纬度上下交替（锯齿形），相邻帧球面空间连续。$N_{yaw} \times N_{pitch}$ 个视角，50%帧间重叠确保完整覆盖
    - 设计动机：列优先保持水平连续性，锯齿避免纬度跳跃，与SAM2的"相邻帧内容变化小"假设完美匹配

2. **ERP-to-Perspective投影与Prompt映射**:

    - 功能：将ERP像素无畸变转换到标准透视视图并映射用户prompt
    - 核心思路：三阶段几何投影：虚拟相机内外参定义 → 透视帧2D网格建立 → 球面坐标采样。Prompt点通过相同变换矩阵投影到可见帧的2D坐标
    - 设计动机：消除ERP畸变，使每个视图与SAM2训练分布一致

3. **大规模合成全景训练数据**:

    - 功能：弥补全景实例分割标注数据的极度稀缺
    - 核心思路：InfiniGen引擎合成183K张4K全景图及实例标签，与SAM2原始数据混合微调。仅微调记忆注意力层，大部分参数冻结以防灾难性遗忘
    - 设计动机：零样本泛化依赖充分的训练数据覆盖；混合训练保持通用分割能力

### 损失函数 / 训练策略

标准SAM2训练损失（focal loss + dice loss）。仅微调记忆注意力层，其余参数冻结。混合训练：合成全景+SAM2原始数据。

## 实验关键数据

### 主实验

| 模型大小 | SAM2原始 | SAP（本文） | 提升 |
|---------|----------|-----------|------|
| Tiny | ~35 mIoU | ~59 mIoU | +24.2 |
| Small | ~38 mIoU | ~57 mIoU | +19 |
| Base | ~41 mIoU | ~58 mIoU | +17 |
| Large | ~44 mIoU | ~52 mIoU | +8 |
| **平均** | - | - | **+17.2** |

### 消融实验

| 配置 | 关键指标 | 说明 |
|------|---------|------|
| 无重叠 vs 50%重叠 | +10.8 mIoU | 重叠是保证边界一致性的关键 |
| 直接ERP切割 vs 透视投影 | +4.7 mIoU | 透视投影消除畸变，匹配SAM2训练分布 |
| 单数据源 vs 混合训练 | 大幅领先 | 合成+SAM2数据混合远优于任一单独 |
| 行优先 vs 列优先扫描 | 列优先更优 | 水平空间连续性对记忆传播更关键 |

### 关键发现

- 接缝/极区分割改善最显著：mIoU 46.1→68.2（+22.1），最难区域获益最大
- InfiniGen合成数据贡献 +19.1 mIoU，是性能提升核心推动力
- 零样本泛化——不需目标域标注即可在真实4K全景上工作

## 亮点与洞察

- 最小改动获得最大收益：不修改SAM2架构，仅改变输入处理方式就获巨大提升。将2D空间问题抽象为拓扑-记忆对齐的视角极具启发性，是"改变输入适配模型"范式的典范。

## 局限性 / 可改进方向

- 列扫描序列长度随分辨率线性增长，推理延迟可能较高
- 极区仍是最难区域——ERP到透视投影在极区信息损失大
- 未扩展到360°视频
- 仅在SAM2上验证，未测试其他foundation model

## 相关工作与启发

- **vs OmniSAM**: 直接在ERP上滑动窗口，接受畸变；SAP通过透视投影消除畸变并恢复SAM2记忆假设
- **vs PanoSAM**: 需在ERP上重训练SAM且不处理拓扑连续性
