# Multimodal RewardBench 2: Evaluating Omni Reward Models for Interleaved Text and Image

**会议**: CVPR 2026  
**论文**: [CVF Open Access](https://openaccess.thecvf.com/content/CVPR2026/html/Hu_Multimodal_RewardBench_2_Evaluating_Omni_Reward_Models_for_Interleaved_Text_CVPR_2026_paper.html)  
**代码**: 待确认  
**领域**: 多模态VLM  
**关键词**: 奖励模型, omni 模型, 偏好评测基准, 图文交错生成, MLLM-as-a-judge

## 一句话总结
MMRB2 是首个面向「omni 模型」（能在同一序列里任意交错读写文本与图像）的奖励模型评测基准，横跨文生图、图像编辑、交错生成、多模态推理四大任务，每任务 1000 对专家标注偏好对，揭示出当前最强裁判（Gemini 3 Pro 仅 76% 平均一致率）与人类（>90%）之间仍有显著差距。

## 研究背景与动机

**领域现状**：奖励模型（Reward Model, RM）是 LLM 后训练的核心组件——它把人类偏好量化成标量分数，既能在 RLHF/RL 阶段引导策略，也能做数据质量筛选和 Best-of-N 采样选择。近期一批「omni 模型」（如 Chameleon、GPT-Image、Gemini 2.5 Flash Image）能在一段任意顺序的序列里同时理解和生成图文，把多模态生成推到了新高度。

**现有痛点**：但针对 omni 模型的奖励建模几乎是空白，根本原因是**没有评测基准**——没有基准就无法衡量一个 RM 的好坏。已有的多模态偏好基准（VL-RewardBench、Multimodal RewardBench 1）只覆盖「图文→文本」（image-text-to-text）这类纯理解任务，无法评价交错图文输出；而文生图侧的自动指标（CLIPScore、VQAScore）和偏好训练 RM（ImageReward、HPSv2）大多在老一代模型（SD 2.1 时代）数据上训练，面对前沿模型输出已经失准。

**核心矛盾**：omni 模型的通用性恰恰给奖励建模带来独有难题——① 图像难以像数学/代码那样自动验证；② 一个序列里可以有任意数量、任意顺序的图文，评判维度爆炸；③ 高质量偏好数据需要精心设计的标注协议；④ omni 模型本身能力还不够强，很难像构造纯文本 RM 基准那样轻松采到高质量回答。

**本文目标**：建一个**全面、可靠、且对下游高度预测**的 omni 奖励模型评测基准，覆盖 omni 模型的主要应用面，并系统刻画当前裁判（无论是 MLLM-as-a-judge 还是偏好训练 RM）到底差在哪。

**切入角度**：把 omni 模型的能力边界拆成四个有代表性的子任务，用「集成过滤先剔除简单对、再上专家标注」的方式把人力集中在最有信息量的难样本上，并用 >90% 专家共识保证标签可靠。

**核心 idea**：用「四任务 × 1000 专家偏好对 + 双向一致评测协议」搭出首个 omni RM 基准 MMRB2，让 omni 奖励建模第一次可被测量、可被对比、可预测下游。

## 方法详解

### 整体框架
MMRB2（Multimodal RewardBench 2）本质是一个**数据基准 + 评测协议**，不训练新模型。它由四个子任务组成（文生图、图像编辑、交错生成、多模态推理），每个子任务 1000 个数据点，每个数据点是一个三元组「任务 prompt + 被选回答（chosen）+ 被拒回答（rejected）」。被评测的 RM 看到 prompt 和两个回答后输出偏好，基准用「RM 偏好与人类偏好是否一致」来打分。

数据管线分四步：① **回答生成**——从 21 个公开/新建任务里按难度分层采样 prompt，再用 7–11 个 SOTA 模型（含 API 与开源，交错/推理任务还额外接入能调 Python 与图像工具的 agent）生成多个候选回答；② **集成过滤**——用 9 个 MLLM 裁判投票，剔除「几乎一边倒」的简单对；③ **人类标注**——专业标注员按任务定制的评判框架打分，再做高分歧/模糊/平局过滤；④ **评测**——用「位置一致双向评测」统计 RM 与人类的一致率。这是纯 benchmark/数据集论文，流程线性清晰但不含可训练模块，故不画框架图，下面按关键设计逐一拆解。

### 关键设计

**1. 四子任务覆盖 omni 模型的能力全谱**

针对「已有基准只测图文→文本，无法评交错图文输出」这一空白，MMRB2 把 omni 模型的应用面切成四个互补子任务：**文生图**（看 prompt 与候选图，判物体组合、空间关系、属性绑定、文字渲染）、**图像编辑**（给 1–3 张输入图 + 编辑指令，判编辑忠实度与无关区域保持的平衡，含单图与多图合成）、**交错生成**（多模态 prompt 触发图文混排输出，如讲故事/how-to/教育内容，判全局规划与图文协调）、**多模态推理**（"thinking with images"，模型可产出中间草图，判推理轨迹与最终答案）。每个子任务 1000 对、合计 4000 对，prompt 来自 21 个来源（含 WISE、Emu-Edit、ISG-Bench、BLINK 等），从而保证基准既宽又贴近真实难度边界。

**2. 集成过滤 + 专家偏好标注：把人力压在难样本上**

针对「omni 偏好数据贵、且大量简单对没有区分度」的痛点，MMRB2 先用一套**集成过滤（ensemble filtering）**把简单对剔掉：收集 9 个跨能力档的多模态裁判（GPT-5/4.1/4o、Gemini 2.5 Flash/Pro、Gemma-3 27B/12B/4B、Qwen2.5-VL-7B），每个裁判对每对都正序（A vs B）和反序（B vs A）各评一次以抵消位置偏置；定义「多数标签在全部裁判评测中占比 ≥ $90\%$」的为简单对并丢弃——因为它们对区分细粒度 RM 几乎无信号。剩下的难样本交给 Surge AI 平台的专业标注员，每对由 3 名不知情标注员在 7 点 Likert 量表上独立打分。再做三重质控：剔除评分跨度 >4 分的高分歧对、剔除均分落在 3.0–4.0 中点附近的模糊对、剔除多数票平局对。三个生成类任务从约 17,700 次判断里筛出每任务 1000 对（约保留初始集 50%），过滤后**专家一致率高达 95.6%**（剔除平局后）。

**3. 多模态推理任务的「逐响应标注→构造偏好对」**

针对「推理任务有标准答案、但推理过程对错需单独判」的特点，推理任务不直接做成对标注，而是**先逐个回答标注、再构造偏好对**。先筛出既含正确答案又含某种推理过程的回答，并平衡「纯文本推理」与「图文混合推理」两类；每个回答收集 3 份人类标注，判断其推理是否有重大错误。构造偏好对时：preferred 取「3 名标注员一致认为推理无重大错误且答案正确」的回答；non-preferred 取两类——「答案对但推理被一致判为有重大错误」与「答案错但带推理」。同一回答不在多对间重复使用。这样把「答案对错」与「推理质量」解耦，逼裁判同时看两者。

**4. 位置一致双向评测协议（positional consistent dual evaluation）**

针对「裁判普遍偏好序列里第一个选项」的位置偏置，MMRB2 规定每对都被每个裁判评两次：原序（A vs B）和交换序（B vs A），两次判断都作为独立数据点保留。这同时起两个作用：把裁判-人类比较的样本量翻倍、提升统计稳定性；并**惩罚位置偏置严重的裁判**（它在正反序上会自相矛盾，从而被一致率拉低）。一致率的计算是二元的——裁判偏好与人类偏好（任务 1–3 取 3 标注员多数票，任务 4 取推理与答案正确性的一致判定）匹配记 1.0、否则 0.0。

### 损失函数 / 训练策略
本文是评测基准，不涉及模型训练，无损失函数。被评测对象分两类：MLLM-as-a-judge（直接 prompt 大模型当裁判）与偏好训练 RM（ImageReward、HPSv2/v3、PickScore、EditReward、UnifiedReward、VQAScore 等）。

## 实验关键数据

### 主实验
MLLM-as-a-judge 在 MMRB2 四任务上的一致率（accuracy，%）。最强的 Gemini 3 Pro 平均仅 76.3%，距人类 >90% 仍有差距；GPT-4o 在推理任务掉到 51.9%，作者据此指出它已不适合再当前沿多模态评测器。

### 消融实验
基准本身无可消融模块，下表给出几项关键分析结论（按论文 §4.3–4.5 整理）：

| 分析维度 | 关键数值 | 说明 |
|----------|---------|------|
| 下游相关性 | Pearson $r$ > 0.8（全任务全模型） | MMRB2 分数与 Best-of-N 下游表现强相关，可当下游代理 |
| 同模型 vs 跨模型对 | 最优裁判差 5–13 个点 | 跨模型对一致率更高（如 Gemini 3 Pro 文生图 79.7% vs 70.4%，差 9.3） |
| 混合模态偏置 | GPT-5 差 49.3 个点 | 推理混合模态对里，裁判强烈偏向「含图」回答；Gemini 3 Pro 仍有 17.9 点偏置 |
| 测试时扩展 | K=9 仅 +0.8–1.2% | 多数票扩展对多模态 RM 收益甚微，Qwen3-VL 无提升 |

### 关键发现
- **多模态推理是四任务里最难的一类**：除 Gemini 3 Pro 外，顶级 API 模型在推理上只有 52–70% 一致率（生成类有 63–75%），可能源于解题路径多样、解释详略人各有偏好、且要同时判答案正确性与推理质量。
- **裁判存在「含图即好」的强偏置**：在混合模态推理对里，当 preferred 回答含图时 GPT-5 准确率 88.2%、含文时仅 38.9%，相差 49.3 个点——说明裁判把「是否带图」误当成了质量信号。
- **偏好训练有效但易过时**：EditReward 在编辑上 +9.4%、UnifiedReward 在文生图上 +9.4%，但多数 RM 因训练数据来自老一代系统而对前沿模型输出失准（分布漂移），强 MLLM 仅靠简单 prompt 就能立起高 baseline。

## 亮点与洞察
- **把「位置偏置」从干扰项变成评测维度**：双向评测不仅抵消偏置，还让偏置严重的裁判自然被一致率惩罚——一举两得的协议设计，可复用到任何成对偏好评测。
- **集成过滤的「先剔简单对」思路**：用 9 个跨能力档裁判 + 90% 阈值把人力集中在难样本上，既省标注成本又保证基准对 RM 有区分度，是构造高信息量偏好数据集的可迁移范式。
- **最"啊哈"的是「含图即好」偏置的量化**：连最强裁判都有近 18 个点偏置，说明当前多模态 RM 还没学会评判「该不该出图」，这是 omni 后训练亟需修的失败模式。
- **强相关下游（$r>0.8$）让基准有实用价值**：MMRB2 分数能预测 Best-of-N 下游收益（如 GPT-5 当选择器把 FLUX 的 GenAI-Bench 从 73% 提到 79%），不是纯学术指标。

## 局限与展望
- **交错生成无专用评测器**：作者承认目前没有专门针对交错图文输出训练的 RM，该子任务只能用 MLLM-as-a-judge 评，缺少偏好训练 RM 的对照。
- **集成过滤可能引入偏置**：虽用多样裁判 + 高阈值缓解，但「被 9 个裁判一致认为简单」的对被剔除，理论上仍可能系统性偏向某类样本（作者放在附录做了消融验证）。
- **测试时扩展收益有限**：多数票扩展对多模态 RM 几乎无效，提示需要为多模态奖励设计不同于纯文本 LLM 的扩展机制——这是开放问题。
- **规模相对小**：每任务 1000 对、合计 4000 对，对覆盖 omni 模型「任意长度任意顺序」的全部组合仍偏小，长序列交错场景可能欠采样。

## 相关工作与启发
- **vs Multimodal RewardBench 1 (MMRB1)**: MMRB1 只覆盖「图文→文本」的 MLLM 理解任务；MMRB2 把场景升到 omni 模型的四类生成/推理任务，难度与覆盖面都显著扩大，是直接的承接与升级。
- **vs RewardBench / RewardBench 2（纯文本）**: 那两者系统比较 LLM 奖励函数（指令遵循、推理、安全等轴），MMRB2 把同样的「系统性评测裁判」思路迁到多模态，并补上图像难自动验证、需专家标注的独有挑战。
- **vs ImageReward / HPSv2 / PickScore（文生图 RM）**: 这些是被评测对象而非竞品，MMRB2 实验证明它们在前沿模型输出上已失准（如 ImageReward 文生图仅 54.0%），凸显出建新基准的必要性。
- **vs VQAScore / CLIPScore（自动指标）**: 这类参考无关指标常漏判组合性错误，MMRB2 显示它们大幅落后于 MLLM 裁判，进一步说明 omni 评测不能只靠启发式指标。

## 评分
- 新颖性: ⭐⭐⭐⭐⭐ 首个面向 omni 模型（交错图文读写）的奖励模型评测基准，填补了明确的空白。
- 实验充分度: ⭐⭐⭐⭐⭐ 覆盖 15+ 裁判 × 4 任务，外加下游相关性、同/跨模型、混合模态偏置、测试时扩展四类深入分析。
- 写作质量: ⭐⭐⭐⭐ 结构清晰、协议交代细致，但部分定量细节散落附录，正文需对照表格才能看全。
- 价值: ⭐⭐⭐⭐⭐ 与下游 Best-of-N 强相关（$r>0.8$），可直接当 omni 后训练的奖励代理与裁判选型依据。
