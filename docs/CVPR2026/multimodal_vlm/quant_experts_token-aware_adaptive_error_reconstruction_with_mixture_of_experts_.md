# Quant Experts: Token-aware Adaptive Error Reconstruction with Mixture of Experts for Large Vision-Language Models Quantization

**会议**: CVPR2026  
**arXiv**: [2602.24059](https://arxiv.org/abs/2602.24059)  
**代码**: 无  
**领域**: multimodal_vlm  
**关键词**: Post-Training Quantization, Vision-Language Models, Mixture of Experts, Low-Rank Adaptation, Token-aware Quantization

## 一句话总结

提出 Quant Experts (QE)，通过将重要通道分为 token 无关和 token 依赖两组，分别用共享专家和路由专家（均为低秩适配器）进行量化误差补偿，实现了对 VLM 的 token 感知自适应量化，在 W4A6 设置下 72B 模型平均精度提升 5.09%。

## 研究背景与动机

后训练量化（PTQ）是部署大规模视觉语言模型（VLM）的关键技术，通过将权重和激活映射到低位宽表示来实现压缩加速。然而现有 PTQ 方法存在一个根本性缺陷：**它们依赖静态的通道重要性估计和全局误差补偿策略**，忽视了重要通道在不同输入下的分布差异。

作者通过系统分析发现了两个关键观察：

**观察一：重要通道的位置随 token 动态变化。** 在同一层权重下，重要通道的位置不仅在不同模态（图像 vs 文本）之间差异显著，甚至在同一模态的不同 token 之间也会发生明显迁移。这是因为 token 语义和上下文信息的差异导致了激活分布的变化。这直接说明了 SmoothQuant、AWQ 等全局校准方法的局限性——用固定的离群值处理方式无法捕获 token 级别的通道重要性变化。

**观察二：重要通道的频率分布极不均匀。** 统计重要通道在所有 token 中的出现频率后发现，只有一小部分重要通道在大多数 token 中持续出现（token 无关型），而大部分重要通道仅在特定 token 中被激活（token 依赖型）。值得注意的是，那些低频出现的 token 依赖通道可能具有更大的离群值幅度，其对量化误差补偿的贡献同样不可忽视。

这两个观察揭示了一个核心洞察：**VLM 中的重要通道同时包含全局和局部两类成分**。仅做全局补偿会损害 token 级语义保真度，仅做局部补偿则削弱整体表征能力。因此需要一种差异化的补偿策略。

## 方法详解

### 整体框架

QE（Quant Experts）是一个基于混合专家（MoE）的 token 感知自适应量化误差重建框架，核心思路是将量化误差分解为全局和局部两部分，分别用不同类型的专家进行补偿：

1. **通道依赖性划分**：从校准数据中统计每个通道作为重要通道的出现频率，按频率降序排列后划分为 token 无关通道（top-k 高频）和 token 依赖通道（后续 Nr*k 个）
2. **共享专家（SE）**：用一个低秩适配器重建 token 无关通道引起的全局量化误差
3. **路由专家（REs）**：将 token 依赖通道通过共现分析聚类为多个子组，每个子组分配一个路由低秩适配器，推理时由轻量路由器动态选择最优专家

量化误差优化目标：最小化全精度与量化模型输出差异，误差近似项分解为 token 无关部分和输入依赖部分之和。

### 关键设计

#### 1. 通道依赖性划分

对校准数据中每个 token 计算其重要通道集合（按激活值与权重行均值的乘积取 top-k），聚合所有 token 的重要通道并统计频率，按降序排列后：
- 前 k 个 → token 无关通道
- 后 Nr*k 个 → token 依赖通道

#### 2. 共享专家（SE）用于 token 无关通道

对 token 无关通道免除直接量化，改用 whitening SVD 的低秩适配器进行重建。同时采用通道缩放降低激活幅度、等比放大权重，缓解 activation 量化误差。共享专家处理后的残差误差交由路由专家进一步精化。

#### 3. 路由专家（REs）用于 token 依赖通道

这是本文最核心的设计，包含以下步骤：

- **共现分析**：构建共现矩阵，记录每个 token 依赖通道在各 token 中是否被选为重要通道
- **谱聚类**：利用归一化逐点互信息（NPMI）将共现矩阵转换为相似度矩阵，通过 Laplacian 特征分解 + K-Means 将 token 依赖通道聚为 Nr 簇
- **加权 SVD**：对每个簇，根据簇内通道的激活统计构建权重向量，对残差误差做加权 SVD 得到低秩适配器
- **路由器构建**：利用每个路由专家的残余误差的绝对均值作为路由参数 R，推理时选择预测误差最小的专家

#### 4. 可选精化（Refinement）

冻结所有参数，仅微调路由专家和路由器。逐层优化（非端到端），损失函数包含：
- 回归损失：选误差最小的专家优化其重建精度
- 分类损失：用 KL 散度对齐路由器预测分布和实际重建误差分布
- 总损失加权系数：alpha=1.0, beta=0.05, tau=0.5

### 复杂度分析

| 指标 | 原始线性层 | QE |
|------|-----------|-----|
| 计算 (FLOPs) | sd^2 | sd^2 + sd(2r + Nr) |
| 参数量 | d^2 | d^2 + rd(1 + Nr) |

由于 r << d 且 Nr << d，额外开销极小。实验中 r=64, Nr=8。

### 消融实验

**各组件贡献（Qwen2VL-2B, W4A6）：**

| 配置 | MMMU | ScienceQA |
|------|------|-----------|
| FP16 | 39.89 | 76.95 |
| 仅 REs | 34.56 | 68.72 |
| 仅 SE | 35.22 | 69.61 |
| SE + 随机路由 | 35.89 | 70.00 |
| SE + 随机聚类 | 35.33 | 69.71 |
| **QE (SE+REs)** | **36.89** | **70.85** |

**关键发现：**

1. **SE 和 REs 缺一不可**：移除任一组件都会导致性能下降，验证了全局+局部双层补偿的必要性
2. **路由策略有效**：随机路由比有策略路由差 1.0/0.85 个点，说明基于残余误差的路由器确实能选到最优专家
3. **共现聚类有效**：随机聚类比 NPMI 谱聚类差 1.56/1.14 个点，验证了共现分析对通道分组的重要性
4. **路由专家数量**：Nr 从 2 增到 16 时 Avg. 从 67.08 升至 68.06，但边际收益递减且内存开销增大，Nr=8 是较好权衡
5. **精化有效**：W4A6 下精化使 OCRBench 提升 1.4 点（7B），TextVQA 提升 5.95 点
6. **硬件加速**：在 NPU 分析模型上，QE 在 prefill 阶段可实现 3.5x-4.5x 加速

## 亮点与洞察

1. **问题发现精准**：两个观察（重要通道位置随 token 变化 + 频率分布不均匀）为方法设计提供了坚实的实证基础，这比直接提出方法更有说服力
2. **MoE 机制巧妙映射**：将 token 无关/依赖通道自然映射到共享/路由专家，概念清晰、设计优雅
3. **NPMI 谱聚类**：用归一化逐点互信息捕获通道共现关系，再用谱聚类分组，比简单的 K-Means 更能捕获非线性相关性
4. **路由器设计无需学习**：用残余误差绝对均值直接构建路由参数，推理时仅需一次矩阵乘法即可选出最优专家，极其轻量
5. **规模泛化性强**：从 2B 到 72B 模型一致有效，且在大模型上优势更明显（72B W4A6 提升 5.09%）
6. **额外开销极小**：由于 r << d，低秩适配器几乎不增加计算和内存成本

## 局限性 / 可改进方向

1. **校准数据依赖**：通道频率统计依赖 128 个校准样本的代表性，在分布外场景可能失效
2. **超参数固定**：k=32, Nr=8, r=64 对所有层使用相同值，层自适应的超参数选择可能进一步提升性能
3. **仅量化 LLM 部分**：实验主要量化语言模型的线性层，Visual Encoder 的联合量化虽在补充实验中提及但未深入
4. **精化阶段仍需训练**：虽然是可选的，但精化带来的提升不可忽视（特别是大模型），这部分仍有计算成本
5. **缺少延迟实测**：硬件加速仅通过分析模型报告，缺少在 GPU 上的实际推理延迟和吞吐量对比
6. **路由专家数量权衡**：Nr 增大带来的提升边际递减，寻找更好的通道-专家映射关系值得探索

## 相关工作与启发

- **SmoothQuant / AWQ**：通道缩放的代表方法，全局固定缩放系数，无法适应 token 级别变化，是 QE 的动机来源
- **LQER / ASER**：用 SVD 低秩适配器重建量化误差的先驱，但仅做全局统一补偿，QE 的共享专家部分延续了这一思路
- **MBQ (CVPR 2025)**：发现模态间通道敏感度差异，引入模态感知缩放，QE 将粒度从模态级进一步细化到 token 级
- **MoE**：混合专家机制在此处用于量化误差补偿而非模型容量扩展，是一个新颖的迁移应用
- **谱聚类 + NPMI**：借鉴 NLP 中的共现分析方法来分析通道关系，跨领域方法迁移
- **启发**：token 级别的自适应处理思想可推广到其他模型压缩任务（如剪枝、蒸馏），分析通道动态特性再设计差异化策略是一个有价值的范式
