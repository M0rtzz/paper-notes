# Physical Simulator In-the-Loop Video Generation

**会议**: CVPR 2026  
**arXiv**: [2603.06408](https://arxiv.org/abs/2603.06408)  
**代码**: https://vcai.mpi-inf.mpg.de/projects/PSIVG  
**领域**: 图像生成 / 物理一致视频  
**关键词**: 物理模拟器, 视频生成, 扩散模型, 测试时优化, 纹理一致性

## 一句话总结
提出 PSIVG，首个将物理模拟器嵌入视频扩散模型推理循环的无训练框架：从模板视频重建 4D 场景和 3D 网格并初始化物理模拟器，生成物理一致轨迹引导视频生成，并通过 Test-Time Texture Consistency Optimization（TTCO）优化前景纹理一致性。用户研究中 82.3% 偏好率远超所有基线。

## 研究背景与动机

1. **领域现状**：扩散模型生成的视频视觉质量惊人，但严重违反基本物理定律（重力、惯性、碰撞）。物体常跨帧不一致、凭空消失或物理上不可能地运动。
2. **现有痛点**：(a) 视频生成模型以去噪/重建为目标训练，无机制强制物理约束；(b) 有些工作用 LLM 推理物理（文本层面），与实际模拟无法相比；(c) PISA 通过微调学习物理但需额外训练数据；(d) PhysGen3D/PhysAnimator 需要用户输入动作/力，不支持开放词汇文本生成。
3. **核心矛盾**：视觉保真度和物理合理性之间存在根本性差距——像素级去噪训练目标无法理解物理定律。
4. **本文要解决什么**：如何在预训练扩散模型中注入物理一致性？
5. **切入角度**：物理模拟器天然产出物理正确的轨迹但缺乏视觉真实感，扩散模型有视觉质量但缺物理——两者互补。
6. **核心idea一句话**：从模板视频感知 → 物理模拟 → 渲染引导 → TTCO 纹理优化的完整回路。

## 方法详解

### 整体框架
输入文本提示 → 生成模板视频 → 感知管线提取 3D 网格/场景/相机 → 初始化物理模拟器 → 模拟产出物理一致轨迹 → 渲染 RGB/mask/像素对应 → 引导视频生成 → TTCO 优化纹理一致性。

### 关键设计

1. **感知管线（Perception Pipeline）**:
   - **前景物体几何**：用 GroundingDINO+SAM 检测分割动态物体，从第一帧裁剪用 InstantMesh 做单图 3D 重建。比多帧重建更可靠因为模板视频本身几何不一致
   - **背景场景几何**：ViPE 做 4D 重建（mask 前景后 bundle adjustment），聚合所有帧的度量深度点云得到 3D 背景
   - **前景动力学**：选两个关键帧，线性速度 = 3D 位移 / 时间间隔；旋转速度通过 SuperGlue 2D 特征匹配后计算相对质心的 2D 流场得到

2. **物理模拟**:
   - 做什么：基于 MPM（Material Point Method）模拟产出物理准确轨迹
   - **模拟域确定**：包围前景运动范围和背景几何，定义 $[0,2]^3$ 归一化域并计算合适的分辨率和度量-模拟尺度
   - **物理属性估计**：用 GPT-5 做分层推理——先推断材质组成/弹性/粗糙度等定性属性，再映射到密度/杨氏模量等数值参数。比直接估数值更可靠
   - 渲染输出：Mitsuba 渲染 RGB 帧 + 分割 mask + 逐帧像素对应关系

3. **物理一致视频生成**:
   - 做什么：用模拟器输出引导扩散模型生成
   - 核心思路：计算混合光流——前景取模拟器渲染 RGB 的 RAFT 光流（物理一致运动），背景取模板视频光流（保留相机动态/自然运动）。用 GwtF（Go-with-the-Flow）做光流条件生成
   - 设计动机：模拟器渲染虽物理正确但视觉不真实（风格化/低分辨率/无光照），不能直接替代
   
4. **Test-Time Texture Consistency Optimization (TTCO)**:
   - 做什么：解决光流引导下前景物体纹理闪烁/颜色漂移问题
   - 核心思路：用模拟器的像素对应将第一帧 warp 到后续帧作为纹理一致目标：$\mathcal{L}_{tex}(t) = \sum_{j=1}^{J}\|[De(h_0(\hat{L}_\tau))]_{q_{t,j}} - [W_t(\hat{I}_1)]_{q_{t,j}}\|_2^2$。总损失 $\mathcal{L}_{TTCO} = \sum_{t=2}^{T}\mathcal{L}_{tex}(t)$
   - **前景定向优化**：添加可学习零初始化 embedding 到物体对应的文本 token + DiT 层特征调制。不使用 LoRA 因会影响背景
   - 训练设置：AdamW, lr=2e-4, 50 迭代，聚焦扩散步 700-1000（高噪声步）
   - 设计动机：直接光流引导不够精确（尤其旋转时），需要额外像素级约束。文本 token 优化可定向影响物体外观而不破坏背景

### 损失函数 / 训练策略
- 完全无需训练数据，测试时优化
- 基座模型冻结，仅优化极少量可学习参数

### 关键发现
- 运动可控性大幅领先：SAM mIoU 0.84 vs PISA-Seg 0.50，Corr MSE 0.007 vs 0.012
- 用户偏好 82.3%，远超所有基线（最高 HunyuanVideo 仅 4.5%）
- PISA-Seg 等方法虽然时间稳定但物体几乎不动——高一致性来自"静态"而非物理正确
- TTCO 的前景定向优化关键：LoRA 优化会退化背景，token 优化精准定向
- GPT-5 的分层物理推理比直接估数值更可靠

## 亮点与洞察
- **模拟器-生成器互补**的洞察深刻：模拟器提供物理正确性，扩散模型提供视觉质量，两者各取所长
- **TTCO 的前景定向优化**：通过优化文本 token embedding 实现对物体外观的精准控制而不影响背景，验证了文本 token 与对应物体的强关联
- **分层物理属性推理**：不直接估数值而先推定性属性再映射，是 VLM 辅助物理理解的实用范式

## 局限性 / 可改进方向
- 感知管线各环节误差会累积（3D 重建/速度估计/场景拼接）
- 仅支持刚体和简单弹性体，不处理流体/布料/烟雾等复杂物理
- 每个视频需要完整的感知+模拟+生成+优化流程，计算开销大
- 物理属性的 GPT-5 估计有不确定性

## 相关工作与启发
- **vs PISA**: PISA 通过微调学习物理，但效果有限（物体静止化）。PSIVG 用真实模拟器更直接
- **vs PhysGen3D/PhysAnimator**: 需要用户提供动作/力，PSIVG 从文本自动生成
- **vs LLM 物理推理**: 仅文本层面的物理推理无法保证精确的时空一致性
