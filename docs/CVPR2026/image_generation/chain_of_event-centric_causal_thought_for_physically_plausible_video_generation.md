# Chain of Event-Centric Causal Thought for Physically Plausible Video Generation

**会议**: CVPR 2026
**arXiv**: [2603.09094](https://arxiv.org/abs/2603.09094)
**代码**: 即将公开
**领域**: 视频生成 / 物理合理性
**关键词**: 物理合理视频生成, 事件链推理, 链式思维, 物理公式约束, 双模态提示

## 一句话总结

将物理现象建模为因果连接的事件序列，通过物理公式驱动的事件链推理分解复杂物理过程，再用渐进式语义-视觉双提示引导现成视频扩散模型生成物理合理的因果演进视频。

## 研究背景与动机

物理合理视频生成（PPVG）要求生成的视频遵循物理定律。现有视频扩散模型（如 CogVideoX、Sora）在生成逼真场景方面很强，但对物理常识的理解不足——不能正确展现物体下沉、光线折射、冰块融化等物理过程。

现有 PPVG 方法（如 PhyT2V、DiffPhy、PAG-SAD）利用 LLM 推理来增强提示的物理概念，但存在两个关键问题：

1. **因果模糊性**：物理现象本质上是因果有序的事件链（如"加热→融化→蒸发"），但现有方法只给模型一个静态语义标签，无法建模这种动态演进过程，导致生成的视频只呈现物理现象的某个瞬间而非完整过程。
2. **物理一致性约束不足**：纯文本无法传达事件之间的因果连续性，虽然视觉线索（参考视频/图像）可以提供可观察的过渡证据，但与特定物理现象紧密对齐的视觉先验往往难以获取。

本文的核心 idea：将物理现象视为一系列因果连接、动态演化的事件序列，而非单一场景。

## 方法详解

### 整体框架

框架分为两个协同模块：(1) PECR 模块通过物理公式约束的链式推理将物理现象分解为事件链；(2) TCP 模块将事件链转化为时序对齐的语义-视觉双提示，引导现成视频扩散模型（如 CogVideoX-5B）生成物理合理的视频。

### 关键设计

1. **物理公式驱动的事件链推理（PECR）**:

   **物理公式锚定（PFG）**: 首先通过问答确定文本描述中涉及的物理定律 $\mathcal{L}$，推理出公式名称 $\mathcal{N}_\mathcal{L}$，然后从在线知识库中检索对应的物理公式 $\mathcal{F}^*$。用常识推理设定公式所需的物理参数。

   **物理现象分解（PPD）**: 基于检索到的物理公式进行计算分析，追踪物理参数的变化。当参数变化超过阈值 $\tau_p$ 时确定事件边界：

   $$\mathcal{C}_t = \{(\mathbf{P}_t, \mathcal{F}^*(\mathbf{P}_t)) \mid \|\mathbf{P}_t - \mathbf{P}_{t-1}\| > \tau_p\}$$

   同时维护动态场景图 $\mathcal{G}_t = \{\mathcal{V}_t, \mathcal{R}_t\}$，根据物理条件更新节点属性（如液体变色）和边关系（如物体间距减小）。相邻事件间的参数通过物理连续性检验来确保一致性。

2. **渐进式叙事修订（PNR，TCP 模块一部分）**: 每个事件的描述不是独立生成，而是在前一个事件描述基础上做最小增量修订：

   $$w_t = \text{LLM}(w_{t-1} + \Delta(w_{t-1}, \mathcal{C}_t, \mathcal{G}_t))$$

   物理条件 $\mathcal{C}_t$ 约束允许的转变（如升温允许"融化"但排除"冻结"），场景图 $\mathcal{G}_t$ 保持物体身份一致。最终通过语义压缩和因果连接词将多个事件描述合并为一个正向语义提示 $w^*_+$，同时构造负向描述 $w^*_-$。

3. **交互式关键帧合成（IKS，TCP 模块一部分）**: 为每个物理事件合成关键帧作为视觉先验。从预定义的编辑操作集（如拖拽、遮罩）中选择合适的操作符 $\mathcal{O}_t$，物理参数变化量作为数值约束控制编辑幅度：

   $$v_t = \text{Edit}(v_{t-1}; \mathcal{O}_t)$$

   关键帧间用线性插值生成过渡帧，编码为 VAE 特征后加噪替代原始高斯噪声，为视频扩散模型提供物理感知先验。

### 损失函数 / 训练策略

本方法是**训练无关的（training-free）**，不训练任何网络。所有组件都通过提示工程调用现成模型：GPT-OSS-20B 做推理，Qwen-Image-Edit 做图像编辑，CogVideoX-5B 做视频生成。

### 主实验

| 基准 | 指标 | 本文 | 之前 SOTA (PhysHPO) | 提升 |
|------|------|------|---------------------|------|
| PhyGenBench | PCA (avg) | **0.66** | 0.61 | +8.19% |
| PhyGenBench | Mechanics PCA | **0.67** | 0.55 | +21.8% |
| PhyGenBench | Optics PCA | **0.72** | 0.68 | +5.9% |
| PhyGenBench | Thermal PCA | **0.65** | 0.50 | +30.0% |

| VideoPhy | Solid-Solid SA∩PC | **40.6%** | ~28.2% | +12.4% |

**物理顺序评估**（PD=现象检测, PO=物理顺序）：

| 领域 | PD | PO | 说明 |
|------|----|----|------|
| Mechanics | 0.79 | **0.79** | PO 远超 DiffPhy (0.53) 和 Kling (0.45) |
| Optics | 0.84 | **0.85** | 因果事件链显著提升物理顺序正确性 |
| Thermal | 0.78 | **0.69** | 热力学过程的因果建模效果明显 |

### 消融实验

| 配置 | Avg PCA | 说明 |
|------|---------|------|
| 完整方法 | 0.66 | - |
| 去 PFG (物理公式) | 0.62 | -6%，缺少定量约束 |
| 去 PPD (现象分解) | 0.59 | -11%，缺少事件链最致命 |
| 去 PNR (叙事修订) | 0.64 | -3%，语义连贯性略降 |
| 去 IKS (关键帧合成) | 0.55 | -17%，视觉锚定最关键 |

### 关键发现

- **事件数量 = 4 最优**: 太少（1-3）时序监督不足，太多（5-6）关键帧编辑累积误差
- IKS（关键帧合成）是最重要的组件（去掉后下降 17%），说明视觉先验比语义提示更能约束物理合理性
- 在物理顺序验证（PO）指标上的巨大提升（如力学 0.45→0.79）充分验证了事件链推理的价值

## 亮点与洞察

- 将物理现象建模为因果事件链是一个深刻的洞察：物理世界本质上就是因果演化的，而非静态场景
- 物理公式作为确定性约束嵌入 CoT 推理是优雅的设计——LLM 负责定性推理，公式负责定量计算，各取所长
- 完全 training-free 的架构让方法可以即插即用到任何视频扩散模型上
- 关键帧插值替代纯高斯噪声初始化是一个简单但有效的思路，为视频扩散模型注入了物理过程先验

## 局限性 / 可改进方向

- 对组合物理定律场景（多个物理定律同时作用）效果较差，因为基础模型的组合物理推理能力有限
- 依赖于在线知识库检索物理公式，覆盖范围受限；罕见物理现象可能找不到对应公式
- 关键帧合成通过编辑传播，累积误差限制了事件数量不能太多
- 方法的整体延迟较高（LLM 推理 + 图像编辑 + 视频生成多次调用）

## 相关工作与启发

- **PhyT2V/DiffPhy/PAG-SAD**: 用 CoT 推理增强物理感知提示，但只描述单一场景而非因果过程
- **Diffusion Forcing**: 类似的"帧级控制"思想也出现在 CMDM 中，但本文从事件链而非帧级做控制
- **LayerCraft/GoT**: 用推理辅助空间布局，本文扩展到时序因果链
