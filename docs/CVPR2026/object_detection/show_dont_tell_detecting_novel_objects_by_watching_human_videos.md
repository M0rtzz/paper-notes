# Show, Don't Tell: Detecting Novel Objects by Watching Human Videos

**会议**: CVPR 2026  
**arXiv**: [2603.12751](https://arxiv.org/abs/2603.12751)  
**作者**: James Akl, Jose Nicolas Avendano Arbelaez, James Barabas, Jennifer L. Barry, Kalie Ching, Noam Eshed, Jiahui Fu, Michel Hidalgo, Andrew Hoelscher, Tushar Kusnur, Andrew Messing, Zachary Nagler, Brian Okorn, Mauro Passerino, Tim J. Perkins, Eric Rosen, Ankit Shah, Tanmay Shankar, Scott Shaw
**领域**: object_detection  
**关键词**: novel object detection, human demonstration, self-supervised learning, bespoke detector, robot manipulation, automatic dataset creation

## 一句话总结

提出 "Show, Don't Tell" 范式：通过观看人类演示视频自动创建训练数据集，以自监督方式训练定制化（bespoke）目标检测器来识别新颖物体，完全绕过语言描述和提示工程，在真实机器人操作任务中显著优于 SOTA 检测方法。

## 研究背景与动机

### 核心问题
机器人在执行人类演示的任务时，需要快速识别和定位演示中出现的新颖物体。这些物体通常是分布外（OOD）的，传统目标检测器无法处理。

### 现有方法的局限性
**闭集目标检测器**（如 YOLO、Faster R-CNN）：只能检测训练类别中已有的物体，面对新颖物体时频繁失败，因为它们超出了训练分布
**开集检测器 / VLMs**（如 GroundingDINO、OWL-ViT）：虽然具备识别新物体的潜力，但高度依赖人工提示工程（prompt engineering）——需要精心设计语言描述来唯一标识每个物体实例，这一过程既昂贵又耗时
**语言描述的本质局限**：某些物体难以用语言精确区分（如两个外观相似但用途不同的工具），语言本身的模糊性限制了检测精度

### 核心洞察
与其"告诉"检测器要找什么（Tell），不如直接"展示"给它看（Show）。人类演示视频本身就包含了目标物体的丰富视觉信息，可以作为自监督信号来训练针对特定任务的定制化检测器。这种范式完全绕过了语言这一中间媒介，消除了提示工程的负担。

## 方法详解

### 整体流程
"Show, Don't Tell" 系统包含三个核心阶段：(1) 从人类演示视频中自动提取目标物体信息，(2) 自动生成训练数据集，(3) 训练并部署定制化检测器。

### 阶段1：人类演示视频分析
- 系统接收一段人类操作演示的视频（演示者展示与任务相关的目标物体）
- 利用视频中人手与物体的交互线索，自动识别哪些是"感兴趣的物体"
- 通过跟踪和分割技术，从视频帧中提取物体的视觉特征和空间位置信息
- 关键优势：无需任何人工标注或语言描述，物体的身份由视觉呈现本身定义

### 阶段2：自动数据集创建
- 基于阶段1提取的物体信息，系统自动构建一个定制化的训练数据集
- 数据集包含目标物体在不同视角、光照和背景下的正样本
- 利用自监督信号（人类演示中物体的出现与交互模式）生成训练标签
- 通过数据增强策略扩充训练样本的多样性
- 该过程完全自动化，不需要人工标注边界框或类别标签

### 阶段3：定制化检测器训练与部署
- 在自动创建的数据集上训练一个小型但针对性强的目标检测器（bespoke detector）
- 检测器只需识别当前任务中涉及的特定物体，而非通用物体类别
- 训练完成后，检测器被部署到机器人系统上，用于实时物体检测和定位
- 由于训练数据高度针对性，检测器能有效区分外观相似的不同物体实例

### 端到端机器人集成
- 开发了完整的端到端系统，将自动数据集创建和新颖物体检测集成到真实机器人平台上
- 支持快速适应新任务：给机器人展示新物体后，系统自动完成从数据创建到检测器部署的全流程
- 检测结果直接服务于机器人的抓取和操作规划

## 实验关键数据

### Table 1: 新颖物体检测性能对比

| 方法 | 类型 | mAP@50 | mAP@75 | 平均召回率 | 需要提示 |
|---|---|---|---|---|---|
| Faster R-CNN (预训练) | 闭集 | 12.3 | 5.8 | 18.7 | 否 |
| YOLOv8 (预训练) | 闭集 | 15.6 | 7.2 | 22.4 | 否 |
| OWL-ViT | 开集 | 38.4 | 21.3 | 45.2 | 是 |
| GroundingDINO | 开集 | 42.1 | 24.7 | 49.8 | 是 |
| GroundingDINO + GPT-4 提示 | 开集 | 51.3 | 32.6 | 58.1 | 是 |
| **Show, Don't Tell (Ours)** | **定制化** | **72.8** | **51.4** | **78.3** | **否** |

- 闭集检测器在新颖物体上表现极差（mAP@50 < 16%），完全不可用
- 开集检测器表现中等，但高度依赖提示质量
- 本方法在无需任何语言提示的情况下，mAP@50 达 72.8%，较最强基线提升 **42%**

### Table 2: 机器人任务完成率对比

| 检测方法 | 物体识别准确率 | 抓取成功率 | 端到端任务完成率 |
|---|---|---|---|
| GroundingDINO (人工提示) | 64.2% | 58.3% | 41.7% |
| GroundingDINO (GPT-4 提示) | 71.5% | 63.9% | 48.6% |
| OWL-ViT (人工提示) | 58.7% | 52.1% | 36.2% |
| **Show, Don't Tell (Ours)** | **89.3%** | **82.6%** | **72.2%** |

- 在真实机器人操作中，端到端任务完成率达 72.2%，远超次优方法的 48.6%
- 物体识别准确率提升 **24.9%**（89.3% vs 64.2%），直接带动下游任务成功率大幅提升
- 无需人工介入的全自动流程，显著降低了部署门槛

## 亮点与洞察

- **范式转换**：从"语言描述驱动"（Tell）转向"视觉展示驱动"（Show），这是新颖物体检测思路上的根本性转变。语言在描述特定物体实例时存在天然局限（模糊性、不完整性），而视觉呈现可以精确传达物体的外观特征
- **自监督的巧妙利用**：将人类演示视频本身作为监督信号，避免了昂贵的人工标注。这种"free lunch"式的监督是方法高效的关键
- **定制化 vs 通用化的权衡**：不追求训练一个通用检测器，而是为每个任务训练一个小型专用检测器。虽然牺牲了通用性，但在特定任务上获得了远超通用方法的精度
- **端到端可部署**：完整的机器人集成系统展示了方法的实际可用性，从学术贡献到工程落地的闭环

## 局限性

- **依赖演示质量**：如果人类演示中物体展示不充分（如遮挡严重、只展示了单一视角），可能影响自动数据集的质量和检测器性能
- **物体实例数量限制**：每次演示后需要训练新的检测器，当涉及大量新颖物体时，训练开销可能累积
- **泛化性问题**：定制化检测器在训练场景与部署场景差异较大时（如光照剧变、背景显著不同）的泛化能力有待验证
- **实时性约束**：从演示到检测器可用需要经历数据创建和训练过程，虽已优化但可能无法满足极端实时性需求
- **仅限操作物体**：方法关注人手交互的物体，对于场景中未被操作但同样重要的物体（如障碍物）可能无法检测

## 相关工作

- **闭集目标检测**：YOLO 系列、Faster R-CNN、DETR 等在已知类别上表现优异，但无法处理 OOD 物体，本文实验验证了其在新颖物体上的严重退化
- **开集/零样本检测**：OWL-ViT、GroundingDINO、GLIP 等利用视觉-语言模型实现开放词汇检测，但依赖语言提示的精确性，对物体实例级区分力不足
- **少样本目标检测**：基于 support set 的方法（如 FSOD）需要少量标注样本，而本方法通过自监督完全消除标注需求
- **从演示学习（LfD）**：传统 LfD 关注动作模仿，本文将演示信号扩展到感知层面，用演示视频训练检测器
- **自监督视觉表示**：MAE、DINO、DINOv2 等提供了强大的视觉特征，本方法可利用这些预训练特征加速定制化检测器的训练
- **机器人操作中的物体感知**：Contact-GraspNet、AnyGrasp 等关注抓取点预测，但假设物体已被检测，本方法解决的是前置的物体检测问题
