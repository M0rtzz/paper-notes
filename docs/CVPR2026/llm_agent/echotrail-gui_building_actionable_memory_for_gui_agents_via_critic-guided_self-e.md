# EchoTrail-GUI: Building Actionable Memory for GUI Agents via Critic-Guided Self-Exploration

**会议**: CVPR 2026
**arXiv**: [2512.19396](https://arxiv.org/abs/2512.19396)
**代码**: 有
**领域**: LLM Agent / GUI自动化
**关键词**: GUI Agent, 可操作记忆, 自我探索, Critic引导, RAG推理

## 一句话总结
提出EchoTrail-GUI，通过"学习-记忆-应用"三阶段闭环为GUI Agent构建可操作记忆——探索Agent自主交互GUI环境生成轨迹→Critic奖励模型过滤高质量轨迹存入记忆库→新任务时混合检索(密集+稀疏)最相关轨迹注入prompt引导推理，在AndroidWorld和AndroidLab上显著提升任务成功率。

## 研究背景与动机

1. **领域现状**：VLM驱动的GUI Agent已能解析截图+执行多步操作。但现有Agent是无状态的——"数字健忘症"→每个任务从零开始，不积累操作知识。
2. **两大障碍**：(1) **经验获取瓶颈**：高质量轨迹数据稀缺——人工标注慢且不可扩展，无引导探索产出噪声轨迹；(2) **知识应用鸿沟**：即使有轨迹库，如何高效检索和应用仍非trivial——静态示例无法动态适配当前任务。
3. **核心idea**：模拟人类"学-记-用"认知循环——(1) 自我探索生成轨迹→(2) Critic过滤存入记忆库→(3) 检索记忆增强推理。全流程无人工监督。

## 方法详解

### 整体框架
三阶段：**Stage I: Critic引导自我探索**(构建记忆库 $D_{mem}$) → **Stage II: 动态记忆注入**(混合检索相关轨迹) → **Stage III: 记忆增强推理**(注入prompt引导Agent)。

### 关键设计

1. **Stage I: Critic引导自我探索**:
   - **探索Agent**(Gemini 2.5 Flash)自主交互GUI环境→渐进意图聚焦(先好奇驱动探索→后目标聚焦执行)→最长30步轨迹
   - **Critic奖励模型**(Gemini 2.5 Flash Lite)评估轨迹质量(1-5分)→阈值$\theta_{good}=4$过滤→仅高质量轨迹存入永久记忆库$D_{mem}$
   - **轨迹抽象化**：不存原始截图→转为结构化表示(界面文本描述+Agent意图摘要+执行动作)→轻量+通用
   - **双数据库系统**：处理库$D_{proc}$(短期，含成功+失败，引导当前探索)+ 记忆库$D_{mem}$(永久，仅高质量)
   - 关键消融证实：注入低质量记忆比无记忆更差→Critic过滤至关重要

2. **Stage II: 动态记忆注入**:
   - 混合检索：$\text{Score}(\tau, I) = \alpha \cdot S_{dense}(\tau, I) + (1-\alpha) \cdot S_{sparse}(\tau, I)$
   - 密集检索(FAISS+嵌入余弦相似度)捕捉语义相关+稀疏检索(BM25)捕捉关键词匹配
   - Top-K=2为最优(消融验证)
   - 记忆格式化：原始日志→结构化步骤(界面描述+意图+动作)→可读的分步指南

3. **Stage III: 记忆增强推理**:
   - 结构化prompt：$P_t = f(I, M_t, H_t, s_t, E_{sum}(s_t))$
   - 任务指令+格式化记忆+当前历史+截图+截图文本摘要→Agent产出下一步动作
   - Plug-and-play：可用于任何现成VLM(Qwen2.5-VL-72B/GPT-4o均验证)

### 数据资产
EchoTrail-4K：4000+高质量Android交互轨迹，支持检索式推理和下游训练。

### 关键发现
- **低质量记忆有害**：无Critic过滤的记忆注入比不注入更差→验证了质量控制的必要性
- **混合检索>单一检索**：密集捕捉语义+稀疏捕捉关键词→互补
- **K=2最优**：过多记忆增加prompt长度和干扰
- **渐进意图聚焦**有效：先探索再聚焦→比纯随机探索质量高
- **模型无关**：对Qwen和GPT-4o都有效→方法通用

## 亮点与洞察
- **"数字健忘症"问题的精准定位和解决**：将GUI Agent从无状态转变为记忆增强→根本性能力提升
- **自我探索的数据飞轮**：Agent探索→Critic过滤→积累高质量记忆→记忆引导更好探索→更多记忆...正反馈循环
- **Critic过滤是核心机制**：不是"有记忆就好"而是"只有好记忆才好"——低质量记忆是毒药
- **Plug-and-play范式**：记忆系统与Agent模型完全解耦→可即插即用到任何VLM backbone

## 局限性 / 可改进方向
- 探索和Critic都依赖大模型API(Gemini)→成本和延迟
- 当前记忆是static的(存入后不更新)→能否随Agent能力提升修剪/更新记忆？
- 轨迹抽象丢失了截图视觉信息→某些依赖精确视觉的任务可能受限
- 跨应用版本更新后，旧记忆中的UI布局可能过时

## 相关工作与启发
- **vs Reflexion/Voyager**: 它们用反思/记忆增强通用Agent，EchoTrail专为GUI场景设计→Critic+轨迹抽象更适合GUI特性
- **vs AgentTrek**: 从网页教程挖掘轨迹，EchoTrail自主探索→更灵活、覆盖更广
- **vs RAG for Agents**: 通用RAG方法无法保证检索到有效的操作序列→EchoTrail的Critic保证了记忆质量
