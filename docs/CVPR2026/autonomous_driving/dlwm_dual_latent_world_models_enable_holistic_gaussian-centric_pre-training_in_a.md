# DLWM: Dual Latent World Models enable Holistic Gaussian-centric Pre-training in Autonomous Driving

**会议**: CVPR 2026  
**arXiv**: [2604.00969](https://arxiv.org/abs/2604.00969)  
**代码**: 无  
**领域**: Autonomous Driving / 自动驾驶预训练  
**关键词**: 世界模型, 3D高斯溅射, 自监督预训练, 占据预测, 运动规划

## 一句话总结
提出 DLWM，一个两阶段的高斯中心自监督预训练范式：第一阶段通过重建深度和语义图学习3D高斯表示，第二阶段训练双隐世界模型——高斯流引导的时序预测（用于占据感知/预测）和自车规划引导的时序预测（用于运动规划），显著提升三大核心任务性能。

## 研究背景与动机

**领域现状**: 基于视觉的自动驾驶中，高斯中心表示（如GaussianFormer）以3D语义高斯提供综合且稀疏的场景描述，在感知/预测/规划的表达力-效率平衡上优于密集BEV和稀疏query。但依赖大量人工标注限制可扩展部署。

**现有痛点**: (a) MAE类预训练不显式学3D几何；(b) 渲染式预训练(UniPAD/ViDAR)依赖LiDAR深度监督；(c) 隐世界模型已用于运动规划，但在感知和预测任务上尚未探索；(d) 高斯查询在帧间缺乏一对一对应（排列等价性），无法直接做时序特征监督。

**核心矛盾**: 如何为高斯中心模型的全生命周期（感知+预测+规划）设计统一自监督预训练？在哪个表示空间做时序预测最合适？

**本文目标**: 为高斯中心自动驾驶设计全面的自监督预训练方案，覆盖3D占据感知、4D占据预测和运动规划三大任务。

**切入角度**: 利用BEV光栅化作为高斯query到密集表示的桥梁——BEV特征保留高度信息、有明确帧间区域对应，完美解决排列等价性挑战。

**核心idea**: Stage 1学高斯表示（自监督重建）→ Stage 2学时序演化（双隐世界模型分别服务感知/预测和规划）。

## 方法详解

### 整体框架
**Stage 1**: 图像编码器(ResNet+FPN) → 高斯编码器(自编码+图像交叉注意力+精化) → 可微渲染重建深度和语义图。
**Stage 2a**: 冻结感知模块 + 高斯流预测 → ego运动对齐 → BEV光栅化 → L2监督预测未来BEV特征（用于感知/预测下游任务）。
**Stage 2b**: 当前高斯隐变量 + 预测ego轨迹 → 条件预测未来BEV特征 → L2监督（用于规划下游任务）。

### 关键设计

1. **高斯表示学习（Stage 1）**:

    - **功能**: 自监督学习3D语义高斯来重建多视角深度和语义图
    - **核心思路**: 高斯编码器逐块精化查询——(1) 自编码模块：高斯间交互 + 4D稀疏卷积做跨帧时序传播；(2) 图像交叉注意力：可变形采样聚合视觉特征（利用相机内外参）；(3) 精化模块：调整高斯属性（位置、协方差、透明度、语义logits）
    - **训练信号**: 深度GT来自LiDAR（稀疏但精准） + Metric3D伪深度（密集但有噪声），语义GT来自Grounded SAM——**全程无需任何人工标注**
    - **设计动机**: 通过渲染约束迫使高斯学习物理先验。LiDAR + Metric3D互补——前者提供准确度，后者提供密集覆盖

2. **高斯流引导的隐世界模型（Stage 2a）**:

    - **功能**: 预测3D高斯的运动流，通过BEV光栅化做时序特征预测
    - **核心思路**:
        - 流预测头估计每个高斯的动态位移 $\Delta\mu_k^t$
        - Ego运动对齐: $\mu_k^{t+1} = \mathbf{T}_{ego}^{t \to t+1}(\mu_k^t + \Delta\mu_k^t)$
        - BEV光栅化：将变换后的3D高斯投影到2D BEV平面，得到预测隐变量 $\hat{B}_{t+1}$
        - 监督: $\mathcal{L}_{bev} = \|\hat{B}_{t+1} - B_{t+1}\|_2$，其中 $B_{t+1}$ 由冻结感知模块从真实 $t+1$ 帧图像产生
    - **设计动机**: 高斯query的排列等价性使直接特征监督不可行。BEV光栅化将稀疏query投影到密集网格，保留高度信息（垂直堆叠）且有明确空间对应——完美桥接query空间和监督空间

3. **Ego规划引导的隐世界模型（Stage 2b）**:

    - **功能**: 以预测ego轨迹为条件预测未来场景，改进运动规划
    - **核心思路**: 当前高斯隐变量 + 预测ego轨迹 → 条件预测未来BEV特征 → L2监督
    - **设计动机**: 规划任务需要"如果我这样走，未来场景会怎样"的条件想象能力，与感知/预测的精确场景流需求不同，因此需要独立的世界模型分支

4. **4D占据预测设计**:

    - **功能**: 流式自回归预测未来占据
    - **核心思路**: (1) 用ego轨迹对齐当前高斯到下一帧；(2) 新进入区域用随机高斯补全；(3) 3D稀疏卷积+精化层同时处理已有和新补的高斯；(4) Gaussian-to-Occupancy Splatting预测下一帧占据
    - **设计动机**: 自回归方式允许长期预测，随机高斯补全新区域支持场景外推

### 损失函数 / 训练策略
- **Stage 1**: $\mathcal{L}_{rec} = 1.0 \cdot \mathcal{L}_d + 0.05 \cdot \mathcal{L}_{pd} + 1.0 \cdot \mathcal{L}_{sem}$
- **Stage 2a**: $\mathcal{L}_{rec} + \mathcal{L}_{bev}$（重建+BEV预测）
- **Stage 2b**: ego规划损失 + $\mathcal{L}_{bev}$
- 两阶段分别预训练，微调时加载预训练权重到对应模块

## 实验关键数据

### 主实验（3D占据感知, SurroundOcc-nuScenes）

| 方法 | IoU | mIoU | 说明 |
|------|-----|------|------|
| GaussianFormer | 29.83 | 19.10 | 基础高斯方法 |
| GaussianFormer-2 | 31.74 | 20.82 | 改进版 |
| GaussianWorld* | 32.77 | 21.79 | 时序建模 |
| Baseline (无预训练) | 31.77 | 20.83 | 本文基线 |
| **DLWM (Ours)** | **34.61** | **21.85** | **+2.84 IoU, +1.02 mIoU** |

### 预训练提升总结

| 任务 | 预训练提升 | 具体数值 |
|------|-----------|----------|
| 3D占据感知 | +1.02 mIoU | IoU: 31.77→34.61 |
| 4D占据预测 | +2.68 mIoU (avg) | 1s/2s/3s均显著提升 |
| 运动规划 | -16% L2 error | 规划精度大幅提升 |

### 消融实验

| 预训练配置 | 感知 | 预测 | 规划 | 说明 |
|-----------|------|------|------|------|
| 无预训练 | baseline | baseline | baseline | - |
| 仅Stage 1 | ++ | + | + | 高斯表示学习有效 |
| Stage 1 + 2a | +++ | +++ | + | 时序特征学习进一步提升感知预测 |
| **Stage 1 + 2a + 2b** | +++ | +++ | +++ | **双世界模型全面最优** |

### 关键发现
- **预训练全面提升三大任务**: 感知+1.02, 预测+2.68, 规划-16%L2，验证了全生命周期预训练的有效性
- **BEV作为隐表示空间**: 解决了高斯query排列等价性的关键技术挑战
- **双世界模型设计合理**: 感知/预测需精确场景流，规划需条件想象——功能解耦带来最优性能
- **自监督标签多源**: LiDAR + Metric3D + Grounded SAM，完全无人工标注

## 亮点与洞察
- **全生命周期预训练**: 首个为高斯中心自动驾驶全部核心任务设计统一预训练的工作
- **BEV桥梁思想**: 用BEV光栅化解决高斯query的permutation equivalence是可推广的技术思路
- **规划L2误差-16%**: 预训练对规划的显著提升说明时序感知能力直接惠及决策模块
- **无人工标注**: 全部使用自动化标签（LiDAR深度、伪深度、自动分割）

## 局限与展望
- 两阶段预训练有顺序依赖，端到端联合训练可能更高效
- 双世界模型独立训练，未探索两者间信息共享
- 当前仅预测一步未来，多步长期预测能力待验证
- 对极端天气、夜间等OOD场景的鲁棒性未讨论

## 相关工作与启发
- 与UniPAD/ViDAR区别：使用体素渲染+需LiDAR深度；本文用高斯渲染+多源自动标签
- 与GaussianWorld区别：做3DGS空间场景演化但缺乏系统预训练策略
- 与SQS区别：仅针对稀疏感知预训练不覆盖规划；DLWM覆盖全生命周期
- 启发：隐世界模型的表示空间选择至关重要——BEV是高斯中心模型的自然桥梁

## 评分
- 新颖性: ⭐⭐⭐⭐ 双隐世界模型和BEV桥梁设计有价值，但整体是现有技术组合
- 实验充分度: ⭐⭐⭐⭐ 三大任务全覆盖，消融各阶段贡献，但规划实验细节略少
- 写作质量: ⭐⭐⭐⭐ 框架图清晰，技术动机阐述合理
- 价值: ⭐⭐⭐⭐⭐ 为高斯中心自动驾驶提供了系统化预训练方案，三大任务全面提升

**关键词**: 世界模型, 3D高斯, 自监督预训练, 占据预测, 运动规划

## 一句话总结
提出DLWM，面向自动驾驶的双潜在世界模型全局高斯中心预训练范式——Stage1自监督学习3D高斯场景表示（渲染多视图语义+深度图），Stage2训练双潜在世界模型：高斯流引导的模型用于下游占据感知/预测(+1.02/+2.68 mIoU)，自车轨迹引导的模型用于运动规划(-16% L2误差)，解决了高斯查询跨帧无法直接监督的排列等价性挑战。

## 研究背景与动机

1. **领域现状**：视觉自动驾驶中，3D高斯表示在感知/预测/规划多任务上较BEV/稀疏查询更达表达-效率平衡。但依赖大量标注限制了扩展部署。
2. **现有痛点**：(1) MAE类预训练不显式学3D几何；(2) 渲染式预训练(UniPAD/ViDAR)需LiDAR深度；(3) 潜在世界模型用于规划有效但**用于感知/预测尚未探索**；(4) 高斯查询的排列等价性——不同帧独立初始化的高斯无一一对应→直接特征监督不可行。
3. **关键突破点**：将高斯query通过BEV栅格化转化为密集网格表示→保留高度信息且有清晰帧间区域对应→可作为潜在空间用于世界模型时间监督。
4. **核心idea**：两阶段——Stage1学高斯空间表示(渲染重建) → Stage2对偶世界模型：(a) 高斯流引导→感知/预测 + (b) 自车轨迹引导→规划。

## 方法详解

### 整体框架
多视图图像 → 2D backbone → 3D高斯query预测 → **Stage 1**: 可微渲染多视图语义+深度图 → 自监督重建损失。用预训练权重初始化 → **Stage 2a**: 高斯流引导世界模型——预测3D高斯流(位移)→当前高斯传播到未来帧→BEV栅格化为潜在表示→与冻结的未来帧感知特征做监督。**Stage 2b**: 自车轨迹引导世界模型——预测自车轨迹→条件化当前高斯潜在→预测未来高斯BEV特征→同样与冻结特征对齐。

### 关键设计

1. **Stage 1: 高斯空间表示学习**:
    - 预测3D高斯(位置/尺度/旋转/颜色/语义属性)→可微渲染多视图语义图+深度图→自监督重建
    - 不需要LiDAR深度——深度也是渲染获得的
    - 学到的高斯query自带精细上下文特征→为Stage2提供良好初始化

2. **Stage 2a: 高斯流引导世界模型(感知/预测)**:
    - 预测每个高斯的3D流(位移向量)→将当前帧高斯传播到未来帧位置
    - BEV栅格化：传播后的高斯通过splatting投影到BEV网格→获得密集潜在表示
    - 用冻结的Stage1感知模块处理未来帧多视图图像→获得"目标"特征→作为潜在表示的监督信号
    - **解决排列等价性**：高斯query排列等变→但BEV栅格化后是固定网格→帧间有清晰的区域对应→时间监督可行

3. **Stage 2b: 自车轨迹引导世界模型(规划)**:
    - 预测自车轨迹→将轨迹编码为运动条件
    - 用运动条件+当前高斯潜在→预测未来场景的高斯BEV特征
    - 同样与冻结感知特征对齐
    - 设计动机：规划和感知需要不同的时间预测信息——规划需要轨迹条件化(自车运动影响观测)，感知需要场景级流

4. **为什么是双世界模型而非单一模型**:
    - 感知/预测关注场景级运动(他车/行人)→高斯流是自然的监督
    - 规划关注自车决策的影响→轨迹条件化是自然的监督
    - 两者优化目标不同→分开训练避免互相干扰

## 实验关键数据

### 主实验(SurroundOcc + nuScenes)

| 下游任务 | 无预训练 | +DLWM预训练 | 提升 |
|----------|:---:|:---:|:---:|
| 3D占据感知(mIoU) | 基准 | +1.02 | 显著 |
| 4D占据预测(mIoU) | 基准 | +2.68 | 更显著 |
| 运动规划(L2 error) | 基准 | -16% | 大幅 |

### 消融实验

| 配置 | 感知 | 预测 | 规划 |
|------|:---:|:---:|:---:|
| 无Stage 1 | 下降 | 下降 | 下降 |
| 无Stage 2a(高斯流WM) | 下降 | 大幅下降 | 不变 |
| 无Stage 2b(轨迹WM) | 不变 | 不变 | 下降 |
| **完整DLWM** | **最优** | **最优** | **最优** |

→ 证实双世界模型各自服务不同下游任务。

### 关键发现
- 4D占据预测获益最大(+2.68 mIoU)→时间预测任务最受益于世界模型预训练
- 高斯流引导WM对感知也有帮助(+1.02)→学到了更好的场景动态表示
- 轨迹引导WM仅对规划有帮助→验证了分开训练的合理性
- BEV栅格化成功解决了高斯排列等价性问题→潜在空间的时间监督可行

## 亮点与洞察
- **排列等价性→BEV栅格化的巧妙解决**：高斯query跨帧无对应→但splatting到BEV后成为固定网格→这一转换是整个框架的关键使能
- **双世界模型的任务分工明确**：感知/预测→场景流驱动；规划→轨迹驱动。不同任务需要不同的时间预测信号→分工优于统一
- **全生命周期预训练**：从空间(Stage1 渲染重建)到时间(Stage2 世界模型)→覆盖感知/预测/规划全链路→一次预训练多任务受益

## 局限与展望
- Stage2的两个世界模型分别训练→联合训练可能发现共享的时间特征
- BEV栅格化丢失了高度维度的精细信息→3D体素栅格化可能更好但计算量更大
- 两阶段训练pipeline较复杂→能否简化为单阶段端到端？
- 当前世界模型仅预测一步未来→多步预测对长视界规划可能更好

## 评分
- 新颖性: ⭐⭐⭐⭐ 双世界模型+高斯流引导的时间预训练
- 实验充分度: ⭐⭐⭐⭐ SurroundOcc+nuScenes三任务验证+消融
- 写作质量: ⭐⭐⭐⭐ pipeline图清晰，排列等价性问题的解释充分
- 价值: ⭐⭐⭐⭐⭐ 对高斯中心自动驾驶方法的预训练有重要推动
