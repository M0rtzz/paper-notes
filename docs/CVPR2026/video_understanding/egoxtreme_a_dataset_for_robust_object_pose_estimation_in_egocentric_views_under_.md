# EgoXtreme: A Dataset for Robust Object Pose Estimation in Egocentric Views under Extreme Conditions

**会议**: CVPR 2026  
**arXiv**: [2603.25135](https://arxiv.org/abs/2603.25135)  
**代码**: [https://taegyoun88.github.io/EgoXtreme/](https://taegyoun88.github.io/EgoXtreme/)  
**领域**: Video Understanding / 6D Pose Estimation  
**关键词**: 第一人称视角, 6D位姿估计, 极端条件, 智能眼镜, 基准数据集

## 一句话总结

提出 EgoXtreme，首个面向极端条件下第一人称视角的大规模 6D 物体位姿估计基准数据集，涵盖严重运动模糊、动态光照和烟雾遮挡三种真实挑战，揭示了当前 SOTA 位姿估计器在这些条件下的严重失效。

## 研究背景与动机

智能眼镜正在成为重要的可穿戴设备，在"双手忙碌、目光专注"的场景中提供丰富的上下文理解。要理解佩戴者的活动环境，第一人称视角下的 6D 物体位姿估计至关重要。

然而，现有 6D 位姿估计基准存在严重的**现实差距**：

**YCB-Video、LineMod、T-LESS** 等标准数据集均在受控实验室条件下采集，光照稳定、运动缓慢
**H2O、HOT3D** 虽引入了第一人称视角，但光照条件温和，缺乏真实的极端环境测试
3. 真实智能眼镜使用场景面临独特挑战：
   - 相机固定在移动的头部，产生快速且频繁的运动
   - 近距离视角导致严重截断
   - 动态/极端光照（头灯、闪光灯、应急灯、安全出口灯）
   - 烟雾、雾霾等视觉遮挡物

核心问题：仅通过模拟（合成数据、帧平均模拟模糊等）来弥合差距是不够的——**需要一个真实世界极端条件下的基准来驱动鲁棒模型的发展**。

## 方法详解

### 整体框架

EgoXtreme 是一个数据集贡献型工作，核心包括：（1）三个挑战场景的设计与数据采集；（2）基于 OptiTrack 动捕系统的高精度 6D 标注流程；（3）对 SOTA 方法的全面基准评估。

### 关键设计

1. **三大挑战场景**：覆盖不同类型的极端条件 → 每个场景针对不同的感知困难 → 确保评估的多维度性

    - **工业维修**：使用锤子、电钻、锯等工具进行精细操作，6 种光照 + 烟雾，测试对精密运动和光照变化的鲁棒性
    - **运动场景**：快速挥动乒乓球拍、棒球棒、高尔夫球杆等（物体速度达 1.37 m/s），4 种光照，测试对极端运动模糊的鲁棒性
    - **应急救援**：搜索急救箱、灭火器等应急物品，6 种光照 + 烟雾，测试在剧烈相机抖动和视觉遮挡下的检测能力
    - **设计动机**：三个场景分别考验精细运动、高速运动和紧急情况下的位姿估计，全面暴露模型的薄弱环节

2. **8 种光照条件**：标准 + 极端全覆盖 → 系统化分析光照影响

    - 标准：正常亮度（normal）、中等亮度（middle）、高亮度（high）
    - 极端：低光照（low）、头灯动态光照（head）、手电筒（flash）、应急旋转灯（warning）、安全出口绿灯（green）
    - 部分场景额外引入烟雾模拟火灾/工业环境

3. **高精度 6D 标注流程**：OptiTrack + Aria 眼镜 → 亚毫米级精度 → 确保 GT 可靠性

    - Aria 眼镜采集 30fps RGB 图像，同步 1000fps SLAM 数据
    - OptiTrack 动捕系统 120fps 跟踪反光标记
    - Umeyama 方法对齐坐标系 + Kalman 滤波修正 SLAM 漂移
    - 手动时间偏移校准补偿曝光延迟

### 损失函数 / 训练策略

数据集工作，无特定损失函数。评估使用标准 BOP 指标：
- ADD(-S) recall @ 阈值 0.1d, 0.2d, 0.3d（d 为物体直径）
- MSSD（最大对称感知表面距离）↓
- MSPD（最大对称感知投影距离）↓

## 实验关键数据

### 主实验

三个 SOTA 零样本 6D 位姿估计器在 EgoXtreme 上的表现：

| 场景 + 条件 | FoundPose @0.3d | GigaPose @0.3d | PicoPose @0.3d |
|------------|----------------|----------------|----------------|
| 运动-标准 | 4.72 | 24.64 | 24.61 |
| 运动-极端 | 2.42 (-49%) | 19.04 (-23%) | 17.86 (-27%) |
| 维修-标准 | 37.61 | 62.77 | 76.84 |
| 维修-极端 | 30.03 (-20%) | 45.52 (-27%) | 64.09 (-17%) |
| 维修-标准+烟 | 30.00 | 52.86 | 59.87 |
| 维修-极端+烟 | 25.63 | 45.11 | 52.30 |
| 应急-标准 | 12.88 | 46.34 | 67.83 |
| 应急-极端 | 0.56 (-96%) | 21.30 (-54%) | 36.23 (-47%) |
| 应急-极端+烟 | 0.76 | 21.54 | 31.54 |

- **运动场景最具挑战**：所有模型的 recall 均不超过 25%，FoundPose 甚至不到 5%
- **应急极端条件下性能暴跌**：PicoPose 从 67.83% 降至 36.23%（降 31.6%p）
- FoundPose 在极端条件下几乎完全失效——其稀疏特征匹配方法在截断和模糊下无法获得足够的对应点

### 消融实验

图像预处理（去模糊/去雾/低光增强）对 PicoPose 的影响：

| 场景 | 无预处理 @0.3d | +去模糊 | +去雾 | +低光增强 | +去模糊+去雾 |
|------|--------------|--------|------|----------|------------|
| 维修 | 63.32 | 57.55 (-5.8) | 57.71 (-5.6) | 58.08 (-5.2) | 53.28 (-10.0) |
| 应急 | 51.70 | 45.51 (-6.2) | 37.74 (-14.0) | 46.75 (-5.0) | 43.65 (-8.1) |
| 运动 | 23.02 | 22.24 (-0.8) | 22.05 (-1.0) | — | 20.99 (-2.0) |

**图像恢复不仅无效，反而会损害位姿估计精度！** 去雾在应急场景中尤其糟糕（@0.1d 仅 4.74%），因为非均匀烟雾条件下去雾引入了严重噪声。

追踪策略对运动场景的影响（GigaPose）：

| 策略 | 棒球棒 @0.3d | 网球拍 @0.3d | 高尔夫 @0.3d |
|------|------------|------------|------------|
| Per-frame | 60.55 | 50.91 | 8.36 |
| Direct temporal | 14.29 (-76%) | 22.77 (-55%) | 3.61 (-57%) |
| Fusion temporal | 60.59 | 49.56 | 13.98 |
| **Hybrid temporal** | **64.46 (+6%)** | **50.55** | **14.35 (+72%)** |

### 关键发现

1. **极端条件下性能全面崩溃**：所有 SOTA 模型在极端光照/运动模糊下都有 20-96% 的性能下降
2. **图像恢复是个陷阱**：去模糊、去雾、低光增强在视觉上可能看起来更好，但下游位姿估计反而更差——这挑战了"先恢复再估计"的直觉
3. **时间信息是关键**：简单的帧间传播（Direct temporal）在高速运动下会恶化，但基于置信度的混合策略（Hybrid temporal）可以选择性利用时间先验，显著提升动态场景的表现
4. **FoundPose 最脆弱**：其稀疏特征匹配在截断和模糊下容易失败，对应点不足导致 PnP-RANSAC 直接没有输出
5. **不同类型的退化是正交的**：光照变化导致分布偏移，运动模糊导致特征丢失——需要不同的策略来应对

## 亮点与洞察

1. **填补了关键的评估空白**：现有数据集无法代表真实智能眼镜使用场景的极端条件，EgoXtreme 是首个系统性地涵盖这些挑战的大规模基准
2. **"图像恢复无益"的发现极具价值**：挑战了领域中"先增强后识别"的常见假设，揭示了恢复方法可能引入的artifacts对下游任务有害
3. **数据规模和质量兼具**：130万帧、775.5分钟、15名参与者、OptiTrack 亚毫米精度标注
4. **实验设计巧妙**：使用 GT bounding box 来解耦检测误差，专注评估位姿估计本身的鲁棒性
5. **时间建模的重要性**：Hybrid temporal 策略的成功暗示了未来应该发展基于视频的位姿估计方法

## 局限性 / 可改进方向

1. **仅限室内**：依赖 OptiTrack 动捕系统，无法在户外采集真实数据
2. **缺乏手部标注**：极端运动模糊下手部标注困难，但手-物交互对理解第一人称操作非常重要
3. **13 个物体**：相比 YCB-V 的 21 个或 HOT3D 的 33 个，物体种类较少
4. **场景局限**：仅有三个场景，缺乏如烹饪、医疗等其他常见的智能眼镜应用场景
5. **未探索深度模态**：仅评估 RGB-only 方法，虽然智能眼镜通常不配深度传感器，但 RGB-D 方法作为上界参考仍有意义

## 相关工作与启发

- **HOT3D**：最大的第一人称物体追踪基准，但光照温和、运动缓慢，与 EgoXtreme 形成互补
- **H2O**：第一人称手-物交互数据集，同样缺乏极端条件
- **GigaPose / FoundPose / PicoPose**：三种代表性零样本位姿估计器，代表了粗精两步法和端到端方法
- **DarkIR**：联合处理低光、噪声和模糊的最新方法，但在位姿估计下游仍然无效
- 启示：位姿估计的鲁棒性不能依赖像素级增强，需要从特征表示和时间建模两个层面根本性地解决
