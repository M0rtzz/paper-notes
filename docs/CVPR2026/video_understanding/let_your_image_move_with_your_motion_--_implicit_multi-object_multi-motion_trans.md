# FlexiMMT: Let Your Image Move with Your Motion! — Implicit Multi-Object Multi-Motion Transfer

**会议**: CVPR2026  
**arXiv**: [2603.01000](https://arxiv.org/abs/2603.01000)  
**代码**: [项目页](https://ethan-li123.github.io/FlexiMMT_page/)  
**领域**: 视频理解 / 可控视频生成  
**关键词**: 运动迁移, 多目标多运动, 隐式运动表示, 扩散模型, 注意力解耦, 掩码传播

## 一句话总结

FlexiMMT 是首个支持隐式多目标多运动迁移的 I2V 框架，通过运动解耦掩码注意力机制 (MDMA) 和差异化掩码提取机制 (DMEM)，将多个参考视频的不同运动独立分配给目标图像中的不同物体，实现灵活组合式运动迁移。

## 背景与动机

1. **运动迁移是可控视频生成的核心方向**：从参考视频中捕捉运动动态并施加于目标主体，是当前视频生成研究的热点任务
2. **显式 vs 隐式运动表示**：显式方法（姿态/光流/轨迹）需要额外估计器且计算昂贵；隐式方法直接从参考视频编码运动到潜在嵌入，更灵活易用
3. **I2V 比 T2V 更具挑战**：I2V 需同时保持给定首帧图像的外观一致性和参考运动的跟随性，对空间和结构一致性要求更高
4. **现有方法局限于单目标场景**：已有隐式运动迁移方法大多只处理单物体或单运动，无法同时将不同运动分配给同一图像中的多个物体
5. **多目标运动纠缠是核心瓶颈**：在全局注意力中，不同物体的运动 token 和文本 token 发生混合交互，导致运动在不同物体间相互干扰
6. **外部分割模型依赖**：训练时使用外部分割模型会带来计算开销且导致训练-推理不一致，需要更高效的掩码获取方案

## 方法详解

### 整体框架

FlexiMMT 基于 CogVideoX-5B-I2V 扩散模型（MM-DiT 架构），引入可训练的运动 token $T_m$ 与文本 token $T_y$ 和视频 token $T_v$ 在 3D 全注意力层中交互。核心创新在于通过物体级掩码约束注意力交互，解耦多物体运动。

**训练阶段**：每个视频仅含单目标，对每个参考视频独立训练运动 token（2000 步，AdamW，lr=3e-3），通过 QK 乘积从注意力激活中提取物体掩码。

**推理阶段**：给定多目标图像和多个预训练运动 token，利用语义分割模型获取首帧掩码，通过动态回归掩码传播机制 (RMPM) 逐帧传播掩码，并在注意力层中应用 MDMA 实现运动解耦。

### 运动解耦掩码注意力 (MDMA)

构建物体特定的掩码矩阵 $\mathcal{M}$，约束注意力交互使运动/文本 token 仅影响其指定区域：

**Motion-to-[X] (M2X) 掩码**：

- **M2V 掩码**：运动 token 仅关注对应物体的视频 token，$\mathcal{M}_{m \to v}^k[p,q] = 1$ 仅当 $T_m[p] \in T_m^k \wedge T_v[q] \in T_v^k$
- **V2M 掩码**：M2V 的转置
- **M2M 掩码**：设为零矩阵，彻底阻断不同物体运动 token 间的交互

**Text-to-[X] (T2X) 掩码**：

- **T2V 掩码**：运动描述文本仅关注对应物体的视频 token
- **T2T 掩码**：阻断不同物体运动描述文本的交叉关注
- **T2M 掩码**：文本 token 仅与对应物体的运动 token 交互

最终掩码矩阵按 $\mathcal{A} \leftarrow \mathcal{A} \odot \mathcal{M}$ 应用于原始注意力图。训练时仅激活 M2M/M2V/T2M 子掩码（单物体）；推理时激活全部子掩码（多物体）。

### 差异化掩码提取机制 (DMEM)

**训练阶段 — QK 乘积提取**：利用模型自身注意力激活，用物体相关的文本查询 $Q_y^k$ 与视频键 $K_v^\top$ 相乘，以均值为阈值二值化得到物体掩码 $\widehat{\mathcal{M}}^k$。优势：(i) 避免外部分割模型的训练开销；(ii) 保持训练-推理一致性。

**推理阶段 — 回归掩码传播 (RMPM)**：QK 方法在多目标场景失效（特征纠缠无法区分物体），故引入 RMPM：

1. 用分割模型 [SegGPT] 提取首帧各物体掩码 $\{\widehat{\mathcal{M}}_1^k\}$
2. 维护滑动窗口锚定特征集 $\mathcal{F}_{\text{anc}}$ 和对应掩码
3. 按帧递推：计算当前帧特征与锚定特征的相关矩阵 $\mathcal{C}_l^k$，传播得到新帧掩码
4. 更新锚定集（窗口大小 $W=2$）

### 动态 RMPM

利用扩散模型早期步骤主要恢复粗结构的特性，每步计算掩码差异，若低于阈值 $\alpha=5\%$ 则停止更新、复用稳定掩码，显著减少推理时间。

## 实验关键数据

### 数据集与设置

200 组视频-图像对，覆盖 20 种运动类型，来源包括 FlexiAct、Pexels、DAVIS 2017 和 Seedream。基模型 CogVideoX-5B-I2V，分辨率 720×480，49 帧，6 张 A800 GPU。

## 亮点

1. **首创性**：首个隐式 I2V 多目标多运动迁移框架，填补该领域空白
2. **设计精巧的 MDMA**：通过 6 类子掩码（M2V/V2M/M2M/T2V/V2T/T2T/T2M/M2T）全面解耦注意力，思路清晰且有效
3. **训练-推理一致的掩码策略**：训练时用 QK 乘积轻量提取掩码避免外部依赖，推理时用 RMPM 递推传播首帧掩码
4. **动态 RMPM 提效**：利用扩散模型生成特性自适应终止掩码更新，工程实用性强
5. **新指标 FF**：提出 Flow Fidelity 度量运动幅度和方向的整体相似度，弥补 TF 不足
6. **人工评价优势显著**：在 20 位评分者中取得 76-89% 的压倒性胜出率

## 局限性 / 可改进方向

1. **首帧依赖分割模型**：推理时需要外部语义分割模型提取首帧掩码，未完全实现端到端
2. **训练局限于单目标视频**：每次训练只用单目标视频学习运动 token，可能限制多物体交互运动（如两人对话、协作动作）的学习
3. **数据集规模有限**：仅 200 组视频-图像对用于评测，泛化性有待更大规模验证
4. **AC/TC 指标偏低的解释**：论文认为高 AC/TC 可能反映静态输出，但这种辩护略有 hand-waving 之嫌
5. **运动 token 需按物体独立训练**：每个运动需单独训练 2000 步，多运动组合时总训练开销与运动数成正比
6. **未探索复杂遮挡/多物体重叠场景**：当物体在运动过程中发生大面积遮挡时 RMPM 的鲁棒性未知
