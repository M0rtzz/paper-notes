# StyleTokenizer: Defining Image Style by a Single Instance for Controlling Diffusion Models

**会议**: ECCV 2024  
**arXiv**: [2409.02543](https://arxiv.org/abs/2409.02543)  
**代码**: 无  
**领域**: Image Generation  
**关键词**: 风格迁移, 风格token, 扩散模型控制, 单图风格学习, 内容-风格分离

## 一句话总结

提出StyleTokenizer，通过将风格定义为可学习的token嵌入，实现仅用一张参考图即可控制扩散模型的风格生成，同时精确分离内容和风格。

## 研究背景与动机

利用扩散模型进行风格化图像生成是一个重要的应用场景。用户通常希望提供一张参考图像来指定所需风格，然后让模型按照该风格生成新内容。然而，当前方法在风格控制方面面临一个核心挑战——**内容与风格的精确分离**。

具体问题包括：(1) 如何从单张参考图像中提取"纯风格"信息，而不混入内容信息？例如，给定一张梵高风格的星空画作为参考，模型应该学到"梵高画风"而非"星空场景"；(2) 现有方法（如LoRA微调、IP-Adapter等）在风格迁移时常常泄露参考图像的内容元素到生成结果中；(3) 使用文本描述风格（如"oil painting style"）过于粗糙，无法捕获细微的风格差异。

本文提出StyleTokenizer，将"风格"定义为一种可以用token嵌入表示的属性。通过专门设计的训练流程，学习一个风格编码器，能从单张参考图像中提取纯风格token，精确排除内容信息。

## 方法详解

### 整体框架

StyleTokenizer的工作流程：(1) 使用风格编码器从参考图像中提取风格token；(2) 将风格token与用户提供的文本提示（描述所需内容）组合；(3) 扩散模型基于组合的风格+内容条件生成目标图像。关键在于风格编码器的训练方式确保它只编码风格、不编码内容。

### 关键设计

1. **风格编码器（Style Encoder）**:
    - 功能：从单张参考图像中提取纯风格token
    - 核心思路：基于预训练的视觉编码器（如CLIP ViT），添加轻量级的投影层将视觉特征映射为风格token。关键训练策略是使用"相同风格、不同内容"的图像对进行训练——让编码器在不同内容但相同风格的图像上产生相似的token，从而迫使编码器忽略内容、聚焦风格
    - 设计动机：直接使用CLIP特征会包含大量内容信息；专门的训练策略是实现内容-风格分离的关键

2. **风格Token注入机制**:
    - 功能：将风格token有效地注入扩散模型的生成过程
    - 核心思路：将风格token作为额外的条件，通过cross-attention注入到U-Net的去噪过程中。风格token与文本token在不同的attention层分别处理——文本token控制内容生成，风格token控制风格化效果。这种分离的注入方式减少了内容-风格的干扰
    - 设计动机：如果将风格token简单拼接到文本token后面，两者会在attention中相互干扰，导致分离不完全

3. **对比式风格学习（Contrastive Style Learning）**:
    - 功能：学习内容无关的风格表示
    - 核心思路：构建正样本对（相同风格、不同内容的图像对）和负样本对（不同风格的图像对），使用对比学习目标训练风格编码器。正样本对可以通过数据增强（如style transfer）或从风格一致的数据集中采样获得
    - 设计动机：对比学习天然适合学习不变性——在这里是对内容的不变性、对风格的保留性

### 损失函数 / 训练策略

- 对比损失：拉近相同风格图像的风格token，推远不同风格的token
- 扩散重建损失：使用风格token和文本条件生成图像，与ground truth计算去噪损失
- 风格一致性损失：确保同一风格图像生成的结果在视觉风格上一致
- 训练策略：先训练风格编码器（冻结扩散模型），然后联合微调

## 实验关键数据

### 主实验

| 数据集 | 指标 | 本文 | StyleAdapter | 提升 |
|--------|------|------|----------|------|
| 52提示×28风格 | 风格一致性 | 最优 | 基线 | +10-15% |
| 风格迁移评估 | 内容保持率 | 最优 | IP-Adapter | 更少内容泄露 |
| 用户研究 | 风格偏好率 | >65% | LoRA微调 | 显著领先 |
| FID | 生成质量 | 有竞争力 | 多个方法 | 保持质量 |

### 消融实验

| 配置 | 关键指标 | 说明 |
|------|---------|------|
| 无对比学习 | 内容泄露 | 风格token包含内容信息 |
| 拼接式注入 | 风格-内容干扰 | 生成结果不可控 |
| 单token vs 多token | 多token更优 | 更丰富的风格表示 |
| 不同参考图数量 | 单图即有效 | 多张略有提升 |

### 关键发现

- 对比训练策略是实现内容-风格分离的关键
- 分离的attention注入比简单的token拼接效果好得多
- 单张参考图即可定义风格，方法使用门槛低
- 风格token可以与任意文本提示组合，实现灵活的风格化内容生成

## 亮点与洞察

- 将风格表示为可学习的token是一个直观且有效的设计思路
- 内容-风格分离的训练策略是方法成功的核心
- 单张参考图即可学习风格，大大降低了使用门槛
- 方法无需为每种风格单独训练（如LoRA），一次训练适用所有风格

## 局限性 / 可改进方向

- 风格的定义和量化仍然是主观的，不同用户对"风格"的理解可能不同
- 对于需要精确模仿参考图中特定纹理细节的场景，token化的风格表示可能信息不足
- 构建"相同风格、不同内容"的训练数据本身需要人工策划或style transfer
- 视频风格化是一个自然的扩展方向
- 可以探索组合多种风格token生成混合风格

## 相关工作与启发

- **IP-Adapter**: 通过图像提示控制扩散模型，但不区分内容和风格
- **StyleAdapter**: 另一种风格适配方法，但风格分离不够精确
- **LoRA微调**: 为每种风格单独微调，效果好但不可扩展
- 启发：将视觉属性（风格、光照、材质等）表示为独立的token是一种通用的可控生成范式
