# MolLangBench: A Comprehensive Benchmark for Language-Prompted Molecular Structure Recognition, Editing, and Generation

**会议**: ICLR 2026  
**arXiv**: [2505.15054](https://arxiv.org/abs/2505.15054)  
**代码**: [GitHub](https://github.com/TheLuoFengLab/MolLangBench) / [HuggingFace](https://huggingface.co/datasets/ChemFM/MolLangBench)  
**领域**: AI for Chemistry / 分子-语言接口  
**关键词**: 分子识别, 分子编辑, 分子生成, 语言-分子对齐, Benchmark

## 一句话总结

提出 MolLangBench，一个全面评估 AI 模型在语言提示下进行分子结构识别、编辑和生成能力的 benchmark，支持 SMILES 字符串、分子图像和分子图三种分子表示，揭示了当前最强模型（GPT-5）在这些对人类直觉简单的任务上仍存在显著缺陷。

## 研究背景与动机

精确的分子识别、编辑和生成是化学家和 AI 系统处理各种化学任务的基本前提。然而，当前缺乏一个全面的 benchmark 来评估 AI 模型在这些基础分子-语言接口任务上的能力。

现有工作的不足：
- 已有分子生成 benchmark 多关注高级任务（如药物设计），缺乏对基础能力的系统评估
- 现有数据集质量不一，可能存在歧义性和不确定性
- 缺少对多种分子表示（字符串、图像、图）的统一评估

核心动机：如果 AI 系统连基础的分子结构识别和操作都做不好，更复杂的化学任务（如药物发现、材料设计）也难以信赖。需要一个高质量、无歧义的 benchmark 来准确测量当前 AI 的分子理解能力。

## 方法详解

### 整体框架

MolLangBench 评估三类核心能力：
1. **分子结构识别 (Recognition)**：给定分子，用自然语言回答关于其结构的问题
2. **分子编辑 (Editing)**：根据语言指令修改给定分子的结构
3. **分子生成 (Generation)**：根据纯文本描述从头生成分子

支持三种分子表示：
- **线性字符串**：SMILES 表示
- **分子图像**：2D 结构图
- **分子图**：图结构表示

### 关键设计

1. **识别任务 (Recognition)**：
   - 通过自动化学信息学工具（RDKit）构建，确保答案的确定性和无歧义性
   - 子任务包括：单跳邻居、键类型、官能团识别、环结构等
   - 既有训练集也有测试集
   - 设计动机：测试模型是否真正"理解"分子结构，而非仅仅模式匹配

2. **编辑任务 (Editing)**：
   - 通过严格的专家标注和验证构建
   - 给定分子 + 编辑指令 → 输出修改后的分子
   - 示例：替换官能团、改变连接方式、添加/删除原子等
   - 仅测试集（防止训练集泄漏）

3. **生成任务 (Generation)**：
   - 同样通过专家标注和验证
   - 仅根据文字描述生成分子（无参考分子输入）
   - 最具挑战性的任务
   - 仅测试集

4. **评估标准**：
   - 识别和编辑：准确率（精确匹配）
   - 生成：准确率（生成的分子是否满足所有描述条件）
   - 对图像模态，使用 Mathpix API 将生成图像转回 SMILES 进行自动评估

### 损失函数 / 训练策略

作为 benchmark 论文，MolLangBench 本身不训练模型，而是评估现有模型：
- 使用 OpenAI 批处理 API 进行大规模评估
- 提供标准化的 prompt 模板
- 评估流程：准备 prompts → 创建批处理任务 → 提交 API → 获取结果 → 计算指标

## 实验关键数据

### 主实验

各任务最强模型表现：

| 任务 | 最强模型 | 准确率 | 说明 |
|------|---------|--------|------|
| 识别 (Recognition) | GPT-5 | **86.2%** | 对人类直觉简单的结构问题 |
| 编辑 (Editing) | GPT-5 | **85.5%** | 根据指令修改分子 |
| 生成 (Generation) | GPT-5 | **43.0%** | 从文本描述生成分子 |

### 消融实验

| 配置 | 关键指标 | 说明 |
|------|---------|------|
| SMILES vs 图像输入 | SMILES 通常更高 | 文本模型在 SMILES 上表现更好 |
| 不同识别子任务 | 难度差异大 | 简单的原子计数 vs 复杂的环识别 |
| o1/o3/o4-mini vs GPT-5 | GPT-5 全面领先 | 推理能力对分子任务有帮助但不够 |
| 专用分子模型 vs 通用 LLM | 差距在缩小 | 通用 LLM 在规模上弥补了领域知识不足 |

### 关键发现

1. **生成任务极具挑战**：最强模型 GPT-5 仅 43.0% 准确率，说明当前 AI 从文本描述构建分子结构的能力严重不足
2. **识别和编辑仍有提升空间**：86.2% 和 85.5% 看似不错，但这些是对人类化学家来说非常基础的任务，理想状态应接近 100%
3. **分子表示方式影响大**：SMILES 字符串作为输入时模型表现通常优于图像输入，暗示当前多模态模型在分子图像理解上的局限
4. **数据质量至关重要**：通过自动化工具和专家标注确保的高质量数据，使得评估结果更可靠

## 亮点与洞察

- **填补重要空白**：首个系统评估 AI 在基础分子操作能力的全面 benchmark
- **三种分子表示**：统一评估 SMILES、图像、图三种模态，暴露不同表示下模型的优劣
- **高质量数据构建**：识别任务用自动化工具确保确定性，编辑和生成任务用严格专家标注
- **揭示当前 AI 的局限**：即使是 GPT-5，在基础分子任务上的表现也令人担忧——这对依赖 AI 进行化学研究发出了警示
- **配套训练数据**：同时发布了 MolLangData 用于训练，形成完整的评估-训练生态

## 局限性 / 可改进方向

1. **分子覆盖范围**：当前主要覆盖常见有机小分子，对金属有机化合物、聚合物、生物大分子的覆盖有限
2. **任务粒度**：编辑和生成任务的复杂度有限，更复杂的多步合成路径设计未涉及
3. **评估局限**：基于精确匹配的评估可能过于严格，同一分子可能有多种有效的 SMILES 表示
4. **图像模态**：依赖 Mathpix 将图像转回 SMILES 进行评估，引入了额外的错误来源
5. **模型覆盖**：主要评估了 OpenAI 模型，对开源模型的评估可以更全面

## 相关工作与启发

- 分子语言模型领域快速发展，但缺乏统一的基础能力评估标准
- 与 NLP benchmark (GLUE/SuperGLUE) 类似的思路——先测基础能力，再测高级任务
- MoleculeNet 等早期分子 benchmark 主要关注属性预测，MolLangBench 关注语言-分子交互
- **启发**：AI for Science 领域需要更多关注基础能力的系统评估，而非直接追求端到端的复杂任务
