# P-GenRM: Personalized Generative Reward Model with Test-time User-based Scaling

**会议**: ICLR 2026  
**arXiv**: [2602.12116](https://arxiv.org/abs/2602.12116)  
**代码**: [GitHub](https://github.com/Tongyi-ConvAI/Qwen-Character/tree/main/Character-GenRM)  
**领域**: LLM对齐/个性化  
**关键词**: 个性化奖励模型, 生成式评判, 用户画像, 测试时scaling, 协同过滤

## 一句话总结
提出P-GenRM——首个个性化生成式奖励模型：将混合偏好信号(显式准则+隐式历史)转化为结构化评价链(用户画像+评分标准)，通过三阶段训练(PSI监督微调→CRE强化学习→课程学习)学习自适应评估，再用双粒度测试时scaling(个体级多次评分聚合+原型级相似用户协同)减少噪声并增强新用户泛化，在个性化奖励基准上SOTA+3%测试时scaling增益。

## 研究背景与动机

1. **领域现状**：RLHF对齐LLM→奖励模型是核心。个性化对齐→需要用户特定的奖励信号。开放场景中偏好信号稀疏/噪声/场景依赖。

2. **现有痛点**：
   - (1) **静态偏好建模**：将动态/场景依赖的偏好简化为固定规则→同一用户不同场景偏好不同(开车时简洁/聊天时详细)
   - (2) **新用户泛化弱**：冷启动问题→有限反馈无法构建可靠奖励
   - (3) 现有方法不利用GenRM的测试时扩展性

3. **切入角度**：生成式奖励模型(GenRM)→可生成评价链→天然支持测试时scaling+可解释性→结合用户画像和协同过滤。

## 方法详解

### 三阶段训练

1. **PSI (Persona-guided Scoring Induction)**：
   - 监督微调→学习从混合偏好信号推断评价链
   - 输出：用户画像+场景自适应评分标准+评分

2. **CRE (Criteria-based Reasoning Enhancement)**：
   - 强化学习→增强评价链生成能力
   - 特别针对偏好信息缺失的场景

3. **课程学习**：
   - 从简单到难的negative example
   - 逐步提升区分困难样本的能力

### 测试时用户基scaling

1. **个体级scaling**：
   - 对同一用户/查询生成多次评分方案
   - 自适应聚合→减少推断偏好中的噪声

2. **原型级scaling**：
   - 将用户聚类为User Prototypes(协同过滤思想)
   - 纳入相似用户的评分→增强冷启动场景

### 评价链结构
```
用户偏好信号 → 场景感知画像 → 动态评分标准(含权重) → 逐条评分 → 最终分数
```

## 实验关键数据

### 个性化奖励模型基准
| 方法 | 准确率 | 新用户泛化 |
|------|--------|----------|
| 通用RM | 基线 | — |
| SynthesizeMe | +1% | 弱 |
| **P-GenRM** | **+2.3%** | **强** |
| **P-GenRM+TUS** | **+5.3%** | **最强** |

### 测试时Scaling效果
| Scaling | 增益 |
|---------|------|
| 无scaling | 基线 |
| 个体级scaling | +1.5% |
| **个体+原型级** | **+3%** |

### 关键发现
- 测试时scaling的+3%增益→证明GenRM的扩展性优势
- 原型级scaling对新用户帮助最大→因为可以借用相似用户的偏好
- 动态评分标准 vs 固定标准→+2%→场景自适应很重要
- 评价链的可解释性→用户可以理解为什么某个回答得分高/低

## 亮点与洞察
- **首个个性化GenRM**：将生成式奖励模型与个性化对齐结合→开辟新方向。
- **协同过滤遇到RLHF**：推荐系统的核心思想(相似用户有相似偏好)→首次用于奖励模型→跨领域创新。
- **测试时Scaling的价值**：与LLM的test-time compute scaling异曲同工→更多推理时间→更好的个性化。
- **"评价链"而非"评分"**：不只输出一个数字→输出完整的评价过程→可调试/可解释/可迭代。
