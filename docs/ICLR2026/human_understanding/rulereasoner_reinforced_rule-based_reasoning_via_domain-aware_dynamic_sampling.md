# RuleReasoner: Reinforced Rule-based Reasoning via Domain-aware Dynamic Sampling

**会议**: ICLR 2026  
**arXiv**: [2506.08672](https://arxiv.org/abs/2506.08672)  
**代码**: 无  
**领域**: LLM推理/规则推理  
**关键词**: 规则推理, RLVR, 域感知采样, 动态权重, OOD泛化

## 一句话总结
提出RuleReasoner——通过大规模规则推理数据集(RuleCollection-32K, 8类任务)+域感知动态采样(Dads, 基于历史奖励动态调整域采样权重)的RLVR方法增强LLM规则推理→8B模型在OOD任务上超越OpenAI-o1(Δ10.4%)和DeepSeek-R1(Δ14%),且训练步数更少(更高效)。

## 研究背景与动机

**领域现状**：规则推理→根据事实+规则得出结论→人类在法律/医学/数学中的核心能力。LRM通过RL激发长思维链推理→但规则推理特有挑战(格式/类型/复杂度变化)未被专门研究。

**现有痛点**：
   - (1) 规则格式多样(符号/自然语言/显式/隐式)→简单的RLVR不够
   - (2) 训练数据域不平衡→某些域欠优化
   - (3) 现有RLVR数据效率低→需大量rollout
   - (4) 缺乏大规模多样的规则推理数据集

**切入角度**：(1) 收集8类多样规则推理任务; (2) 域感知动态采样→自适应平衡各域。

## 方法详解

### RuleCollection-32K数据集
- 8类规则推理任务(逻辑/法律/数学/符号等)
- 显式和隐式规则
- 多种复杂度级别
- 支持ID和OOD评估

### 域感知动态采样(Dads)

1. 初始：标准RLVR训练(等权采样)
2. 每N步：更新域权重
   - 低奖励域→权重↑(需要更多训练)
   - 高奖励域→权重↓(已充分学习)
3. 按新权重采样训练batch→继续训练

**关键**：基于**历史奖励**→不需要proxy模型/人工先验→自适应

### RLVR训练
- GRPO(不带KL loss)→鼓励探索
- 精确匹配奖励(+1/-1)
- 严格on-policy→每次rollout后一次梯度更新

## 实验关键数据

### OOD(3个未见推理基准)
| 模型 | 平均Pass@1 | vs o1 |
|------|----------|-------|
| DeepSeek-R1 | 中 | -14% |
| OpenAI-o1 | 高 | 基线 |
| **RuleReasoner-8B** | **最高** | **+10.4%** |
| **RuleReasoner-4B** | 较高 | +7.3% |

### ID(8个训练域)
- RuleReasoner-8B→所有8个域SOTA
- 比标准RLVR→+4.1%平均

### 训练效率
| 方法 | 达到目标性能步数 |
|------|-------------|
| 标准RLVR | 基线 |
| DAPO | 0.8x |
| **RuleReasoner+Dads** | **0.5x** |

### 关键发现
- Dads→域间性能更平衡→弱域显著提升+强域不下降
- 4B模型也能有效学规则→不需要超大模型
- OOD泛化强→说明学到了可迁移的规则推理能力(非记忆)
- 无KL loss→更大的探索空间→对规则推理有利

## 亮点与洞察
- **"8B超越o1在规则推理上"**：规则推理可能是小模型的sweet spot→因为需要精确而非广博。
- **Dads的自适应简单性**：仅用历史奖励→无需额外模型/人工标注→self-adaptive且开销为零。
- **"规则推理=可控AI的基础"**：真实世界需要遵守规则(法律/安全/合规)→增强规则推理=增强可控性。
- **训练效率的bonus**：Dads不仅提高性能→还加速训练→因为减少了在"已学好"域上的浪费。
