# In Agents We Trust, but Who Do Agents Trust? Latent Source Preferences Steer LLM Generations

**会议**: ICLR 2026  
**arXiv**: [2602.15456](https://arxiv.org/abs/2602.15456)  
**代码**: [GitHub](https://github.com/) (作者声明公开)  
**领域**: 推荐系统 / LLM偏见分析  
**关键词**: LLM Agent, 信息源偏好, 信任偏见, 品牌感知, 推荐系统

## 一句话总结
通过对12个LLM在新闻、学术、电商三个领域的大规模控制实验，揭示了LLM存在系统性的**隐式信息源偏好**（latent source preferences），这种偏好可以压倒内容本身的影响，且无法通过简单提示消除。

## 研究背景与动机
基于大语言模型的智能体正在被广泛部署为各类在线平台的用户接口——新闻聚合、学术搜索、电商推荐等。在这些场景中，LLM Agent 充当了用户与后端信息检索系统之间的中介角色，负责过滤、优先排序和综合来自不同来源的信息。

**核心问题**：这些 LLM 在处理带有来源标签的信息时，是否会对某些来源产生系统性偏好？例如，面对语义完全相同但分别标注为 NYTimes 和 Fox News 的新闻，LLM 会如何选择？

现有研究大多集中在 LLM 生成内容时的偏见（政治偏见、性别偏见等），但鲜有工作关注 LLM 在**选择和呈现**已有信息时的偏见——本文作者将这种偏见命名为"隐式信息源偏好"（latent source preferences），并通过系统性实验证明其存在和影响。

## 方法详解

### 整体框架
- **输入**：构造控制实验——语义相同但标注不同来源的内容对
- **被测模型**：来自6家提供商的12个LLM（GPT-4.1-Mini/Nano, Llama-3.1/3.2, Phi-4/Mini, Mistral-Nemo/Ministral, Qwen2.5-7B/1.5B, DeepSeek-R1-Llama/Qwen）
- **评估域**：新闻（政治倾向集+世界新闻集）、学术（50个顶级期刊会议）、电商（70个平台）
- **输出**：来源偏好排名 + Kendall Tau 相关系数

### 关键设计
1. **直接评估（Direct Evaluation）**：
   - 做什么：显式询问模型对两个信息源的偏好，例如"请比较 CNN 和 Fox News 的新闻标准"
   - 核心思路：类似 LLM-as-a-judge 评估范式，让模型直接表达偏好
   - 设计动机：捕获模型显式声称的偏好

2. **间接评估（Indirect Evaluation）**：
   - 做什么：在实际任务场景中，给模型语义相同但来源不同的内容，观察其选择
   - 核心思路：构造控制变量实验——两篇语义等价的文章分别标注不同来源，让模型选择"质量更高"的那篇。如果模型一致性地偏好某个来源的文章，说明存在隐式偏好
   - 设计动机：模型的显式声明可能与实际行为不一致，间接评估更能反映部署时的真实行为

3. **身份一致性分析**：
   - 分析模型是否能将同一信息源的不同身份（品牌名、X/Twitter账号、网站URL、关注者数量等）关联起来并分配类似偏好
   - 发现：多数大模型能识别不同身份形式的同一来源，但当表面形式差异过大时（如从品牌名到社交媒体ID），关联能力会下降

4. **凭证理性分析**：
   - 测试模型对来源凭证（关注者数量、H5-Index、成立年份）的偏好是否理性
   - 发现：H5-Index是相对稳定的信号，但关注者数量和成立年份的解读存在不一致性和非理性行为

### 案例研究

**案例1：AllSides新闻选择**
- 使用AllSides.com的3855条新闻事件数据
- 六组实验：隐藏来源、显示来源、要求不偏见、左右互换等
- 核心发现：来源信息对选择的影响巨大，仅交换来源标签就能显著改变选择分布

**案例2：Amazon卖家选择**
- 使用真实Amazon卖家数据，让LLM代替用户选择卖家
- 评估模型在价格、评分、配送速度等维度的权重
- 发现模型行为与Amazon BuyBox算法明显不同，更偏向评分和价格

## 实验关键数据

### 主实验
| 数据集 | 指标 | 主要发现 |
|--------|------|----------|
| 政治倾向新闻集 | 偏好%标准差 | 大模型（GPT-4.1-Mini, Phi-4）偏好方差大，小模型偏好弱 |
| 世界新闻集 | Kendall Tau相关 | 不同模型间偏好排名高度相关（共享训练数据效应） |
| 学术研究集 | 偏好%（分领域） | NEJM在医学类96%被选，CV类仅19%——验证了上下文依赖性 |
| 电商平台集 | 偏好%（分品类） | BestBuy在电子品类97%被选，食品杂货类仅51% |

### 消融实验
| 配置 | 关键指标 | 说明 |
|------|---------|------|
| Source Hidden | 接近均匀 | 隐藏来源信息时选择接近随机 |
| Source Shown | 偏好显著 | 显示来源后选择分布显著倾斜 |
| Source Swap | 偏好反转 | 交换来源标签后选择跟随来源 |
| Do Not Be Biased | 无效 | 提示消偏无法覆盖隐式偏好 |

### 关键发现
- **大模型偏好更强**：GPT-4.1-Mini、Phi-4等大模型比小模型表现出更强、更异质的来源偏好
- **偏好是上下文相关的**：同一模型对同一来源在不同话题领域中的偏好可以完全不同
- **后训练影响偏好**：DeepSeek-R1-Distill-Llama-8B与Llama-3.1-8B-Instruct来自同一基础模型但偏好截然不同
- **提示消偏无效**：指示模型"不要偏见"几乎无法减弱甚至可能加强偏好
- **来源偏好可压倒内容**：在新闻选择中，来源偏好的影响力超过了内容本身

## 亮点与洞察
- **首次系统性研究**：第一个系统性地揭示和量化 LLM 隐式信息源偏好的工作
- **实验设计精巧**：通过控制变量（语义等价内容、只替换来源标签）严格分离了来源效应和内容效应
- **实际影响深远**：如果 LLM Agent 系统性地偏好某些信息源，可能导致信息茧房、品牌不公平竞争等问题
- **安全隐患**：攻击者可能利用 LLM 的来源偏好，通过模仿高信任度来源的标识来操纵推荐结果
- **多利益方启示**：对平台方、内容提供方、用户和政策制定者都有重要启示

## 局限性 / 可改进方向
- 仅研究了三个应用领域（新闻、学术、电商），未来可扩展到医疗、法律等高风险领域
- 未深入探究偏好的因果来源——是预训练数据频率驱动还是后训练导致
- 实验分析了共现频率与偏好的关系但发现不完全解释，需要更深入的机制研究
- 未提出有效的去偏方法——仅发现提示无效，但没有探索训练时干预或推理时控制策略
- 研究仅涉及文本模态，多模态LLM是否存在类似偏好值得探索

## 相关工作与启发
- **LLM偏见**：政治偏见（Feng et al., 2023）、地理偏见（Manvi et al., 2024）、文化偏见（Baker et al., 2023）——本文揭示了一种新型的"来源偏见"
- **LLM认知偏差**：Itzhak et al.（2024）研究了LLM的认知偏差起源，本文发现后训练也能改变偏好
- **推荐系统公平性**：传统推荐系统的公平性研究可以类比到LLM Agent场景
- **对未来Agent设计的启示**：需要设计透明可控的Agent，让用户能理解和调整Agent的信息源偏好
