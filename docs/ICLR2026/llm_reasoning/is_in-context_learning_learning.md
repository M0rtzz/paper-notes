# Is In-Context Learning Learning?

**会议**: ICLR 2026  
**arXiv**: [2509.10414](https://arxiv.org/abs/2509.10414)  
**代码**: 无  
**领域**: LLM 理论 / 上下文学习  
**关键词**: 上下文学习, 学习理论, 记忆化, 泛化能力, 分布敏感性

## 一句话总结

通过大规模实验分析 ICL 的本质，从数学定义和实证两个层面回答"上下文学习是否真正在学习"：数学上 ICL 符合学习的定义，但实证表明其学习和泛化到未见任务的能力有限，更多依赖于提示中的规律性模式推断而非真正的归纳学习。

## 研究背景与动机

In-Context Learning（ICL）允许自回归模型通过下一个 token 预测来解决任务，无需进一步训练。这导致了关于模型能力的大量论断——声称它们能够用少量 shot（示例）"学习"未见过的任务。然而：

**推理 ≠ 学习**：ICL 不显式编码给定的观察，模型依赖的是先验知识和给定的示例
**ICL 的机制不明**：尽管 ICL 被广泛使用，但其底层机制仍未被充分理解
**性能不一致**：ICL 在不同任务、模型和提示风格下表现差异巨大
**记忆化的干扰**：预训练数据中可能包含测试任务的相关知识，难以区分 ICL 的真实学习能力和记忆化效果

核心问题：当我们排除记忆化、预训练知识、分布偏移和提示风格的影响后，ICL 是否真正在"学习"？

## 方法详解

### 整体框架

本文采用**分析性研究**框架，分为两部分：
1. **数学分析**：从学习理论出发，讨论 ICL 是否满足"学习"的形式化定义
2. **大规模实证分析**：系统地消融或控制各个混淆因素，评估 ICL 的真实学习能力

### 关键设计

1. **数学层面的论证**：

    - 作者论证 ICL 在数学上确实符合学习的形式化定义
    - 但要完整刻画其能力需要实证工作
    - 关键区分：满足学习的数学定义 ≠ 具有强大的泛化学习能力

2. **消融记忆化的实验设计**：

    - 通过控制测试任务与预训练数据的重叠度
    - 使用新颖的/人工构造的任务来测试纯粹的 ICL 能力
    - 评估模型在完全未见过的任务分布上的表现

3. **分布偏移分析**：

    - 改变示例的分布（如随机化标签、打乱顺序等）
    - 观察模型性能如何随分布变化而变化
    - 测试 ICL 对输入分布的敏感性

4. **提示风格和措辞分析**：

    - 系统测试不同的提示格式（零样本、少样本、思维链等）
    - 分析语言特征（如措辞变化）对性能的影响
    - Chain-of-Thought 的分布敏感性分析

### 实验设计要点

- **多模型对比**：测试多个不同规模和架构的自回归模型
- **多任务覆盖**：在形式上相似但语义不同的任务上进行测试
- **控制变量**：逐一控制记忆化、预训练、分布偏移、提示风格等因素
- **极限行为分析**：当示例数量趋近于很多时，观察性能的渐近行为

## 实验关键数据

### 主实验

作者在大规模实验中发现了一系列反直觉的结果：

| 发现 | 详细说明 | 影响 |
|------|---------|------|
| 示例增多时的饱和 | 准确率对示例分布、模型、提示风格和输入语言特征不敏感 | ICL 的"学习"有明确的天花板 |
| 分布敏感性 | ICL 从提示中的规律性推断模式，而非学习抽象规则 | 易受提示格式影响 |
| CoT 的脆弱性 | Chain-of-Thought 提示对分布偏移尤为敏感 | CoT 可能更多是格式匹配而非推理 |
| 形式相似任务的差异 | 在形式上相似的任务上表现差异大 | 自回归的临时编码不够鲁棒 |

### 消融实验

| 消融条件 | 观察结果 | 结论 |
|---------|---------|------|
| 排除记忆化 | 性能显著下降 | 大量"ICL 能力"来自预训练记忆 |
| 控制预训练分布 | ICL 对预训练数据分布高度依赖 | 非真正的分布无关学习 |
| 随机化示例标签 | 某些情况下性能影响不大 | 模型可能忽略示例内容 |
| 变化提示措辞 | 性能波动显著 | 表面格式比语义内容更重要 |

### 关键发现

1. **在极限情况下（示例充分），准确率对以下因素不敏感**：
   - 示例的分布
   - 模型选择
   - 提示风格
   - 输入的语言特征

2. **ICL 更像是模式匹配而非学习**：模型从提示的规律性中推断模式，而非从示例中归纳出抽象规则

3. **Chain-of-Thought 的特殊脆弱性**：CoT 提示对分布偏移特别敏感，暗示 CoT 的"推理"可能更多是格式匹配

4. **自回归的临时编码机制不够鲁棒**：这限制了 ICL 作为通用学习机制的潜力

## 亮点与洞察

- **填补了 ICL 理解的关键空白**：之前的工作要么从纯理论角度分析 ICL（如证明 Transformer 能模拟梯度下降），要么从经验角度展示 ICL 的成功案例，本文是首次系统地从"ICL 是否真正在学习"的角度进行大规模消融分析
- **"数学上可以学习"与"实际上在学习"的区分**：即使 ICL 满足 PAC 学习的形式定义，其实际表现也远未达到强泛化学习器的水平
- **对 CoT 的质疑有深远意义**：如果 CoT 更多是格式匹配而非推理，那么当前基于 CoT 的复杂推理流水线可能建立在不稳固的基础上
- **挑战了"LLM 是少样本学习器"的叙事**：提供了实证证据表明 LLM 的"少样本能力"很大程度上来自预训练记忆而非真正的上下文学习

## 局限性 / 可改进方向

1. **对最新模型的覆盖**：论文可能未覆盖最新的模型（如 o1、Claude 3.5 等），这些模型的 ICL 能力可能有质的提升
2. **任务域的局限**：主要关注 NLP 任务，对多模态、数学推理等任务的 ICL 分析可能有不同结论
3. **"学习"的定义争议**：不同的学习理论框架可能给出不同的结论
4. **缺乏正面建议**：论文主要揭示问题，但未提出增强 ICL 学习能力的方案
5. 可以更深入地分析不同预训练数据规模和质量对 ICL 能力的影响
6. 对 scaling 效应的分析不够充分——更大的模型是否能真正突破 ICL 的局限？

## 相关工作与启发

- **理论方面**：与 Garg et al. (2022) "What can transformers learn in-context?" 形成对话，后者从构造性角度证明 Transformer 能做 ICL，本文则从实证角度质疑其实际能力
- **实践方面**：对 Prompt Engineering 社区的重要提醒——精心设计的提示可能更多是在匹配预训练分布，而非真正教会模型新知识
- **ICL vs 微调**：本文的发现间接支持了微调（如 MAPD 等方法）作为比 ICL 更可靠的少样本适应方式的路线
- 启发：未来的 LLM 评估基准应更注重区分"记忆化复述"和"真正的泛化学习"
