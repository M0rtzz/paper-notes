# The First Impression Problem: Internal Bias Triggers Overthinking in Reasoning Models

**会议**: ICLR 2026 (Poster)  
**arXiv**: [2505.16448](https://arxiv.org/abs/2505.16448)  
**代码**: 无  
**领域**: LLM 推理 (LLM Reasoning)  
**关键词**: 过度思考, 内部偏差, 推理模型, 因果干预, 注意力机制

## 一句话总结

发现推理模型（如 o1 风格模型）的过度思考（overthinking）现象源于模型在看到问题后立即形成的"内部偏差"（preliminary guess），当这种初始猜测与后续推理冲突时会触发过度反思，通过反事实干预实验证明了因果关系，并发现现有缓解方法均无法消除此偏差影响。

## 研究背景与动机

推理模型（Reasoning Models）是近期 LLM 领域最重要的进展之一。以 OpenAI o1 为代表的模型通过长链式思考来提升复杂任务的解题能力。然而，这类模型普遍存在一个令人困扰的问题——**过度思考（overthinking）**，即生成大量冗余的推理步骤，浪费计算资源但无法提升结果质量，有时甚至降低准确率。

此前关于 overthinking 的研究主要从**外部表现**层面进行分析（如推理 token 数量与问题难度的不匹配），但对其**内在触发机制**缺乏深入理解。

本文提出了一个全新的视角——**"第一印象问题"（The First Impression Problem）**：

- 当模型遇到一个问题时，在进行系统性推理**之前**，它的内部表示已经形成了关于答案的初步猜测
- 这种猜测称为**"内部偏差"（internal bias）**，因为它可能不会被显式生成，而是隐含在模型的隐状态中
- 当这种初始猜测与模型随后通过规范推理得出的结论产生**冲突**时，模型会陷入反复的自我修正和反思循环，产生冗余的推理步骤
- 核心类比：就像人类的"第一印象"会影响后续判断，模型的"第一印象"也会干扰其推理过程

## 方法详解

### 整体框架

本文的研究框架采用了**"发现问题→验证关联→证明因果→解释机制→尝试缓解"**的递进式分析结构，结合了观测分析、因果干预和可解释性实验。

### 关键设计

1. **内部偏差的识别与定义**：
   - 做什么：定义并量化推理模型的"内部偏差"
   - 核心思路：模型在接收到输入问题后、开始系统推理之前，其隐状态中已编码了对答案的初步猜测。通过探测模型在仅看到问题（未开始生成推理链）时的内部表示来提取这种偏差
   - 设计动机：在认知心理学中，"锚定效应"（anchoring effect）描述了初始信息如何不成比例地影响后续判断。模型可能存在类似的"认知锚"

2. **关联性验证**：
   - 做什么：在多个模型和多样化推理任务上验证内部偏差与过度思考之间的统计关联
   - 核心思路：对比分析——当内部偏差与最终正确答案一致时，推理过程是否简洁；当两者冲突时，是否出现冗余推理步骤
   - 关键发现：在多个推理模型（包括不同规模和架构）和多种推理任务（数学、编程、逻辑等）上，内部偏差与 overthinking 行为之间存在显著的统计关联

3. **反事实因果干预实验**：
   - **干预一：移除输入问题**。在模型开始生成推理链之后，从上下文中移除原始输入问题。如果 overthinking 确实由内部偏差（源自对问题的初始处理）触发，那么移除问题应该减少后续的冗余推理
   - 实验结果：移除输入问题后，冗余推理步骤**显著减少**，证明了内部偏差的因果作用
   - **干预二：手动注入偏差**。通过人为地向模型注入与正确答案冲突或一致的偏差信息，观察 overthinking 行为的变化
   - 实验结果：注入冲突偏差**增加** overthinking，注入一致偏差**减少** overthinking，进一步证实了因果关系

4. **注意力机制分析**：
   - 做什么：通过可解释性实验揭示内部偏差影响推理轨迹的具体机制
   - 核心发现：模型在推理过程中对输入问题的**过度注意力**（excessive attention）是内部偏差影响推理轨迹的关键机制
   - 解释：模型在生成推理链时，反复"回看"原始问题，这种回看行为强化了初始的内部偏差，导致模型在偏差与推理结论冲突时无法有效"放手"，不断尝试协调两者

### 缓解方法评估

作者评估了多种旨在缓解 overthinking 的现有方法：

- 推理步骤预算限制
- 早停策略
- 思考 token 限制
- 提示工程优化

然而，**所有方法都未能有效消除内部偏差的影响**。这表明内部偏差是一个深层的模型特性，而非可以通过简单后处理消除的表面现象。

## 实验关键数据

### 主实验

在多个推理模型和多样化任务上验证内部偏差与 overthinking 的关联。

| 实验条件 | Overthinking 程度 | 说明 |
|---------|:----------------:|------|
| 偏差与答案一致 | 低 | 推理过程简洁高效 |
| 偏差与答案冲突 | 高 | 大量冗余反思步骤 |
| 无偏差（移除问题后） | 显著降低 | 验证因果关系 |
| 注入冲突偏差 | 增加 | 因果干预确认 |
| 注入一致偏差 | 减少 | 因果干预确认 |

| 不同任务类型 | 关联均显著 | 数学/编程/逻辑推理等任务上均观测到 |

### 关键发现

1. **内部偏差的普遍性**：所有测试的推理模型都展现出内部偏差现象，不因模型规模或架构而消失，表明这是推理模型的一个**内在特性**而非训练缺陷。

2. **因果关系明确**：两种反事实干预实验（移除问题和注入偏差）从正反两个方向证明了内部偏差与 overthinking 之间的**因果关系**，而不仅仅是相关性。

3. **注意力机制的关键角色**：过度关注输入问题是内部偏差影响推理的具体机制，提示了模型在推理过程中的信息流动模式。

4. **现有方法的根本性不足**：所有尝试的缓解方法都无法消除内部偏差的影响，说明需要从模型架构或训练范式的更根本层面来解决这个问题。

## 亮点与洞察

1. **认知心理学视角的原创性**：将"第一印象效应"和"锚定效应"等认知心理学概念引入 LLM 推理分析，提供了理解 overthinking 的全新框架。这不仅是一个诊断工具，更是一种思维方式的转变。

2. **严格的因果推断方法**：不满足于观测性关联分析，而是通过反事实干预实验建立因果关系。两种互补的干预设计（移除偏差源 + 注入偏差）形成了严密的因果推断链条，这在 LLM 行为分析中相对稀少。

3. **深层洞察的实用后果**：发现所有现有缓解方法无效，这一"负面结果"本身具有重大价值——它告诉社区表面的修补行不通，需要从更根本的层面（如模型架构、训练目标、推理范式）来解决 overthinking 问题。

4. **对推理模型设计的启示**：内部偏差的发现暗示，推理模型可能需要设计一种机制来明确区分"初始直觉"和"系统推理"，类似于 Kahneman 的 System 1/System 2 理论中对两种思维模式的区分。

## 局限性 / 可改进方向

1. **缺少有效解决方案**：论文很好地诊断了问题并证明了因果关系，但坦诚承认没有找到有效的缓解方法。这留下了重要的开放问题。

2. **内部偏差的量化不够精确**：当前的偏差提取方法依赖于对模型隐状态的探测，可能无法完全捕捉偏差的全部信息。

3. **模型覆盖面**：主要在公开可用的推理模型上验证，对商业闭源模型（如 o1、Claude 等）的分析有限。

4. **任务类型扩展**：主要聚焦于有明确答案的推理任务，对开放式生成、创意写作等任务中的 overthinking 尚未探索。

5. **解决方案方向**：
   - 可以探索在训练阶段就纳入"偏差解耦"目标
   - 可以设计显式的"System 1 / System 2"推理架构
   - 可以研究注意力正则化技术来减少对输入问题的过度关注

## 相关工作与启发

- **Overthinking 分析**：已有工作关注推理模型的 token 浪费问题，但多从外部行为角度分析，本文首次深入内部机制
- **认知启发的 AI 分析**：System 1/System 2 理论、锚定效应等认知科学概念在 AI 中的应用
- **模型可解释性**：注意力分析、探测分类器等技术用于理解模型内部表示
- **推理效率优化**：如 Token 预算控制、早停机制等，本文证明这些方法在面对内部偏差时的局限性
- **因果推断在 NLP 中的应用**：反事实干预作为分析模型行为的工具
- 启发方向：**从认知科学借鉴理论框架**来理解和改善 LLM 行为是一个值得深耕的方向
