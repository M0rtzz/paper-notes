# Skirting Additive Error Barriers for Private Turnstile Streams

**会议**: ICLR 2026  
**arXiv**: [2602.10360](https://arxiv.org/abs/2602.10360)  
**代码**: 无  
**领域**: 差分隐私/流算法  
**关键词**: 差分隐私, 连续发布, 旋转门流, 不同元素计数, F2矩

## 一句话总结

证明差分隐私旋转门流(支持插入和删除)中的多项式加性误差下界可以通过引入乘性误差来绕过——对不同元素计数问题实现polylog(T)乘性+polylog(T)加性误差(而非此前的Ω(T^{1/4})纯加性误差)，对F2矩实现1+o(1)乘性+polylog(T)加性误差(而非Ω(T)纯加性误差)，且仅需polylog空间。

## 研究背景与动机

**领域现状**：差分隐私连续发布(continual release)=数据流中每步都需私密发布统计量。旋转门模型允许插入和删除→比仅插入流困难得多。两个基础问题：不同元素计数和F2矩估计。

**现有痛点**：
   - 不同元素计数：最佳纯加性误差O(T^{1/3})，下界Ω(T^{1/4})→多项式级误差不可避免
   - F2矩：灵敏度Ω(T)→任何纯加性算法必须误差≥Ω(T)
   - 但这些下界实例中真实值远大于误差→暗示乘性误差可能有帮助

**切入角度**：(α,β)混合误差模型→当真实值>>β时相当于α倍近似→下界不再约束。

## 方法详解

### 核心概念：(α,β)混合误差

$$Y_t/p - r \leq \hat{Y}_t \leq q \cdot Y_t + s$$

其中 $pq = \alpha$, $r+s = \beta$。当 $Y_t \gg \beta$ 时相当于乘性α近似。

### 不同元素计数：两种算法

1. **最小哈希算法(Theorem 3.1)**：
   - 经典idea：集合A的最小哈希值估计|A|
   - 隐私约束下不能精确计算最小哈希→创建基于hash LSB的桶
   - 桶中期望元素数几何增长→用私密连续计数确定min hash
   - 限制：严格旋转门流(频率非负)
   - 误差：polylog(T)乘性+polylog(T)加性

2. **域缩减算法(Theorem 4.1)**：
   - 哈希到足够小的域→使碰撞频繁
   - 私密连续计数检测碰撞
   - 缩减后域大小=不同元素数估计
   - 适用：一般旋转门流
   - 误差：polylog(T)乘性+polylog(T)加性

### F2矩估计

- 利用AMS草图(Alon-Matias-Szegedy)
- 将F2分解为"少数大元素"+"大量小元素"
- 大元素：直接跟踪(Count-Sketch)
- 小元素：草图+私密计数
- 误差：(1+o(1))乘性+polylog(T)加性→远优于Ω(T)纯加性

### 空间复杂度

- 所有算法：polylog(n,T)空间→对比先前方法的多项式空间

## 亮点与洞察

- **"绕过"而非"突破"下界**：多项式加性误差下界是信息论级的→但只约束纯加性误差→混合误差模型打开新空间。
- **混合误差的自然性**：在流算法中即使没有隐私约束也需要乘性误差(空间限制)→引入乘性误差是自然的。
- **polylog空间的意义**：先前方法用多项式空间→本文用polylog空间→真正可用于大规模流。
- **统一视角**：纯加性和纯乘性误差是(1,β)和(α,0)的特例→(α,β)误差模型为理解隐私-精度权衡提供更细粒度的框架。
