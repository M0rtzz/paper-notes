# Human-LLM Collaborative Feature Engineering for Tabular Data

**会议**: ICLR 2026  
**arXiv**: [2601.21060](https://arxiv.org/abs/2601.21060)  
**领域**: 表格学习/人机协作  
**关键词**: 特征工程, LLM, 贝叶斯优化, 人类反馈, 表格数据

## 一句话总结
提出人-LLM协作特征工程框架——解耦LLM的特征操作提议和选择过程：LLM仅负责生成候选操作→贝叶斯优化(建模效用+不确定性)引导选择→当效用估计不可靠时(早期轮次)→选择性征询人类偏好反馈(成对比较)→合成研究和真实用户研究均证明提升性能+降低认知负担。

## 研究背景与动机

1. **领域现状**：表格数据仍是最常见的ML数据形态→特征工程质量决定性能。LLM(CAAFE等)→利用语义理解自动提议特征→但同时负责提议+选择→缺乏校准的效用估计→重复探索低收益操作。

2. **现有痛点**：
   - (1) LLM同时提议+选择→黑箱优化→无法校准效用/不确定性
   - (2) 早期迭代→观测数据少→效用估计不准→误导选择
   - (3) 人类专家有直觉(什么变换有用)→但没有高效整合机制
   - (4) AutoML方法(固定算子)→缺乏任务理解→生成冗余特征

3. **切入角度**：解耦提议(LLM)和选择(贝叶斯优化)→不确定性高时引入人类偏好。

## 方法详解

### 框架设计

**LLM: 操作提议器**
- 输入：任务描述+列描述+历史(操作,效用)
- 输出：N个候选特征变换操作
- 不负责选择→避免LLM的uncalibrated heuristics

**贝叶斯优化: 操作选择器**
- 代理模型$\hat{g}(e)$→估计每个操作的效用+不确定性
- 操作编码：语义嵌入(text-embedding) + 列使用向量(binary)
- 采集函数→平衡探索-利用

**人类偏好反馈(选择性)**
- 早期→效用估计不准→此时人类反馈最有价值
- 形式：成对比较("操作A vs B哪个更好？")→比绝对评分更自然
- 选择性征询→仅当期望收益>认知成本时→不过度打扰

### 反馈整合
- 人类偏好→Bayesian更新代理模型
- 随迭代→观测增多→代理模型自身准确→人类反馈需求减少

## 实验关键数据

### 合成研究(多数据集)
| 方法 | 特征工程性能 | 说明 |
|------|-----------|------|
| AutoML(固定算子) | 基线 | |
| CAAFE(LLM提议+选择) | +中 | LLM heuristic |
| LLM提议+BO选择 | +较好 | 无人类 |
| **LLM+BO+人类偏好** | **+最好** | full framework |

### 真实用户研究
| 指标 | 改善 |
|------|------|
| 特征工程性能 | 提升(所有用户) |
| 人类认知负担 | 降低 |
| 征询次数 | 少(选择性) |

### 关键发现
- 解耦提议/选择→比LLM全权→稳定+有效
- 贝叶斯优化→校准的效用估计→避免重复探索低收益
- 人类偏好→早期最有价值→后期几乎不需要→高效利用人力
- 成对比较→比绝对评分→人类更容易且更准确

## 亮点与洞察
- **"LLM创造+人类+BO判断"的三方协作**：各方做最擅长的→LLM的创造力+BO的理性+人类的直觉。
- **选择性征询的经济学**：不是"人类在不在loop"→而是"什么时候需要人类"→成本-收益分析。
- **解耦的通用原则**：提议和选择→不应由同一个agent负责→这对所有LLM-as-optimizer场景适用。
- **真实用户研究的稀缺性**：大多ML工作→只做offline实验→本文有真人验证→更可信。
