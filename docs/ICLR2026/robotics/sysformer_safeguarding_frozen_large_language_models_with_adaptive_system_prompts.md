# Sysformer: Safeguarding Frozen Large Language Models with Adaptive System Prompts

**会议**: ICLR 2026  
**arXiv**: [2506.15751](https://arxiv.org/abs/2506.15751)  
**代码**: [GitHub](https://github.com/Ksartik/sysformer)  
**领域**: LLM安全/模型安全  
**关键词**: 系统提示, 自适应安全, LLM冻结, 模块化防御, 越狱防护

## 一句话总结
提出Sysformer——轻量级Transformer模块附着在冻结LLM输入端,根据用户提示自适应修改系统提示嵌入：保持LLM参数不变→Sysformer学习在嵌入空间中将固定系统提示转化为更鲁棒的版本,在5个LLM×2个基准上有害提示拒绝率提升80%+安全提示合规率提升90%,对越狱攻击鲁棒性提升100%。

## 研究背景与动机

1. **领域现状**：LLM用于安全关键场景→需要防止有害内容生成。微调(RLHF/安全训练)不完美→深层漏洞仍可被越狱攻击利用。

2. **现有痛点**：
   - (1) 微调→不随模型规模扩展+可能擦除有用知识+导致过度拒绝
   - (2) 额外LLM调用(smoothening)→推理成本翻倍
   - (3) 提示过滤→可能误过滤有用内容
   - (4) 手动系统提示→劳动密集+消耗上下文长度+容易泄露

3. **切入角度**：学习自适应修改系统提示→而非修改模型本身→轻量+不影响预训练知识。

## 方法详解

### Sysformer架构

```
用户提示 ──→ [跨注意力] ──→ 修改后的系统提示嵌入 ──→ LLM(冻结) ──→ 输出
                ↑
固定系统提示嵌入 ─┘
```

1. 固定系统提示→初始嵌入
2. 用户提示→编码为查询
3. Sysformer(小型Transformer)→跨注意力融合用户提示和系统提示
4. 输出：修改后的系统提示嵌入→与用户提示拼接→送入冻结LLM

### 训练目标
- 有害提示→学习生成使LLM拒绝的系统提示嵌入
- 安全提示→学习生成使LLM正常回答的系统提示嵌入
- 双目标优化→不过度拒绝

### 越狱防护
- 训练时增加少量越狱攻击样本→增强泛化
- 不需要覆盖所有攻击类型→少量样本即可提升

### 关键特性
- LLM参数完全冻结→不影响预训练知识
- 无额外LLM调用→推理开销最小
- 在嵌入空间而非文本空间操作→更细粒度的控制

## 实验关键数据

### 5个LLM × 2个基准
| 指标 | 改善 |
|------|------|
| 有害提示拒绝率 | ↑ 80% |
| 安全提示合规率 | ↑ 90% |
| 越狱攻击鲁棒性 | ↑ 100% |

### 与其他方法对比
| 方法 | 不修改LLM | 无额外调用 | 自适应 | 拒绝差距↓ |
|------|---------|----------|--------|---------|
| 微调 | ✗ | ✓ | ✗ | 中 |
| Smoothening | ✓ | ✗ | ✓ | 大 |
| 过滤 | ✓ | ✓ | ✗ | 中 |
| SysEmbedder(非自适应) | ✓ | ✓ | ✗ | 中 |
| **Sysformer** | **✓** | **✓** | **✓** | **最小** |

### 关键发现
- 自适应 >> 固定系统提示→不同用户提示需要不同安全策略
- 在嵌入空间操作→比文本空间更有效→因为连续调整更精细
- 跨LLM家族泛化→在未见LLM上也有效(某些程度)
- 少量越狱样本(~100)→对新型攻击也增强→泛化好

## 亮点与洞察
- **"系统提示也可以学习"**：之前系统提示是人工写的fixed文本→Sysformer证明可以根据每个输入动态生成→更有效。
- **嵌入空间的安全操作**：不在文本空间(人可读)而在嵌入空间(模型空间)操作→更powerful且不易被绕过。
- **"拒绝差距"最小化**：不只是提高拒绝率→同时提高合规率→真正的challenge是两者兼顾。
- **模块化防御**：Sysformer是独立模块→可以给任何LLM装上→即插即用的安全层。
