# FedDAG: Clustered Federated Learning via Global Data and Gradient Integration for Heterogeneous Environments

**会议**: ICLR 2026  
**arXiv**: [2602.23504](https://arxiv.org/abs/2602.23504)  
**代码**: [GitHub](https://tinyurl.com/2rbkb3zu)  
**领域**: 联邦学习/聚类优化  
**关键词**: 聚类联邦学习, 数据+梯度相似度, 双编码器, 自适应聚类, 异构数据

## 一句话总结
提出FedDAG——融合数据和梯度信息的聚类联邦学习框架：通过加权类别级(class-wise)相似度同时考虑数据分布和梯度方向进行更精确的客户端聚类,配合双编码器架构(主编码器+辅助编码器)实现跨簇知识共享而保留簇内特化,并设计联邦感知度量自动确定最优簇数,统一处理标签偏斜/特征偏斜/概念偏移/数量偏移四种异构类型。

## 研究背景与动机

1. **领域现状**：非IID数据导致FL性能下降→聚类FL将相似客户端分组各自训练。现有方法按相似度来源分为数据导向(PACFL)和梯度导向(CFL/IFCA)两派。

2. **现有痛点**：
   - (1) **不完整相似度**：仅用数据或梯度→评估不全面。梯度方法受高维/多种偏斜影响; 数据方法仅考虑标签偏斜忽略概念偏移
   - (2) **簇间隔离**：现有方法限制知识共享在同簇内→其他簇的有用低层表示被浪费
   - (3) **簇数预设**：大多方法需预设簇数→不自适应
   - (4) **偏斜类型不全**：仅处理标签偏斜→不处理概念偏移/数量偏移

3. **切入角度**：同时利用数据和梯度→类别级加权融合→更全面; 双编码器→跨簇共享; 自适应簇数。

## 方法详解

### 三大创新

1. **加权类别级数据-梯度相似度融合**：
   - **类别级数据相似度**：比PACFL更精细→按类别比较子空间(而非整体)→自然处理概念偏移
   - **压缩梯度相似度**：客户端每轮最多对一个模型计算梯度→传输压缩梯度
   - **加权融合**：每个客户端学习权重ω→控制数据vs梯度信息的比重
   - **熵损失优化**：最小化熵→使最终邻接矩阵更锐利(清晰聚类)

2. **双编码器跨簇共享**：
   - **主编码器**：用自己簇内客户端数据优化→簇内特化
   - **辅助编码器**：用互补簇的梯度更新→跨簇特征迁移
   - 两个编码器输出拼接→分类器在联合表示上训练
   - 不是noisy blend(如FedSoft)→显式分离→更干净

3. **联邦感知自适应簇数**：
   - 层次聚类→生成候选聚类方案
   - 新联邦感知度量→奖励紧凑簇+惩罚过度分裂
   - 自动确定最优簇数(支持split+merge)

## 实验关键数据

### CIFAR-10/FMNIST (标签偏斜20% + 数量偏移)
| 方法 | CIFAR-10 | FMNIST | 技术 |
|------|---------|--------|------|
| PACFL | 90.45 | 94.41 | 数据(D) |
| CFL | 72.80 | 86.97 | 梯度(G) |
| IFCA | — | — | 梯度(G) |
| **FedDAG** | **92.5+** | **95.5+** | **D+G** |

### 四种异构类型覆盖
| 异构类型 | PACFL | CFL | FedDAG |
|---------|-------|-----|--------|
| 标签偏斜 | ✓ | ✓ | **✓** |
| 特征偏斜 | ✗ | △ | **✓** |
| 概念偏移 | ✗ | ✗ | **✓** |
| 数量偏移 | ✗ | ✗ | **✓** |

### 关键发现
- 数据+梯度融合比单独使用任一方法→在所有偏斜类型上都更好
- 类别级比较对概念偏移至关重要→整体比较会被少数相似类主导
- 双编码器 vs 软聚类(FedSoft)→双编码器更干净→避免了无关簇的噪声
- 自适应簇数in实验中与手动调优的最优值吻合

## 亮点与洞察
- **唯一处理四种异构的方法**：之前的聚类FL方法最多处理1-2种偏斜→FedDAG首次统一处理所有常见异构类型。
- **"类别级比较"的洞察**：PACFL的子空间最小角度→一个共享类就能造成高相似度→FedDAG按类别分别比较→更准确。
- **双编码器是"selective知识共享"**：不是混合所有模型→而是让辅助编码器选择性地从最互补的簇学习→保留主编码器的纯度。
- **联邦感知度量的必要性**：传统聚类指标(如silhouette)不适合FL→FedDAG设计了考虑FL特性(客户端数/通信/隐私)的度量。
