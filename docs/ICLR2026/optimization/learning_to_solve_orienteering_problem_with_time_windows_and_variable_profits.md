# Learning to Solve Orienteering Problem with Time Windows and Variable Profits

**会议**: ICLR 2026  
**arXiv**: [2603.06260](https://arxiv.org/abs/2603.06260)  
**代码**: 无  
**领域**: 组合优化/车辆路径  
**关键词**: 定向问题, 时间窗, 可变利润, 离散-连续解耦, 线性规划

## 一句话总结
提出DeCoST——学习式两阶段方法解决带时间窗和可变利润的定向问题(OPTWVP)：第一阶段并行解码器预测路径和初始服务时间分配,第二阶段用线性规划在固定路径上全局最优化服务时间(证明全局最优性),通过利润加权时间分配比(pTAR)提供跨阶段反馈→在解质量和效率上超越SOTA构造式和元启发式方法(最高6.6x推理加速)。

## 研究背景与动机

1. **领域现状**：OPTWVP=在时间预算内选择节点子集访问+决定每个节点的服务时长→最大化总奖励(奖励随服务时长增加)。经典VRP扩展→但离散决策(路线)+连续决策(服务时间)耦合。

2. **现有痛点**：
   - (1) NCO方法→仅处理离散路线→不处理连续服务时间分配
   - (2) 离散+连续双变量→联合搜索空间指数膨胀
   - (3) 分解方法→第一阶段短视(不考虑后续时间分配)→局部调整不足
   - (4) 元启发式→手动设计+穷举→不可扩展

3. **切入角度**：智能解耦+跨阶段协调→第一阶段感知连续变量→第二阶段精确优化。

## 方法详解

### 两阶段框架: DeCoST

**阶段1: 并行解码(路线+初始时间)**
- 路线解码器: 预测访问顺序(标准注意力机制)
- 服务时间解码器(STD): 预测每个节点的初始时间分配
- 并行结构→两者同时考虑→避免纯路线决策的短视

**阶段2: 服务时间优化(STO)**
- 固定路线后→服务时间分配=约束线性规划
- **全局最优性证明**: LP有闭式最优解→无需迭代求解
- 并行计算→对所有采样路线同时优化
- 利用LP解提供长程结构信息反馈到阶段1

### 跨阶段反馈: pTAR

利润加权时间分配比(profit-weighted Time Allocation Ratio):
- 阶段2的最优时间分配→计算pTAR
- pTAR作为监督信号→指导阶段1的时间预测
- 防止过早收敛到条件最优→保持探索灵活性

### 兼容性
- DeCoST的第二阶段可与不同的第一阶段构造式求解器组合
- 一致提升各种基线的解质量

## 实验关键数据

### OPTWVP实例(100-500节点)
| 方法 | 解质量 | 推理时间 | 说明 |
|------|--------|---------|------|
| 最新元启发式 | 中 | 慢(基线) | 手动设计 |
| POMO(NCO) | 差(不处理时间) | 快 | 仅路线 |
| **DeCoST** | **最好** | **6.6x快** | 路线+时间 |

### 消融
| 组件 | 贡献 |
|------|------|
| 仅路线解码 | 基线 |
| +STD(服务时间解码) | +5%解质量 |
| +STO(LP优化) | +8%解质量 |
| +pTAR反馈 | +3%解质量(防止过早收敛) |

### 关键发现
- 全局最优服务时间→比近似/启发式好→LP的精确性很重要
- 跨阶段反馈→阶段1的路线决策质量提升→长程视野
- 500节点→DeCoST仍高效→元启发式已很慢
- DeCoST的第二阶段可增强各种第一阶段基线

## 亮点与洞察
- **"解耦但协调"**：不是简单分开→通过pTAR反馈实现两阶段的互信→比联合优化更高效比独立优化更好。
- **LP全局最优性的利用**：连续部分有精确解→不需要近似→这是OPTWVP特有结构的Lucky。
- **"第一阶段需要远见"**：仅路线最优≠路线+时间最优→pTAR让路线决策能预见时间后果。
- **对NCO的扩展**：NCO大多只处理纯离散→DeCoST是离散-连续混合优化的NCO→填补空白。
