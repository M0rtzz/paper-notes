# EgoWorld: Translating Exocentric View to Egocentric View using Rich Exocentric Observations

**会议**: ICLR 2026  
**arXiv**: [2506.17896](https://arxiv.org/abs/2506.17896)  
**代码**: [有](https://redorangeyellowy.github.io/EgoWorld/)  
**领域**: 3D视觉  
**关键词**: 视角转换, 第三人称到第一人称, 扩散模型, 手物交互, 点云投影

## 一句话总结

提出 EgoWorld 框架，通过从外部视角提取点云、3D 手部姿态和文本描述等多模态观测，利用两阶段管线将单张第三人称图像转换为高质量的第一人称视图。

## 研究背景与动机

第一人称视觉对于理解手物交互至关重要，在 AR/VR 和机器人领域有广泛应用。然而大多数现有视频资源来自第三人称视角，因此从第三人称到第一人称的视角转换具有重要实用价值。这一任务面临两大核心困难：

**几何差异巨大**：第三人称视角提供全局上下文，而第一人称视角聚焦于手部和物体细节，两者之间存在严重的遮挡和外观变化
**现有方法限制多**：依赖多视图输入、已知相对相机位姿、初始第一人称参考帧，或依赖不可靠的 2D 手部布局预测（如 Exo2Ego），泛化能力差

EgoWorld 的核心动机是：利用丰富的外部视角 3D 观测信息（而非 2D 布局），通过点云重投影加扩散模型来弥合视角鸿沟。

## 方法详解

### 整体框架

EgoWorld 采用两阶段管线设计：

- **阶段一：外部视角观测**：从单张第三人称图像中提取稀疏第一人称 RGB 映射 S_ego、3D 第一人称手部姿态 P_ego 和文本描述 T_exo
- **阶段二：第一人称视图重建**：基于上述观测信息，利用扩散模型生成稠密的第一人称图像

### 关键设计

**（1）深度估计与尺度对齐**

使用 off-the-shelf 深度估计器从外部视角图像获取相对深度图。由于相对深度存在尺度歧义，利用 3D 外部手部姿态（由手部姿态估计器提取的 MANO mesh）进行尺度校准。具体是通过手部区域的深度中值比来计算全局尺度因子 s*，校准后得到度量级深度图。

**（2）点云构建与视角变换**

结合外部图像、校准深度图和相机内参构建 RGBD 点云。关键步骤是计算视角变换矩阵：

- 训练一个 ViT 骨干 + MLP 回归器的 3D 第一人称手部姿态估计器，从外部视角图像直接预测第一人称手部姿态（创新点：此前无模型可做此任务）
- 使用 Umeyama 算法从外部和第一人称手部姿态之间计算刚体变换矩阵
- 将点云变换到第一人称视角得到稀疏 RGB 映射

**（3）文本描述提取**

使用 VLM（Qwen-VL）对外部图像生成场景和手物交互的文本描述，为重建提供语义先验。

**（4）基于扩散模型的重建**

采用预训练的 Latent Diffusion Model 进行图像修复式重建：

- 将稀疏映射编码为 4 通道潜空间嵌入
- 将手部姿态投影为 2D 姿态图，经 VAE 编码并通道降维得到 1 通道姿态嵌入
- 将稀疏嵌入、姿态嵌入和噪声嵌入拼接为 9 通道输入送入 U-Net
- 文本通过 CLIP 编码后作为交叉注意力引导

### 损失函数 / 训练策略

- **手部姿态估计器**：L2 loss，ViT-224 + MLP，batch size 64，lr 1e-4，Adam，100 epochs
- **扩散模型**：标准 LDM 去噪目标，微调预训练 LDM inpainting 模型，batch size 3，lr 1e-5，AdamW，5 epochs
- 推理时采用 Classifier-Free Guidance (CFG) 增强文本引导
- 全部在单张 NVIDIA RTX 4090 GPU 上训练

### 关键发现

1. **文本描述贡献最大**：单独加入文本将 FID 降幅 21%，而单独加入 pose 仅有微弱提升
2. **多模态互补显著**：同时使用 pose + text 时 PSNR 比 text-only 额外提升 2.6 dB
3. **真实场景泛化**：在 wild 场景中仅用手机拍摄即可生成合理结果
4. **几何-语义解耦**：即使提供错误文本描述，模型仍保持正确的几何结构

## 亮点与洞察

- **从 2D 布局到 3D 点云的范式转变**：3D 点云重投影提供了比 2D 手部布局更鲁棒的结构先验
- **多模态观测互补性**：点云提供几何结构、手部姿态引导手部细节、文本描述补充语义
- **端到端设计**：无需预定义相机位姿或参考帧，所有中间表示由现成模型自动推断
- **首次实现从外部视角直接预测第一人称 3D 手部姿态**

## 局限性 / 可改进方向

1. 稀疏 RGB 映射质量高度依赖深度估计和手部姿态估计的准确性
2. 对于在外部视角完全不可见的区域，仍需依赖扩散模型的想象能力
3. 未利用视频序列的时序信息，可扩展至视频级一致性生成
4. 需要外部和第一人称相机内参估计，实际应用中可能引入误差

## 相关工作与启发

- **Exo2Ego**：2D 手部布局 + 扩散，EgoWorld 用 3D 点云取代 2D 布局
- **4Diff**：使用点云但不包含 pose 和 text 条件
- **LDM Inpainting**：EgoWorld 的重建骨干，将视角转换问题转化为图像修复问题
