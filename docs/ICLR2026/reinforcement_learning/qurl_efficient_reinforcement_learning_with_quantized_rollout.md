# QuRL: Efficient Reinforcement Learning with Quantized Rollout

**会议**: ICLR 2026  
**arXiv**: [2602.13953](https://arxiv.org/abs/2602.13953)  
**领域**: LLM训练/RL效率  
**关键词**: 量化rollout, RLVR, 自适应裁剪, 权重更新感知, 推理加速

## 一句话总结
提出QuRL——用量化actor加速RL训练的rollout阶段：量化actor生成序列(占训练70%时间)+全精度actor做梯度更新→提出自适应裁剪范围(ACR)防止长期训练崩溃(量化/全精度策略分歧累积)+更新感知量化(UAQ用不变缩放放大权重变化超过量化粒度)→INT8/FP8量化实现20-80%rollout加速且性能不降甚至微升。

## 研究背景与动机

**领域现状**：推理LLM的RL训练(RLVR)→rollout(自回归解码)占训练70%时间→是效率瓶颈。

**现有痛点**：
   - (1) 自回归解码→逐token串行→密集访存→GPU利用率低
   - (2) 推理任务需长CoT→rollout更慢
   - (3) 朴素量化actor+解耦PPO→长时间训练后出现策略崩溃
   - (4) RL权重更新极小(信赖域约束)→远小于量化噪声→量化后更新"消失"

**切入角度**：量化rollout加速(本质是off-policy→需正确的重要性采样)+解决两个技术挑战。

## 方法详解

### 自适应裁剪范围(ACR)
- 问题：量化actor和全精度actor的策略分布不断分歧→固定裁剪比率下重要性采样比率可能无限增长→训练崩溃
- 解决：动态调整裁剪范围ε→基于两个actor的策略分歧程度
- 分歧大→ε增大→允许更大更新→追赶分歧
- 分歧小→ε回归正常

### 更新感知量化(UAQ)
- 问题：RL每步权重变化ΔW≈10^{-5}→而量化误差≈10^{-3}→量化后ΔW被"吞掉"
- 解决：不变缩放(invariant scaling)→同时减小量化噪声+放大有效权重变化
- 确保权重变化超过量化粒度→被正确传递

### 训练流程
```
每步: θ_old → 量化Q(θ_old) → 生成rollout(快) → 
      用θ(全精度)计算概率 → 重要性比率R = π_θ/π_Q(θ_old) → 
      ACR裁剪 → GRPO更新θ
```

## 实验关键数据

| 配置 | rollout加速 | 性能(DeepScaleR) |
|------|-----------|----------------|
| 基线(FP16 rollout) | 1x | 53.8% |
| 朴素INT8 rollout | 1.5x | 崩溃 |
| **QuRL INT8** | **1.5x** | **55.5%(+1.7%)** |
| **QuRL FP8** | **1.8x** | **54.2%** |

### 关键发现
- ACR防止了训练崩溃→无ACR约200步后reward骤降
- UAQ让量化后的权重更新被正确传播→無UAQ性能下降~2%
- INT8竟然微升+1.7%→可能量化噪声起到了正则化作用
- 7B/14B/32B模型均有效→方法可扩展

## 亮点与洞察
- **"70%时间在rollout→量化加速是最直接的"**：不改RL算法→只加速推理→正交于其他优化。
- **ACR的必要性**：off-policy RL+量化→策略漂移是inevitable→需要动态调整信赖域。
- **"权重更新消失"的发现**：RL更新远小于量化噪声→这是之前量化RL失败的根本原因→UAQ是直接解。
