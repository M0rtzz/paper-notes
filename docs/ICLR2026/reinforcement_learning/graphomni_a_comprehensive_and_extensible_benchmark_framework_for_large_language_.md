# GraphOmni: A Comprehensive and Extensible Benchmark Framework for Large Language Models on Graph-theoretic Tasks

**会议**: ICLR 2026  
**arXiv**: [2504.12764](https://arxiv.org/abs/2504.12764)  
**代码**: [GitHub](https://github.com/GAI-Community/GraphOmni)  
**领域**: 图推理 / LLM评估  
**关键词**: 图推理, LLM基准测试, 序列化格式, 提示策略, 强化学习

## 一句话总结

提出 GraphOmni，一个全面评估 LLM 在图论任务上推理能力的基准框架，系统考察图类型、序列化格式和提示策略三个维度的交互影响，并提出基于 RL 的自适应最优因子选择方法。

## 研究背景与动机

LLM 在自然语言处理方面取得了巨大成功，但在结构化图数据上的推理能力仍有待深入探索。现有的图推理基准存在以下不足：

1. **覆盖维度单一**：大多数工作仅考虑图类型、序列化格式或提示策略中的某一个维度，缺乏对三者交互作用的系统分析
2. **图类型有限**：通常只包含 1-4 种图类型，不足以反映真实世界网络的多样性
3. **缺乏随机基线**：许多基准没有随机基线作为参考，难以准确评估 LLM 的真实能力
4. **框架不可扩展**：现有基准通常是固定的，研究者难以添加新的图生成器、序列化方法或提示策略

GraphOmni 旨在填补这一空白，提供一个覆盖面最广、可扩展性最强的图推理评估框架。

## 方法详解

### 整体框架

GraphOmni 包含四个核心组件：**Benchmark Tasks（基准任务）**、**Graph Types（图类型）**、**Prompt Schemes（提示方案）** 和 **Serialization Formats（序列化格式）**，总计生成 241,726 个查询。

### 关键设计

1. **6 类图论任务**：
   - BFS 遍历序列：测试顺序推理能力
   - 连通性判断：测试局部链接理解
   - 环检测：测试循环模式识别
   - 直径计算：测试全局结构把握
   - 最短路径：测试路径规划
   - 三角形计数：测试精确数值枚举

2. **7 种图类型**：涵盖 ER 图（ERM/ERP）、二部 ER 图（BERM/BERP）、BA 图（BAG/BAF）和无标度图（SF），覆盖从稀疏双部图到高连通图的多种拓扑结构

3. **9 种提示策略**：Algorithm、CoT、K-Shot、Instruct、0-Shot、0-CoT、0-Instruct、0-Algorithm、LTM，系统考察显式指导程度对结果的影响

4. **7 种序列化格式**：GMoL、Adjacency Set/List/Matrix、Edge Set/List、GMaL，探索不同文本表示方式对 LLM 理解图结构的影响

5. **三级难度设计**：Easy（0-10 节点）、Medium（10-20 节点）、Hard（20-30 节点）

### RL-based Prompt Search（RL-Opt）

受基准测试结果启发，提出 RL 引导的最优序列化策略选择方法：
- 将序列化过程建模为序列决策问题，决策变量包括序列化格式、提示策略和 LLM 选择
- 使用 DQN 学习 Q 函数来逼近最优组合
- 采用 ε-greedy 探索策略，ε 从 1.0 逐渐衰减到 0.01
- 搜索空间 E = 7 × 9 × 5 = 315 种组合

## 实验关键数据

### 主实验

评估了 9 个 LLM（5 个开源 + 4 个闭源）：

| 任务 | 难度 | Claude-3.5 | GPT-4o | Qwen2.5 | 随机基线 |
|------|------|-----------|--------|---------|---------|
| BFS | Easy | 91.42 | 90.31 | 33.03 | ≈0% |
| Connectivity | Easy | 98.38 | 92.61 | 56.29 | 67.49 |
| Cycle | Easy | 78.95 | 62.37 | 53.37 | 50.00 |
| Diameter | Easy | 83.71 | 79.14 | 42.81 | 11.20 |
| Shortest Path | Easy | 94.35 | 81.75 | 42.61 | - |
| Triangle | Easy | 43.41 | 50.33 | 12.88 | 2.13 |

### RL-Opt 结果

| 指标 | 各任务平均值 |
|------|------------|
| Cost（搜索成本比） | ≈0.22（减少约 25%） |
| Rate（准确率比） | ≈0.90 |

### 消融实验

| 配置 | 关键发现 | 说明 |
|------|---------|------|
| 开源 vs 闭源 | 性能差距 15-20% | 复杂任务（直径、三角形）差距尤为显著 |
| 简洁序列化 vs 结构化 | 开源偏好简洁，闭源兼容两者 | 开源模型受限于长上下文处理能力 |
| 带示例 vs 不带示例 | 开源受益于多示例，闭源更复杂 | 复杂提示可能反而干扰闭源模型 |

### 关键发现

1. **整体性能中等，仍有很大提升空间**：即使是 Claude-3.5 和 GPT-4o 也远未达到人类水平（接近 100%）
2. **高度变异性**：不同序列化-提示组合可导致同一模型同一任务上高达 40% 的准确率差异
3. **常见错误类型**：序列化格式误读、图论概念理解错误（如把"直径"当"最长路径"而非"最长最短路径"）
4. **RL 方法在 25% 的搜索成本下可保持约 90% 的最优准确率**

## 亮点与洞察

- **三维度交互分析**是本文最大贡献，揭示了序列化格式和提示策略之间存在复杂的非线性交互，没有单一最优组合
- **随机基线的引入**使评估更加公平，避免了在类别不平衡任务（如环检测）上的误导性结论
- **RL-Opt 框架**具有实用价值：当面对新的图任务或新模型时，可以用较低成本找到接近最优的序列化-提示组合
- 错误分析非常细致，包括 6 种常见错误模式的实例分析

## 局限性 / 可改进方向

1. 仅考虑合成图数据，未涉及真实世界图（如社交网络、分子图）
2. 开源模型仅使用 7B-14B 规模，未评估更大模型（如 70B+）
3. RL-Opt 假设所有基准结果不可用从头搜索，但实际中可能存在部分先验知识
4. 未探索图的多模态表示（如图像形式的图可视化）
5. 可扩展到更大规模的图（当前最大 30 节点）

## 相关工作与启发

- **NLGraph**、**GraphQA** 等早期工作只考虑单一维度，GraphOmni 的多维度交叉评估提供了更完整的画面
- 对下游应用的启发：在实际使用 LLM 处理图任务时，应该根据具体任务特点选择合适的序列化-提示组合，而非使用固定配置
- RL-Opt 的思路可推广到其他需要组合优化的 LLM 任务配置场景
