# On Discovering Algorithms for Adversarial Imitation Learning

**会议**: ICLR 2026  
**arXiv**: [2510.00922](https://arxiv.org/abs/2510.00922)  
**代码**: 无  
**领域**: 模仿学习/元学习  
**关键词**: 对抗性模仿学习, 奖励赋值函数, LLM引导进化, 元学习, 训练稳定性

## 一句话总结
用LLM引导的进化搜索自动发现对抗性模仿学习(AIL)的奖励赋值(RA)函数——将AIL分解为密度比估计(判别器)+奖励赋值(密度比→标量奖励)两阶段，发现的DAIL算法在未见环境和策略优化器上泛化且超越人工设计的GAIL/AIRL/FAIRL，分析揭示DAIL通过提供更informative的梯度信号实现更稳定训练。

## 研究背景与动机

1. **领域现状**：AIL(如GAIL)将模仿学习形式化为对抗博弈→判别器估计密度比→奖励赋值(RA)函数将密度比转化为策略训练的奖励。现有RA函数(GAIL/AIRL/FAIRL)都是从散度最小化目标人工推导。

2. **现有痛点**：
   - (1) AIL训练不稳定→如同GAN的不稳定性
   - (2) 大量研究关注改进判别器(阶段1)→RA函数(阶段2)被严重忽视
   - (3) 人工设计RA函数依赖人类直觉→可能不是最优选择

3. **切入角度**：不再人工设计RA函数→用LLM引导进化搜索直接从数据/性能驱动发现最优RA函数→元学习AIL算法。

## 方法详解

### AIL的两阶段分解
1. **密度比估计**：判别器分类专家 vs 策略的(s,a)对→logits≈log(ρ_E/ρ_π)
2. **奖励赋值**：r(s,a) = r_f(ρ_E/ρ_π)，其中r_f是RA函数

### 现有RA函数(人工设计)
| 散度 | 算法 | RA函数 |
|------|------|--------|
| Forward KL | FAIRL | -ℓ·e^ℓ |
| Backward KL | AIRL | ℓ |
| Jensen-Shannon | GAIL | softplus(ℓ) |

### LLM引导进化搜索
- LLM生成候选RA函数(Python代码)
- 在一组训练环境上评估→适应度=模仿策略性能
- 进化操作：变异/交叉→LLM建议改进
- 迭代直到收敛→产出DAIL的RA函数

### DAIL(Discovered AIL)
- 发现的RA函数形式不同于任何已知散度→真正的新发现
- 无需修改AIL框架→直接替换RA函数

## 实验关键数据

### Brax连续控制(未见环境)
| 方法 | 平均回报 | 训练稳定性 |
|------|---------|----------|
| GAIL | 基线 | 不稳定 |
| AIRL | 中 | 较稳定 |
| FAIRL | 中 | 中 |
| **DAIL** | **最高** | **最稳定** |

### Minatar离散控制(未见环境)
- DAIL同样超越所有人工设计基线→跨离散/连续环境泛化

### 跨策略优化器泛化
- 在PPO上发现→在SAC上也有效→算法级泛化

### 关键发现
- DAIL提供更informative的梯度→策略更新更有效
- DAIL的奖励信号在训练后期不collapse→避免了GAIL的模式坍塌
- 发现的RA函数具有可解释的数学结构

## 亮点与洞察
- **"首个元学习的AIL算法"**：之前AIL算法都是人工设计→DAIL是第一个数据驱动发现的。
- **RA函数的被忽视的重要性**：大家都在改进判别器→本文表明RA函数同等重要甚至更重要。
- **LLM作为算法搜索引擎**：比传统进化搜索更高效→LLM的代码生成+领域知识加速搜索。
- **与FunSearch的类比**：类似DeepMind用LLM发现数学函数→这里用LLM发现RL的奖励函数。
