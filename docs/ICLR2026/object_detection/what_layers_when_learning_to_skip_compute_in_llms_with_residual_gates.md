# What Layers When: Learning to Skip Compute in LLMs with Residual Gates

**会议**: ICLR 2026  
**arXiv**: [2510.13876](https://arxiv.org/abs/2510.13876)  
**领域**: LLM效率/自适应计算  
**关键词**: 残差门控, token级层跳过, 自适应深度, GateSkip, 推理加速

## 一句话总结
提出GateSkip——在每个Attention/MLP分支输出添加sigmoid-linear门控→训练门控学习token级重要性→推理时按门控值排序跳过低重要性token→在指令微调模型上全计算提升精度+50%计算节省仍匹配基线→可与量化/剪枝/推测解码组合→门控分析揭示Transformer信息流(BOS token=锚点)。

## 研究背景与动机

1. **领域现状**：LLM统一分配计算到每个token每层→浪费。自适应计算→按需分配→两大方向：路由式(MoD)/早退出。

2. **现有痛点**：
   - (1) 路由式(MoD)→离散决策→不稳定+需从头训练
   - (2) 早退出→加辅助LM头→改变预训练隐藏状态+校准难
   - (3) 两者→通常需要预训练阶段引入→不能后训练添加

3. **切入角度**：残差流门控→可微+在已有模型上微调→推理时确定性跳过。

## 方法详解

### GateSkip机制

$$h_{\ell+1} = h_\ell + o_\ell \odot g_\ell(h_\ell)$$

其中$g_\ell(h_\ell) = \sigma(W_G h_\ell + b)$ → sigmoid门控

- 训练时→门控学习哪些token对哪些层重要→并行可微
- 推理时→按门控值排序→低于分位数阈值的token→跳过该层

### 跳过机制
- 跳过的token→隐藏状态+KV cache直接复制到下一层
- 保留token→正常计算
- 每层预算可独立设置

### 稀疏性损失
$$\mathcal{L} = \mathcal{L}_{LM} + \lambda ||\mathbf{g}||_1$$
→ 鼓励门控稀疏→更多token可被跳过

### 关键特性
- 可微→在预训练模型上直接微调(非从头训练)
- 轻量→每层仅增加一个线性投影
- 与量化/剪枝/推测解码正交→可组合

## 实验关键数据

### 长程推理(Llama 3.1 8B)
| 计算节省 | 精度保持率 |
|---------|----------|
| 10% | >95% |
| **15%** | **>90%** |
| 20% | ~85% |

### 指令微调模型
| 设置 | 效果 |
|------|------|
| 全计算 | **精度提升**(vs无门控) |
| 25%节省 | 仍好于无门控基线 |
| **~50%节省** | **匹配基线** |

### 门控分析洞察
- 早期层→BOS token门控值最高→作为"信息锚点"
- 深层→逐渐选择性→聚焦内容词
- 标点符号→跨层高门控→可能用于信息聚合

### 关键发现
- 指令微调时门控→竟然提升精度→因为门控起到了正则化作用
- 50%节省仍匹配→说明大量计算确实是冗余的
- 与INT4量化组合→进一步2x加速→乘法效应

## 亮点与洞察
- **"残差流=控制机制"**：不只是skip connection→而是通过门控调节信息流→新的理解视角。
- **指令微调时精度提升**：反直觉→减少某些计算反而更好→门控=自适应正则化。
- **可解释性bonus**：门控值→直接揭示"什么token在什么层重要"→Transformer可解释性工具。
- **后训练可加**：不需要从头训练→已有模型→微调加门控→即可获得自适应深度。
