# Generate Any Scene: Scene Graph Driven Data Synthesis for Visual Generation Training

**会议**: ICLR 2026  
**arXiv**: [2412.08221](https://arxiv.org/abs/2412.08221)  
**代码**: [GitHub](https://github.com/RAIVNLab/GenerateAnyScene)  
**领域**: 图像生成/数据合成  
**关键词**: 场景图, 组合生成, 数据引擎, 自我改进, 奖励模型

## 一句话总结
提出Generate Any Scene——基于场景图的数据引擎系统性枚举可能的视觉场景(28K物体×1.5K属性×10K关系→近乎无限场景图)→翻译为标题+VQA对实现自动评测和奖励建模→用于四个应用:(1)自我改进(SD1.5+4%),(2)定向蒸馏(从DALL-E3→SD1.5+10% TIFA),(3)场景图奖励模型(+5% DPG-Bench vs CLIP),(4)内容审核增强。

## 研究背景与动机

**领域现状**：T2I生成模型质量高→但组合泛化和语义对齐差。"a black dog chasing rabbit in Van Gogh style"→可能缺兔子/错风格。

**现有痛点**：
   - (1) 训练数据(LAION/CC3M)→网络爬取→噪声/弱组合/偏向单物体粗描述
   - (2) 人工密集标注→不可扩展
   - (3) VLM自动标注→幻觉/语义噪声
   - (4) 缺乏系统性覆盖组合空间的方法

**切入角度**：场景图→结构化表示视觉空间→系统枚举组合→自动生成标题+评测→数据引擎。

## 方法详解

### Generate Any Scene数据引擎

**视觉元素分类法(Taxonomy)**：
- 28,787个物体 + 1,494个属性 + 10,492个关系 + 2,193个场景属性
- 层次化组织(flower→daisy→white daisy)
- 从多种来源收集(Visual Genome/GQA/ConceptNet等)

**场景图生成**：
- 可配置结构复杂度(1-N个物体/0-M个关系)
- 可提供种子场景图扩展
- 近乎无限的组合空间

**场景图→标题翻译**：
- 结构化场景图→自然语言标题
- 可控制描述粒度

**场景图→VQA对生成**：
- 为每个物体/属性/关系生成问答对
- 支持自动评测生成图像的语义对齐
- 支持构建奖励模型

### 四个应用

1. **自我改进框架**：
    - 生成多张图→选VQA得分最高的→作为微调数据→迭代改进
    - SD1.5: +4%平均提升→甚至好于CC3M真实数据微调

2. **定向蒸馏**：
    - 评估发现SD1.5的弱点(如组合生成)→用DALL-E 3擅长的方面→定向蒸馏
    - 仅用<800条合成标题→SD1.5 TIFA +10%

3. **场景图奖励模型**：
    - 用VQA对构建细粒度奖励→每个物体/属性/关系都检查
    - GRPO微调SimpleAR-0.5B→+5% DPG-Bench vs CLIP奖励

4. **内容审核**：
    - 用合成数据增强ViT-T检测器→跨模型/跨数据集泛化更好

## 实验关键数据

### 自我改进
| 方法 | TIFA | T2I-CompBench |
|------|------|------------|
| SD1.5(原始) | 基线 | 基线 |
| +CC3M真实数据微调 | +2% | +2% |
| **+Generate Any Scene自我改进** | **+4%** | **+4%** |

### 定向蒸馏
- 800条合成标题→TIFA +10%→极少数据即高效

### 奖励模型
| 奖励类型 | DPG-Bench |
|---------|----------|
| CLIP奖励 | 基线 |
| **场景图奖励** | **+5%** |

### 关键发现
- 场景图结构→确保评测的完备性→不遗漏任何物体/属性/关系
- 合成数据→可以替代甚至超越真实数据→因为更丰富更结构化
- 定向蒸馏→仅需少量数据→精准弥补特定弱点
- 奖励模型→场景图级细粒度>>CLIP的整体打分

## 亮点与洞察
- **"数据引擎而非模型改进"**：不改模型→改数据→通过更好的训练数据改善生成质量→数据中心范式。
- **"场景图=组合空间的结构化表示"**：之前的evaluation也用场景图(DSG/DPG)→本文更进一步用于训练数据生成。
- **800条数据定向蒸馏的效率**：找到弱点→在弱点上蒸馏→极少数据极大改善→比海量数据微调更高效。
- **自我改进的可持续性**：模型自己生成数据→筛选→自己训练→迭代改进→无需外部数据来源。
