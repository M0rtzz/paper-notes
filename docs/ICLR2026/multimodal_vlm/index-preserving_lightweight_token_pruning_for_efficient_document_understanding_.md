# Index-Preserving Lightweight Token Pruning for Efficient Document Understanding

**会议**: ICLR 2026 (Workshop on MM Intelligence)  
**arXiv**: [2509.06415](https://arxiv.org/abs/2509.06415)  
**代码**: 无  
**领域**: 多模态VLM / 文档理解  
**关键词**: token pruning, document understanding, VLM efficiency, patch classifier, spatial coherence

## 一句话总结

提出一种轻量级的 token 剪枝框架，通过二值 patch 分类器移除文档图像中的非文本背景区域，并用 max-pooling 细化步骤恢复碎片化文本区域的空间连贯性，在保持准确率的同时大幅降低 VLM 的计算开销。

## 研究背景与动机

视觉语言模型（VLM）在文档理解任务中取得了令人瞩目的成果，但其高计算需求仍然是实际部署的主要障碍。文档图像有一个鲜明特点：与自然图像不同，文档中包含大量空白背景、装饰性元素等非信息区域。这些区域在经过 ViT 编码器切分为 patch token 后，仍然会占用后续 LLM 推理的大量计算资源，造成严重的计算浪费。

现有的 token 压缩方法（如 token merging、attention-based pruning）通常需要在模型训练过程中集成，或依赖复杂的注意力分数计算。这些方法要么需要重新训练模型，要么引入额外的计算开销，对于即插即用的部署场景不够友好。

本文寻求一种更轻量的方案：**在 VLM 处理之前**，通过一个简单的预处理步骤过滤掉明显无用的 patch token，类似于为文档图像做一次快速的"关注区域检测"。关键挑战在于如何在保持简单性的同时确保不误删重要的文本区域。

## 方法详解

### 整体框架

整体流程分为三步：
1. 将文档图像切分为 patch 网格
2. 使用轻量级二值分类器预测每个 patch 是否包含有意义的文本内容
3. 用 max-pooling 细化处理恢复被误删的碎片化文本区域
4. 只将保留的 patch token 送入 VLM 处理

核心创新在于"index-preserving"设计：剪枝操作保留了 patch 的原始位置索引信息，确保送入 VLM 的 token 仍然携带正确的位置编码。这对文档理解至关重要，因为文本的空间位置（如表格中的行列关系、段落的上下文关系）是关键信息。

### 关键设计

1. **二值 Patch 级分类器（Binary Patch-Level Classifier）**: 
   - 对每个 patch 预测二值标签：文本区域（保留）vs 非文本背景（剪除）
   - 分类器非常轻量，可以是浅层 CNN 或简单的线性分类器
   - 训练数据可以通过自动化方式获取：对文档图像运行 OCR，有文本检测结果的 patch 标注为正，其余为负
   - 设计动机：文档图像中文本 vs 背景的区分相对简单，不需要复杂的特征提取

2. **Max-Pooling 细化步骤（Max-Pooling Refinement）**: 
   - 问题：简单的二值分类可能导致文本区域的碎片化——一个连续的文本行中间的某些 patch 被误删，破坏了文本的空间连贯性
   - 解决方案：在分类器输出的二值 mask 上应用 max-pooling 操作
   - 效果：如果一个 patch 的邻域中存在被识别为文本的 patch，它也会被保留
   - 这确保了文本区域的空间连续性，避免因个别 patch 的误分类而丢失上下文

3. **索引保持策略（Index Preservation）**: 
   - 剪枝后的 token 序列中，每个 token 保留其在原始 patch 网格中的位置索引
   - 确保位置编码（positional embedding）仍然正确反映空间关系
   - 对于基于 RoPE 或可学习位置编码的 VLM，这是保证性能的关键

### 损失函数 / 训练策略

分类器的训练使用标准的二值交叉熵损失：
- 正样本：包含文本的 patch
- 负样本：纯背景 patch
- 训练数据通过 OCR 自动标注，无需人工标注
- 分类器轻量且快速收敛，训练成本可忽略

推理时，分类器作为预处理模块运行，VLM 本身不需要任何修改或重训练。

## 实验关键数据

### 主实验

在多个真实世界文档数据集上评估：

| 评估维度 | 说明 |
|---------|------|
| 计算节省 | 大幅减少送入 VLM 的 token 数量 |
| 准确率保持 | 在文档理解任务上与完整输入相当 |
| 通用性 | 适用于不同类型的文档（扫描件、PDF 截图等） |

### 消融实验

| 配置 | 说明 |
|------|------|
| 有/无 max-pooling 细化 | 验证空间连贯性恢复的重要性 |
| 不同剪枝阈值 | 控制保留 token 的比例 |
| 有/无 index preservation | 验证位置信息保持的必要性 |

### 关键发现

- 文档图像中通常有大量（可能 40-70%）的 patch 是纯背景区域，剪枝潜力巨大
- 二值分类器虽然简单，但在文档场景下分类精度很高
- Max-pooling 细化对保持连续文本区域的完整性至关重要
- 索引保持确保了位置信息的正确传递，移除后性能明显下降

## 亮点与洞察

1. **极简但有效**: 方法设计极其简洁——一个二值分类器 + max-pooling，但充分利用了文档图像的领域特性（大量空白背景）
2. **即插即用**: 不需要修改或重训练 VLM，作为预处理步骤即可使用，实际部署友好
3. **领域针对性强**: 专门针对文档理解场景设计，比通用的 token 压缩方法更高效
4. **Index preservation 的重要性**: 强调了在 token 剪枝时保持位置信息的必要性，这一点在很多 token 压缩工作中被忽视
5. **训练数据自动获取**: 利用 OCR 自动标注分类器训练数据，避免了人工标注的需求

## 局限性 / 可改进方向

1. **Workshop paper**: 作为 ICLR 2026 Workshop 论文，实验规模和深度可能有限
2. **仅适用于文档**: 方法高度依赖"文档有大量空白背景"这一假设，不适用于自然图像场景
3. **二值分类的局限**: 对于密集排版的文档（如报纸、杂志），有效剪枝空间可能有限
4. **与更复杂方法的对比**: 未与训练时集成的 token 压缩方法（如 FastV、LLaVA-PruMerge 等）深入对比
5. **多语言支持**: 分类器在不同语言/字体的文档上的泛化性有待验证
6. **动态剪枝率**: 当前可能使用固定阈值，根据文档类型自适应调整剪枝率可能效果更好

## 相关工作与启发

- **Token Merging（ToMe）**: 通过合并相似 token 来减少计算，但需要在 ViT 层间操作
- **FastV、SparseVLM**: VLM 内部的 attention-based token 剪枝，计算更重但适用性更广
- **文档 AI**: LayoutLM 系列、Donut 等对文档布局的利用给出了启发
- 本文的方法可以与上述内部剪枝方法组合使用：先用外部分类器做粗剪枝，再在模型内部做精细化压缩
