# MoMa: A Modular Deep Learning Framework for Material Property Prediction

**会议**: ICLR 2026  
**arXiv**: [2502.15483](https://arxiv.org/abs/2502.15483)  
**代码**: [GitHub](https://github.com/GenSI-THUAIR/MoMa)  
**领域**: 材料科学/模块化学习  
**关键词**: 材料属性预测, 模块化, 自适应组合, 知识复用, 少样本

## 一句话总结
提出MoMa——材料属性预测的模块化框架：先为多样化材料任务训练专用模块(full模块或adapter)集中到MoMa Hub,再通过自适应模块组合(AMC, 基于kNN表示传播+凸优化)为新任务选择最优模块加权组合后微调,在17个数据集上平均超越最强基线14%,少样本场景增益更大。

## 研究背景与动机

1. **领域现状**：深度学习预测材料属性→加速材料发现。预训练力场模型(MACE/JMP)在PES相关属性上表现好。

2. **现有痛点**：
   - (1) **多样性**：材料任务跨越晶体/有机分子/多种属性→单一预训练模型难以覆盖
   - (2) **异质性**：不同任务受不同物理定律支配→联合训练产生知识冲突
   - (3) 力场模型仅在PES属性上预训练→热稳定性/电子性质等任务泛化受限

3. **切入角度**：模块化→每个任务一个模块(避免冲突)+跨任务自适应组合(利用共性)。

## 方法详解

### 阶段1: 模块训练与集中化

- 为多种材料任务训练专用模块→两种形式：
  - Full模块：完整模型微调→性能最优
  - Adapter模块：轻量适配器→内存高效
- 集中到MoMa Hub→知识复用+数据隐私保护

### 阶段2: 自适应模块组合(AMC) + 微调

**AMC算法(无训练)**：
1. kNN标签传播→估计每个模块在目标任务上的性能
2. 凸优化→推断最优加权组合
3. 组合后微调→进一步适应

### 为什么AMC优于现有方法？
- 搜索式方法→噪声不稳定(因为任务异质性)
- 路由器方法→数据不足训练(材料数据稀缺)
- AMC→表示驱动+无训练→避免两者的问题

## 实验关键数据

### 17个下游任务
| 方法 | 获胜任务数 | 平均改善 |
|------|----------|---------|
| MACE预训练 | 中 | 基线 |
| JMP预训练 | 较好 | +5% |
| MoE方法 | 中 | 0% |
| **MoMa** | **16/17** | **+14%** |

### 少样本(材料科学常见)
- MoMa的优势在少样本下更大→因为模块组合利用了跨任务知识

### 模块数量缩放
- Hub中module越多→平均性能越好→可持续扩展

### 关键发现
- AMC权重具有可解释性→揭示材料属性间的关系
- Adapter模块→仅少量额外参数→性能接近full模块
- 联合训练(单模型)→在某些任务上性能下降(冲突)→模块化避免

## 亮点与洞察
- **"模块化=材料科学的正确范式"**：材料任务的多样性+异质性→单一模型必然妥协→模块化是自然解。
- **AMC的优雅**：不需要训练路由器→仅用表示空间的kNN→凸优化有理论保证→计算简单。
- **MoMa Hub的生态愿景**：开源模块库→社区可贡献新任务的模块→累积知识→加速材料发现。
- **14%的平均改善**：在成熟基线上→这个幅度非常显著→说明之前的预训练范式确实有根本限制。
