# AdaBlock-dLLM: Semantic-Aware Diffusion LLM Inference via Adaptive Block Size

**会议**: ICLR 2026  
**arXiv**: [2509.26432](https://arxiv.org/abs/2509.26432)  
**代码**: [https://github.com/lgxi24/AdaBlock-dLLM](https://github.com/lgxi24/AdaBlock-dLLM)  
**领域**: LLM 推理优化  
**关键词**: 扩散语言模型, 半自回归解码, 自适应块大小, 语义感知调度, 推理加速

## 一句话总结
首次系统性挑战扩散语言模型（dLLM）中固定块大小的半自回归解码设定，发现"波动带"（Volatility Band）编码了局部语义结构，并提出 AdaBlock-dLLM——一个无需训练、即插即用的自适应块大小调度器，在相同吞吐量下实现最高 5.3% 的准确率提升。

## 研究背景与动机

扩散语言模型（dLLMs）因其并行解码能力受到越来越多关注，成为自回归 LLM 的有力替代。在多种解码策略中，块级半自回归（semi-AR）方法因支持 KV 缓存和良好的准确率-速度折衷而被广泛采用。Semi-AR 解码的核心思想是：每步解码一个固定大小的 token 块，通过多步去噪揭示（unmask）这些 token。

**现有痛点**：本文发现固定块大小存在两个根本性缺陷：
1. **延迟解码开销（Late Decoding Overhead）**：当前块外的 token 即使已经具有高置信度，也被不必要地延迟揭示。模型已经"知道"这些 token 是什么，但必须等到处理到对应块才能输出，浪费了计算资源。
2. **过早解码错误（Premature Decoding Error）**：当前块内的低置信度 token 被过早提交（commit），导致生成错误的 token。这些 token 尚未通过足够的去噪步骤，就被固定为最终结果。

这两个问题的根源在于：固定块大小无法匹配文本的自然语义边界——语义单元（如短语、从句）的长度是可变的。

**核心 idea**：通过分析去噪过程中的置信度动态，发现一个"波动带"区域对应语义步骤边界，据此自适应调整块大小，使解码边界与语义步骤对齐。

## 方法详解

### 整体框架
输入：待生成的 masked token 序列。输出：通过自适应块大小的半自回归解码得到的完整文本。核心是在运行时根据 token 置信度动态决定每步解码的块大小，而非使用预设的固定值。

### 关键设计

1. **波动带（Volatility Band, VB）发现**：
   - 通过统计分析去噪过程中 token 置信度的变化模式
   - 发现 token 的置信度在解码过程中呈现出局部随机的波动区域
   - 这个波动带（VB）区域与文本的局部语义结构（如短语、子句边界）高度相关
   - VB 内的 token 通常跨越语义步骤的边界——它们的置信度还在波动中，不应被过早提交
   - VB 外高置信度的 token 已经稳定，可以安全揭示
   - 设计动机：利用置信度动态作为语义结构的代理信号，无需额外的语义分析模块

2. **自适应块大小调度器**：
   - 核心机制：使用 delimiter token（分隔符 token，如标点、换行）的置信度作为块边界信号
   - 当 delimiter token 的置信度超过阈值时，将其作为当前块的边界
   - 这使得块大小自然地与语义步骤（句子、短语等）对齐
   - 具体步骤：（a）检测当前块范围内每个 token 的置信度；（b）识别落入 VB 区域的 delimiter positions；（c）在最近的 delimiter 处截断块或扩展块
   - 两个关键超参数：threshold（token 转移置信度阈值，默认 0.9）和 delimiter_threshold（分隔符置信度阈值，LLaDA=0.3, Dream=0.5）

3. **与现有推理框架的集成**：
   - 完全无需训练，不修改模型权重
   - 作为即插即用调度器集成到现有推理框架（如 Fast-dLLM）
   - 与 dual cache 策略结合时效果最佳
   - 支持 LLaDA 和 Dream 两大主流 dLLM 架构
   - 设计动机：保持部署友好性，不增加训练开销

### 损失函数 / 训练策略
无需训练。纯推理时优化策略。关键在于运行时的调度逻辑：
- 对每个去噪步骤，根据当前 token 置信度分布动态决定块大小
- 高置信度 token 立即揭示（解决延迟解码开销）
- 低置信度区域的 token 保留为 masked（解决过早解码错误）

## 实验关键数据

### 主实验
在多个基准上评估，涵盖数学推理、代码生成和指令遵循：

| 任务 | 模型 | 固定块大小 | AdaBlock-dLLM | 提升 |
|------|------|-----------|--------------|------|
| GSM8K (5-shot) | LLaDA-8B | 基线 | +提升 | 数学推理改善 |
| MATH (4-shot) | LLaDA-8B | 基线 | +提升 | 数学推理改善 |
| HumanEval (0-shot) | LLaDA-8B | 基线 | +提升 | 代码生成改善 |
| MBPP (3-shot) | LLaDA-8B | 基线 | +提升 | 代码生成改善 |
| IFEval (0-shot) | LLaDA-8B | 基线 | +提升 | 指令遵循改善 |
| 综合 | 多模型 | 基线 | **最高+5.3%** | 相同吞吐量预算下 |

支持的模型包括 LLaDA-8B-Base/Instruct、LLaDA-1.5、Dream-v0-Base/Instruct-7B。

### 消融实验
| 配置 | 关键指标 | 说明 |
|------|---------|------|
| 不同默认块大小 (16/32/64) | 32 最优 | 块大小基线影响自适应效果 |
| threshold 变化 | 0.9 最优 | 过低导致过早提交，过高导致过多延迟 |
| delimiter_threshold（LLaDA vs Dream） | LLaDA=0.3, Dream=0.5 | LLaDA 从头训练方差低，Dream 从 AR 适配方差高 |
| 无 VB 感知（随机调整） | 性能下降 | VB 的语义信号是关键 |

### 关键发现
- 固定块大小确实是 dLLM 半自回归解码的瓶颈，自适应调整带来一致性改善
- 波动带（VB）是一个有意义的信号：它编码了去噪过程中的语义结构信息
- delimiter token 的置信度是有效的块边界检测器
- LLaDA（从头训练）和 Dream（从 AR 模型适配）表现不同的置信度分布特征
- 在相同吞吐量预算下获取准确率提升，说明块大小调整带来的是"更聪明"的计算分配
- 该方法的收益与 dual cache 等其他推理优化正交，可叠加使用

## 亮点与洞察
- 首次系统性研究 dLLM 中固定块大小的问题，开辟了新的优化方向
- 波动带（VB）的发现很有洞察力——置信度动态作为语义结构的代理信号
- 方法简洁优雅：无需训练、即插即用、仅靠置信度分析就能显著改善
- Late decoding overhead 和 premature decoding error 的分析清晰有力
- 对 LLaDA 和 Dream 两种架构的差异分析提供了对 dLLM 训练方式影响的洞察
- 超参数设置简单（仅两个阈值），且提供了清晰的选择指南

## 局限性 / 可改进方向
- 当前仅在 semi-AR 解码范式上验证，未探索对全并行解码的适用性
- 依赖 delimiter token 作为边界信号，在某些语言或格式中可能不够鲁棒
- 两个阈值需要根据模型特性手动选择，缺乏自动调参机制
- 未分析在更大规模模型（如 70B+）上的表现
- 5.3% 的提升虽然显著，但绝对值有限
- 对非英语语言的适用性未验证
- 未来可探索将 VB 洞察反馈到训练阶段，设计更好的 dLLM 训练策略

## 相关工作与启发
- 与 Fast-dLLM 等推理优化工作互补，AdaBlock 是正交的调度层优化
- 与 LLaDA、Dream 等 dLLM 架构工作相关，提供了推理阶段的优化视角
- 自适应计算分配的思想在 AR 模型中也有（如 early exit），本文首次应用于 dLLM
- 启发：1）其他 dLLM 训练方法的设计可考虑语义步骤边界的对齐；2）VB 分析方法可推广到理解 dLLM 的去噪行为
