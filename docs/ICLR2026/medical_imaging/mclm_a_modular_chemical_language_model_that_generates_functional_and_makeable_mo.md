# mCLM: A Modular Chemical Language Model that Generates Functional and Makeable Molecules

**会议**: ICLR 2026  
**arXiv**: [2505.12565](https://arxiv.org/abs/2505.12565)  
**代码**: 有  
**领域**: AI4Science/药物发现  
**关键词**: 化学语言模型, 模块化分子, 构建块, 可合成性, 双语模型

## 一句话总结
提出mCLM——模块化化学语言模型将分子在功能构建块(而非原子)级别tokenize：用自动化合成兼容的构建块(酰胺偶联/Suzuki/Buchwald反应)作为化学词汇+GNN编码块+自然语言描述功能→形成code-switch双语训练，前置合成可行性同时改善分子功能预测，在430个FDA药物上显著改善关键药物属性，3B参数超越GPT-5的合成可及性。

## 研究背景与动机

1. **领域现状**：LLM用于分子生成→但SMILES原子级tokenization=字符级NLP→泛化差+生成的分子难以实际合成。块化学(block chemistry)→用预制构建块自动化合成→但AI未充分利用。

2. **现有痛点**：
   - (1) 原子级tokenization→open vocabulary→类似字符级语言模型→泛化差
   - (2) 生成的分子大多不可合成→AI提案和实验室之间有巨大鸿沟
   - (3) SMILES破坏分子图结构→相邻原子在字符串中可能很远
   - (4) 改善一个药物属性常牺牲另一个→多目标优化困难

3. **切入角度**：分子tokenization从原子→构建块=从字符→子词→自然对齐NLP的tokenization哲学+保证自动化合成可行性。

## 方法详解

### 模块化化学词汇

**合成保证tokenizer**：
- 仅在自动化兼容反应处断键(酰胺偶联/Suzuki/Buchwald)
- 产出的构建块→保证可被合成机器人组装
- ~80万独特构建块(~20万合成保证)

**规则tokenizer(备选)**：
- 相同断键类型但无额外规则→确保训练覆盖更多分子

### 双语化学-语言建模

```
输入：[自然语言功能描述] [构建块GNN编码] [自然语言反应约束]
→ Transformer解码器 → 预测下一个token(块或文字)
```

- 构建块用GNN编码→捕获图结构
- 构建块嵌入插入到自然语言文本中(实体名位置)
- 形成"code-switch"训练数据→模型学会两种语言

### 迭代自改进推理

1. 给定目标功能→生成候选分子
2. 评估多个属性(BBB穿透/毒性/溶解度等)
3. 基于评估反馈→重新生成改进版
4. 迭代直到多目标满足

## 实验关键数据

### 430个FDA药物
| 方法 | 功能改善 | 合成可及性 |
|------|---------|----------|
| GPT-5 | 中 | 差 |
| SMILES-LLM | 中 | 差 |
| Graph-LLM | 较好 | 中 |
| **mCLM(3B)** | **最好** | **最好(超越GPT-5)** |

### 122个分布外药物(仅用自动化兼容块)
- mCLM在所有基线中属性分数和合成可及性均最优

### "Fallen Angels"(临床失败药物候选)拯救
- mCLM迭代改进→保留有效功能同时修复导致失败的属性
- 例：改善Imatinib的BBB穿透性同时保留抗癌活性

### 关键发现
- 块级tokenization vs 原子级→合成可及性提升显著(SA score↓)
- 双语训练→分子功能理解更深→比单模态好
- 迭代自改进→多目标优化有效→避免"改善一个牺牲另一个"
- 3B参数→远小于GPT-5但化学任务上更好→domain-specific优势

## 亮点与洞察
- **"分子tokenization = NLP tokenization"的类比**：字符→子词的提升→类比原子→构建块→这个insight指导了整个方法设计。
- **"合成可行性前置"**：不是后验检查→而是tokenizer设计保证→生成的分子天然可在自动化平台合成。
- **code-switch双语模型**：类似人类双语者自然切换→分子块嵌入和自然语言无缝混合→优雅的多模态。
- **"Fallen Angels拯救"的实际价值**：挽救临床失败的药物候选→直接影响患者→最有说服力的应用。
