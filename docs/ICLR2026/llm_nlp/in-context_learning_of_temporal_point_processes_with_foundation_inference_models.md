# In-Context Learning of Temporal Point Processes with Foundation Inference Models

**会议**: ICLR 2026 / **arXiv**: [2509.24762](https://arxiv.org/abs/2509.24762) / **代码**: [OpenFIM](https://fim4science.github.io/OpenFIM/intro.html) / **领域**: llm_nlp / **关键词**: temporal point process, foundation model, in-context learning, Hawkes process, amortized inference, conditional intensity

## 一句话总结

提出 FIM-PP——首个面向时间点过程的基础推断模型，通过在大规模合成 MTPP 数据上预训练 Transformer，实现对条件强度函数的上下文学习推断，零样本即可匹配专用模型性能，微调后在多个真实数据集上达到 SOTA。

## 研究背景与动机

标记时间点过程（MTPP）是建模异步、不规则事件序列的标准框架。现有神经 MTPP 方法存在根本限制：

1. **缺乏迁移性**：每个新数据集需从头训练，无法跨系统复用。
2. **基础模型缺位**：NLP/ODE/SDE 已有基础模型，但事件序列至今没有。
3. **可解释性需求**：基于条件强度函数的方法保留激发/抑制关系可读性。

## 方法详解

### 整体框架

两阶段：(1) 合成数据生成——定义广泛条件强度函数族并采样大量 MTPP；(2) 预训练 Transformer 从上下文事件序列推断强度函数。

### 关键设计

#### 合成数据生成

涵盖五类过程：经典 Hawkes（指数核）、Poisson、周期（正弦基强度）、高初始激发（Gamma 基强度）、非单调偏移核（Rayleigh）。随机采样 z in {-1,0,1} 涵盖激发/抑制/无影响。共 72K 点过程、1440 万事件。

#### 模型架构

**上下文编码器**：层次化。每事件嵌入 u = phi_t(t) + phi_k(k) + phi_dt(dt)，序列经 Transformer + 固定查询注意力压缩，再聚合。

**历史编码器**：事件历史作 Transformer 解码器查询，上下文为键值。

**强度参数化**：三非负参数 (alpha, beta, mu)：lambda = mu + (alpha-mu) exp(-beta(t-t_last))。事件后跳至 alpha，以 beta 弛豫至 mu。参数是历史/标记依赖的。

### 损失函数 / 训练策略

下一事件负对数似然。16M 参数，最多 22 种标记。微调几分钟、11GB 显存。

**多事件预测 N=20**：

### 消融实验

- 预训练 vs 从头训练：预训练收敛更快、性能更优
- 上下文数量：远少于 2000 条即可
- 零样本对未见幂律核也可泛化

### 关键发现

1. **零样本竞争力强**：匹配需训练数小时的专用模型
2. **微调极快**：几分钟全面超越基线
3. **可解释性保留**：估计强度函数
4. **标记预测有空间**：交替/单标记主导模式未被覆盖

## 亮点与洞察

- 🌟 **首个点过程基础模型**：填补事件序列领域空白
- 🧪 **合成预训练 + 上下文推断**：广泛先验实现零样本推断
- 🔬 **可解释强度参数化**：三参数指数衰减兼顾灵活性与可解释性
- 🏗️ **层次化编码**：每序列独立编码再聚合，避免超长序列瓶颈

## 局限性 / 可改进方向

1. **预训练分布覆盖不足**：某些模式未被覆盖
2. **自回归误差累积**
3. **标记数上限 22**
4. **指数衰减对长程依赖不足**
