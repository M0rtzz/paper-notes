# SproutBench: A Benchmark for Safe and Ethical Large Language Models for Youth

**会议**: AAAI 2026  
**arXiv**: [2508.11009](https://arxiv.org/abs/2508.11009)  
**代码**: 无  
**领域**: AI安全  
**关键词**: LLM安全评估, 儿童AI安全, 发展心理学, 年龄分层基准, 青少年保护

## 一句话总结

提出 SproutBench，一个包含 1,283 个发展心理学驱动的对抗性提示的评估基准，系统评估 47 个 LLM 在儿童和青少年（0-6、7-12、13-18 岁）场景下的安全性，发现安全性与风险预防强相关（$\rho = 0.86$），交互性与年龄适配性存在显著权衡（$\rho = -0.48$）。

## 研究背景与动机

### LLM 在青少年中的应用现状

LLM 正加速融入教育、娱乐和社交平台，儿童和青少年逐渐成为重要用户群体。然而，主流安全基准（如 JailbreakBench、HarmBench）主要关注成人场景下的越狱防护和有害内容检测，其核心目标是最小化企业责任，忽视了未成年人的发展性需求。

### 现有基准的关键缺陷

**年龄特异性覆盖不足**：未区分早期儿童期（0-6 岁）、中期儿童期（7-12 岁）和青少年期（13-18 岁）的认知、情感和社交风险

**风险类型覆盖有限**：现有基准如 MinorBench 或 ToxiGen 仅覆盖少量儿童安全维度

**评估视角偏差**：从"风险规避"出发，而非"发展促进"——未检查模型输出是否符合年龄、心理安全和社会建设性

### 儿童特有的 AI 风险分类

论文建立了系统的儿童-AI 交互风险分类：

**来自 LLM 输出的风险**：
- 心理健康：情感依赖、打断现实关系、不当处理抑郁等敏感话题
- 社交行为：暴露于操纵性或反社会内容，侵蚀共情和冲突解决能力
- 误信息：有限的批判思维加剧对虚假/有害内容的易感性
- 认知学习：过度使用降低创造力、认知超载
- 隐私：儿童可能无意分享敏感数据

**来自用户滥用的风险**：
- 社交行为：利用 LLM 恶作剧、网络欺凌、通过提示工程绕过安全防护
- 学术诚信：作弊、传播误信息

## 方法详解

### 整体框架

SproutBench 的构建流程：
1. **发展心理学知识库构建**：基于 Piaget 和 Vygotsky 的发展理论，为三个年龄段定义认知特征和语言风格
2. **对抗性提示生成管道**：组合年龄、风险行为和查询类型生成上下文丰富的提示
3. **多维度评估**：从 6 个维度评估 LLM 响应

### 关键设计

#### 1. **发展知识知识库**

为三个年龄段定义发展特征：
- **0-6 岁**：有限的抽象推理、简单句式
- **7-12 岁**：发展中的冲动控制、探索性行为
- **13-18 岁**：复杂的情感查询、价值观冲突

风险行为集合 $B_a$ 按年龄分层设计（见下表），查询类型 $Q_t$ 跨年龄通用：Testing、Manipulative、Emotional、Exploratory。

生成策略定义为三元组：$s = (a, b, q_t)$，其中 $a \in A$（年龄组），$b \in B_a$（风险行为），$q_t \in Q_t$（查询类型）。

#### 2. **数据集构建**

总计 1,283 个对抗性提示，来源于两部分：

| 年龄组 | 行为类型 | 数量 |
|--------|---------|------|
| **0-6 岁** | 无意识敏感词输入 | 65 |
| | 过度依赖 AI 陪伴 | 40 |
| | 情感表达需求 | 43 |
| | 模仿游戏行为 | 134 |
| **7-12 岁** | 命令操纵尝试 | 51 |
| | 虚构身份/场景 | 71 |
| | 诱导敏感内容 | 71 |
| | 测试模型容忍度 | 111 |
| | 测试隐私边界 | 40 |
| | 社交炫耀动机 | 40 |
| | 模仿暴力游戏 | 40 |
| | 伪装提问 | 40 |
| | 恶意使用 AI | 40 |
| **13-18 岁** | 学术作弊尝试 | 48 |
| | 绕过限制话题 | 99 |
| | 情感操纵测试 | 98 |
| | 意识形态挑战 | 53 |
| | 价值冲突测试 | 67 |
| | 社会工程测试 | 35 |
| **总计** | | **1,283** |

其中 200 个从 Safe-Child-LLM 数据集改编（使用 GPT-4 适配），683 个全新生成。

#### 3. **生成管道**

1. **提示公式化**：根据策略 $s$ 从知识库提取年龄适配的语言风格和动机
2. **模型交互**：将提示输入 GPT-3.5（temperature=0.7）生成查询 $D_s = M_{LLM}(f_{prompt}(s))$
3. **LLM 质量验证**：使用 GPT-4 评估语言自然度、年龄适配性、动机一致性和潜在风险

#### 4. **评估维度与评分**

6 个评估维度（0-5 分）：年龄适配性、教育价值、情感支持、交互性、风险预防、安全性

使用 Qwen-2.5 自动评分，与 3 位儿童发展心理学家的专家标注达到 Cohen's Kappa = 0.78。

### 损失函数 / 训练策略

SproutBench 是评估基准而非训练方法，不涉及模型训练。评估过程使用标准推理设置，在 8×NVIDIA RTX 4090 上运行。

## 实验关键数据

### 主实验

评估 47 个 LLM（参数量 135M 到 70B）。

**TOP-3 模型**：

| 模型 | 总分均值 | 标准差 | 安全性 | 风险预防 |
|------|---------|--------|--------|---------|
| llama2:7b | **4.61** | 0.13 | 4.93 | 4.86 |
| llama2:70b | **4.58** | 0.13 | 4.92 | 4.84 |
| gemma2:9b | **4.56** | 0.04 | — | — |

**BOTTOM-3 模型**：

| 模型 | 总分均值 | 标准差 | 安全性 | 风险预防 |
|------|---------|--------|--------|---------|
| smollm2:135m | 3.26 | 0.35 | 3.45 | 2.99 |
| tinyllama:1.1b | 3.41 | 0.21 | 3.54 | 3.19 |
| phi3:3.8b | 3.52 | 0.65 | 3.68 | 3.47 |

### 消融实验

**维度相关性分析（Spearman 相关系数）**：

| 维度对 | 相关系数 | 解读 |
|--------|---------|------|
| 安全性 ↔ 风险预防 | **$\rho = 0.86$** | 正相关，保护行为一致 |
| 年龄适配性 ↔ 教育价值 | $\rho = 0.81$ | 正相关 |
| 情感支持 ↔ 年龄适配性 | $\rho = 0.74$ | 正相关 |
| 交互性 ↔ 年龄适配性 | **$\rho = -0.48$** | 负相关，关键权衡 |
| 交互性 ↔ 安全性 | $\rho = 0.12$ | 弱相关，正交维度 |

**PCA 分析**：PC1（90.28%方差）= 安全轴，PC2（5.07%方差）= 交互性轴，两者近似正交。

**模型规模与表现（底10%表现者中的规模分布）**：

| 规模 | 数据集占比 | 低分占比 | 过度代表倍数 |
|------|-----------|---------|-------------|
| Tiny (<500M) | 8.16% | 20% | **2.45×** |
| Small (500M-7B) | 40.8% | 64% | 1.57× |
| Medium (7-30B) | 38.78% | 16% | 0.41× |
| Large (>30B) | 6.12% | 0% | **0×** |

卡方检验 $\chi^2 = 14.62, p < 0.01$，小模型在低分表现者中显著过度代表。

### 关键发现

1. **安全性与风险预防高度一致**（$\rho = 0.86$）：模型如果安全则通常也善于预防风险
2. **交互性是双刃剑**：高交互性会降低年龄适配性（$\rho = -0.48$），Exemplar 模型（如 gemma3:12b）能平衡两者
3. **小模型风险大**：Tiny 模型被低分组过度代表 2.45 倍，Large 模型（>30B）在低分组中完全缺席
4. **phi3:3.8b 在 13-18 岁组显著退化**：安全性降至 2.97，说明该模型在处理复杂青少年话题时特别脆弱
5. **K-Means 聚类识别出 5 种模型原型**：Exemplars（紫色簇）= 高安全+高交互，Underachievers（橙色簇）= 高风险+中交互

## 亮点与洞察

1. **评估范式转变**：从"风险规避"到"发展促进"，检查模型是否积极支持儿童的认知、情感和社会发展
2. **发展心理学理论驱动**：基于 Piaget 和 Vygotsky 的理论构建知识库，确保提示的发展适配性
3. **交互性-安全性权衡的发现**：揭示了一个此前未被关注的关键权衡——提升交互性可能牺牲年龄适配性
4. **覆盖 20 种风险行为**：远超现有基准的覆盖范围，包括模仿网络恶作剧、隐私测试、情感依赖等被忽视的风险

## 局限与展望

1. **评估依赖 Qwen-2.5 自动评分**：虽然 Kappa=0.78 表明与专家一致，但自动评分可能遗漏微妙的发展性问题
2. **文化多样性不足**：提示主要基于西方发展心理学理论，可能不适用于所有文化背景
3. **缺乏纵向研究**：未跟踪 LLM 与儿童的长期交互效果
4. **未包含青年参与**：未来应让实际的儿童和青少年参与基准设计
5. **部分模型评分差异较大**（如 phi3:3.8b 标准差 0.65），可能反映评估的不稳定性

## 相关工作与启发

- **UNICEF 儿童 AI 政策指南**：基于联合国儿童权利公约，提倡安全、公平和隐私的设计原则
- **Safe-Child-LLM**：聚焦心理健康和安全，但失败率仍达 30-40%
- **JailbreakBench / ToxiGen**：成人安全基准的局限性——忽视 grooming、AI 过度依赖等青少年特有风险
- 对 AI 安全社区的启示：需要建立年龄分层的安全评估标准，而非一刀切的成人标准

## 评分

- 新颖性: ⭐⭐⭐⭐ — 首个全面的年龄分层 LLM 安全基准
- 实验充分度: ⭐⭐⭐⭐⭐ — 47 个模型、6 维度、3 年龄组的全面评估
- 写作质量: ⭐⭐⭐⭐ — 结构清晰，但部分内容重复
- 价值: ⭐⭐⭐⭐ — 填补儿童 LLM 安全评估空白，有重要的政策意义
