# Evaluating LLMs for Police Decision-Making: A Framework Based on Police Action Scenarios

**会议**: AAAI 2026  
**arXiv**: [2601.03553](https://arxiv.org/abs/2601.03553)  
**代码**: https://github.com/Heedou/PASFramework (有)  
**领域**: Recommender / NLP Evaluation  
**关键词**: LLM评估, 警务决策, 领域特定评估, 场景化评估, LLM-as-Judge

## 一句话总结
提出 PAS（Police Action Scenarios）评估框架，一个面向警务场景的 LLM 评估体系，涵盖场景定义、参考答案构建、LLM 响应生成、核心指标提取和性能解读五个阶段，基于 8000+ 韩国警察官方文件构建评估数据集，发现商用 LLM（GPT-4、Gemini、Claude）在警务任务上显著低于参考答案，尤其在事实性和逻辑正确性方面。

## 研究背景与动机

LLM 在警务领域的应用日益广泛（交通事故分析、自动报告生成、犯罪调查等），但存在**严重的验证缺口**：

- **现有评估的不足**：传统评估仅关注信息准确性（如法条匹配、犯罪分类），无法捕捉警务工作中的关键需求——情境适配、法律程序合规、判断合理性
- **实际风险**：LLM 生成的响应可能不是法律意义上的"错误"，但未经验证直接使用可能导致非法逮捕、不当证据收集等严重后果
- **已知偏差问题**：LLM 调度建议已被发现存在区域和种族偏见

**核心矛盾**：缺乏一个专为警务领域设计的、系统性的 LLM 评估框架。现有基准（BLEU、ROUGE、多选题）无法评估警务场景下的推理质量和操作合规性。

**核心 idea**：建立一个五阶段的可扩展评估框架 PAS，将真实警务场景作为评估载体，通过警察专家参与的指标验证流程，筛选出真正能反映警务任务质量的评估指标。

## 方法详解

### 整体框架

PAS 框架形式化为 E_police = f(S, R, G, M, P)，包含五个阶段：
1. S（场景定义）→ 2. R（参考答案）→ 3. G（LLM 响应生成）→ 4. M（核心指标）→ 5. P（综合评估）

### 关键设计

1. **场景定义（S）—— 情境 × 行动的双维度设计**:
   - 做什么：定义警务评估的任务载体
   - 核心思路：将场景分解为 Situation（部门、角色、任务、事件类型）和 Action（法律判断、案件分类、报告撰写、市民互动等）
   - 设计动机：基于法律对警务职责的定义和警务模型理论，确保评估覆盖真实工作中的多种情境
   - 本文实例化为**执勤前思维训练**场景——模拟警察在值班前心理预演应对各种事件的过程

2. **两阶段指标筛选**:
   - 做什么：从 15 个候选指标中筛选出真正与警务任务质量相关的核心指标
   - 核心思路：
     * **第一阶段——多元回归**：哪些指标能显著预测整体质量分（p < .05）？
     * **第二阶段——专家相关性验证**：筛选出的指标是否与人类警察专家判断正相关（Spearman ρ, p < .05）？
   - 设计动机：单靠统计拟合可能选出与领域无关的指标，必须用领域专家判断做二次验证
   - 最终保留 5 个核心指标：逻辑正确性、完整性、事实性、逻辑效率、逻辑鲁棒性

3. **混合评估方法（LLM-as-Judge + 人类专家）**:
   - 做什么：平衡自动化评估的规模化能力与人类专家的领域洞察
   - 核心思路：225 个响应（75 题 × 3 LLM）同时经 3 次自动评审 + 2 次专家评审
   - 设计动机：结合先前关于 LLM 评审员可靠性的研究发现，通过结构化提示（参考答案 + 推理说明）提升一致性

### 损失函数 / 训练策略

本文非训练类工作。数据收集和构建流程：
- **数据源**：韩国国家警察署 1,602 本官方警察手册（PDF），涵盖调查、执法、交通管制、紧急响应等领域
- **预处理**：按标题分段、格式化为 {title, content, question} 结构
- **质量过滤**：在专家指导下筛选，最终得到 8,348 条高质量条目
- **评估集**：75 对问答对，覆盖 7 大类、3 种流程类型
- **LLM 测试**：零样本，temperature=0.8，仅给出简短场景描述

## 实验关键数据

### 主实验

**核心指标上 LLM 与参考答案的差距（5 分制）**：

| 指标 | 参考答案 | GPT-4 | Gemini | Claude | 平均差 |
|------|---------|-------|--------|--------|--------|
| 逻辑正确性 | 4.14 | 2.61 | 3.04 | 2.80 | **-1.32** |
| 事实性 | 4.15 | 2.60 | 2.98 | 2.72 | **-1.38** |
| 完整性 | 3.81 | 2.76 | 3.30 | 2.78 | **-0.86** |
| 逻辑效率 | 3.84 | 2.63 | 2.87 | 2.77 | **-1.09** |
| 逻辑鲁棒性 | 3.81 | 2.72 | 3.09 | 2.84 | **-0.93** |
| 整体质量 | 3.88 | 2.69 | **3.06** | 2.83 | **-1.02** |

### 消融实验（指标筛选过程）

| 指标 | 回归系数 β | 回归 p 值 | 专家相关 ρ | 相关 p 值 | 是否保留 |
|------|-----------|----------|-----------|----------|---------|
| 逻辑正确性 | 0.116 | .018 | 0.560 | <.001 | ✓ |
| 完整性 | 0.141 | .001 | 0.409 | <.001 | ✓ |
| 事实性 | 0.247 | <.001 | 0.330 | .002 | ✓ |
| 逻辑效率 | 0.137 | .004 | 0.330 | .002 | ✓ |
| 逻辑鲁棒性 | 0.167 | .002 | 0.315 | .003 | ✓ |
| 逻辑解释 | 0.108 | .003 | 0.203 | .055 | ✗（专家相关不显著） |

### 关键发现

- **所有商用 LLM 在核心指标上均显著低于参考答案**（p < .001），平均差距 -1.115（核心指标）> -0.828（全指标），说明关键能力短板被准确捕捉
- **Gemini 整体最优**（3.06），GPT-4 最差（2.69），但三者差距不大
- **LLM 擅长的是文本质量**（可读性、无害性），而非**领域知识**（事实性、逻辑正确性）
- **按警务领域分析**：所有 LLM 在 112 紧急呼叫（2.37）、交通（2.55）、安保（2.61）方面得分最低
- **回归模型拟合度极高**（R²=0.856），说明 5 个核心指标确实能有效预测整体质量

## 亮点与洞察

- **方法论贡献突出**：不只是评估一个领域，而是提出了一个**可扩展的评估框架设计方法论**，适用于任何需要领域特定 LLM 评估的场景
- **两阶段指标筛选的严谨性**：统计显著性 + 专家验证的双重门槛，避免了指标选择的主观性
- **"逻辑解释"被剔除的案例**非常有说服力——它通过了回归检验但未通过专家相关验证，说明两阶段缺一不可
- **数据集建设的实效性**：8000+ 真实官方文件 → 75 组高质量问答对，展示了领域数据集构建的规范流程

## 局限性 / 可改进方向

- **数据集规模有限**：75 个问题可能无法覆盖所有警务场景的复杂性
- **语言局限**：基于韩国警务法律和文件，直接迁移到其他国家需要重新构建参考答案
- **零样本设定**：未测试 few-shot 或微调后的 LLM 表现（可能显著提升）
- **LLM-as-Judge 的固有偏差**：相关系数中等（ρ=0.315-0.560），说明自动评估与专家判断仍有显著差距
- **未探索 RAG 增强**：将警察手册作为检索知识库可能大幅改善 LLM 在事实性上的表现
- **评估集仅覆盖"执勤前准备"场景**：其他场景（如现场处置、审讯、取证）需要单独验证

## 相关工作与启发

- FLASK 框架为候选指标提供了起点，但本文的两阶段筛选方法是重要的方法论进步
- LLM-as-Judge 在专业领域的应用需要结构化提示 + 参考答案 + 评分标准三重保障
- 本文的框架设计可以直接迁移到**医疗**（紧急分诊）、**法律**（案件分析）、**金融**（合规审查）等领域
- 揭示了一个重要洞察：LLM 在通过标准化考试（如律师资格考试）上的优秀表现并不能说明其在真实场景中的实用性
