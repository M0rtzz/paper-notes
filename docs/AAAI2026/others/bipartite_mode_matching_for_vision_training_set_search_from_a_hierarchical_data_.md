# Bipartite Mode Matching for Vision Training Set Search from a Hierarchical Data Server

**会议**: AAAI 2026  
**arXiv**: [2601.09531](https://arxiv.org/abs/2601.09531)  
**代码**: [github.com/yorkeyao/BMM](https://github.com/yorkeyao/BMM)  
**领域**: 其他  
**关键词**: 训练集搜索, 域适应, 二部图匹配, 层级聚类, 数据中心方法

## 一句话总结

提出层级数据服务器 + 二部图模式匹配（BMM）框架，通过多粒度层级聚类组织大规模源数据、用匈牙利算法一对一匹配源域和目标域的语义模式（modes），从而搜索出与目标域分布差距最小的训练集，在行人重识别和目标检测任务上显著优于已有训练集搜索方法。

## 研究背景与动机

### 问题场景

在自动驾驶、医学影像、大规模监控等实际应用中，目标域的数据可获取但实时标注不可行（成本高、需专家知识）。一个替代方案是从大规模预标注的数据服务器中搜索合适的训练集来训练模型。

### 核心挑战——模式粒度不匹配

目标域通常包含多个不同的语义模式（modes），如果训练集未覆盖这些模式，模型性能就会下降。关键问题在于**粒度不匹配**：如目标域有"苹果"模式，简单匹配可能只对应"真苹果"或更宽泛的"水果"，而非语义层级相当的"所有形式的苹果"。

### 已有方法的局限

现有训练集搜索方法（如 SnP、NDS 等）主要通过优化算法来减小分布差异或利用模型反馈，但**忽视了数据服务器本身结构的优化潜力**。且这些方法多为特定任务设计（如 SnP 仅针对 re-ID，TL;DR 仅针对语言处理）。

### 本文的灵感来源

类比搜索引擎的层级组织结构（如 Google 的 PageRank 通过链接分析构建网页层次），作者提出将数据服务器组织为层级结构，以支持不同粒度的模式匹配。

## 方法详解

### 整体框架

BMM 框架包含三个核心步骤：
1. **层级数据服务器构建**：对源数据进行平衡 k-means + 自底向上凝聚聚类，构建多层级语义模式树
2. **目标域模式提取**：对目标数据进行平坦聚类，提取不同语义模式
3. **二部图匹配**：用 FID 距离连接源模式和目标模式，通过匈牙利算法实现最优一对一匹配

### 关键设计

#### 1. **层级数据服务器（Hierarchical Data Server）**

**做什么**：将源数据组织为多层级的语义模式树，提供不同粒度的匹配候选。

**核心思路**：
- 首先用 ImageNet 预训练特征提取器提取所有源图像特征 $\mathcal{F}_s = \{F(x_i)\}_{i \in [m_s]}$
- 用**平衡 k-means** 将特征空间划分为 $J$ 个等大小的基础聚类 $\{C^1, \ldots, C^J\}$，最小化 SSE：

$$SSE = \sum_{k=1}^{J} \sum_{x_i \in C_k} \|x_i - \theta_k\|^2 \quad \text{s.t.} \quad |C_k| = \frac{m_t}{K}, \forall k$$

- 在此基础上通过**凝聚聚类（agglomerative clustering）**自底向上逐步合并最近的聚类对，最终生成树状图（dendrogram）
- 合并过程中产生的所有不同大小的聚类都被视为数据服务器中的候选模式，最终得到 $H$ 个模式

**设计动机**：使用平衡 k-means（而非普通 k-means）的关键原因是避免同一层级模式大小差异过大导致的长尾分布问题，确保后续匹配质量。实验验证了这一改进带来 FID 从 51.93 降至 50.48、mAP 从 26.08% 提升至 28.16%。

#### 2. **目标域模式提取与二部图构建**

**做什么**：提取目标域的语义模式，构建源-目标模式间的完全二部图。

**核心思路**：
- 对目标域用**平坦聚类**提取 $L$ 个模式 $\{T^1, \ldots, T^L\}$（使用平坦而非层级聚类，因为层级聚类产生的高层/低层模式高度相关，不利于模式选择）
- 构建二部图 $G = (V, E)$：顶点集为所有源模式和目标模式，每条边的权重为源-目标模式对的 FID 距离：

$$\text{FID}(x, y) = \|\mu_x - \mu_y\|_2^2 + \text{Tr}(\Sigma_x + \Sigma_y - 2(\Sigma_x \Sigma_y)^{1/2})$$

其中 $\mu$ 和 $\Sigma$ 分别为聚类内图像描述子的均值和协方差矩阵

#### 3. **匈牙利算法最优匹配**

**做什么**：在二部图中找到全局最优的一对一模式匹配。

**核心思路**：寻找排列 $\sigma^*$ 使总匹配代价最小：

$$\sigma^* = \arg\min_{\sigma \in \mathfrak{S}_L} \sum_{i}^{L} \text{FID}(T^i, S^{\sigma(i)})$$

然后将所有匹配到的源聚类合并（去重）形成搜索到的训练集 $S^* = \{S^{\sigma(i)}\}_{i=1}^L$。

**为什么用匈牙利而非贪心直接匹配**：直接匹配可能导致多个目标模式匹配到同一源模式（实验中 20 个 case 有 4 次发生），降低数据多样性。允许重复虽然 FID 低但 SSIM 降低（15.85 vs 20.45），mAP 也降低（20.14 vs 26.08）。

### 损失函数 / 训练策略

BMM 本身不涉及神经网络训练，而是一个数据搜索框架。搜索到训练集后，可直接用于训练任务模型（如 IDE 用于 re-ID、RetinaNet 用于检测），也可与伪标签等 UDA 方法联合使用获得进一步提升。

**时间复杂度**：总体 $\mathcal{O}(J^3)$，其中层级构建是一次性离线过程；对不同目标域的匹配复杂度为 $\mathcal{O}(\log J \cdot J \cdot L)$，部署友好。

### 消融实验

| 配置 | FID↓ | mAP↑ | 说明 |
|------|------|------|------|
| 基线（平坦聚类 + 贪心搜索） | 60.64 | 25.45 | SnP 方法 |
| 仅层级聚类（无模式匹配） | ~60 | ~25 | 单独使用无显著提升 |
| 仅模式匹配（平坦聚类） | ~60 | ~25 | 单独使用无显著提升 |
| **层级聚类 + 模式匹配** | **51.93** | **26.08** | 两者协同才有效 |
| 普通 k-means | 51.93 | 26.08 | 基线 |
| **平衡 k-means** | **50.48** | **28.16** | 简单改进带来显著提升 |
| 直接匹配（允许重复） | 51.07 | 20.14 | SSIM 低，多样性不足 |
| 直接匹配（删除重复） | 60.84 | 22.07 | 部分目标模式未匹配 |
| **BMM（匈牙利匹配）** | **51.93** | **26.08** | 平衡多样性与域差距 |

### 关键发现

1. **层级结构免除超参调优**：平坦聚类需要精确找到源/目标聚类数的"甜点"，而层级聚类只要聚类数不太小即可保持稳定性能（源聚类 >200、目标聚类 >20 即可）
2. **与 UDA 方法正交互补**：BMM + MMT 伪标签将 mAP 从 26.08% 提升到 78.95%（targeting Market），远超单独使用任一方法
3. **跨任务泛化**：同一框架在 re-ID 和检测任务上均有效，无需针对任务修改

## 亮点与洞察

1. **数据中心 vs 模型中心**：不同于大量关注模型改进的 UDA 工作，BMM 从数据服务器结构优化的角度出发，提供了正交于模型改进的提升路径
2. **层级结构的优雅性**：一次离线构建层级树，不同目标域可复用，部署高效
3. **模式匹配的可解释性**：可视化匹配结果直观展示搜索到的训练集如何覆盖目标域的各种模式（如暗光环境匹配）
4. **平衡 k-means 的巧妙设计**：简单的等尺寸约束即带来有意义的提升，体现"任务感知的数据结构设计"理念

## 局限性 / 可改进方向

1. **特征提取器依赖**：当前使用 ImageNet 预训练模型提取特征，对与 ImageNet 语义差异大的域可能效果有限
2. **FID 作为模式距离的局限**：FID 假设特征分布为高斯，对复杂多模态分布可能不够准确
3. **扩展到更复杂任务**：当前仅验证了 re-ID 和检测，分割、关键点检测等密集预测任务的适用性有待验证
4. **数据服务器规模的可扩展性**：匈牙利算法复杂度为 $O(J^3)$，超大规模服务器可能需要近似算法
5. **动态更新**：数据服务器添加新数据后需要重新构建层级结构，增量更新机制值得研究

## 相关工作与启发

- **训练集搜索**：SnP (Yao et al. 2023) 是最相关的前序工作，提出搜索+剪枝框架；BMM 在搜索阶段显著改进
- **无监督域适应（UDA）**：BMM 与 MMT、Adaptive Teacher 等 UDA 方法正交互补
- **数据估值**：Data Shapley (Ghorbani & Zou 2019) 从样本价值角度选择数据，与 BMM 的模式匹配视角互补
- **对实际部署的启示**：在真实场景中，构建结构化数据服务器可能是提升迁移学习效果的低成本高收益策略
